Inter-VLAN Routing

Comunicación entre VLANs: Router-on-a-Stick · Subinterfaces 802.1Q · Layer 3 Switch + SVIs

CCNA 200-301 Módulo 3 · Switching ⚠ Muy preguntado en examen

🔁 ¿Qué es Inter-VLAN Routing?

Por diseño, los dispositivos en VLANs diferentes no pueden comunicarse entre sí a nivel de Capa 2 (switching). Para que el tráfico cruce entre VLANs, se necesita un proceso de enrutamiento de Capa 3. A esta operación se le llama Inter-VLAN Routing.

📌 Principio fundamental

Cada VLAN es un dominio de broadcast independiente y una subred IP diferente. Para que un host en la VLAN 10 (192.168.10.0/24) hable con un host en la VLAN 20 (192.168.20.0/24), el tráfico debe pasar por un router o un switch de capa 3 que tenga una interfaz en cada subred.

📐 Los 3 métodos de Inter-VLAN Routing

🔌 Método Legado (Legacy)

Una interfaz física del router por cada VLAN. Simple pero costoso en interfaces. Obsoleto.

🔁 Router-on-a-Stick

Una sola interfaz física del router dividida en subinterfaces 802.1Q. El método más común en el examen CCNA.

⚡ Switch Layer 3 (MLS)

El switch enruta internamente usando SVIs (Switch Virtual Interfaces). El método más eficiente y moderno.

🧠 Conceptos clave antes de configurar

🔌 Enlace Trunk (Troncal) 802.1Q

Un enlace trunk transporta tráfico de múltiples VLANs usando etiquetas 802.1Q de 4 bytes insertadas en la trama Ethernet. Es el vínculo entre el switch y el router (o el switch L3). Sin trunk, no hay Inter-VLAN Routing.

📌 Subinterfaz (Router-on-a-Stick)

Una subinterfaz es una interfaz lógica y virtual creada sobre una interfaz física del router. Se identifica con la notación GigabitEthernet0/0.10 (la parte después del punto = número de VLAN). Cada subinterfaz recibe su propia dirección IP (gateway de la VLAN).

⚡ SVI – Switch Virtual Interface

En un switch de Capa 3, una SVI es una interfaz virtual asociada a una VLAN. Se configura con interface vlan X y se le asigna la IP que actuará como default gateway para todos los hosts de esa VLAN.

📖 Referencia CCNA 200-301

Capítulo 17 del Official Cert Guide Vol. 1 (Wendell Odom): "Inter-VLAN Routing". Capítulo correspondiente en Daniel Pérez Torres: "Enrutamiento entre VLANs – Router-on-a-Stick y Switch Layer 3".

🔌 Método 1 · Legacy (Interfaz por VLAN)

⚠ Método obsoleto – solo conceptual en CCNA

Requiere una interfaz física del router conectada a un puerto de acceso (access) del switch por cada VLAN. Si tienes 10 VLANs, necesitas 10 interfaces del router. Raramente se implementa hoy en día, pero puede aparecer en preguntas conceptuales.

🔁 Método 2 · Router-on-a-Stick (ROAS)

Una única interfaz física del router se conecta al switch mediante un enlace trunk 802.1Q. Sobre esa interfaz se crean subinterfaces virtuales, una por cada VLAN. Cada subinterfaz actúa como el default gateway de su VLAN.

📌 Flujo del tráfico en Router-on-a-Stick

  • PC-A (VLAN 10) envía un paquete a PC-C (VLAN 20)
  • El paquete sale por el trunk del switch con etiqueta VLAN 10
  • El router lo recibe en la subinterfaz G0/0.10
  • El router enruta el paquete y lo reenvía por G0/0.20 con etiqueta VLAN 20
  • El switch lo entrega a PC-C (VLAN 20) quitando la etiqueta

⚡ Método 3 · Switch Layer 3 con SVIs

Un switch de Capa 3 (Multilayer Switch) puede enrutar entre VLANs internamente, sin necesidad de un router externo. Se crean SVIs (Switch Virtual Interfaces) — una por VLAN — que actúan como gateways. Se habilita el enrutamiento IP con ip routing.

💻 Configuración · Router-on-a-Stick

Escenario: VLAN 10 (Administración 192.168.10.0/24) y VLAN 20 (Ventas 192.168.20.0/24). El router se conecta al switch por GigabitEthernet0/0.

PASO 1 · Configurar el Switch (trunk + VLANs) 

SW1# configure terminal

! Crear las VLANs
SW1(config)# vlan 10
SW1(config-vlan)# name ADMINISTRACION
SW1(config-vlan)# exit
SW1(config)# vlan 20
SW1(config-vlan)# name VENTAS
SW1(config-vlan)# exit

! Configurar el puerto hacia el router como TRUNK
SW1(config)# interface GigabitEthernet0/1
SW1(config-if)# description TRUNK_HACIA_ROUTER
SW1(config-if)# switchport mode trunk
SW1(config-if)# switchport trunk encapsulation dot1q
SW1(config-if)# switchport trunk allowed vlan 10,20
SW1(config-if)# no shutdown
SW1(config-if)# exit

! Configurar puertos de acceso VLAN 10
SW1(config)# interface range FastEthernet0/2 - 3
SW1(config-if-range)# switchport mode access
SW1(config-if-range)# switchport access vlan 10
SW1(config-if-range)# no shutdown

! Configurar puertos de acceso VLAN 20
SW1(config)# interface range FastEthernet0/4 - 5
SW1(config-if-range)# switchport mode access
SW1(config-if-range)# switchport access vlan 20
SW1(config-if-range)# no shutdown
SW1(config-if-range)# end
SW1# write memory

PASO 2 · Configurar el Router (subinterfaces) 

R1# configure terminal

! Activar la interfaz física (sin IP, sin encapsulación)
R1(config)# interface GigabitEthernet0/0
R1(config-if)# description TRUNK_HACIA_SW1
R1(config-if)# no ip address
R1(config-if)# no shutdown
R1(config-if)# exit

! Subinterfaz para VLAN 10
R1(config)# interface GigabitEthernet0/0.10
R1(config-subif)# description GATEWAY_VLAN10_ADMIN
R1(config-subif)# encapsulation dot1Q 10
R1(config-subif)# ip address 192.168.10.254 255.255.255.0
R1(config-subif)# exit

! Subinterfaz para VLAN 20
R1(config)# interface GigabitEthernet0/0.20
R1(config-subif)# description GATEWAY_VLAN20_VENTAS
R1(config-subif)# encapsulation dot1Q 20
R1(config-subif)# ip address 192.168.20.254 255.255.255.0
R1(config-subif)# exit
R1(config)# end
R1# write memory

⚠ Orden obligatorio: primero encapsulation, luego ip address

Si intentas asignar una dirección IP a una subinterfaz sin antes configurar encapsulation dot1Q [vlan-id], IOS te dará un error. El número de subinterfaz no necesita coincidir con el número de VLAN, pero por convención siempre se hace coincidir para facilitar la administración.

⚡ Configuración · Switch Layer 3 con SVIs

MLS# configure terminal

! Habilitar enrutamiento IP en el switch L3 (CRÍTICO)
MLS(config)# ip routing

! Crear VLANs
MLS(config)# vlan 10
MLS(config-vlan)# name ADMINISTRACION
MLS(config)# vlan 20
MLS(config-vlan)# name VENTAS

! Crear SVI para VLAN 10 (gateway de VLAN 10)
MLS(config)# interface vlan 10
MLS(config-if)# description GATEWAY_VLAN10
MLS(config-if)# ip address 192.168.10.254 255.255.255.0
MLS(config-if)# no shutdown
MLS(config-if)# exit

! Crear SVI para VLAN 20 (gateway de VLAN 20)
MLS(config)# interface vlan 20
MLS(config-if)# description GATEWAY_VLAN20
MLS(config-if)# ip address 192.168.20.254 255.255.255.0
MLS(config-if)# no shutdown
MLS(config-if)# exit

! Asignar puertos de acceso
MLS(config)# interface range FastEthernet0/1 - 2
MLS(config-if-range)# switchport mode access
MLS(config-if-range)# switchport access vlan 10

MLS(config)# interface range FastEthernet0/3 - 4
MLS(config-if-range)# switchport mode access
MLS(config-if-range)# switchport access vlan 20
MLS(config-if-range)# end

! Verificar SVIs
MLS# show ip interface brief
Interface              IP-Address      OK? Method Status    Protocol
Vlan10                 192.168.10.254  YES manual up        up
Vlan20                 192.168.20.254  YES manual up        up

! Verificar routing (rutas conectadas)
MLS# show ip route connected
C    192.168.10.0/24 is directly connected, Vlan10
C    192.168.20.0/24 is directly connected, Vlan20

⚠ El comando ip routing es indispensable

Sin ip routing, el switch solo actúa como dispositivo de Capa 2 aunque tenga SVIs configuradas. El comando activa el motor de enrutamiento del MLS. Es el error más común en el examen.

📋 Tabla de Comandos Clave

🧪 Laboratorio · Router-on-a-Stick en Packet Tracer

🗺 Topología del Laboratorio

1 Router Cisco ISR 4331 · 1 Switch Cisco 2960 · 4 PCs
VLANs: 10 (Administración) y 20 (Ventas)
Conexión Router ↔ Switch: GigabitEthernet0/0 (trunk) ↔ Gi0/1

Abrir Packet Tracer y construir la topología

Agrega los siguientes dispositivos al área de trabajo:

  • 1x Router ISR4331 (o cualquier router con interfaz Gi)
  • 1x Switch 2960-24TT
  • 4x PC genéricas

Conexiones a realizar:

  • Router GigabitEthernet0/0 → Switch GigabitEthernet0/1 (cable directo)
  • PC0 → Switch Fa0/2 · PC1 → Switch Fa0/3 (serán VLAN 10)
  • PC2 → Switch Fa0/4 · PC3 → Switch Fa0/5 (serán VLAN 20)

Configurar el Switch SW1

Haz clic en el switch → pestaña CLI y escribe:

enable
configure terminal
hostname SW1
vlan 10
 name ADMIN
vlan 20
 name VENTAS
interface GigabitEthernet0/1
 switchport mode trunk
 no shutdown
interface range Fa0/2 - 3
 switchport mode access
 switchport access vlan 10
interface range Fa0/4 - 5
 switchport mode access
 switchport access vlan 20
end
write memory

Configurar el Router R1 (subinterfaces)

Haz clic en el router → pestaña CLI:

enable
configure terminal
hostname R1
interface GigabitEthernet0/0
 no ip address
 no shutdown
interface GigabitEthernet0/0.10
 encapsulation dot1Q 10
 ip address 192.168.10.254 255.255.255.0
interface GigabitEthernet0/0.20
 encapsulation dot1Q 20
 ip address 192.168.20.254 255.255.255.0
end
write memory

Asignar IPs a las PCs

Haz clic en cada PC → Desktop → IP Configuration:

  • PC0: IP 192.168.10.1 · Máscara 255.255.255.0 · Gateway 192.168.10.254
  • PC1: IP 192.168.10.2 · Máscara 255.255.255.0 · Gateway 192.168.10.254
  • PC2: IP 192.168.20.1 · Máscara 255.255.255.0 · Gateway 192.168.20.254
  • PC3: IP 192.168.20.2 · Máscara 255.255.255.0 · Gateway 192.168.20.254

Verificar la conectividad

Desde la PC0 → Desktop → Command Prompt:

# Ping a PC en VLAN 10 (misma VLAN – debe funcionar sin router)
C:\> ping 192.168.10.2
Reply from 192.168.10.2: bytes=32 time<1ms TTL=128

# Ping al gateway de VLAN 10 (subinterfaz del router)
C:\> ping 192.168.10.254
Reply from 192.168.10.254: bytes=32 time=1ms TTL=255

# Ping a PC en VLAN 20 (cruzar VLANs – Inter-VLAN Routing)
C:\> ping 192.168.20.1
Reply from 192.168.20.1: bytes=32 time=2ms TTL=127

¡ÉXITO! El tráfico cruzó de VLAN 10 a VLAN 20 a través del router.

Troubleshooting – Si el ping falla

  • Verifica que interface GigabitEthernet0/0 esté en no shutdown
  • Verifica que la subinterfaz tenga encapsulation dot1Q [VLAN] antes de la IP
  • Confirma que el puerto del switch esté en modo trunk con show interfaces trunk
  • Confirma que las VLANs están permitidas en el trunk: show interfaces Gi0/1 trunk
  • Verifica que el default gateway de la PC sea la IP de la subinterfaz correcta

📋 Resumen Ejecutivo

🎯 ¿Qué resuelve?

Permite la comunicación entre dispositivos en VLANs (subredes) diferentes, que por diseño están aisladas a nivel de Capa 2.

🔁 Router-on-a-Stick

Una interfaz física → varias subinterfaces lógicas. Cada subinterfaz tiene encapsulation dot1Q [vlan] y su propia IP gateway.

⚡ L3 Switch + SVI

El switch enruta internamente. Se habilita con ip routing. Las SVIs (interface vlan X) actúan como gateways.

⚠ Errores comunes

  • Olvidar no shutdown en la interfaz física
  • Poner IP en la interfaz física en vez de en subinterfaces
  • Olvidar ip routing en switch L3
  • Puerto del switch no configurado como trunk

📌 Checklist de configuración Router-on-a-Stick

  • ☐ Switch: VLANs creadas (vlan 10, vlan 20)
  • ☐ Switch: Puerto hacia router en modo trunk
  • ☐ Switch: Puertos de host en modo access con VLAN asignada
  • ☐ Router: Interfaz física en no shutdown (sin IP)
  • ☐ Router: Subinterfaces con encapsulation dot1Q [vlan-id]
  • ☐ Router: IP asignada en cada subinterfaz
  • ☐ PCs: Default gateway = IP de la subinterfaz de su VLAN

📖 Para el examen CCNA 200-301

  • Saber identificar la configuración correcta vs incorrecta de subinterfaces
  • Conocer el orden: primero encapsulation dot1Q, luego ip address
  • Saber cuándo usar Router-on-a-Stick vs Switch L3
  • El número de subinterfaz no tiene que coincidir con el ID de VLAN, pero por convención SÍ
  • La VLAN nativa NO necesita encapsulation dot1Q en la subinterfaz
