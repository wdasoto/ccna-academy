NAPALM — Multi-Vendor

Network Automation and Programmability Abstraction Layer. Una API unificada que funciona idénticamente en Cisco IOS, NX-OS, Juniper JunOS, Arista EOS y más. Un código para gobernarlos a todos: get_facts, get_interfaces, configuración idempotente y validación automática.

🌐 Multi-vendor 🎯 CCNA Domain 6.1 ⏱ ~90 min lectura 📊 Abstracción unificada ✅ Config idempotente

¿Por qué NAPALM?

Netmiko te permite hablar con dispositivos, pero cada uno habla un dialecto diferente. NAPALM (Network Automation and Programmability Abstraction Layer) es el traductor universal: le pides "dame los facts" o "aplica esta config" y él se encarga de traducir a los comandos específicos de Cisco, Juniper, Arista o Arista.

Desarrollado por David Barroso y el equipo de Spotify, es la librería preferida cuando tienes redes heterogéneas y necesitas consistencia absoluta.

🎯

Abstracción Total

Mismo código Python para IOS, NXOS, JunOS, EOS. Cambias solo el driver, el resto es idéntico.

🔄

Configuración Idempotente

Aplica cambios solo si son necesarios. Valida antes de commitear. Rollback automático si falla.

📋

Datos Estructurados

Todo devuelve diccionarios Python listos para usar. No más parsear strings de "show" manualmente.

🔒

Validación Integrada

Compara configuración candidata vs running antes de aplicar. Evita errores humanos en producción.

Código Multi-Vendor Real

multivendor_template.py
from napalm import get_network_driver

def obtener_info_router(vendor, ip, user, pwd):
    """
    Funciona idénticamente para Cisco, Juniper o Arista.
    Cambia solo el driver, el código permanece igual.
    """
    driver = get_network_driver(vendor)
    device = driver(hostname=ip, username=user, password=pwd)
    
    try:
        device.open()
        facts = device.get_facts()
        interfaces = device.get_interfaces()
        
        return {
            "vendor": vendor,
            "hostname": facts["hostname"],
            "model": facts["model"],
            "interface_count": len(interfaces),
            "uptime": facts["uptime"],
        }
    finally:
        device.close()

# Uso con Cisco IOS
cisco_data = obtener_info_router("ios", "192.168.1.1", "admin", "pass")

# Uso con Juniper (misma función)
juniper_data = obtener_info_router("junos", "10.0.0.1", "admin", "pass")

# Uso con Arista EOS (misma función)
arista_data = obtener_info_router("eos", "172.16.0.1", "admin", "pass")

get_config() — Backups

backup_napalm.py
from napalm import get_network_driver
import datetime

driver = get_network_driver("ios")
device = driver("192.168.1.1", "admin", "pass")

device.open()
config = device.get_config()  # Devuelve running y startup

# Guardar running-config
timestamp = datetime.datetime.now().strftime("%Y%m%d_%H%M")
with open(f"backup_{device.get_facts()['hostname']}_{timestamp}.cfg", "w") as f:
    f.write(config["running"])

device.close()

load_merge_candidate() — Cambios parciales

El método más usado: aplica configuración adicional sin borrar lo existente. Ideal para añadir VLANs, interfaces, o rutas.

merge_config.py
from napalm import get_network_driver

driver = get_network_driver("ios")
device = driver("10.0.0.1", "admin", "pass")

config_nueva = """
vlan 50
 name VLAN_PRUEBA_NAPALM
interface GigabitEthernet0/1
 description CONFIGURADA_VIA_NAPALM
"""

device.open()

# Cargar cambios candidatos
device.load_merge_candidate(config=config_nueva)
# O desde archivo: device.load_merge_candidate(filename="cambios.cfg")

# REVISAR SIEMPRE ANTES DE COMMITEAR
diff = device.compare_config()
print("Cambios a aplicar:\n", diff)

if "vlan 50" in diff:
    print("✅ Vlan 50 será creada")
    device.commit_config()
    print("🎉 Cambios aplicados")
else:
    print("⚠️ No hay cambios detectados o ya existe")
    device.discard_config()

device.close()
⚠️ Idempotencia

NAPALM es idempotente: si intentas crear VLAN 50 y ya existe, no falla ni duplica. Pero siempre usa compare_config() para auditar antes de commitear.

compare_config() y confirmación

Flujo de trabajo profesional: cargar → comparar → validar → commitear (o rollback).

workflow_seguro.py
def aplicar_cambios_seguro(device, config, confirmar=False):
    """
    Workflow seguro con validación manual o automática.
    """
    device.open()
    
    try:
        # 1. Cargar candidato
        device.load_merge_candidate(config=config)
        
        # 2. Obtener diff
        diff = device.compare_config()
        
        if not diff:
            print("ℹ️ No hay cambios para aplicar")
            return "no_changes"
        
        print("Diff detectado:\n", diff)
        
        # 3. Validación manual o automática
        if confirmar or input("¿Aplicar cambios? (y/n): ") == "y":
            device.commit_config()
            print("✅ Configuración aplicada")
            
            # 4. Verificación post-commit (opcional)
            new_facts = device.get_facts()
            print(f"Uptime tras cambio: {new_facts['uptime']}")
            return "success"
        else:
            device.discard_config()
            print("❌ Cambios descartados")
            return "discarded"
            
    except Exception as e:
        device.discard_config()
        print(f"🚨 Error: {e}. Config descartada.")
        return "error"
        
    finally:
        device.close()

🚀 Proyecto: Cambio multi-vendor masivo

Script que aplica la misma configuración (NTP, SNMP, banner) a routers Cisco, Juniper y Arista simultáneamente usando la misma función.

proyecto_multivendor.py
from napalm import get_network_driver
import yaml

# Inventario heterogéneo
inventario = [
    {"vendor": "ios", "ip": "192.168.1.1", "user": "admin", "pass": "cisco123"},
       {"vendor": "junos", "ip": "10.0.0.1", "user": "root", "pass": "juniper123"},
    {"vendor": "eos", "ip": "172.16.0.1", "user": "admin", "pass": "arista123"},
]

# Configuración deseada (plantilla Jinja2 o string simple)
plantilla_ntp = """
ntp server {{ ntp_server }}
"""

def configurar_dispositivo(device_info):
    """Aplica config estándar a cualquier vendor"""
    vendor = device_info["vendor"]
    
    # Obtener driver apropiado
    driver = get_network_driver(vendor)
    device = driver(
        hostname=device_info["ip"],
        username=device_info["user"],
        password=device_info["pass"],
        optional_args={"port": 22 if vendor != "eos" else 443}
    )
    
    try:
        device.open()
        print(f"\n🔌 Conectado a {device_info['ip']} ({vendor})")
        
        # Obtener facts antes
        facts = device.get_facts()
        print(f"   Hostname: {facts['hostname']}, Modelo: {facts['model']}")
        
        # Preparar config específica por vendor
        if vendor == "ios":
            config = "ntp server 10.0.0.1\nntp server 10.0.0.2\n"
        elif vendor == "junos":
            config = "set system ntp server 10.0.0.1\nset system ntp server 10.0.0.2\n"
        elif vendor == "eos":
            config = "ntp server 10.0.0.1\nntp server 10.0.0.2\n"
        
        # Aplicar
        device.load_merge_candidate(config=config)
        diff = device.compare_config()
        
        if diff:
            print(f"   Cambios detectados:\n{diff}")
            device.commit_config()
            print("   ✅ Configuración aplicada")
        else:
            print("   ℹ️ Sin cambios (ya estaba configurado)")
            
        return {"ip": device_info["ip"], "status": "success"}
        
    except Exception as e:
        print(f"   ❌ Error: {e}")
        return {"ip": device_info["ip"], "status": "error", "error": str(e)}
    finally:
        device.close()

# Ejecutar en todos los dispositivos
print("🚀 Iniciando configuración multi-vendor...\n")
resultados = [configurar_dispositivo(d) for d in inventario]

# Reporte final
exitos = [r for r in resultados if r["status"] == "success"]
print(f"\n🏁 Completado: {len(exitos)}/{len(inventario)} dispositivos configurados")

Un solo script Python gestionó equipos de 3 vendors diferentes sin duplicar código. La lógica de negocio (qué configurar) está separada de la implementación del vendor gracias a NAPALM.
