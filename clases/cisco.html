M2.5 · COMANDOS CISCO · IOS CLI FUNDAMENTALS

Comandos de Configuración Básica

Domina la Interfaz de Línea de Comandos (CLI) de Cisco IOS desde cero. Modos de operación, comandos esenciales, configuración de interfaces, gestión de archivos de configuración y verificación. La base fundamental para cualquier ingeniero de redes Cisco CCNA.

📋 IOS 15.x / IOS XE 🎯 CCNA 200-301 Exam Topic 1.4, 2.5 ⏱ ~90 min lectura 🧪 Lab: Router + Switch

Los 3 Modos Fundamentales de Cisco IOS

El sistema operativo Cisco IOS (Internetwork Operating System) organiza sus comandos en una jerarquía de modos. Comprender esta jerarquía es esencial — cada modo tiene su propio conjunto de comandos disponibles y el prompt cambia para indicarte dónde estás.

Jerarquía de Modos IOS

1. User EXEC Mode (Modo Usuario)

Es el primer modo al que accedes al conectarte al dispositivo. Tiene privilegios mínimos — solo permite comandos de visualización básica y no permite cambios de configuración.

Prompt y características
! Prompt: Router> o Switch>
Router> (sin #, el > indica modo usuario)

! Comandos disponibles limitados:
Router> show version        ! Sí: información básica del sistema
Router> show interfaces       ! Sí: estado de interfaces
Router> ping 192.168.1.1      ! Sí: prueba de conectividad
Router> enable                ! Sí: entrar a modo privilegiado

! NO disponibles (requieren privilegios):
Router> configure terminal    ! ERROR: comando no encontrado
Router> show running-config   ! ERROR: información restringida

2. Privileged EXEC Mode (Modo Privilegiado)

También llamado enable mode o modo habilitación. Aquí tienes acceso completo a comandos de visualización, diagnóstico y debug. Es el "modo de lectura completa" del dispositivo.

Acceso y comandos clave
! Entrar desde User EXEC:
Router> enable
Router#                        ! Prompt cambia a #

! Si hay contraseña enable configurada, la pedirá:
Router> enable
Password: ********
Router#

! Comandos exclusivos de Privileged EXEC:
Router# show running-config     ! Configuración activa en RAM
Router# show startup-config     ! Configuración de inicio en NVRAM
Router# show ip route           ! Tabla de routing
Router# show ip interface brief ! Resumen de interfaces IP
Router# debug ip rip            ! Debugging de protocolos
Router# configure terminal      ! Entrar a modo configuración

! Salir a User EXEC:
Router# disable
Router>
⚠️ Seguridad del modo enable

El acceso al modo privilegiado debe estar siempre protegido con contraseña. Sin protección, cualquier usuario con acceso físico o remoto puede tomar control total del dispositivo. Usa siempre enable secret en lugar de enable password (la primera está encriptada con MD5, la segunda en texto plano).

3. Global Configuration Mode (Modo Configuración Global)

Desde aquí realizas todos los cambios de configuración del dispositivo. Cada comando que ejecutas afecta el funcionamiento global del router o switch. Desde este modo puedes acceder a submodos específicos.

Entrada y navegación entre modos
! Entrar desde Privileged EXEC:
Router# configure terminal
Router(config)#                ! Prompt indica modo configuración

! Abreviatura común:
Router# conf t                 ! Cisco acepta abreviaturas únicas

! === SUBMODOS DE CONFIGURACIÓN ===

! 1. Interface Configuration Mode:
Router(config)# interface GigabitEthernet0/0
Router(config-if)#             ! Configuración de interfaz específica

Router(config-if)# ip address 192.168.1.1 255.255.255.0
Router(config-if)# no shutdown
Router(config-if)# exit        ! Volver a Global Config
Router(config)#

! 2. Line Configuration Mode (líneas de acceso):
Router(config)# line console 0
Router(config-line)#           ! Configuración de puerto de consola

Router(config-line)# password cisco
Router(config-line)# login
Router(config-line)# exit

! 3. Router Configuration Mode (protocolos de routing):
Router(config)# router rip
Router(config-router)#         ! Configuración de protocolo RIP

Router(config-router)# version 2
Router(config-router)# network 192.168.1.0
Router(config-router)# exit

! Salir completamente al modo privilegiado:
Router(config-router)# end     ! o Ctrl+Z — vuelve directo a #
Router#

Comandos Básicos de Configuración

Estos son los comandos fundamentales que aparecen en tu imagen de referencia, organizados por función y explicados en profundidad con ejemplos prácticos de CCNA.

🚀

enable

Entra al modo privilegiado (EXEC). El primer comando que ejecutas para obtener control total.

Router> enable
Router#
⚙️

configure terminal

Entra al modo de configuración global. Abreviado como conf t.

Router# conf t
Router(config)#
🏷️

hostname

Establece el nombre del dispositivo. Aparece en el prompt y es crucial para identificación.

Router(config)# hostname R1
R1(config)#
🔌

interface

Accede al modo de configuración de interfaz específica (FastEthernet, GigabitEthernet, Serial, etc.).

R1(config)# int g0/0
R1(config-if)#
🌐

ip address

Asigna dirección IP y máscara de subred a una interfaz. Requiere no shutdown para activar.

ip address 192.168.1.1 255.255.255.0
✅

no shutdown

Activa (administratively up) una interfaz. Por defecto, las interfaces de router están apagadas.

R1(config-if)# no shutdown
%LINK-5-CHANGED: Interface... up

Comando enable — Modo de Habilitación

Uso y comportamiento
! Desde User EXEC, elevar privilegios:
Router> enable
Router#

! Si existe enable password/secret configurado:
Router> enable
Password: (no se muestra al escribir)
Router#

! Verificación de seguridad:
Router# show running-config | include enable
enable secret 5 $1$mERr$8j9...
! enable secret 5 = encriptado con MD5 (seguro)
! enable password = texto plano (INSEGURO)
💡 Diferencia crítica: enable password vs enable secret

Si configuras ambos, enable secret tiene prioridad. Nunca uses enable password en producción — se almacena en texto plano en la configuración. enable secret usa hash MD5 irreversible.

Comando configure terminal — Modo Configuración

Variantes y accesos
! Forma completa:
Router# configure terminal

! Abreviaturas aceptadas (única por contexto):
Router# conf t           ! Más común
Router# config t
Router# conf term

! Otras variantes de configure:
Router# configure memory    ! Carga configuración desde NVRAM
Router# configure network   ! Carga desde servidor TFTP/FTP
Router# configure overwrite ! Sobrescribe con archivo externo
Router# configure replace   ! Reemplazo atómico de configuración

Comando hostname — Identidad del Dispositivo

Buenas prácticas de nomenclatura
! Convención recomendada para CCNA y producción:
! [Tipo]-[Ubicación]-[Número] o [Función]-[Site]-[ID]

Router(config)# hostname R1-HQ-CORE      ! Router 1, HQ, Core
R1-HQ-CORE(config)# hostname SW-Floor2-01  ! Switch piso 2, ID 01
SW-Floor2-01(config)# hostname RT-BRANCH-A ! Router sucursal A

! Reglas: máximo 64 caracteres, sin espacios, case-sensitive
! Caracteres válidos: A-Z, a-z, 0-9, . _ -

! El hostname afecta:
! 1. El prompt de comandos
! 2. Los mensajes de syslog (logs del sistema)
! 3. El nombre en CDP/LLDP (descubrimiento de vecinos)
! 4. El nombre de archivo en backups automáticos

Comandos de Interfaz: interface, ip address, no shutdown

Configuración completa de interfaz
! === CONFIGURACIÓN DE INTERFAZ DE ROUTER ===

R1(config)# interface GigabitEthernet0/0    ! Forma completa
R1(config)# interface gig0/0                ! Abreviatura común
R1(config)# int g0/0                        ! Más corta
R1(config-if)#

! Asignar IP y máscara:
R1(config-if)# ip address 192.168.1.1 255.255.255.0
! o notación CIDR (en IOS moderno):
R1(config-if)# ip address 192.168.1.1/24

! Activar interfaz (CRÍTICO):
R1(config-if)# no shutdown
%LINK-5-CHANGED: Interface GigabitEthernet0/0, changed state to administratively up
%LINEPROTO-5-UPDOWN: Line protocol on Interface GigabitEthernet0/0, changed state to up

! Descripción de la interfaz (opcional pero recomendado):
R1(config-if)# description LINK-TO-CORE-SW-PORT-Gi0/1
R1(config-if)# desc WAN-ISP-PRIMARY         ! abreviado

! Salir:
R1(config-if)# exit
R1(config)#

! === DIFERENCIA CRÍTICA: ROUTER vs SWITCH ===
! ROUTER: Interfaces están administratively down por defecto
! SWITCH: Interfaces están up por defecto al conectar cable

! === CONFIGURACIÓN DE INTERFAZ DE SWITCH (Capa 2) ===
SW1(config)# interface FastEthernet0/1
SW1(config-if)# switchport mode access
SW1(config-if)# switchport access vlan 10
SW1(config-if)# description PC-CONTABILIDAD-01
! En switch L2, NO usamos "ip address" en interfaces de acceso

Comandos de gestión de configuración: copy y erase

Guardar, respaldar y eliminar configuración
! === GUARDAR CONFIGURACIÓN (CRÍTICO PARA EXAMEN) ===

! Copiar configuración activa (RAM) a configuración de inicio (NVRAM):
R1# copy running-config startup-config
! Abreviatura universalmente aceptada:
R1# copy run start
! Versión más corta (IOS moderno):
R1# wr                                        ! write memory

Destination filename [startup-config]?     (Enter para confirmar)
Building configuration...
[OK]

! ⚠️ ADVERTENCIA: Si no ejecutas copy run start, TODO se pierde al reiniciar

! === RESPALDAR EN SERVIDOR TFTP ===
R1# copy running-config tftp:
Address or name of remote host []? 192.168.1.100
Destination filename [R1-confg]? R1-backup-2024.cfg
!!
618 bytes copied in 0.052 secs

! === RESTAURAR DESDE TFTP ===
R1# copy tftp: running-config
! o para reemplazar completamente:
R1# copy tftp: startup-config

! === ELIMINAR CONFIGURACIÓN DE INICIO (⚠️ PELIGRO) ===
R1# erase startup-config
Erasing the nvram filesystem will remove all configuration files! Continue? [confirm]y
[OK]
Erase of nvram: complete

! Reiniciar para arrancar en modo setup:
R1# reload
! O forzar factory reset completo:
R1# write erase                               ! = erase startup-config

! === FUSIONAR CONFIGURACIONES (COMANDO PELIGROSO) ===
R1# copy startup-config running-config
! ⚠️ CUIDADO: Fusiona startup con running, NO reemplaza completamente
! Puede dejar comandos obsoletos de sesiones anteriores
⚠️ ZONA DE PELIGRO — Comandos que destruyen configuración
1 erase startup-config — Elimina configuración de inicio. Al reiniciar, el dispositivo arranca en modo setup o vacío.
2 write erase — Alias de erase startup-config. Mismo efecto destructivo.
3 reload sin guardar — Si olvidaste copy run start, perdiste toda la configuración desde el último guardado.
4 configure replace — Reemplazo atómico. Si el archivo fuente está mal, puedes quedar sin acceso remoto.

Comandos adicionales de la imagen: reload, ip default-gateway, show version

Comandos de sistema y verificación
! === RELOAD — Reinicio del dispositivo ===
R1# reload
System will be restarted, continue? [confirm]y

! Programar reload para evitar quedar bloqueado:
R1# reload in 10                             ! Reinicia en 10 minutos
R1# reload at 14:00                          ! Reinicia a las 2 PM
R1# reload cancel                            ! Cancelar reload programado

! === IP DEFAULT-GATEWAY (solo en switches) ===
! Los switches L2 necesitan gateway para management remoto:
SW1(config)# ip default-gateway 192.168.1.1
! Los routers NO usan este comando — usan rutas estáticas o dinámicas

! === SHOW VERSION — Información del sistema ===

Configuración completa de interfaz de Router

Ejemplo práctico: Router con LAN y WAN
! === INTERFAZ LAN (GigabitEthernet) ===
R1(config)# interface GigabitEthernet0/0
R1(config-if)# description LAN-INTERNAL-VLAN10
R1(config-if)# ip address 192.168.10.1 255.255.255.0
R1(config-if)# ip address 192.168.10.1 255.255.255.0 secondary  ! IP secundaria
R1(config-if)# no shutdown
R1(config-if)# duplex auto
R1(config-if)# speed auto
! o forzar valores:
R1(config-if)# duplex full
R1(config-if)# speed 1000

! === INTERFAZ WAN (Serial con HDLC/PPP) ===
R1(config)# interface Serial0/0/0
R1(config-if)# description WAN-LINK-TO-ISP
R1(config-if)# ip address 203.0.113.2 255.255.255.252
R1(config-if)# no shutdown
R1(config-if)# clock rate 64000        ! Solo en lado DCE
R1(config-if)# bandwidth 1544          ! Métrica para routing

! === INTERFAZ LOOPBACK (Virtual) ===
R1(config)# interface Loopback0
R1(config-if)# description ROUTER-ID-OSPF
R1(config-if)# ip address 1.1.1.1 255.255.255.255  ! /32 host route
! Loopback siempre está up/up, nunca falla físicamente

Configuración de interfaces de Switch

Switch L2: Access, Trunk, y SVI
! === PUERTO DE ACCESO (Access Port) ===
SW1(config)# interface FastEthernet0/1
SW1(config-if)# description PC-VENTAS-01
SW1(config-if)# switchport mode access
SW1(config-if)# switchport access vlan 10
SW1(config-if)# spanning-tree portfast      ! Para puertos de host
SW1(config-if)# no shutdown

! === PUERTO TRUNK (802.1Q) ===
SW1(config)# interface GigabitEthernet0/1
SW1(config-if)# description TRUNK-TO-CORE-SW
SW1(config-if)# switchport mode trunk
SW1(config-if)# switchport trunk allowed vlan 10,20,30,99
SW1(config-if)# switchport trunk native vlan 99  ! VLAN nativa diferente a 1
SW1(config-if)# no shutdown

! === SVI — Switch Virtual Interface (Capa 3 en switch) ===
SW1(config)# interface Vlan10
SW1(config-if)# description MANAGEMENT-VLAN10
SW1(config-if)# ip address 192.168.10.2 255.255.255.0
SW1(config-if)# no shutdown
! La SVI está up solo si hay al menos un puerto en VLAN 10 up/up

! Verificar estado de VLANs:
SW1# show ip interface brief
SW1# show vlan brief
💡 Diferencia clave: Router vs Switch en interfaces

Router: Las interfaces están administratively down por defecto. Debes ejecutar no shutdown explícitamente. Las interfaces son Capa 3 por defecto (rutables).

Switch: Las interfaces están up por defecto al conectar cable. Son Capa 2 por defecto (switching). Para IP en switch, usas SVI (VlanX) o configuras el switch como L3 con ip routing.

Gestión de Archivos de Configuración

Comprender la arquitectura de almacenamiento de configuración en Cisco es fundamental para el examen CCNA y para la administración en producción.

Arquitectura de Memoria Cisco

Comandos de gestión de configuración en profundidad

Flujo completo de gestión de configuración
! === FLUJO NORMAL DE TRABAJO ===

! 1. Realizar cambios de configuración:
R1(config)# hostname CORE-R1
CORE-R1(config)# interface g0/0
CORE-R1(config-if)# ip address 10.1.1.1 255.255.255.0
CORE-R1(config-if)# no shutdown

! 2. Verificar cambios en configuración activa:
CORE-R1# show running-config interface g0/0

! 3. Guardar cambios permanentemente:
CORE-R1# copy running-config startup-config
! o:
CORE-R1# write memory
! o en IOS moderno:
CORE-R1# wr

! === COMANDOS AVANZADOS DE COPIA ===

! Copiar configuración a servidor TFTP (backup):
R1# copy running-config tftp:
Address or name of remote host []? 192.168.1.100
Destination filename [R1-confg]? R1-backup-Jan15.cfg
!!
1256 bytes copied in 0.048 secs

! Restaurar desde TFTP:
R1# copy tftp: running-config
! o reemplazar startup completamente:
R1# copy tftp: startup-config

! Copiar a USB (en routers con puerto USB):
R1# copy running-config usbflash0:backup.cfg

! === COMANDOS DE ELIMINACIÓN ===

! Eliminar startup-config (factory reset al reiniciar):
R1# erase startup-config
Erasing the nvram filesystem will remove all configuration files! Continue? [confirm]y
[OK]

! Eliminar VLANs en switch (archivo separado):
SW1# delete flash:vlan.dat
! ⚠️ erase startup-config NO borra VLANs en switches

! Formatear flash (EXTREMO — elimina IOS):
R1# format flash:

! === CONFIGURACIÓN DE RECUPERACIÓN ===

! Configurar registro de inicio (boot):
R1(config)# config-register 0x2102    ! Normal: carga startup-config
R1(config)# config-register 0x2142    ! Recuperación: ignora startup

! Ver registro actual:
R1# show version | include register
Configuration register is 0x2102
⚠️ ERROR COMÚN EN EXAMEN CCNA

Situación: Configuras un router, todo funciona, pero al reiniciar pierdes todo.

Causa: Olvidaste ejecutar copy running-config startup-config (o wr).

Solución: Siempre, SIEMPRE, guarda después de cambios exitosos. En Packet Tracer y exámenes, esto te quita puntos.

Comandos de Verificación (Show Commands)

Los comandos show son tu herramienta principal para troubleshooting y verificación. Dominarlos es esencial para el examen CCNA y para el trabajo real.

📊

show running-config

Muestra la configuración activa en RAM. Incluye TODOS los comandos aplicados actualmente.

show run
show run | section router
show run | begin interface
💾

show startup-config

Muestra la configuración guardada en NVRAM. Lo que se cargará al reiniciar.

show start
🌐

show ip interface brief

Resumen rápido de todas las interfaces: IP, estado administrativo y protocolo.

show ip int brief
sh ip int b
🗺️

show ip route

Tabla de routing completa. Muestra cómo llegar a cada red conocida.

show ip route
show ip route 192.168.1.0
🔍

show interfaces

Estado detallado de interfaces: estadísticas, errores, duplex, velocidad.

show int g0/0
show int | include error
⚙️

show version

Información del sistema: modelo, IOS, memoria, interfaces, config-register.

show ver | i register

Show running-config — Análisis profundo

Variantes y filtros útiles
! === FORMAS BÁSICAS ===
R1# show running-config              ! Configuración completa
R1# show run                          ! Abreviatura

! === FILTROS CON PIPE (|) — CRÍTICO PARA EXAMEN ===

! begin: muestra desde la primera coincidencia:
R1# show run | begin interface GigabitEthernet0/0
! Muestra toda la config empezando desde Gi0/0

! include: solo líneas que contienen el texto:
R1# show run | include ip address
 ip address 192.168.1.1 255.255.255.0
 ip address 10.0.0.1 255.255.255.252

! exclude: elimina líneas con el texto:
R1# show run | exclude !               ! Quita líneas de comentario

! section: muestra sección completa:
R1# show run | section router ospf      ! Todo el bloque OSPF
R1# show run | section vty              ! Líneas VTY (telnet/ssh)

! === COMPARAR RUNNING VS STARTUP ===
R1# show archive config differences nvram:startup-config system:running-config
! Muestra línea por línea qué cambió desde el último guardado

Show ip interface brief — El comando más usado

Interpretación de estados
R1# show ip interface brief
Interface              IP-Address      OK? Method Status                Protocol
GigabitEthernet0/0     192.168.1.1     YES manual up                    up    
GigabitEthernet0/1     10.0.0.1        YES manual up                    up    
GigabitEthernet0/2     unassigned      YES unset  administratively down down  
Serial0/0/0            203.0.113.2     YES manual up                    down  
Loopback0              1.1.1.1         YES manual up                    up    

! === INTERPRETACIÓN DE ESTADOS ===

! Status (estado administrativo):
! - administratively down = ejecutaste "shutdown" o nunca "no shutdown"
! - up = interfaz administrativamente activada

! Protocol (estado de capa 2):
! - up = hay conectividad física (carrier detect) y keepalives
! - down = no hay cable, cable desconectado, o vecino apagado

! === COMBINACIONES COMUNES ===

! up / up      = Todo bien, interfaz funcionando
! up / down    = Cable conectado pero problema de capa 2 (mismatch, keepalives)
! down / down  = No hay cable o interfaz shutdown
! administratively down / down = Interfaz explícitamente apagada

! Ejemplo Serial0/0/0 arriba: up/down = cable conectado pero vecino no responde

O     192.168.3.0/24 [110/3] via 10.0.0.2, 00:15:23, GigabitEthernet0/1

! === ANÁLISIS DE ENTRADAS ===

! S* 0.0.0.0/0 = Ruta por defecto (static, candidate default)
! [1/0] = [AD/métrica] → AD=1 (static), métrica=0

! C = Connected: red directamente conectada a interfaz local
! L = Local: dirección IP específica asignada a la interfaz (/32)

! O = OSPF: ruta aprendida por OSPF
! [110/2] = [AD=110/métrica=2] → costo OSPF de 2
! via 10.0.0.2 = next-hop (IP del router vecino)
! 00:15:23 = tiempo desde que se aprendió la ruta
! GigabitEthernet0/1 = interfaz de salida

💡 Técnica de troubleshooting: Divide y vencerás

Usa show commands en secuencia lógica:

  1. show ip interface brief — ¿Las interfaces están up/up?
  2. show ip route — ¿Conozco la red destino?
  3. show running-config | section [protocolo] — ¿La configuración es correcta?
  4. ping y traceroute — ¿Llego al destino?
  5. debug — ¿Qué está pasando en tiempo real? (con precaución)

Seguridad Básica de Dispositivos Cisco

Proteger el acceso a los dispositivos de red es fundamental. Estos comandos son requeridos en cualquier implementación de producción y frecuentemente evaluados en el examen CCNA.

Jerarquía de contraseñas en Cisco IOS

Configuración completa de seguridad de acceso
! === 1. CONTRASEÑA DE CONSOLA (acceso físico) ===
R1(config)# line console 0
R1(config-line)# password ConsoleSec123
R1(config-line)# login                    ! Exige login con contraseña
R1(config-line)# logging synchronous      ! Evita que mensajes corten comandos
R1(config-line)# exec-timeout 5 30        ! Timeout: 5 min 30 seg de inactividad

! === 2. CONTRASEÑA DE VTY (acceso remoto Telnet/SSH) ===
R1(config)# line vty 0 4                  ! 5 sesiones simultáneas (0-4)
R1(config-line)# password VtySec456
R1(config-line)# login
R1(config-line)# transport input ssh      ! Solo SSH, no Telnet (seguro)
! o para permitir ambos (menos seguro):
R1(config-line)# transport input telnet ssh

! === 3. ENABLE PASSWORD vs ENABLE SECRET ===

! FORMA INSEGURA (texto plano):
R1(config)# enable password AdminPass789

! FORMA SEGURA (hash MD5 irreversible):
R1(config)# enable secret SuperSecret999
! Si ambos existen, enable secret tiene prioridad

! === 4. ENCRIPTAR TODAS LAS CONTRASEÑAS ===
R1(config)# service password-encryption
! Aplica cifrado Vigenère débil a passwords en texto plano
! NO encripta enable secret (ya está hasheado)
! Muestra "7" en show run: password 7 0822455D0A16

! Verificación:
R1# show running-config | include password
enable secret 5 $1$mERr$8j9...
password 7 0822455D0A16       ! Encriptado con service password-encryption

Configuración de SSH (Secure Shell)

SSH v2 completo — Reemplazo seguro de Telnet
! === REQUISITOS PARA SSH ===
! 1. Hostname diferente de "Router" o "Switch"
! 2. Domain name configurado
! 3. Generar claves RSA (512-4096 bits)
! 4. Crear usuario local con privilegio
! 5. Habilitar SSH en líneas VTY

! Paso 1: Hostname (ya debería estar configurado)
R1(config)# hostname R1-HQ

! Paso 2: IP domain name (obligatorio para generar certificado)
R1(config)# ip domain-name cisco.local

! Paso 3: Generar claves RSA (mínimo 768, recomendado 2048)
R1(config)# crypto key generate rsa
The name for the keys will be: R1-HQ.cisco.local
Choose the size of the key modulus in the range of 360 to 4096 for your
  General Purpose Keys. Choosing a key modulus greater than 512 may take
  a few minutes.

How many bits in the modulus [512]: 2048
% Generating 2048 bit RSA keys, keys will be non-exportable...
[OK] (elapsed time was 3 seconds)

! Paso 4: Crear usuario local con privilegio máximo
R1(config)# username admin privilege 15 secret AdminSSH2024!
! privilege 15 = nivel de acceso enable automático al loguear
! secret = almacenado con hash (no password)

! Paso 5: Configurar VTY para SSH
R1(config)# line vty 0 4
R1(config-line)# login local              ! Usar base de datos local de usuarios
R1(config-line)# transport input ssh      ! Solo SSH, bloquear Telnet
R1(config-line)# exec-timeout 3 0         ! 3 minutos timeout

! Opcional: Versión SSH 2 (más segura que v1)
R1(config)# ip ssh version 2

! Verificación:
R1# show ip ssh
SSH Enabled - version 2.0
Authentication timeout: 120 secs; Authentication retries: 3

R1# show crypto key mypubkey rsa

Usage: General Purpose Key
Key is not exportable.
Key Data:
  30820122 300D0609 2A864886 F70D0101 01050003 82010F00 3082010A...

Seguridad de puertos de switch (Port Security)

Protección contra MAC flooding y unauthorized access
! === PORT SECURITY EN INTERFAZ DE ACCESO ===

SW1(config)# interface FastEthernet0/1
SW1(config-if)# switchport mode access
SW1(config-if)# switchport access vlan 10

! Habilitar port security:
SW1(config-if)# switchport port-security
! Por defecto: máximo 1 MAC, violation = shutdown

! Configurar MAC específica (estática):
SW1(config-if)# switchport port-security mac-address 0000.0c12.3456
! o aprender la primera MAC conectada (sticky):
SW1(config-if)# switchport port-security mac-address sticky

! Permitir hasta 3 MACs (útil para IP phones con PC detrás):
SW1(config-if)# switchport port-security maximum 3

! Configurar acción ante violación:
SW1(config-if)# switchport port-security violation ?
  protect   ! Descarta paquetes desconocidos, no alerta
  restrict  ! Descarta + envía SNMP trap + log
  shutdown  ! Pone interfaz en error-disabled (default)

SW1(config-if)# switchport port-security violation restrict

! Verificación:
SW1# show port-security
SW1# show port-security interface fa0/1

🧪 Laboratorio Práctico: Configuración Inicial Completa

Este laboratorio integra todos los comandos aprendidos en una configuración realista de router y switch desde cero. Ideal para practicar en Packet Tracer o GNS3.

Topología del Laboratorio

Objetivos del laboratorio

  • Configurar router R1 con hostname, interfaces, SSH y OSPF
  • Configurar switch SW-ACCESS con VLANs, trunk y port security
  • Establecer conectividad completa PC1 → Internet
  • Verificar con show commands y guardar configuración
Parte 1
Configuración inicial de Router R1
! === CONFIGURACIÓN BÁSICA ===
Router> enable
Router# configure terminal
Router(config)# hostname R1-HQ

! Seguridad básica
R1-HQ(config)# enable secret CiscoLab2024!
R1-HQ(config)# service password-encryption
R1-HQ(config)# banner motd #
ACCESO AUTORIZADO UNICAMENTE
#

! === CONFIGURACIÓN DE INTERFACES ===

! Interfaz LAN (hacia switch)
R1-HQ(config)# interface GigabitEthernet0/0
R1-HQ(config-if)# description LAN-TO-SW-ACCESS
R1-HQ(config-if)# ip address 192.168.10.1 255.255.255.0
R1-HQ(config-if)# no shutdown
R1-HQ(config-if)# exit

! Interfaz WAN (hacia ISP/Internet)
R1-HQ(config)# interface GigabitEthernet0/1
R1-HQ(config-if)# description WAN-TO-ISP
R1-HQ(config-if)# ip address 10.0.0.1 255.255.255.252
R1-HQ(config-if)# no shutdown
R1-HQ(config-if)# exit

! === CONFIGURACIÓN SSH ===
R1-HQ(config)# ip domain-name lab.local
R1-HQ(config)# crypto key generate rsa modulus 2048
R1-HQ(config)# username admin privilege 15 secret Admin1234!
R1-HQ(config)# ip ssh version 2

R1-HQ(config)# line vty 0 4
R1-HQ(config-line)# login local
R1-HQ(config-line)# transport input ssh
R1-HQ(config-line)# exit

! === CONFIGURACIÓN DE ROUTING (OSPF) ===
R1-HQ(config)# router ospf 1
R1-HQ(config-router)# router-id 1.1.1.1
R1-HQ(config-router)# network 192.168.10.0 0.0.0.255 area 0
R1-HQ(config-router)# network 10.0.0.0 0.0.0.3 area 0
R1-HQ(config-router)# passive-interface GigabitEthernet0/0
R1-HQ(config-router)# exit

! Ruta por defecto hacia Internet
R1-HQ(config)# ip route 0.0.0.0 0.0.0.0 10.0.0.2
R1-HQ(config)# router ospf 1
R1-HQ(config-router)# default-information originate
R1-HQ(config-router)# end

! === VERIFICACIÓN Y GUARDADO ===
R1-HQ# show ip interface brief
R1-HQ# show ip route
R1-HQ# show ip ospf neighbor
R1-HQ# copy running-config startup-config
