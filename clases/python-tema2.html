Netmiko — SSH Automation

La librería que convirtió SSH en una API de redes. Netmiko abstrae toda la complejidad de los prompts de IOS, el timing de comandos y las diferencias entre plataformas — y te da métodos Python simples para conectar, leer y configurar cualquier dispositivo de red. Usado en producción por Netflix, Google, Cisco TAC y miles de equipos NOC en todo el mundo.

🔌 Netmiko 4.x+ 🎯 CCNA Domain 6.1 & 6.2 ⏱ ~95 min lectura 💻 10 laboratorios código 🚀 Proyecto producción real 🎯 12 preguntas Quiz

¿Por qué Netmiko existe — y por qué lo necesitas?

Antes de Netmiko, conectarse a un dispositivo Cisco con Python requería usar Paramiko directamente — un cliente SSH de bajo nivel que no entiende nada de CLIs de red. El ingeniero tenía que gestionar manualmente los prompts del router (Router>, Router#, Router(config)#), los tiempos de espera de cada comando, las diferencias entre plataformas, los banners de login y docenas de casos especiales. Medio día de código para enviar un solo show ip route.

Netmiko (creada por Kirk Byers en 2014) resuelve exactamente eso. Es una capa de abstracción construida sobre Paramiko que entiende los dispositivos de red como ciudadanos de primera clase. Maneja los prompts automáticamente, sabe cuándo un comando terminó de ejecutarse, normaliza las diferencias entre IOS, NXOS, XR, EOS y 30+ plataformas más, y expone una API limpia de Python.

🎯

Prompt Intelligence

Detecta automáticamente los prompts de cada modo IOS: user exec, privileged, config, config-if. Nunca necesitas manejar esto manualmente.

Anatomía de ConnectHandler — El corazón de Netmiko

ConnectHandler es la clase central de Netmiko. Al instanciarla, establece la sesión SSH, maneja el login, navega al modo correcto (user exec o privileged) y deja la sesión lista para recibir comandos. Cada parámetro tiene un propósito específico y entenderlos todos es clave para escribir scripts robustos.

connecthandler_completo.py — Todos los parámetros explicados
from netmiko import ConnectHandler

# ── PARÁMETROS FUNDAMENTALES ─────────────────────────────────
dispositivo = {
    # OBLIGATORIOS
    "device_type" : "cisco_ios",      # Plataforma — determina el comportamiento
    "host"        : "192.168.1.1",    # IP o FQDN del dispositivo
    "username"    : "netops",         # Usuario SSH
    "password"    : "Cisco123",       # Contraseña SSH

    # OPCIONALES — AUTENTICACIÓN
    "secret"      : "EnablePass",     # Enable password (si el usuario no es priv 15)
    "key_file"    : "~/.ssh/id_rsa", # Autenticación por llave SSH (más seguro)
    "passphrase"  : "miPassphrase",  # Passphrase de la llave privada

    # OPCIONALES — CONEXIÓN
    "port"        : 22,              # Puerto SSH (default: 22)
    "timeout"     : 60,              # Timeout de TCP handshake en segundos
    "auth_timeout": 30,              # Timeout de autenticación SSH
    "banner_timeout": 15,            # Tiempo para recibir el banner SSH
    "conn_timeout": 10,              # Timeout de establecimiento de conexión

    # OPCIONALES — COMPORTAMIENTO
    "session_log" : "session_R1.log", # Guarda TODA la sesión en un archivo
    "fast_cli"    : True,             # Desactiva delays extra — más rápido
    "global_delay_factor": 1,        # Multiplicador global de delays (default: 1)

    # OPCIONALES — SEGURIDAD
    "use_keys"    : False,            # Forzar autenticación por password
    "allow_agent" : False,            # No usar SSH agent del sistema
}

# ── FORMA 1: Context Manager (RECOMENDADA) ────────────────────
# Cierra la conexión automáticamente al salir del bloque 'with'
# aunque haya una excepción — igual que 'with open()' para archivos
with ConnectHandler(**dispositivo) as conn:
    output = conn.send_command("show ip interface brief")
    print(output)
# conn.disconnect() ejecutado automáticamente aquí

# ── FORMA 2: Manual (cuando necesitas más control) ────────────
conn = ConnectHandler(**dispositivo)
try:
    conn.enable()   # Entrar al modo privilegiado
    output = conn.send_command("show version")
    print(output)
finally:
    conn.disconnect()   # Siempre desconectar en un bloque finally

# ── FORMA 3: Usando **unpacking con update ─────────────────────
# Útil cuando el diccionario base es reutilizable
base = {"device_type":"cisco_ios","username":"netops","password":"Cisco123"}
r1 = {**base, "host":"10.0.0.1"}
r2 = {**base, "host":"10.0.0.2"}
r3 = {**base, "host":"10.0.0.3"}

for router in [r1, r2, r3]:
    with ConnectHandler(**router) as conn:
        print(conn.send_command("show ip route summary"))

Flujo interno de una conexión Netmiko

Entender qué ocurre internamente ayuda a diagnosticar problemas de conexión. El proceso tiene 6 fases bien definidas:

1
TCP Handshake — Puerto 22
Python abre una conexión TCP al host en el puerto especificado. Si falla aquí: el dispositivo está apagado, el puerto está bloqueado por ACL, o el IP es incorrecto. Timeout controlado por conn_timeout.
2
Negociación SSH — Intercambio de llaves y algoritmos
Paramiko negocia algoritmos de cifrado, MAC e intercambio de llaves con el dispositivo. Si falla aquí: versiones incompatibles de SSH o algoritmos. Timeout controlado por banner_timeout.
3
Autenticación — Usuario y contraseña (o llave)
SSH verifica las credenciales contra la configuración del dispositivo (username secret en IOS). Si falla: NetmikoAuthenticationException. Timeout: auth_timeout.
4
Detección del prompt — Identificar modo actual
Netmiko envía caracteres y analiza la respuesta para detectar el prompt (Router> o Router#). Identifica si está en user exec o privileged mode. Controla el comportamiento futuro.
5
Enable mode (opcional) — Elevación de privilegios
Si el usuario no es privilege 15, Netmiko ejecuta automáticamente enable usando el campo secret del diccionario. Si conn.enable() falla: secret incorrecto o usuario ya está en privilege 15.
6
Sesión lista — ConnectHandler disponible para uso
La sesión está completamente establecida en modo privilegiado. Los métodos send_command() y send_config_set() están disponibles. La sesión se mantiene hasta llamar disconnect().
⚠️ El parámetro session_log — Tu mejor aliado en debugging

Agrega "session_log": "debug_R1.log" al diccionario de conexión y Netmiko guardará absolutamente todo lo que se envía y recibe en la sesión SSH, incluyendo prompts, comandos y outputs. Es el primer paso cuando una conexión no funciona como esperas. Recuerda eliminarlo o proteger el archivo en producción ya que contiene contraseñas en texto plano.

send_command() — Todos los parámetros y casos de uso

send_command() es el método más usado en Netmiko. Envía un comando de lectura (show, ping, traceroute) al dispositivo y espera hasta que el output esté completo, detectando el prompt de retorno. Tiene muchos más parámetros de los que la documentación básica muestra.

send_command_completo.py — Todos los parámetros
from netmiko import ConnectHandler

router = {"device_type":"cisco_ios","host":"10.0.0.1","username":"netops","password":"Cisco123"}

with ConnectHandler(**router) as conn:

    # ── BÁSICO ───────────────────────────────────────────────────
    output = conn.send_command("show ip interface brief")

    # ── STRIP_PROMPT y STRIP_COMMAND ─────────────────────────────
    # Por defecto ambos son True — el output devuelto NO incluye el
    # prompt del router ni el comando enviado. Solo el output limpio.
    output_raw = conn.send_command(
        "show version",
        strip_prompt=False,   # Incluir "Router#" al final
        strip_command=False,  # Incluir "show version" al inicio
    )

    # ── DELAY_FACTOR — Para comandos lentos ──────────────────────
    # Multiplica el tiempo de espera base (default 1 = ~2 segundos)
    # show tech-support puede tardar 2-3 minutos en dispositivos grandes
    output_tech = conn.send_command(
        "show tech-support",
        delay_factor=8,  # Espera 8x el tiempo base = ~16 segundos
        max_loops=1000,  # Número máximo de iteraciones de espera
    )

    # ── EXPECT_STRING — Cuando el output NO termina en prompt ─────
    # Ping extendido, confirmaciones, wizards interactivos
    output_ping = conn.send_command(
        "ping 8.8.8.8 repeat 100",
        expect_string=r"Success rate",  # Esperar esta regex en el output
    )

    # ── READ_TIMEOUT — Timeout absoluto por comando ───────────────
    output_large = conn.send_command(
        "show running-config",
        read_timeout=120,  # Máximo 120 segundos esperando
    )

    # ── CMD_VERIFY — Verificar que el comando apareció en output ──
    output = conn.send_command(
        "show ip route",
        cmd_verify=True,  # Default True — detecta si el comando no fue aceptado
    )

    # ── USE_TEXTFSM — Output estructurado directamente ───────────
    # Devuelve lista de diccionarios en lugar de string plano
    interfaces = conn.send_command(
        "show ip interface brief",
        use_textfsm=True,  # Requiere textfsm + ntc-templates instalados
    )
    # interfaces = [{'intf':'Gi0/0','ipaddr':'192.168.1.1','status':'up',...}]
    for intf in interfaces:
        print(f"{intf['intf']:25} {intf['ipaddr']:16} {intf['status']}")

Extraer datos estructurados de show commands

extraer_datos.py — Parseo manual de outputs comunes
from netmiko import ConnectHandler

router = {"device_type":"cisco_ios","host":"10.0.0.1","username":"netops","password":"Cisco123"}

with ConnectHandler(**router) as conn:

    # ── Extraer versión IOS ───────────────────────────────────────
    version_output = conn.send_command("show version")
    version_ios = "desconocida"
    for linea in version_output.splitlines():
        if "Cisco IOS Software" in linea:
            version_ios = linea.split(",")[1].strip()
            break
    print(f"IOS Version: {version_ios}")

    # ── Extraer estado de interfaces ─────────────────────────────
    intf_output = conn.send_command("show ip interface brief")
    interfaces_up   = []
    interfaces_down = []

    for linea in intf_output.splitlines():
        if "Interface" in linea or not linea.strip():
            continue
        campos = linea.split()
        if len(campos) >= 6:
            intf, ip, _, _, status, prot = campos[:6]
            if status == "up" and prot == "up":
                interfaces_up.append({"intf":intf, "ip":ip})
            else:
                interfaces_down.append({"intf":intf, "status":status})

    print(f"\n✅ Interfaces UP ({len(interfaces_up)}):")
    for i in interfaces_up:
        print(f"   {i['intf']:25} {i['ip']}")

    print(f"\n❌ Interfaces DOWN ({len(interfaces_down)}):")
    for i in interfaces_down:
        print(f"   {i['intf']:25} {i['status']}")

    # ── Extraer vecinos OSPF ─────────────────────────────────────
    ospf_output = conn.send_command("show ip ospf neighbor")
    vecinos_full = []

    for linea in ospf_output.splitlines():
        if "FULL" in linea:
            campos = linea.split()
            vecinos_full.append({
                "neighbor_id": campos[0],
                "state"      : campos[2],
                "address"    : campos[5],
                "interface"  : campos[6],
            })

    print(f"\n⭕ Vecinos OSPF FULL: {len(vecinos_full)}")
    for v in vecinos_full:
        print(f"   {v['neighbor_id']:16} via {v['address']:16} on {v['interface']}")

    # ── Obtener uptime del router ─────────────────────────────────
    version_out = conn.send_command("show version | include uptime")
    print(f"\n⏱  {version_out.strip()}")
🔥 Pro tip: Usar | include y | section en los comandos

Netmiko envía el comando tal cual al dispositivo. Aprovecha los pipes de IOS para filtrar directamente en el dispositivo y reducir el texto a parsear en Python: "show run | include hostname", "show ip route | include O", "show interface status | include connected". Esto acelera el script y reduce la carga en la sesión SSH.

send_config_set() — Configurar dispositivos en producción

send_config_set() es el método de escritura de Netmiko. Acepta una lista de comandos, entra automáticamente a config terminal, los ejecuta en orden y regresa al modo privilegiado. Es la base de toda automatización de configuración masiva.

send_config_completo.py — Todos los escenarios de uso
from netmiko import ConnectHandler

router = {"device_type":"cisco_ios","host":"10.0.0.1","username":"netops","password":"Cisco123"}

with ConnectHandler(**router) as conn:

    # ── BÁSICO: lista de strings ─────────────────────────────────
    config_cmds = [
        "interface GigabitEthernet0/1",
        "description ENLACE-WAN-ISP",
        "ip address 200.0.0.1 255.255.255.252",
        "no shutdown",
    ]
    output = conn.send_config_set(config_cmds)
    print(output)

    # ── GUARDAR — Método nativo Netmiko ──────────────────────────
    # Usa "write memory" en IOS, "copy run start" en otros — automático
    conn.save_config()

    # ── CONFIGURAR VLANS DINÁMICAMENTE ───────────────────────────
    vlans = {10: "DATOS", 20: "VOZ", 30: "SERVIDORES", 99: "MGMT"}
    cmds_vlans = []
    for vid, nombre in vlans.items():
        cmds_vlans.append(f"vlan {vid}")
        cmds_vlans.append(f" name {nombre}")
    conn.send_config_set(cmds_vlans)

    # ── DESDE ARCHIVO .cfg ───────────────────────────────────────
    conn.send_config_from_file("config_ospf.cfg")

    # ── VERIFICAR ESTADO ANTES DE GUARDAR ────────────────────────
    print(f"Prompt actual: {conn.find_prompt()}")
    print(f"En enable mode: {conn.check_enable_mode()}")
    print(f"En config mode: {conn.check_config_mode()}")

    # ── DELAY_FACTOR para comandos que tardan (ej: crypto key) ───
    conn.send_config_set(
        ["crypto key generate rsa modulus 2048"],
        cmd_verify=False,    # crypto key no hace echo del comando
        delay_factor=4,
    )

Funciones de configuración para producción

config_produccion.py — Casos reales empresariales
from netmiko import ConnectHandler
import datetime, os

# ── FUNCIÓN 1: Habilitar SSH en routers ──────────────────────
def habilitar_ssh(dispositivo, hostname, dominio="empresa.local"):
    config = [
        f"hostname {hostname}",
        f"ip domain-name {dominio}",
        "crypto key generate rsa modulus 2048",
        "ip ssh version 2",
        "ip ssh time-out 60",
        "ip ssh authentication-retries 2",
        "line vty 0 4",
        " transport input ssh",
        " login local",
        " exec-timeout 10 0",
    ]
    with ConnectHandler(**dispositivo) as conn:
        conn.send_config_set(config, cmd_verify=False, delay_factor=4)
        conn.save_config()
    print(f"✅ SSH listo en {dispositivo['host']}")

# ── FUNCIÓN 2: Aplicar hardening CIS-benchmark ───────────────
def aplicar_hardening(dispositivo):
    config = [
        "no ip http server",
        "no ip http secure-server",
        "no cdp run",
        "no ip source-route",
        "service password-encryption",
        "login block-for 300 attempts 5 within 60",
        "ip access-list standard MGMT-ACL",
        " permit 10.0.0.0 0.0.0.255",
        " deny any log",
        "line vty 0 4",
        " access-class MGMT-ACL in",
    ]
    with ConnectHandler(**dispositivo) as conn:
        conn.send_config_set(config)
        conn.save_config()
    print(f"✅ Hardening aplicado en {dispositivo['host']}")

# ── FUNCIÓN 3: Backup running-config con fecha ───────────────
def backup_config(dispositivo, directorio="backups"):
    os.makedirs(directorio, exist_ok=True)
    fecha = datetime.datetime.now().strftime("%Y%m%d_%H%M%S")

    with ConnectHandler(**dispositivo) as conn:
        hostname = conn.send_command("show run | include ^hostname").split()[1]
        config   = conn.send_command("show running-config", read_timeout=120)

    fname = f"{directorio}/{hostname}_{fecha}.cfg"
    with open(fname, "w") as fout:
        fout.write(config)
    print(f"✅ Backup: {fname} ({len(config):,} chars)")
    return fname

# ── FUNCIÓN 4: Configurar OSPF con verificación ──────────────
def configurar_ospf(dispositivo, proc, redes):
    """redes = [("10.0.0.0","0.0.0.3","0"), ("192.168.0.0","0.0.0.255","0")]"""
    config = [f"router ospf {proc}"]
    for red, wc, area in redes:
        config.append(f" network {red} {wc} area {area}")
    config += ["passive-interface default", "no passive-interface GigabitEthernet0/0"]

    with ConnectHandler(**dispositivo) as conn:
        conn.send_config_set(config)
        conn.save_config()
        verify = conn.send_command("show ip protocols | include OSPF")
        estado = "✅ OSPF activo" if "OSPF" in verify else "❌ OSPF NO detectado"
        print(f"{estado} en {dispositivo['host']}")
⚠️ Regla de oro: Verificar siempre antes de save_config()

Nunca llames save_config() sin antes verificar que la configuración aplicó correctamente. Usa send_command() con el show correspondiente después de cada bloque de configuración. Si algo falló, no guardas y puedes hacer rollback con send_config_set(["reload in 1"]) en casos extremos de laboratorio.

TextFSM — De texto plano a datos estructurados

TextFSM es un motor de parsing de texto desarrollado por Google y ahora mantenido por la comunidad de red. En lugar de escribir código de parsing manual para cada show command, TextFSM usa plantillas predefinidas que describen el patrón del output. ntc-templates es el repositorio con +500 plantillas listas para Cisco, Arista, Juniper y más.

La integración en Netmiko es tan profunda que un solo parámetro activa todo: use_textfsm=True en send_command(). El resultado es una lista de diccionarios con las claves extraídas automáticamente.

textfsm_basico.py — Parsing estructurado automático
from netmiko import ConnectHandler

router = {"device_type":"cisco_ios","host":"10.0.0.1","username":"netops","password":"Cisco123"}

with ConnectHandler(**router) as conn:

    # ── Sin TextFSM: string plano ─────────────────────────────
    raw = conn.send_command("show ip interface brief")
    print(type(raw))   # <class str>
    print(raw[:200])

    # ── Con TextFSM: lista de diccionarios ────────────────────
    parsed = conn.send_command("show ip interface brief", use_textfsm=True)
    print(type(parsed))   # <class list>
    print(parsed[0])
    # {
    #   "intf"   : "GigabitEthernet0/0",
    #   "ipaddr" : "192.168.1.1",
    #   "status" : "up",
    #   "proto"  : "up"
    # }

    # Iterar y filtrar directo — sin escribir ni una linea de parsing
    interfaces_up = [i for i in parsed if i["status"] == "up" and i["proto"] == "up"]
    interfaces_dn = [i for i in parsed if i["status"] != "up"]

    print(f"UP: {len(interfaces_up)} | DOWN: {len(interfaces_dn)}")
    for intf in interfaces_up:
        print(f"  {intf['intf']:<25} {intf['ipaddr']:<16} UP/UP")

    # ── show ip ospf neighbor estructurado ────────────────────
    ospf = conn.send_command("show ip ospf neighbor", use_textfsm=True)
    full_neighbors = [n for n in ospf if "FULL" in n.get("state","")]
    print(f"Vecinos FULL: {len(full_neighbors)}")

    # ── show ip route estructurado ────────────────────────────
    rutas = conn.send_command("show ip route", use_textfsm=True)
    rutas_ospf = [r for r in rutas if r.get("protocol") == "O"]
    print(f"Rutas OSPF: {len(rutas_ospf)}")
    for r in rutas_ospf:
        print(f"  O {r['network']}/{r['mask']} via {r['nexthop']} [{r['distance']}/{r['metric']}]")

Manejo profesional de excepciones Netmiko

En un entorno de producción, los scripts no corren en un lab controlado — los dispositivos tienen latencia variable, credenciales rotadas, interfaces caídas o actualizaciones en progreso. Un script sin manejo de errores fallará silenciosamente o tirará excepciones no manejadas que detienen todo el proceso. El manejo robusto de errores es la diferencia entre un script de laboratorio y uno de producción.

excepciones_netmiko.py — Árbol completo de excepciones
from netmiko import ConnectHandler
from netmiko.exceptions import (
    NetmikoAuthenticationException,   # Usuario/password incorrecto
    NetmikoTimeoutException,          # Timeout de conexión TCP/SSH
    ReadTimeout,                      # Timeout esperando output de comando
    NetmikoBaseException,             # Clase base — captura todas las anteriores
)
from paramiko.ssh_exception import (
    AuthenticationException,          # Paramiko: auth fallida
    NoValidConnectionsError,          # No se puede establecer TCP
    SSHException,                     # Error genérico de SSH
)
import socket, time

def conectar_robusto(dispositivo, reintentos=3, espera=5):
    """
    Conecta con reintentos automáticos y manejo granular de errores.
    Retorna (conn, error_msg) — conn es None si falló.
    """
    ip = dispositivo["host"]

    for intento in range(1, reintentos + 1):
        print(f"[{intento}/{reintentos}] Conectando a {ip}...")
        try:
            conn = ConnectHandler(**dispositivo)
            print(f"✅ Conectado a {ip}")
            return conn, None

        except NetmikoAuthenticationException:
            # Credenciales incorrectas — NO tiene sentido reintentar
            msg = f"❌ AUTH FALLIDA en {ip} — verificar usuario/password"
            print(msg)
            return None, msg  # Retorno inmediato sin reintentos

        except NetmikoTimeoutException:
            msg = f"⏱  TIMEOUT en {ip} — dispositivo no responde"
            print(msg)
            # Sí tiene sentido reintentar — puede ser congestión transitoria
            if intento < reintentos:
                print(f"   Esperando {espera}s antes del próximo intento...")
                time.sleep(espera)

        except (NoValidConnectionsError, socket.timeout) as e:
            msg = f"🔌 SIN CONECTIVIDAD TCP a {ip}:{dispositivo.get('port',22)}"
            print(msg)
            if intento < reintentos:
                time.sleep(espera * 2)  # Espera doble para problemas de red

        except SSHException as e:
            msg = f"🔐 ERROR SSH en {ip}: {e}"
            print(msg)
            if intento < reintentos:
                time.sleep(espera)

        except Exception as e:
            msg = f"💥 ERROR INESPERADO en {ip}: {type(e).__name__}: {e}"
            print(msg)
            if intento < reintentos:
                time.sleep(espera)

    return None, f"Fallaron los {reintentos} reintentos para {ip}"


# ── TIMEOUT en send_command ───────────────────────────────────
def comando_robusto(conn, comando, timeout=60):
    try:
        return conn.send_command(comando, read_timeout=timeout)
    except ReadTimeout:
        print(f"⏱  Timeout ejecutando: {comando}")
        return None
    except Exception as e:
        print(f"Error en comando '{comando}': {e}")
        return None


# ── PATRÓN COMPLETO CON REPORTE ──────────────────────────────
dispositivos = [
    {"device_type":"cisco_ios","host":"10.0.0.1","username":"netops","password":"Cisco123"},
    {"device_type":"cisco_ios","host":"10.0.0.2","username":"admin","password":"WrongPass"},
    {"device_type":"cisco_ios","host":"10.0.0.99","username":"netops","password":"Cisco123"},
]

reporte = {"exitosos":[], "fallidos":[]}

for disp in dispositivos:
    conn, error = conectar_robusto(disp)
    if conn:
        try:
            hostname = conn.send_command("show run | include ^hostname").split()[-1]
            reporte["exitosos"].append(hostname)
        finally:
            conn.disconnect()  # SIEMPRE desconectar en finally
    else:
        reporte["fallidos"].append({"ip": disp["host"], "error": error})

print(f"\nExitosos ({len(reporte['exitosos'])}): {reporte['exitosos']}")
print(f"Fallidos ({len(reporte['fallidos'])}):")
for f in reporte["fallidos"]:
    print(f"  {f['ip']}: {f['error']}")
🔥 Pro Pattern: Logging estructurado para auditoría

En producción usa el módulo logging de Python en lugar de print(). Configura un log file con timestamp, nivel de severidad y formato estructurado. Cada conexión, comando y error queda registrado para auditoría. Esto es crítico en entornos con cumplimiento SOX, ISO 27001 o PCI-DSS donde se requiere trazabilidad de cambios.

ThreadPoolExecutor — Conectar a 100 routers en paralelo

Secuencialmente, conectar a 100 dispositivos tomando 3 segundos cada uno = 5 minutos. Con threading, los 100 van en paralelo = 3-5 segundos. El módulo concurrent.futures de Python estándar es la forma más limpia y segura de hacer esto, y Netmiko es thread-safe.

concurrencia_netmiko.py — 100 dispositivos en paralelo
from netmiko import ConnectHandler
from concurrent.futures import ThreadPoolExecutor, as_completed
from netmiko.exceptions import NetmikoBaseException
import time

# ── FUNCIÓN QUE CORRE EN CADA THREAD ────────────────────────
def verificar_dispositivo(dispositivo):
    ip = dispositivo["host"]
    try:
        with ConnectHandler(**dispositivo) as conn:
            hostname = conn.send_command("show run | include ^hostname").split()[-1]
            version  = conn.send_command("show version | include Version")
            ospf_nb  = conn.send_command("show ip ospf neighbor | count FULL")
            return {
                "ip"      : ip,
                "hostname": hostname,
                "version" : version.strip(),
                "ospf_nb" : ospf_nb.strip(),
                "estado"  : "✅ OK",
            }
    except NetmikoBaseException as e:
        return {"ip": ip, "estado": f"❌ {type(e).__name__}: {str(e)[:60]}"}
    except Exception as e:
        return {"ip": ip, "estado": f"💥 {type(e).__name__}: {str(e)[:60]}"}


# ── INVENTARIO DE DISPOSITIVOS ───────────────────────────────
base_creds = {"device_type":"cisco_ios","username":"netops","password":"Cisco123"}
hosts = [f"10.0.{i//256}.{i%256}" for i in range(1, 11)]  # 10 dispositivos de prueba
dispositivos = [{**base_creds, "host": h} for h in hosts]

# ── EJECUCIÓN EN PARALELO ────────────────────────────────────
MAX_THREADS = 10   # Ajustar según los recursos del sistema y la red
resultados  = []
inicio = time.time()

print(f"Verificando {len(dispositivos)} dispositivos con {MAX_THREADS} threads...")

with ThreadPoolExecutor(max_workers=MAX_THREADS) as executor:
    # Submeter todas las tareas
    futures = {
        executor.submit(verificar_dispositivo, disp): disp["host"]
        for disp in dispositivos
    }

    # Procesar resultados a medida que terminan (no en orden de envio)
    for future in as_completed(futures):
        resultado = future.result()
        resultados.append(resultado)
        print(f"  {resultado['ip']:<16} → {resultado['estado']}")

duracion = time.time() - inicio
print(f"\n⏱  Completado en {duracion:.1f}s para {len(dispositivos)} dispositivos")

# ── REPORTE FINAL ───────────────────────────────────────────
exitosos  = [r for r in resultados if "✅" in r["estado"]]
fallidos  = [r for r in resultados if "✅" not in r["estado"]]

print(f"\n{'='*60}")
print(f"  RESUMEN: {len(exitosos)} OK | {len(fallidos)} FALLIDOS")
print(f"{'='*60}")

if exitosos:
    print(f"\n✅ EXITOSOS:")
    print(f"{'IP':<16} {'Hostname':<15} {'OSPF Neighbors'}")
    print(f"{'-'*50}")
    for r in sorted(exitosos, key=lambda x: x["ip"]):
        print(f"{r['ip']:<16} {r.get('hostname','?'):<15} {r.get('ospf_nb','?')}")

if fallidos:
    print(f"\n❌ FALLIDOS:")
    for r in fallidos:
        print(f"  {r['ip']:<16} {r['estado']}")
⚠️ Calibrar MAX_THREADS según el entorno
  • Lab/desarrollo: 5-10 threads son suficientes
  • Producción WAN: 10-20 threads, considerando latencia y ancho de banda
  • Producción LAN grande: 50-100 threads si los dispositivos tienen buena conectividad
  • Demasiados threads simultaneos pueden causar que los dispositivos rechacen conexiones SSH (VTY limit en IOS)
  • Configura en IOS: line vty 0 15 para permitir hasta 16 sesiones simultáneas

Credenciales seguras — Nunca hardcodear passwords

El error más común en scripts de red en producción: las credenciales en texto plano dentro del código fuente. Si ese script llega a un repositorio git público (incluso por accidente), las credenciales de acceso a toda tu infraestructura quedan expuestas. Hay al menos 4 formas mejores de manejar esto.

import os, getpass

# ── MÉTODO 1: Variables de entorno (recomendado para CI/CD) ──

Este es el tipo de script que un ingeniero de redes entregaría en su primer día como Network Automation Engineer.

noc_system.py — Sistema NOC completo (130 líneas)
#!/usr/bin/env python3
"""
noc_system.py — Sistema de monitoreo y backup para NOC
Uso: python noc_system.py --hosts 10.0.0.1,10.0.0.2 --action check
"""
import os, json, datetime, time, logging, argparse
from concurrent.futures import ThreadPoolExecutor, as_completed
from netmiko import ConnectHandler
from netmiko.exceptions import NetmikoAuthenticationException, NetmikoTimeoutException

# ══════════════════════════════════════════════════════════════
# CONFIGURACIÓN GLOBAL
# ══════════════════════════════════════════════════════════════
MAX_THREADS = 10
BASE_DIR    = os.path.dirname(os.path.abspath(__file__))
BACKUP_DIR  = os.path.join(BASE_DIR, "backups")
LOG_FILE    = os.path.join(BASE_DIR, "noc.log")

logging.basicConfig(
    level=logging.INFO,
    format="%(asctime)s [%(levelname)s] %(message)s",
    handlers=[logging.FileHandler(LOG_FILE), logging.StreamHandler()]
)
log = logging.getLogger(__name__)

# ══════════════════════════════════════════════════════════════
# CREDENCIALES (desde variables de entorno)
# ══════════════════════════════════════════════════════════════
CREDS = {
    "device_type": "cisco_ios",
    "username"   : os.environ.get("NET_USER", "netops"),
    "password"   : os.environ.get("NET_PASS", "Cisco123"),
    "secret"     : os.environ.get("NET_SECRET", ""),
    "timeout"    : 30,
    "fast_cli"   : True,
}

# ══════════════════════════════════════════════════════════════
# FUNCIÓN DE AUDITORÍA COMPLETA POR DISPOSITIVO
# ══════════════════════════════════════════════════════════════
def auditar_dispositivo(ip):
    """Conecta, audita y hace backup de un dispositivo. Thread-safe."""
    resultado = {"ip": ip, "timestamp": datetime.datetime.now().isoformat()}
    disp = {**CREDS, "host": ip}

    try:
        log.info(f"Conectando a {ip}...")
        with ConnectHandler(**disp) as conn:

            # — HOSTNAME ──────────────────────────────────────────
            hn_raw = conn.send_command("show run | include ^hostname")
            resultado["hostname"] = hn_raw.split()[-1] if hn_raw else ip

            # — VERSIÓN Y HARDWARE (TextFSM) ─────────────────────
            ver = conn.send_command("show version", use_textfsm=True)
            if isinstance(ver, list) and ver:
                v = ver[0]
                resultado["ios_version"] = v.get("version","?")
                resultado["modelo"]      = v.get("hardware","?")
                resultado["serial"]      = v.get("serial","?")
                resultado["uptime"]      = v.get("uptime","?")

            # — INTERFACES (TextFSM) ─────────────────────────────
            intfs = conn.send_command("show ip interface brief", use_textfsm=True)
            if isinstance(intfs, list):
                resultado["intfs_up"]  = [i["intf"] for i in intfs if i["status"]=="up"]
                resultado["intfs_dn"]  = [i["intf"] for i in intfs if i["status"]!="up"]

            # — OSPF NEIGHBORS ────────────────────────────────────
            ospf = conn.send_command("show ip ospf neighbor", use_textfsm=True)
            if isinstance(ospf, list):
                resultado["ospf_full"] = [n["neighbor_id"] for n in ospf if "FULL" in n.get("state","")]

            # — CPU y MEMORIA ─────────────────────────────────────
            cpu_raw = conn.send_command("show processes cpu | include CPU utilization")
            if cpu_raw:
                resultado["cpu"] = cpu_raw.split(":")[-1].strip()[:30]

            # — BACKUP running-config ─────────────────────────────
            os.makedirs(BACKUP_DIR, exist_ok=True)
            fecha  = datetime.datetime.now().strftime("%Y%m%d_%H%M%S")
            cfg    = conn.send_command("show running-config", read_timeout=120)
            bkpath = os.path.join(BACKUP_DIR, f"{resultado['hostname']}_{fecha}.cfg")
            with open(bkpath, "w") as fout:
                fout.write(cfg)
            resultado["backup"] = bkpath
            log.info(f"Backup guardado: {bkpath}")

            resultado["estado"] = "OK"

    except NetmikoAuthenticationException:
        resultado["estado"] = "AUTH_ERROR"
        log.error(f"Auth fallida en {ip}")
    except NetmikoTimeoutException:
        resultado["estado"] = "TIMEOUT"
        log.warning(f"Timeout en {ip}")
    except Exception as e:
        resultado["estado"] = f"ERROR: {type(e).__name__}"
        log.error(f"Error en {ip}: {e}")

    return resultado


# ══════════════════════════════════════════════════════════════
# MAIN — ORQUESTADOR PARALELO
# ══════════════════════════════════════════════════════════════
def main(hosts):
    inicio = time.time()
    log.info(f"Iniciando auditoría de {len(hosts)} dispositivos")

    resultados = []
    with ThreadPoolExecutor(max_workers=MAX_THREADS) as executor:
        futures = {executor.submit(auditar_dispositivo, ip): ip for ip in hosts}
        for future in as_completed(futures):
            r = future.result()
            resultados.append(r)
            estado = "✅" if r["estado"] == "OK" else "❌"
            print(f"  {estado} {r['ip']:<16} {r.get('hostname','?'):<15} {r['estado']}")

    # Guardar reporte JSON
    reporte_path = f"reporte_{datetime.datetime.now().strftime('%Y%m%d_%H%M%S')}.json"
    with open(reporte_path, "w") as fout:
        json.dump(resultados, fout, indent=2)

    duracion = time.time() - inicio
    ok_count = len([r for r in resultados if r["estado"]=="OK"])
    print(f"\n⏱  {duracion:.1f}s | ✅ {ok_count} OK | ❌ {len(resultados)-ok_count} FAIL")
    print(f"📄 Reporte: {reporte_path}")
    log.info(f"Auditoría completa: {ok_count}/{len(resultados)} exitosos en {duracion:.1f}s")


if __name__ == "__main__":
    parser = argparse.ArgumentParser(description="NOC System con Netmiko")
    parser.add_argument("--hosts", required=True, help="IPs separadas por coma")
    args = parser.parse_args()
    hosts_list = [h.strip() for h in args.hosts.split(",")]
    main(hosts_list)
