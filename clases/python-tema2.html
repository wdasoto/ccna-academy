Netmiko — SSH Automation

La librería Python más usada para conectarse a dispositivos Cisco por SSH. Envía comandos show, configura interfaces y automatiza tareas de red con solo unas líneas de código. Compatible con IOS, IOS-XE, NXOS, XR y 30+ plataformas.

🔌 SSH + IOS 🎯 CCNA Domain 6.1 ⏱ ~90 min lectura 💻 Labs prácticos 🔐 Seguridad SSH

¿Por qué Netmiko?

Netmiko es la librería estándar de oro para automatización de red en Python. Desarrollada por Kirk Byers, simplifica el uso de Paramiko (SSH puro) agregando inteligencia específica para equipos de red: maneja automáticamente los prompts de enable mode, soporta diferentes plataformas (Cisco, Arista, Juniper, etc.) y hace que enviar comandos sea tan fácil como llamar una función.

Si en el Tema 1 aprendiste a manipular datos de red, en este tema aprenderás a tocar equipos reales vía SSH. Es el puente entre Python y tu infraestructura física.

🔐

SSH Seguro

Conexiones SSHv2 con manejo automático de claves, banners y timeouts. Más robusto que Telnet y scriptable.

📱

Multi-Plataforma

Soporta Cisco IOS, IOS-XE, NXOS, XR, Arista EOS, Juniper JunOS, HP Comware y 30+ sistemas más.

⚡

Simple pero Poderoso

Una sola función send_command() ejecuta show commands y devuelve el output listo para parsear.

🏭

Producción Real

Usado por Netflix, Google, Cisco TAC y miles de empresas para gestionar miles de dispositivos diariamente.

💡 Instalación rápida

Instala Netmiko desde pip (requiere Python 3.7+):

pip install netmiko

También instala dependencias automáticamente: Paramiko (SSH), SCP, y TextFSM para parseo de templates.

Arquitectura de Netmiko

conceptos_clave.py

from netmiko import ConnectHandler

# Diccionario con credenciales y parámetros de conexión
cisco3 = {

    "password": "Cisco123",        # SSH password
    "secret": "EnablePass",        # Enable password (opcional)
    "port": 22,                     # Puerto SSH (default 22)
    "timeout": 60,                 # Timeout de conexión
}

# 2. CONTEXT MANAGER (recomendado) — Cierra automático

    output = conn.send_command("show ip int brief")
    print(output)

Tu primera conexión SSH

Vamos a conectar a un router Cisco real (o Packet Tracer con SSH habilitado) y extraer información básica. Este es el patrón fundamental que usarás en el 90% de tus scripts.

primer_ssh.py — Conexión básica y show version
from netmiko import ConnectHandler

# Definir parámetros del dispositivo
router = {
    "device_type": "cisco_ios",
    "host": "192.168.1.1",
    "username": "admin",
    "password": "Cisco123",
    "secret": "enablepass",  # Si requiere enable
}

# Establecer conexión usando context manager (with)
with ConnectHandler(**router) as conn:
    # Si el dispositivo requiere modo enable (privileged)
    conn.enable()  # Ejecuta "enable" usando el secret
    
    # Enviar comando show
    output = conn.send_command("show version")
    
    # Parsear información básica
    if "Cisco IOS Software" in output:
        print("✅ Conexión exitosa — Dispositivo Cisco identificado")
    
    # Guardar output en archivo
    with open("backup_version.txt", "w") as f:
        f.write(output)

print("🔒 Sesión cerrada automáticamente por el context manager")
⚠️ Requisitos previos en el router

El dispositivo debe tener configurado:

  • Hostname e IP reachable desde tu PC
  • SSH habilitado: ip ssh version 2, crypto key generate rsa
  • Usuario local con privilegio: username admin privilege 15 secret Cisco123
  • VTY lines: line vty 0 4 → transport input ssh → login local

send_command() — Obtener información

El método más usado en Netmiko. Ejecuta comandos show, espera el prompt automáticamente y devuelve el output como string listo para procesar.

show_commands.py — Parseo de información de red
from netmiko import ConnectHandler

cisco_router = {
    "device_type": "cisco_ios",
    "host": "10.0.0.1",
    "username": "admin",
    "password": "Cisco123",
}

with ConnectHandler(**cisco_router) as net_connect:
    # --- Comando básico ---
    interfaces = net_connect.send_command("show ip interface brief")
    print(interfaces)
    
    # --- Uso de strip_prompt para limpieza ---
    hostname = net_connect.send_command(
        "show run | i hostname", 
        strip_prompt=True, 
        strip_command=True
    )
    # strip_prompt=True elimina "Router#>" del final
    # strip_command=True elimina el comando del inicio del output
    
    # --- Parámetros avanzados ---
    # expect_string: cambia el prompt esperado (útil para submenús)
    output = net_connect.send_command(
        "show running-config",
        expect_string="Router#",  # Espera este prompt específico
        delay_factor=2               # Espera 2x el tiempo default
    )
    
    # --- Verificar estado específico ---
    if "up" in interfaces and "down" not in interfaces.split("\n")[1]:
        print("✅ Interfaz principal está UP")

Automatización de verificaciones

network_audit.py — Script de auditoría de red
def auditar_router(ip, usuario, password):
    """Retorna diccionario con datos del router"""
    device = {
        "device_type": "cisco_ios",
        "host": ip,
        "username": usuario,
        "password": password,
    }
    
    with ConnectHandler(**device) as conn:
        conn.enable()
        
        # Recolectar datos
        version = conn.send_command("show version")
        interfaces = conn.send_command("show ip interface brief")
        routes = conn.send_command("show ip route")
        
        # Parsear versión IOS
        ios_version = "Unknown"
        for line in version.split("\n"):
            if "Version" in line and "Software" in line:
                ios_version = line.split("Version")[1].split()[0]
                break
        
        # Contar interfaces UP
        up_count = interfaces.count("up                     up")
        
        return {
            "ip": ip,
            "ios_version": ios_version,
            "interfaces_up": up_count,
            "has_routes": "C" in routes,  # Tiene rutas conectadas
        }

# Usar la función
resultado = auditar_router("192.168.1.1", "admin", "Cisco123")
print(f"Versión IOS: {resultado['ios_version']}")
print(f"Interfaces activas: {resultado['interfaces_up']}")

send_config_set() — Cambios de configuración

Envía comandos de configuración al dispositivo. Netmiko maneja automáticamente el cambio al modo configuration (configure terminal) y el retorno al modo privileged.

configuracion_basica.py — Configurar interfaces y VLANs
from netmiko import ConnectHandler

cisco_switch = {
    "device_type": "cisco_ios",
    "host": "10.0.0.10",
    "username": "admin",
    "password": "Cisco123",
}

comandos_vlan = [
    "vlan 10",
    " name VLAN_DATOS",
    "vlan 20",
    " name VLAN_VOZ",
    "vlan 30",
    " name VLAN_MGMT",
]

comandos_interfaz = [
    "interface GigabitEthernet0/1",
    " description CONEXION-SERVIDOR",
    " switchport mode access",
    " switchport access vlan 10",
    " spanning-tree portfast",
    " no shutdown",
]

with ConnectHandler(**cisco_switch) as net_connect:
    # Enviar configuración de VLANs
    output_vlan = net_connect.send_config_set(comandos_vlan)
    print("VLANs configuradas:")
    print(output_vlan)
    
    # Enviar configuración de interfaz
    output_int = net_connect.send_config_set(comandos_interfaz)
    
    # Verificar cambios
    verificacion = net_connect.send_command("show vlan brief")
    print(verificacion)
💡 Modo exit_config_mode

Por defecto, Netmiko ejecuta end o exit automáticamente después de send_config_set() para salir del modo configuración. Si necesitas permanecer en config mode (raro), usa exit_config_mode=False.

Excepciones y troubleshooting

En redes reales, los dispositivos fallan, las credenciales expiran y los timeouts ocurren. Netmiko proporciona excepciones específicas para manejar estos casos elegantemente.

manejo_errores.py — Script robusto para producción
from netmiko import ConnectHandler, NetmikoTimeoutException, NetmikoAuthenticationException

def conectar_seguro(device_params):
    """Intenta conectar con manejo completo de errores"""
    try:
        connection = ConnectHandler(**device_params)
        return connection, "success"
        
    except NetmikoTimeoutException:
        # Dispositivo no responde (apagado, firewall, ruta incorrecta)
        print(f"⏱️  Timeout: {device_params['host']} no responde (puerto 22 cerrado o host down)")
        return None, "timeout"
        
    except NetmikoAuthenticationException:
        # Credenciales inválidas
        print(f"🔐 Auth Failed: Usuario o password incorrecto para {device_params['host']}")
        return None, "authentication"
        
    except Exception as e:
        # Otros errores (DNS, red, etc.)
        print(f"❌ Error general en {device_params['host']}: {str(e)}")
        return None, "error"

# Uso en producción
router = {"device_type": "cisco_ios", "host": "192.168.1.1", "username": "admin", "password": "Cisco123"}

conn, status = conectar_seguro(router)
if status == "success":
    with conn:
        output = conn.send_command("show uptime")
        print(output)
else:
    # Lógica de fallback: email, log, siguiente router, etc.
    print("Saltando dispositivo...")

Delay Factor y Timing

Dispositivos lentos o comandos largos (como show tech-support) pueden necesitar más tiempo.

timing_avanzado.py
with ConnectHandler(**device) as conn:
    # Comando largo que tarda en ejecutarse
    output = conn.send_command(
        "show tech-support",
        delay_factor=4,           # Espera 4x el tiempo normal
        max_loops=500,            # Más iteraciones de búsqueda de prompt
        strip_prompt=False,       # Mantener el prompt en el output
        strip_command=False      # Mantener el comando en el output

Automatización a escala

El verdadero poder de Netmiko se revela cuando configuras cientos de dispositivos simultáneamente usando loops y funciones.

bulk_config.py — Cambio masivo de contraseñas
from netmiko import ConnectHandler
import getpass  # Oculta password en input

# Inventario de dispositivos
sucursales = [
    {"device_type": "cisco_ios", "host": "10.1.1.1", "username": "admin"},
    {"device_type": "cisco_ios", "host": "10.1.2.1", "username": "admin"},
    {"device_type": "cisco_ios", "host": "10.1.3.1", "username": "admin"},
]

password = getpass.getpass("Ingrese password SSH: ")
nuevo_banner = "Acceso solo autorizado - Area Tecnica"

for sw in sucursales:
    sw["password"] = password  # Agregar password al dict
    
    try:
        with ConnectHandler(**sw) as conn:
            print(f"\n🔧 Procesando {sw['host']}...")
            
            # Cambiar banner
            commands = [
                f"banner motd #{nuevo_banner}#",
                "service password-encryption",
            ]
            output = conn.send_config_set(commands)
            
            # Verificar cambio
            verify = conn.send_command("show banner motd")
            
            if nuevo_banner in verify:
                print(f"✅ {sw['host']}: Banner actualizado")
                conn.save_config()
            else:
                print(f"⚠️  {sw['host']}: No se confirmó el cambio")
                
    except Exception as e:
        print(f"❌ {sw['host']}: Error - {e}")

print("\n🏁 Proceso completado")
💡 Consejo para producción

Para cientos de dispositivos, considera usar multiprocessing o threading para conectar en paralelo. Netmiko soporta esto nativamente, pero recuerda que cada thread mantiene una sesión SSH separada.

🚀 Proyecto: Backup masivo de configuraciones

Script completo que conecta a múltiples dispositivos, extrae la configuración running-config, la guarda con timestamp y genera un reporte de estado.

backup_masivo.py — Solución de producción
import datetime
from netmiko import ConnectHandler
from netmiko.exceptions import NetmikoTimeoutException, NetmikoAuthenticationException
import os

# Crear directorio de backups si no existe
backup_dir = "backups"
os.makedirs(backup_dir, exist_ok=True)

# Inventario
devices = [
    {"device_type": "cisco_ios", "host": "192.168.1.1", "username": "admin", "password": "Cisco123", "name": "R1-CORE"},
    {"device_type": "cisco_ios", "host": "192.168.1.2", "username": "admin", "password": "Cisco123", "name": "R2-DIST"},
    {"device_type": "cisco_nxos", "host": "10.0.0.10", "username": "admin", "password": "Cisco123", "name": "NX9K-1"},
]

timestamp = datetime.datetime.now().strftime("%Y%m%d_%H%M")

print(f"🚀 Iniciando backup masivo - {timestamp}\n")

for dev in devices:
    try:
        print(f"📡 Conectando a {dev['name']} ({dev['host']})...")
        
        with ConnectHandler(**dev) as conn:
            # Obtener configuración
            if dev["device_type"] == "cisco_nxos":
                config = conn.send_command("show running-config")
            else:
                config = conn.send_command("show running-config")
            
            # Guardar archivo
            filename = f"{backup_dir}/{dev['name']}_{timestamp}.cfg"
            with open(filename, "w") as f:
                f.write(config)
            
            print(f"   ✅ Backup guardado: {filename}")
            
    except (NetmikoTimeoutException, NetmikoAuthenticationException) as e:
        print(f"   ❌ Error en {dev['name']}: {str(e)}")

print(f"\n🏁 Proceso completado. Archivos guardados en ./{backup_dir}/")
📝 Mejoras sugeridas para producción
  • Agregar logging a archivo en lugar de print
  • Usar getpass para ocultar contraseñas
  • Leer inventario desde CSV o YAML
  • Agregar verificación de checksum (MD5) de la configuración
  • Integrar con Git para versionado de configuraciones
