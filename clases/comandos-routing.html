Comandos de Routing

Domina la configuración, administración y troubleshooting de protocolos de enrutamiento en Cisco IOS. Desde rutas estáticas hasta OSPF, EIGRP, BGP y NAT/PAT. La base fundamental para conectividad inter-redes en routers Cisco CCNA.

📋 IOS 15.x / IOS XE 🎯 CCNA 200-301 Exam Topic 3.1, 3.2, 3.3, 3.4, 3.5 ⏱ ~120 min lectura 🧪 Lab: 3 Routers + 2 Switches + 1 ISP

Fundamentos de Routing en Cisco IOS

El routing (enrutamiento) es el proceso de seleccionar el mejor camino para enviar paquetes IP desde una red origen hacia una red destino. Los routers Cisco utilizan tablas de enrutamiento que pueden aprenderse dinámicamente mediante protocolos o configurarse manualmente.

Distancia Administrativa (AD) — Prioridad de Fuentes

Jerarquía de confianza en Cisco IOS
! === DISTANCIA ADMINISTRATIVA POR DEFECTO ===
! Valor más bajo = Más confiable

! Protocolo                    AD    Descripción
! --------------------------  ----  --------------------------------
! Connected (C)                0    Interfaz física conectada
! Local (L)                    0    Dirección IP de la interfaz
! Static (S)                   1    Ruta configurada manualmente
! eBGP                        20    BGP externo (entre AS)
! EIGRP Internal (D)          90    Rutas aprendidas por EIGRP
! OSPF (O)                   110    Rutas aprendidas por OSPF
! IS-IS (i)                  115    Intermediate System to Intermediate System
! RIP (R)                    120    Routing Information Protocol
! EIGRP External (D EX)      170    Redistribuidas en EIGRP
! iBGP                       200    BGP interno (dentro del AS)
! Unknown/Unusable           255    Ruta descartada

! === CAMBIAR DISTANCIA ADMINISTRATIVA ===
R1(config)# router ospf 1
R1(config-router)# distance 150
! Cambia AD de OSPF de 110 a 150 (menos preferido que RIP)

R1(config)# ip route 192.168.100.0 255.255.255.0 10.1.1.2 5
! Ruta estática flotante con AD=5 (más preferida que default 1)
! Útil para backup: si la principal falla, esta entra en acción
⚠️ Conflicto de Rutas: AD vs Métrica

Cuando un router aprende la misma red por múltiples fuentes, primero compara la Distancia Administrativa (AD). La ruta con menor AD gana y se instala en la tabla de routing. Solo si la AD es igual, se compara la métrica (costo del protocolo).

Ejemplo: Si OSPF (AD 110) y RIP (AD 120) anuncian la misma red, OSPF gana aunque RIP tenga "mejor" métrica (menos saltos). Para forzar el uso de RIP, cambia su AD o usa rutas estáticas con AD menor.

Configuración de Rutas Estáticas

Las rutas estáticas son configuradas manualmente por el administrador. No consumen ancho de banda por actualizaciones de routing, son seguras (no se anuncian), pero no se adaptan automáticamente a cambios en la topología.

Ruta Estática Básica (Next-Hop IP)

Configuración con dirección IP del siguiente salto
! === RUTA ESTÁTICA CON NEXT-HOP ===
! Formato: ip route [red-destino] [máscara] [next-hop-ip]

R1(config)# ip route 192.168.2.0 255.255.255.0 10.1.1.2
! Para llegar a 192.168.2.0/24, enviar paquetes a 10.1.1.2
! Requiere que 10.1.1.2 sea alcanzable (resoluble ARP en segmento local)

! === RUTA ESTÁTICA CON INTERFAZ DE SALIDA ===
R1(config)# ip route 192.168.2.0 255.255.255.0 Serial0/0/0
! Para llegar a 192.168.2.0/24, usar interfaz Serial0/0/0
! El router no necesita resolver ARP, usa directamente la interfaz
! ⚠️ Solo recomendado en interfaces punto-a-punto (seriales, tunnel)

! === RUTA ESTÁTICA CON NEXT-HOP + INTERFAZ (RECOMENDADO) ===
R1(config)# ip route 192.168.2.0 255.255.255.0 Serial0/0/0 10.1.1.2
! Combina interfaz de salida + next-hop
! Ventaja: No depende de resolución ARP y valida next-hop
! Mejor práctica para interfaces multiacceso (Ethernet)

Ruta por Defecto (Default Route / Gateway of Last Resort)

Configuración de ruta 0.0.0.0/0
! === RUTA POR DEFECTO ===
! Se usa cuando no hay coincidencia específica en la tabla de routing

R1(config)# ip route 0.0.0.0 0.0.0.0 203.0.113.1
! Todo tráfico no conocido se envía al ISP (203.0.113.1)
! Equivalente a "Gateway of Last Resort"

! === VERIFICAR RUTA POR DEFECTO ===
R1# show ip route
Gateway of last resort is 203.0.113.1 to network 0.0.0.0

S*    0.0.0.0/0 [1/0] via 203.0.113.1
! S* = Static default route
! [1/0] = AD=1, Métrica=0

! === RUTA POR DEFECTO CON INTERFAZ ===
R1(config)# ip route 0.0.0.0 0.0.0.0 GigabitEthernet0/0 203.0.113.1
! Especificar interfaz evita problemas de next-hop inalcanzable

Rutas Estáticas Flotantes (Floating Static Routes)

Backup automático con distancia administrativa modificada
! === RUTA ESTÁTICA FLOTANTE (BACKUP) ===
! Si la ruta principal falla, la backup entra automáticamente

! Ruta principal (AD por defecto = 1):
R1(config)# ip route 192.168.100.0 255.255.255.0 10.1.1.2

! Ruta backup (AD = 10, mayor que 1):
R1(config)# ip route 192.168.100.0 255.255.255.0 10.1.2.2 10
! Solo se instala si la principal (AD 1) desaparece

! === VERIFICAR RUTA FLOTANTE ===
R1# show ip route
! Cuando principal está activa:
S    192.168.100.0/24 [1/0] via 10.1.1.2

! Cuando principal falla:
S    192.168.100.0/24 [10/0] via 10.1.2.2

Rutas Estáticas Recursivas y Null0

Descarte de tráfico y agregación de rutas
! === RUTA ESTÁTICA A NULL0 (BLACKHOLE) ===
! Descarta tráfico hacia una red específica

R1(config)# ip route 192.168.200.0 255.255.255.0 Null0
! Todo tráfico a 192.168.200.0/24 se descarta silenciosamente
! Útil para bloquear rangos de IPs maliciosas o no utilizadas

! === AGREGACIÓN DE RUTAS (SUMARIZACIÓN) ===
! Resumir múltiples redes en una sola ruta estática

! Redes individuales:
! 192.168.0.0/24, 192.168.1.0/24, 192.168.2.0/24, 192.168.3.0/24

! Ruta agregada:
R1(config)# ip route 192.168.0.0 255.255.252.0 10.1.1.2
! /22 cubre 192.168.0.0 - 192.168.3.255 (4 redes /24)

! === RUTA ESTÁTICA RECURSIVA ===
R1(config)# ip route 10.0.0.0 255.0.0.0 192.168.1.1
! El next-hop 192.168.1.1 debe ser alcanzable por otra ruta
! Si 192.168.1.1 no es accesible, esta ruta desaparece de la tabla
💡 Best Practice: Rutas Estáticas
  • Usa next-hop + interfaz en redes multiacceso (Ethernet) para evitar problemas de ARP.
  • Usa solo interfaz de salida en enlaces punto-a-punto (seriales, tunnels).
  • Configura rutas flotantes para redundancia en enlaces críticos.
  • Documenta cada ruta estática con description o comentarios.
  • En redes grandes, preferir protocolos dinámicos para mantenibilidad.

Routing Information Protocol (RIP)

RIP es un protocolo de vector de distancia que usa el número de saltos (hop count) como métrica. Es simple pero limitado (máximo 15 saltos). RIPv2 soporta VLSM, CIDR y autenticación. RIPng es la versión para IPv6.

Configuración Básica RIPv2

Activar RIP y anunciar redes
! === CONFIGURACIÓN RIPv2 BÁSICA ===

R1(config)# router rip
R1(config-router)# version 2
! Fuerza uso de RIPv2 (soporta VLSM, CIDR, multicast)

R1(config-router)# no auto-summary
! Desactiva sumarización automática por clase (A/B/C)
! OBLIGATORIO para redes con VLSM/subredes discontinuas

R1(config-router)# network 192.168.1.0
! Anuncia todas las subredes de 192.168.1.0/24
! NO se escribe la máscara en RIP (usa clase por defecto)

R1(config-router)# network 10.0.0.0
! Anuncia todas las subredes de 10.0.0.0/8 (clase A)

! === CONFIGURACIÓN COMPLETA ===
R1(config)# router rip
R1(config-router)# version 2
R1(config-router)# no auto-summary
R1(config-router)# network 192.168.1.0
R1(config-router)# network 10.0.0.0
R1(config-router)# passive-interface GigabitEthernet0/0
! No envía actualizaciones RIP por Gi0/0 (red de usuarios)
! Pero sí recibe y procesa actualizaciones entrantes

Rutas por Defecto en RIP (Default-Information Originate)

Propagar ruta 0.0.0.0/0 por RIP
! === PROPAGAR RUTA POR DEFECTO VÍA RIP ===

! Paso 1: Configurar ruta estática default en el router frontera
R1(config)# ip route 0.0.0.0 0.0.0.0 203.0.113.1

! Paso 2: Propagarla por RIP a todos los routers vecinos
R1(config)# router rip
R1(config-router)# default-information originate
! Inyecta la ruta 0.0.0.0/0 en las actualizaciones RIP

! === VERIFICAR EN ROUTERS INTERNOS ===
R2# show ip route
Gateway of last resort is 192.168.12.1 to network 0.0.0.0

R*    0.0.0.0/0 [120/1] via 192.168.12.1, 00:00:15, GigabitEthernet0/0
! R* = RIP default route, [120/1] = AD 120, 1 hop

Passive-Interface y Control de Actualizaciones

Optimizar anuncios RIP
! === PASSIVE-INTERFACE ===
! Evita enviar actualizaciones innecesarias (seguridad + ancho de banda)

R1(config)# router rip
R1(config-router)# passive-interface GigabitEthernet0/0
! Gi0/0 conecta a usuarios: no envía RIP, pero sí escucha

R1(config-router)# passive-interface default
! Hace TODAS las interfaces pasivas por defecto

R1(config-router)# no passive-interface Serial0/0/0
! Excepto Serial0/0/0 (enlace a otro router): sí envía RIP

! === TIMERS RIP (Avanzado) ===
R1(config-router)# timers basic 30 180 180 240
! Update(30s) Invalid(180s) Holddown(180s) Flush(240s)
! Default: 30 180 180 240. No modificar sin justificación.

Autenticación RIPv2

Seguridad básica con contraseña
! === AUTENTICACIÓN RIPv2 (TEXT PLAIN - NO RECOMENDADO) ===
R1(config)# key chain RIP_AUTH
R1(config-keychain)# key 1
R1(config-keychain-key)# key-string Cisco123

R1(config)# interface Serial0/0/0
R1(config-if)# ip rip authentication key-chain RIP_AUTH
R1(config-if)# ip rip authentication mode text

! === AUTENTICACIÓN RIPv2 (MD5 - RECOMENDADO) ===
R1(config)# key chain RIP_AUTH_MD5
R1(config-keychain)# key 1
R1(config-keychain-key)# key-string SecureRIP2024!

R1(config)# interface Serial0/0/0
R1(config-if)# ip rip authentication key-chain RIP_AUTH_MD5
R1(config-if)# ip rip authentication mode md5
! Ambos lados del enlace deben tener la misma key-chain y modo

RIPng (IPv6)

Configuración RIP para IPv6
! === RIPng (IPv6) ===
! No usa comando network, se habilita por interfaz

R1(config)# ipv6 unicast-routing
! Habilitar routing IPv6 globalmente

R1(config)# interface GigabitEthernet0/0
R1(config-if)# ipv6 address 2001:DB8:ACAD:1::1/64
R1(config-if)# ipv6 rip RIPNG enable
! Habilita RIPng en esta interfaz con nombre de proceso "RIPNG"

R1(config)# interface Serial0/0/0
R1(config-if)# ipv6 address 2001:DB8:ACAD:12::1/64
R1(config-if)# ipv6 rip RIPNG enable

! === VERIFICAR RIPng ===
R1# show ipv6 rip
R1# show ipv6 route rip
⚠️ Limitaciones de RIP
  • Máximo 15 saltos: Redes mayores a 15 routers son inalcanzables.
  • Convergencia lenta: Holddown timer de 180 segundos.
  • Métrica simple: Solo cuenta saltos, ignora ancho de banda.
  • Broadcast RIPv1: No soporta VLSM. Siempre usa RIPv2.
  • Looping: Split horizon y poison reverse mitigan pero no eliminan loops.

Para redes modernas, usar OSPF o EIGRP. RIP solo para compatibilidad legacy o laboratorios educativos.

Open Shortest Path First (OSPF)

OSPF es un protocolo de estado de enlace (link-state) que usa el algoritmo Dijkstra para calcular el camino más corto. Soporta VLSM, CIDR, autenticación, jerarquía por áreas y converge rápidamente. Es el protocolo IGP más usado en redes empresariales.

Configuración OSPFv2 Básica (Single-Area)

Activar OSPF y anunciar redes con wildcard mask
! === CONFIGURACIÓN OSPFv2 SINGLE-AREA ===

R1(config)# router ospf 1
! Process ID 1 (local al router, puede diferir entre routers)

R1(config-router)# router-id 1.1.1.1
! ID único del router en formato IPv4
! Si no se configura: loopback más alta > IP más alta activa

R1(config-router)# network 192.168.1.0 0.0.0.255 area 0
! Wildcard mask: 0.0.0.255 = /24 inverso (255.255.255.0)
! Área 0 = Backbone (obligatoria en multi-area)

R1(config-router)# network 10.1.1.0 0.0.0.3 area 0
! /30 entre routers: wildcard 0.0.0.3

! === CONFIGURACIÓN COMPLETA ===
R1(config)# router ospf 1
R1(config-router)# router-id 1.1.1.1
R1(config-router)# network 192.168.1.0 0.0.0.255 area 0
R1(config-router)# network 10.1.1.0 0.0.0.3 area 0
R1(config-router)# passive-interface GigabitEthernet0/0

! === VERIFICAR VECINOS ===
R1# show ip ospf neighbor
Neighbor ID     Pri   State           Dead Time   Address         Interface
2.2.2.2           1   FULL/DR         00:00:35    10.1.1.2        GigabitEthernet0/1
! FULL = Base de datos LSDB sincronizada
! DR = Designated Router (red multiacceso)

Costo OSPF y Ancho de Banda de Referencia

Optimizar métricas para enlaces modernos
! === FÓRMULA DE COSTO OSPF ===
! Costo = Ancho de Banda de Referencia / Ancho de Banda Interfaz
! Default reference-bandwidth = 100 Mbps (10^8 bps)

! Problema: Enlaces de 1 Gbps o 10 Gbps costan 1 (mínimo)
! 100Mbps/1Gbps = 1, 100Mbps/10Gbps = 1 (igual costo, incorrecto)

! === SOLUCIÓN: Cambiar ancho de banda de referencia ===
R1(config-router)# auto-cost reference-bandwidth 10000
! 10000 = 10 Gbps (en Mbps)
! Ahora: 10Gbps/1Gbps = 10, 10Gbps/100Mbps = 100
! ⚠️ Configurar en TODOS los routers del área OSPF

! === ALTERNATIVA: Costo manual por interfaz ===
R1(config)# interface GigabitEthernet0/0
R1(config-if)# ip ospf cost 10
! Fuerza costo específico (ignora ancho de banda)
! Útil para influir en ruta preferida sin cambiar reference-bandwidth

OSPF Multi-Area y Tipos de Áreas

Jerarquía OSPF para redes grandes
! === OSPF MULTI-AREA ===
! Backbone (Área 0) + Áreas regulares + Áreas especiales

! Router ABR (Area Border Router):
R1(config)# router ospf 1
R1(config-router)# router-id 1.1.1.1
R1(config-router)# network 10.0.0.0 0.0.0.3 area 0
R1(config-router)# network 10.0.1.0 0.0.0.3 area 1
! Conecta Área 0 con Área 1

! === ÁREA STUB (No recibe rutas externas ASBR) ===
R2(config)# router ospf 1
R2(config-router)# area 1 stub
! Reduces tamaño de LSDB en routers de área stub
! Solo reciben ruta por defecto para salir del área

! === ÁREA TOTALLY STUBBY ===
R2(config-router)# area 1 stub no-summary
! No recibe rutas externas NI inter-área
! Solo ruta por defecto 0.0.0.0/0 del ABR

! === ÁREA NSSA (Not-So-Stubby Area) ===
R2(config-router)# area 1 nssa
! Permite importar rutas externas con tipo LSA 7
! Útil cuando hay un ASBR dentro de un área stub

Autenticación OSPF

Seguridad en adyacencias OSPF
! === AUTENTICACIÓN OSPF POR ÁREA (MD5) ===
R1(config)# router ospf 1
R1(config-router)# area 0 authentication message-digest

R1(config)# interface GigabitEthernet0/1
R1(config-if)# ip ospf message-digest-key 1 md5 CiscoOSPF2024!
! Key ID 1, contraseña CiscoOSPF2024!
! Todos los vecinos en área 0 deben tener la misma key

! === AUTENTICACIÓN POR INTERFAZ (MÁS GRANULAR) ===
R1(config)# interface Serial0/0/0
R1(config-if)# ip ospf authentication message-digest
R1(config-if)# ip ospf message-digest-key 1 md5 SecretKey123

! === AUTENTICACIÓN SHA (OSPFv3 / IOS moderno) ===
R1(config)# interface GigabitEthernet0/1
R1(config-if)# ip ospf authentication key-chain OSPF_SHA
! Requiere configurar key-chain con algoritmo SHA previamente

OSPFv3 (IPv6)

Configuración OSPF para IPv6
! === OSPFv3 (IPv6) ===
! Process ID separado de OSPFv2, se habilita por interfaz

R1(config)# ipv6 unicast-routing

R1(config)# router ospfv3 1
R1(config-router)# router-id 1.1.1.1
! OSPFv3 requiere router-id manual si no hay IPv4 en el router

R1(config)# interface GigabitEthernet0/0
R1(config-if)# ipv6 address 2001:DB8:ACAD:1::1/64
R1(config-if)# ipv6 ospf 1 area 0
! Habilita OSPFv3 process 1, área 0 en esta interfaz

R1(config)# interface Serial0/0/0
R1(config-if)# ipv6 address 2001:DB8:ACAD:12::1/64
R1(config-if)# ipv6 ospf 1 area 0

! === VERIFICAR OSPFv3 ===
R1# show ipv6 ospf neighbor
R1# show ipv6 route ospf
R1# show ipv6 ospf interface brief
💡 OSPF Designated Router (DR/BDR)

En redes multiacceso (Ethernet), OSPF elige un DR (Designated Router) y BDR (Backup DR) para reducir flooding de LSAs.

  • Prioridad: Valor 0-255. Mayor prioridad gana. 0 = nunca será DR.
  • Router ID: Si prioridad es igual, gana el Router ID más alto.
  • Comando: ip ospf priority 100 en interfaz.
  • Forzar reelección: clear ip ospf process (reinicia OSPF).

Enhanced Interior Gateway Routing Protocol (EIGRP)

EIGRP es un protocolo híbrido (vector de distancia avanzado) propietario de Cisco. Usa el algoritmo DUAL para cálculo sin loops, converge extremadamente rápido y soporta múltiples protocolos de capa de red (IP, IPX, AppleTalk). EIGRP para IPv6 usa el mismo process ID.

Configuración EIGRP Básica (Named Mode - Moderno)

Configuración recomendada en IOS 15.x+
! === EIGRP NAMED MODE (RECOMENDADO) ===
! Más flexible, soporta IPv4 e IPv6 en un solo proceso

R1(config)# router eigrp CORPORATIVO
! "CORPORATIVO" = nombre del proceso (case-sensitive)

R1(config-router)# address-family ipv4 autonomous-system 100
! AS 100 debe coincidir en TODOS los routers EIGRP

R1(config-router-af)# af-interface GigabitEthernet0/0
R1(config-router-af-interface)# passive-interface
! Interfaz pasiva (no envía hellos EIGRP)

R1(config-router-af-interface)# exit
R1(config-router-af)# network 192.168.1.0 0.0.0.255
R1(config-router-af)# network 10.1.1.0 0.0.0.3

R1(config-router-af)# eigrp router-id 1.1.1.1
! Router ID único (formato IPv4)

! === CONFIGURACIÓN CLÁSICA (Legacy - Aún válida) ===
R1(config)# router eigrp 100
R1(config-router)# network 192.168.1.0 0.0.0.255
R1(config-router)# network 10.1.1.0 0.0.0.3
R1(config-router)# no auto-summary
R1(config-router)# eigrp router-id 1.1.1.1
R1(config-router)# passive-interface GigabitEthernet0/0

Métrica EIGRP y Ancho de Banda/Delay

Fórmula y tuning de métrica compuesta
! === FÓRMULA DE MÉTRICA EIGRP ===
! Métrica = [K1*Bandwidth + K3*Delay] * 256
! Por defecto: K1=1, K2=0, K3=1, K4=0, K5=0
! Solo se usan Bandwidth y Delay por defecto

! === VERIFICAR VALORES K ===
R1# show ip protocols
EIGRP-IPv4 Protocol for AS(100)
  Metric weight K1=1, K2=0, K3=1, K4=0, K5=0

! === MODIFICAR ANCHO DE BANDA (NO RECOMENDADO PARA ROUTING) ===
! Mejor usar "bandwidth" solo para cálculo de métrica, no limita tráfico real
R1(config)# interface Serial0/0/0
R1(config-if)# bandwidth 1544
! En Kbps. Usado por EIGRP/OSPF para calcular métrica.

! === MODIFICAR DELAY ===
R1(config-if)# delay 100
! En microsegundos (usec). Influye directamente en métrica EIGRP.
! Útil para influir en selección de ruta sin afectar QoS.

! === VARIANZA (BALANCEO DE CARGA DESIGUAL) ===
R1(config-router-af)# variance 2
! Permite usar rutas con métrica hasta 2x peor que la mejor
! Útil para balancear tráfico por múltiples enlaces de diferente capacidad

EIGRP Stub y Seguridad

Optimización en redes hub-and-spoke
! === EIGRP STUB ===
! Router stub solo anuncia redes conectadas y sumarizadas
! Reduce consultas DUAL y mejora convergencia

R2(config)# router eigrp CORPORATIVO
R2(config-router)# address-family ipv4 autonomous-system 100
R2(config-router-af)# eigrp stub connected summary
! Opciones: connected, summary, static, redistributed, receive-only

! === AUTENTICACIÓN EIGRP (MD5) ===
R1(config)# key chain EIGRP_AUTH
R1(config-keychain)# key 1
R1(config-keychain-key)# key-string SecureEIGRP2024!

R1(config)# interface Serial0/0/0
R1(config-if)# ip authentication key-chain eigrp 100 EIGRP_AUTH
R1(config-if)# ip authentication mode eigrp 100 md5

! === AUTENTICACIÓN EIGRP (SHA-256 - Named Mode) ===
R1(config)# router eigrp CORPORATIVO
R1(config-router)# address-family ipv4 autonomous-system 100
R1(config-router-af)# af-interface Serial0/0/0
R1(config-router-af-interface)# authentication mode hmac-sha-256 SecureKey123

EIGRP para IPv6 (Named Mode)

Configuración unificada IPv4/IPv6
! === EIGRP PARA IPv6 EN NAMED MODE ===

R1(config)# router eigrp CORPORATIVO
R1(config-router)# address-family ipv6 autonomous-system 100
R1(config-router-af)# af-interface GigabitEthernet0/0
R1(config-router-af-interface)# no shutdown
! EIGRP para IPv6 requiere "no shutdown" por interfaz

R1(config-router-af)# interface GigabitEthernet0/0
R1(config-if)# ipv6 enable
R1(config-if)# ipv6 eigrp 100

! === VERIFICAR EIGRP IPv6 ===
R1# show ipv6 eigrp neighbors
R1# show ipv6 route eigrp
R1# show eigrp address-family ipv6 neighbors
⚠️ EIGRP: Requisitos de Adyacencia

Para que dos routers formen adyacencia EIGRP, deben coincidir en:

  1. Autonomous System (AS): Mismo número de proceso.
  2. K values: Mismos valores K1-K5 (métrica).
  3. Autenticación: Misma key-chain y modo (si está configurada).
  4. Subnet mask: Misma máscara de subred en el enlace.
  5. Timers: Hello/Hold (auto-negociados generalmente).

Si alguno falla, los routers no forman adyacencia. Usa show ip eigrp neighbors para diagnosticar.

Border Gateway Protocol (BGP)

BGP es el protocolo de routing exterior (EGP) que interconecta sistemas autónomos (AS) en Internet. Usa atributos de ruta para tomar decisiones de routing basadas en políticas, no solo en métrica. Es fundamental para conectividad WAN e Internet.

Configuración BGP Básica (eBGP)

Conexión entre dos AS diferentes
! === CONFIGURACIÓN eBGP BÁSICA ===
! Router en AS 65001 conectado a ISP en AS 65002

R1(config)# router bgp 65001
! Número de AS local (1-65535 público, 64512-65535 privado)

R1(config-router)# bgp router-id 1.1.1.1
! Router ID único (igual formato que OSPF)

R1(config-router)# neighbor 203.0.113.2 remote-as 65002
! IP del vecino BGP y su AS (diferente = eBGP)

R1(config-router)# neighbor 203.0.113.2 description ISP-PRIMARY

! === ANUNCIAR REDES PROPIAS ===
R1(config-router)# network 192.168.1.0 mask 255.255.255.0
! La red debe existir en la tabla de routing (connected/static/IGP)
! La máscara es obligatoria en BGP (no usa clases)

R1(config-router)# network 10.0.0.0 mask 255.255.0.0
! Anuncia red corporativa a Internet

! === VERIFICAR VECINO BGP ===
R1# show ip bgp summary
Neighbor        V    AS MsgRcvd MsgSent   TblVer  InQ OutQ Up/Down  State/PfxRcd
203.0.113.2     4 65002    1256    1253       12    0    0 5d12h    152
! State/PfxRcd = 152 rutas recibidas del ISP

iBGP (Dentro del mismo AS)

Full-mesh entre routers del mismo AS
! === iBGP ENTRE ROUTERS INTERNOS ===
! Tres routers en AS 65001

R1(config)# router bgp 65001
R1(config-router)# neighbor 10.1.1.2 remote-as 65001
! Mismo AS = iBGP
R1(config-router)# neighbor 10.1.1.2 update-source Loopback0
! Usar loopback como origen (más estable que interfaz física)

R1(config-router)# neighbor 10.1.2.2 remote-as 65001
R1(config-router)# neighbor 10.1.2.2 update-source Loopback0

! === REQUISITO iBGP: FULL-MESH O ROUTE REFLECTOR ===
! iBGP no reenvía rutas iBGP a otros vecinos iBGP

! Solución 1: Full-mesh (n(n-1)/2 sesiones)
! Solución 2: Route Reflector (recomendado)

R1(config)# router bgp 65001
R1(config-router)# neighbor 10.1.1.2 route-reflector-client
R1(config-router)# neighbor 10.1.2.2 route-reflector-client
! R1 actúa como Route Reflector, reenvía rutas entre clientes

Atributos BGP y Políticas de Routing

Influir en selección de rutas BGP
! === ATRIBUTOS BGP MÁS IMPORTANTES ===
! 1. WEIGHT (Cisco propietario, local al router, mayor=gana)
! 2. LOCAL_PREF (local al AS, mayor=gana)
! 3. AS_PATH (más corto=gana)
! 4. ORIGIN (IGP < EGP < INCOMPLETE)
! 5. MED (menor=gana, influencia desde AS externo)

! === LOCAL PREFERENCE (Preferir un ISP sobre otro) ===
R1(config)# route-map PREFER-ISP1 permit 10
R1(config-route-map)# set local-preference 200

R1(config)# router bgp 65001
R1(config-router)# neighbor 203.0.113.2 route-map PREFER-ISP1 in
! Rutas recibidas de ISP1 tendrán LOCAL_PREF 200 (mayor preferencia)

! === AS_PATH PREPEND (Hacer ruta menos atractiva) ===
R1(config)# route-map LONGER-ASPATH permit 10
R1(config-route-map)# set as-path prepend 65001 65001 65001

R1(config-router)# neighbor 198.51.100.2 route-map LONGER-ASPATH out
! Añade AS 65001 tres veces al anuncio hacia ISP2
! ISP2 verá path más largo y preferirá otra ruta

! === MED (Multi-Exit Discriminator) ===
R1(config)# route-map SET-MED permit 10
R1(config-route-map)# set metric 100

R1(config-router)# neighbor 203.0.113.2 route-map SET-MED out
! MED menor es preferido por el AS vecino

BGP Path Selection (Algoritmo de Decisión)

Orden de preferencia de rutas BGP
! === ALGORITMO DE SELECCIÓN BGP (SIMPLIFICADO) ===
! 1.  WEIGHT más alto (Cisco local)
! 2.  LOCAL_PREF más alto (local al AS)
! 3.  Local origin (redes originadas localmente)
! 4.  AS_PATH más corto
! 5.  ORIGIN code (IGP < EGP < INCOMPLETE)
! 6.  MED más bajo
! 7.  eBGP sobre iBGP
! 8.  IGP costo más bajo al next-hop
! 9.  Router ID más bajo del anunciador

! === VERIFICAR MEJOR RUTA BGP ===
R1# show ip bgp
   Network          Next Hop            Metric LocPrf Weight Path
*> 8.8.8.0/24       203.0.113.2              0    200      0 65002 i
! *> = Mejor ruta seleccionada, válida, instalada en RIB
! i  = Origin IGP (red networkada manualmente)
! e  = Origin EGP (obsoleto)
! ?  = Origin INCOMPLETE (redistribuida)
💡 BGP Best Practices
  • Usa loopbacks como update-source para sesiones iBGP (estabilidad).
  • Filtra rutas con prefix-lists o route-maps para evitar propagación incorrecta.
  • Configura BGP TTL Security (neighbor x.x.x.x ttl-security hops 1) para proteger sesiones eBGP.
  • Usa MD5 authentication en sesiones BGP críticas.
  • Monitorea con show ip bgp neighbors y show ip bgp summary regularmente.

Network Address Translation (NAT) y PAT

NAT traduce direcciones IP privadas a públicas (y viceversa). PAT (Port Address Translation) permite que múltiples hosts compartan una sola IP pública usando diferentes números de puerto. Esencial para conectividad IPv4 con direccionamiento limitado.

NAT Estático (One-to-One)

Mapeo fijo privado-público
! === NAT ESTÁTICO ===
! Un servidor interno necesita IP pública fija

! Paso 1: Definir interfaces inside/outside
R1(config)# interface GigabitEthernet0/0
R1(config-if)# ip nat inside
! Interfaz hacia red interna (privada)

R1(config)# interface GigabitEthernet0/1
R1(config-if)# ip nat outside
! Interfaz hacia Internet (pública)

! Paso 2: Crear mapeo estático
R1(config)# ip nat inside source static 192.168.1.10 203.0.113.10
! 192.168.1.10 (privada) ↔ 203.0.113.10 (pública)
! Útil para servidores web, mail, DNS accesibles desde Internet

! === VERIFICAR NAT ESTÁTICO ===
R1# show ip nat translations
Pro Inside global      Inside local       Outside local      Outside global
--- 203.0.113.10       192.168.1.10       ---                ---

NAT Dinámico (Pool de Direcciones)

Pool de IPs públicas para hosts internos
! === NAT DINÁMICO CON POOL ===
! Múltiples hosts internos comparten pool de IPs públicas

! Paso 1: Definir ACL con hosts permitidos para NAT
R1(config)# access-list 1 permit 192.168.1.0 0.0.0.255
! Red interna 192.168.1.0/24

! Paso 2: Crear pool de direcciones públicas
R1(config)# ip nat pool PUBLIC-POOL 203.0.113.10 203.0.113.20 netmask 255.255.255.0
! 11 direcciones públicas disponibles (10-20)

! Paso 3: Asociar ACL con pool
R1(config)# ip nat inside source list 1 pool PUBLIC-POOL
! Hosts en ACL 1 usarán IPs del pool dinámicamente

! === VERIFICAR NAT DINÁMICO ===
R1# show ip nat translations
Pro Inside global      Inside local       Outside local      Outside global
tcp 203.0.113.10:1024  192.168.1.50:49152 8.8.8.8:443        8.8.8.8:443
udp 203.0.113.11:512   192.168.1.51:33434 1.1.1.1:53         1.1.1.1:53

PAT / NAT Overload (Muchos a Uno)

Una sola IP pública para toda la red interna
! === PAT (PORT ADDRESS TRANSLATION) ===
! La forma más común de NAT en redes SOHO y empresariales

! Método 1: Usar IP de interfaz outside (más común)
R1(config)# access-list 1 permit 192.168.0.0 0.0.255.255
! Todas las redes RFC 1918 (10/8, 172.16/12, 192.168/16)

R1(config)# ip nat inside source list 1 interface GigabitEthernet0/1 overload
! "overload" = PAT (múltiples hosts, un puerto por sesión)
! Usa la IP de Gi0/1 como dirección pública compartida

! Método 2: Usar pool con overload
R1(config)# ip nat pool PAT-POOL 203.0.113.10 203.0.113.10 netmask 255.255.255.252
R1(config)# ip nat inside source list 1 pool PAT-POOL overload
! Pool de una sola IP con overload = PAT

NAT para Redundancia y Políticas

NAT con route-maps y múltiples ISPs
! === NAT BASADO EN RUTA (ROUTE-BASED NAT) ===
! Diferente NAT según el ISP de salida

R1(config)# access-list 10 permit 192.168.1.0 0.0.0.255
R1(config)# access-list 20 permit 192.168.2.0 0.0.0.255

! ISP Principal:
R1(config)# route-map ISP1-NAT permit 10
R1(config-route-map)# match ip address 10
R1(config-route-map)# match interface GigabitEthernet0/1

R1(config)# ip nat inside source route-map ISP1-NAT interface GigabitEthernet0/1 overload

! ISP Backup:
R1(config)# route-map ISP2-NAT permit 10
R1(config-route-map)# match ip address 20
R1(config-route-map)# match interface GigabitEthernet0/2

R1(config)# ip nat inside source route-map ISP2-NAT interface GigabitEthernet0/2 overload

! === CLEAR NAT TRANSLATIONS ===
R1# clear ip nat translation *
! Borra todas las traducciones dinámicas (cuidado en producción)

R1# clear ip nat translation inside 192.168.1.10 203.0.113.10
! Borra traducción específica
⚠️ Problemas Comunes de NAT
  • ACL incorrecta: Si la ACL no incluye al host, no se traduce.
  • Interfaces inside/outside no definidas: NAT no funciona sin ellas.
  • Pool agotado: En NAT dinámico sin overload, si se acaban las IPs, nuevos hosts fallan.
  • ALG (Application Layer Gateway): FTP, SIP, etc. necesitan inspección especial. Usa ip nat service o deshabilita ALG si causa problemas.
  • IP Routing: Asegúrate de que ip routing esté habilitado (por defecto en routers).

Troubleshooting de Routing — Guía de Diagnóstico

Los problemas de routing pueden afectar toda la red. Esta guía sistemática cubre los escenarios más frecuentes en exámenes y producción.

Problema 1: No hay conectividad entre redes

1
Verificar interfaces y capa física
R1# show ip interface brief
Confirmar que las interfaces están up/up y tienen IP correcta.
2
Verificar tabla de routing local
R1# show ip route
¿Existe ruta hacia la red destino? ¿Es la correcta (next-hop alcanzable)?
3
Verificar next-hop alcanzable
R1# ping [next-hop-ip]
Si el next-hop no responde, el problema está en el enlace o capa 2.
4
Traceroute para identificar punto de falla
R1# traceroute [destino]
Identifica en qué router/salto se detiene el tráfico.

Problema 2: OSPF no forma adyacencia

Diagnóstico paso a paso
! === SÍNTOMA: No aparece vecino en "show ip ospf neighbor" ===

! Paso 1: Verificar configuración de interfaz
R1# show ip ospf interface GigabitEthernet0/1
! ¿Está habilitado OSPF en la interfaz?
! ¿El área coincide con el vecino?

! Paso 2: Verificar conectividad IP con vecino
R1# ping 10.1.1.2
! Si no hay ping, problema de capa 2 o IP

! Paso 3: Verificar timers y tipo de red
R1# show ip ospf interface GigabitEthernet0/1 | include Timer
! Hello/Dead deben coincidir (10/40 en broadcast, 30/120 en NBMA)

! Paso 4: Verificar autenticación
R1# show ip ospf interface GigabitEthernet0/1 | include auth
! Ambos lados deben tener misma autenticación (o ninguna)

! Paso 5: Verificar máscara de subred
R1# show ip interface GigabitEthernet0/1 | include Internet
! Ambos routers deben estar en la misma subred

! Paso 6: Verificar si interfaz está passive
R1# show ip protocols | include Passive
! Passive interface no envía/recibe hellos OSPF

! Paso 7: Verificar ACL que bloquee OSPF (protocolo 89)
R1# show access-lists
! OSPF usa IP protocol 89. Asegurar que no esté bloqueado.

Problema 3: EIGRP no forma adyacencia

Checklist de requisitos EIGRP
! === REQUISITOS PARA ADYACENCIA EIGRP ===

! 1. Mismo AS number:
R1# show ip protocols | include AS
! AS 100 en ambos routers

! 2. Mismos K values:
R1# show ip protocols | include K1
! K1=1, K2=0, K3=1, K4=0, K5=0 (por defecto)

! 3. Misma autenticación (si está configurada):
R1# show running-config interface Serial0/0/0 | include auth
! Misma key-chain, mismo modo (MD5/SHA)

! 4. Misma subred:
R1# show ip interface Serial0/0/0 | include Internet
! IP y máscara deben coincidir

! 5. No passive interface:
R1# show ip protocols | include Passive
! Interfaz no debe estar en lista passive

! 6. Capa 2 funcionando:
R1# show cdp neighbors
! ¿Se ve el vecino vía CDP?

Problema 4: BGP Session Down

Diagnóstico de sesiones BGP
! === VERIFICAR ESTADO BGP ===
R1# show ip bgp summary

203.0.113.2     4 65002       0       0        0    0    0 never    Idle
! Idle = No se ha iniciado conexión TCP (puerto 179)

! === CAUSAS COMUNES ESTADO IDLE ===
! 1. No hay conectividad IP con vecino
R1# ping 203.0.113.2

! 2. Puerto 179 bloqueado por ACL/firewall
R1# show access-lists
! BGP usa TCP 179. Debe estar permitido bidireccionalmente.

! 3. AS number incorrecto en neighbor
R1# show running-config | section router bgp
! Verificar remote-as coincide con el AS real del vecino

! 4. Interface shutdown o IP incorrecta
R1# show ip interface brief

! === ESTADO ACTIVE ===
! El router intenta establecer conexión TCP pero falla
! Causa común: Vecino no tiene ruta de retorno hacia este router

! === DEBUG BGP (USAR CON CUIDADO) ===
R1# debug ip bgp
R1# debug ip tcp transactions
! Muestra negociación TCP y mensajes BGP en tiempo real

Diagnóstico de NAT/PAT

  1. ¿Las interfaces están up/up con IP correcta?
  2. ¿Existe ruta en show ip route hacia el destino?
  3. ¿El next-hop es alcanzable (ping)?
  4. ¿Los protocolos de routing están activos (show ip protocols)?
  5. ¿Los vecinos están establecidos (OSPF/EIGRP/BGP)?
  6. ¿Las máscaras de subred coinciden en el enlace?
  7. ¿Hay ACLs bloqueando protocolos de routing?
  8. ¿La autenticación coincide (si está configurada)?
  9. ¿Hay rutas flotantes o administrative distance causando conflicto?
  10. ¿El NAT está configurado correctamente (inside/outside, ACL, overload)?

🧪 Laboratorio Práctico: Routing Completo Enterprise

Este laboratorio integra rutas estáticas, OSPF, EIGRP, BGP, NAT/PAT y troubleshooting en una topología realista de red empresarial con conexión dual-ISP.

Topología del Laboratorio de Routing

Objetivos del Laboratorio

  • Configurar R1-EDGE con rutas estáticas, BGP dual-homed y NAT/PAT
  • Implementar OSPF Area 0 en R2-CORE y redistribución hacia R1
  • Configurar EIGRP AS 100 entre R2-CORE y R3-BRANCH
  • Configurar RIPv2 en R3-BRANCH y R4-REMOTE
  • Redistribuir rutas entre protocolos (EIGRP ↔ OSPF, RIP ↔ EIGRP)
  • Implementar NAT overload para salida a Internet
  • Verificar conectividad end-to-end y documentar troubleshooting
Parte 1
Configuración R1-EDGE (Static + BGP + NAT)
! === CONFIGURACIÓN INICIAL ===
Router> enable
Router# configure terminal
Router(config)# hostname R1-EDGE

! === INTERFACES ===
R1-EDGE(config)# interface GigabitEthernet0/0
R1-EDGE(config-if)# description LAN-CORE
R1-EDGE(config-if)# ip address 192.168.1.1 255.255.255.0
R1-EDGE(config-if)# ip nat inside
R1-EDGE(config-if)# no shutdown

R1-EDGE(config)# interface GigabitEthernet0/1
R1-EDGE(config-if)# description ISP-1-PRIMARY
R1-EDGE(config-if)# ip address 203.0.113.1 255.255.255.252
R1-EDGE(config-if)# ip nat outside
R1-EDGE(config-if)# no shutdown

R1-EDGE(config)# interface GigabitEthernet0/2
R1-EDGE(config-if)# description ISP-2-BACKUP
R1-EDGE(config-if)# ip address 198.51.100.1 255.255.255.252
R1-EDGE(config-if)# ip nat outside
R1-EDGE(config-if)# no shutdown

! === RUTAS ESTÁTICAS POR DEFECTO (DUAL-HOMED) ===
R1-EDGE(config)# ip route 0.0.0.0 0.0.0.0 203.0.113.2
R1-EDGE(config)# ip route 0.0.0.0 0.0.0.0 198.51.100.2 10
! Primaria por ISP-1 (AD 1), backup por ISP-2 (AD 10)

! === BGP CONFIGURATION ===
R1-EDGE(config)# router bgp 65000
R1-EDGE(config-router)# bgp router-id 1.1.1.1
R1-EDGE(config-router)# neighbor 203.0.113.2 remote-as 65001
R1-EDGE(config-router)# neighbor 203.0.113.2 description ISP-1
R1-EDGE(config-router)# neighbor 198.51.100.2 remote-as 65002
R1-EDGE(config-router)# neighbor 198.51.100.2 description ISP-2

! Anunciar redes corporativas
R1-EDGE(config-router)# network 192.168.1.0 mask 255.255.255.0
R1-EDGE(config-router)# network 192.168.2.0 mask 255.255.255.0
R1-EDGE(config-router)# network 192.168.3.0 mask 255.255.255.0

! Política: Preferir ISP-1
R1-EDGE(config)# route-map PREFER-ISP1 permit 10
R1-EDGE(config-route-map)# set local-preference 200
R1-EDGE(config-router)# neighbor 203.0.113.2 route-map PREFER-ISP1 in

! === NAT OVERLOAD (PAT) ===
R1-EDGE(config)# access-list 1 permit 192.168.0.0 0.0.255.255
R1-EDGE(config)# ip nat inside source list 1 interface GigabitEthernet0/1 overload

! Backup NAT por ISP-2
R1-EDGE(config)# route-map NAT-ISP2 permit 10
R1-EDGE(config-route-map)# match interface GigabitEthernet0/2
R1-EDGE(config)# ip nat inside source list 1 route-map NAT-ISP2 interface GigabitEthernet0/2 overload

! === VERIFICACIÓN ===
R1-EDGE# show ip route
R1-EDGE# show ip bgp summary
R1-EDGE# show ip nat translations
R1-EDGE# show ip nat statistics
R1-EDGE# copy running-config startup-config
Parte 2
Configuración R2-CORE (OSPF + EIGRP Redistribución)
! === CONFIGURACIÓN INICIAL ===
Router(config)# hostname R2-CORE

! === INTERFACES ===
R2-CORE(config)# interface GigabitEthernet0/0
R2-CORE(config-if)# description TO-R1-EDGE
R2-CORE(config-if)# ip address 192.168.1.2 255.255.255.0
R2-CORE(config-if)# no shutdown

R2-CORE(config)# interface GigabitEthernet0/1
R2-CORE(config-if)# description TO-R3-BRANCH
R2-CORE(config-if)# ip address 10.1.2.1 255.255.255.252
R2-CORE(config-if)# no shutdown

R2-CORE(config)# interface GigabitEthernet0/2
R2-CORE(config-if)# description LAN-CORE-USERS
R2-CORE(config-if)# ip address 192.168.10.1 255.255.255.0
R2-CORE(config-if)# no shutdown

! === OSPF AREA 0 ===
R2-CORE(config)# router ospf 1
R2-CORE(config-router)# router-id 2.2.2.2
R2-CORE(config-router)# network 192.168.1.0 0.0.0.255 area 0
R2-CORE(config-router)# network 192.168.10.0 0.0.0.255 area 0
R2-CORE(config-router)# passive-interface GigabitEthernet0/2

! Ruta por defecto desde OSPF (R1 la inyecta)
! O configurar ruta estática default en R2 si R1 no la redistribuye

! === EIGRP AS 100 ===
R2-CORE(config)# router eigrp 100
R2-CORE(config-router)# network 10.1.2.0 0.0.0.3
R2-CORE(config-router)# network 192.168.10.0 0.0.0.255
R2-CORE(config-router)# no auto-summary
R2-CORE(config-router)# eigrp router-id 2.2.2.2
R2-CORE(config-router)# passive-interface GigabitEthernet0/2

! === REDISTRIBUCIÓN OSPF → EIGRP ===
R2-CORE(config)# router eigrp 100
R2-CORE(config-router)# redistribute ospf 1 metric 10000 100 255 1 1500
! metric: bandwidth delay reliability load MTU

! === REDISTRIBUCIÓN EIGRP → OSPF ===
R2-CORE(config)# router ospf 1
R2-CORE(config-router)# redistribute eigrp 100 subnets
! "subnets" obligatorio para preservar máscaras VLSM

! === VERIFICACIÓN ===
R2-CORE# show ip route
R2-CORE# show ip ospf neighbor
R2-CORE# show ip eigrp neighbors
R2-CORE# show ip protocols
R2-CORE# copy running-config startup-config
Parte 3
Configuración R3-BRANCH (EIGRP + RIP Redistribución)
! === CONFIGURACIÓN INICIAL ===
Router(config)# hostname R3-BRANCH

! === INTERFACES ===
R3-BRANCH(config)# interface GigabitEthernet0/0
R3-BRANCH(config-if)# description LAN-BRANCH
R3-BRANCH(config-if)# ip address 192.168.2.1 255.255.255.0
R3-BRANCH(config-if)# no shutdown

R3-BRANCH(config)# interface GigabitEthernet0/1
R3-BRANCH(config-if)# description TO-R2-CORE
R3-BRANCH(config-if)# ip address 10.1.2.2 255.255.255.252
R3-BRANCH(config-if)# no shutdown

R3-BRANCH(config)# interface Serial0/0/0
R3-BRANCH(config-if)# description TO-R4-REMOTE
R3-BRANCH(config-if)# ip address 10.1.3.1 255.255.255.252
R3-BRANCH(config-if)# no shutdown

! === EIGRP AS 100 ===
R3-BRANCH(config)# router eigrp 100
R3-BRANCH(config-router)# network 10.1.2.0 0.0.0.3
R3-BRANCH(config-router)# network 192.168.2.0 0.0.0.255
R3-BRANCH(config-router)# no auto-summary
R3-BRANCH(config-router)# eigrp router-id 3.3.3.3
R3-BRANCH(config-router)# passive-interface GigabitEthernet0/0

! === RIPv2 ===
R3-BRANCH(config)# router rip
R3-BRANCH(config-router)# version 2
R3-BRANCH(config-router)# no auto-summary
R3-BRANCH(config-router)# network 10.0.0.0
! 10.1.3.0/30 está dentro de 10.0.0.0/8

! === REDISTRIBUCIÓN RIP → EIGRP ===
R3-BRANCH(config)# router eigrp 100
R3-BRANCH(config-router)# redistribute rip metric 10000 100 255 1 1500

! === REDISTRIBUCIÓN EIGRP → RIP ===
R3-BRANCH(config)# router rip
R3-BRANCH(config-router)# redistribute eigrp 100 metric 3
! Métrica RIP = hop count (máximo 15)

! === VERIFICACIÓN ===
R3-BRANCH# show ip route
R3-BRANCH# show ip eigrp neighbors
R3-BRANCH# show ip rip database
R3-BRANCH# copy running-config startup-config

Verificación End-to-End y Troubleshooting
! === DESDE PC EN LAN-CORE (192.168.10.10) ===
PC> ping 192.168.2.1        ! R3-BRANCH (vía OSPF→EIGRP)

! Debe mostrar: 192.168.10.1 → 192.168.1.1 → 203.0.113.2 → ...

! === DESDE PC EN LAN-REMOTE (192.168.3.10) ===
PC> ping 192.168.10.10      ! LAN-CORE (vía RIP→EIGRP→OSPF)
PC> ping 203.0.113.2        ! ISP-1 (vía RIP→EIGRP→OSPF→BGP)

! === VERIFICAR REDISTRIBUCIÓN ===

! Debe ver rutas O (OSPF), D (EIGRP), R (RIP), B (BGP)

! Verificar "Routing Information Sources" y redistribuciones
