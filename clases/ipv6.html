IPv6 — Internet Protocol Version 6

Estructura · Tipos de Dirección · EUI-64 · SLAAC · NDP · DHCPv6 · Routing IPv6 · OSPFv3 · Troubleshooting

CCNA 200-301 Módulo 1 · Fundamentos 6️⃣ El Futuro de Internet

📖 ¿Por qué IPv6? — El problema de IPv4

IPv4 usa direcciones de 32 bits — un espacio máximo de 4.294.967.296 direcciones. En los años 90 se identificó que ese espacio sería insuficiente con el crecimiento de Internet, dispositivos móviles, IoT y el mundo hiperconectado. En 2011 la IANA asignó el último bloque IPv4. Los RIRs regionales agotaron sus reservas entre 2012 y 2019.

IPv6 (RFC 2460, actualizado en RFC 8200) usa direcciones de 128 bits — 340 undecillones de direcciones (3.4×10³⁸). Suficiente para asignar miles de IPs a cada grano de arena de la Tierra.

🌍 Espacio de direcciones

2¹²⁸ = 340,282,366,920,938,463,463,374,607,431,768,211,456 direcciones. Nunca se agotará. No necesita NAT — cada dispositivo puede tener su propia IP pública global.

⚡ Cabecera simplificada

La cabecera IPv6 es fija en 40 bytes (vs variable en IPv4). Menos campos = procesamiento más rápido en los routers. Los campos opcionales usan Extension Headers separados.

🔒 IPsec nativo

En IPv6, IPsec (cifrado y autenticación) es parte del diseño base del protocolo, no un añadido opcional. Aunque en la práctica su uso sigue siendo configurable, está contemplado desde el diseño.

⚙️ Autoconfiguración

SLAAC (Stateless Address Autoconfiguration) permite a los hosts configurar su propia dirección IPv6 sin servidor DHCP. Solo necesitan recibir el prefijo del router mediante RA (Router Advertisement).

🔢 Estructura de una dirección IPv6

Una dirección IPv6 tiene 128 bits escritos como 8 grupos de 4 dígitos hexadecimales separados por dos puntos (:). Cada grupo representa 16 bits (2 bytes).

📌 Anatomía de una dirección IPv6

2001 : 0DB8 : 0000 : 0000 : 0000 : FF00 : 0042 : 8329
  ↑      ↑      ↑      ↑      ↑      ↑      ↑      ↑
G-1    G-2    G-3    G-4    G-5    G-6    G-7    G-8
16bit  16bit  16bit  16bit  16bit  16bit  16bit  16bit
←──────────── 128 bits en total ──────────────────────→

✂️ Reglas de compresión — Cómo simplificar una IPv6

Regla 1 — Omitir ceros a la izquierda en cada grupo

📌 Los ceros iniciales de cada grupo son opcionales

Original:  2001:0DB8:0000:0000:0000:FF00:0042:8329
Paso 1:    2001: DB8:    0:    0:    0: FF00:   42:8329
                ↑       ↑     ↑     ↑      ↑    ↑
           0DB8→DB8  0000→0  0000→0 FF00→FF00  0042→42

Resultado después de la Regla 1: 2001:DB8:0:0:0:FF00:42:8329

Regla 2 — Reemplazar grupos consecutivos de ceros con "::"

📌 "::" puede usarse UNA SOLA VEZ para representar uno o más grupos de 0000

Después Regla 1: 2001:DB8: 0:0:0 :FF00:42:8329
                            ↑
                 3 grupos consecutivos de 0 → reemplaza con ::

Resultado final: 2001:DB8::FF00:42:8329

Nótese: si hay dos grupos de ceros en diferentes posiciones, el :: reemplaza al grupo más largo. Si son iguales, se reemplaza el primero.

🔄 Expandir una dirección comprimida

📌 Cómo expandir 2001:DB8::FF00:42:8329 a forma completa

  1. Contar los grupos explícitos: 2001, DB8, FF00, 42, 8329 → son 5 grupos
  2. Una dirección IPv6 tiene 8 grupos. Faltan 8 - 5 = 3 grupos
  3. El "::" representa esos 3 grupos, todos = 0000
  4. Resultado: 2001:0DB8:0000:0000:0000:FF00:0042:8329
  5. Añadir ceros para completar a 4 dígitos: 2001:0DB8:0000:0000:0000:FF00:0042:8329

⚠️ Error frecuente: "::" más de una vez

La dirección 2001::FF::1 es INVÁLIDA porque hay dos "::". Sería imposible saber cuántos grupos hay en cada posición. Si necesitas comprimir dos grupos separados de ceros, comprime solo el más largo.

🏷️ Tipos de direcciones IPv6

A diferencia de IPv4 que usaba Unicast, Multicast y Broadcast, IPv6 usa tres categorías: Unicast, Multicast y Anycast. El broadcast desaparece completamente — todo lo que hacía el broadcast en IPv4 lo hace ahora el multicast en IPv6 de forma más eficiente.

📡 Multicast — Un emisor, múltiples receptores

En IPv6, el multicast reemplaza al broadcast. Todos los mensajes "para todos" o "para un grupo específico" usan multicast. El prefijo multicast siempre comienza con FF.

📌 Solicited-Node Multicast Address — El reemplazo de ARP

Cada dirección IPv6 unicast tiene una Solicited-Node Multicast Address asociada: FF02::1:FF + últimos 3 bytes (24 bits) de la dirección unicast.

Ejemplo: Para la IP 2001:DB8::123:4567
Solicited-Node: FF02::1:FF23:4567

Solo el dispositivo con esa IP escucha en esa dirección multicast. Cuando otro host necesita la MAC de 2001:DB8::1:23:4567, envía un Neighbor Solicitation a FF02::1:FF23:4567 — solo el dueño de esa IP responde. Mucho más eficiente que el broadcast ARP de IPv4.

🎯 Anycast — Un emisor, el receptor más cercano

📌 Anycast en IPv6

Una dirección anycast es una dirección unicast asignada a múltiples interfaces (normalmente en diferentes routers). Cuando un paquete se envía a una dirección anycast, se enruta al dispositivo más cercano topológicamente que tiene esa dirección.

Uso típico: servidores DNS redundantes, servidores de autorización, CDNs. El cliente envía la solicitud a la IP anycast y automáticamente llega al servidor más cercano.

🔧 EUI-64 — Generar el Interface ID desde la MAC

EUI-64 (Extended Unique Identifier — 64 bits) es el proceso mediante el cual un host convierte su dirección MAC de 48 bits en un Interface ID de 64 bits para construir su dirección IPv6. Es usado por SLAAC para generar IPs sin servidor DHCP.

📌 El proceso EUI-64 — 3 pasos

  1. Dividir la MAC en dos mitades de 24 bits (OUI + Device ID)
  2. Insertar FF:FE en el centro — entre las dos mitades
  3. Invertir el bit U/L (Universal/Local) — el 7° bit del primer octeto se complementa (0→1 o 1→0)

🧮 Ejemplo completo de cálculo EUI-64

MAC address: 00:1A:2B:3C:4D:5E

Paso 1 — MAC original (48 bits):
   00:1A:2B  :  3C:4D:5E
   OUI         Device ID

Paso 2 — Insertar FF:FE en el centro:
   00:1A:2B:FF:FE:3C:4D:5E  (64 bits = 8 octetos)

Paso 3 — Invertir el bit U/L (bit 7 del primer octeto):
   00 en binario = 0000 0000
   Bit 7 (de derecha a izquierda) posición: 0000 00[0]0
   El bit U/L es el bit 7 (segundo desde la izquierda): 0[0]000000
   0[0]000000 → 0[0]000000? No: en EUI-64 el bit U/L es el bit 6
   00 = 0000 0000 → bit posición 6 = 0 → invertir → 0000 0010 = 02

   00 → 02 (bit U/L invertido)

Interface ID final: 02:1A:2B:FF:FE:3C:4D:5E
En formato IPv6:     021A:2BFF:FE3C:4D5E

Si el prefijo de red es 2001:DB8:1::/64, la dirección GUA SLAAC será:
2001:DB8:1::021A:2BFF:FE3C:4D5E → comprimida: 2001:DB8:1:0:21A:2BFF:FE3C:4D5E

¿Por qué se invierte el bit U/L?

📌 El bit U/L en IEEE 802 / EUI-64

En el estándar IEEE 802 (MAC addresses), el bit U/L (Universal/Local) en el primer octeto indica si la dirección es universalmente administrada (U=0, asignada por el fabricante) o localmente administrada (U=1, configurada manualmente).

Las MACs de los dispositivos tienen U=0. Al invertirlo en EUI-64, el Interface ID resultante tiene U=1, indicando que fue derivado de una MAC universal. Esto permite distinguir entre IDs generados por EUI-64 y los configurados manualmente (donde U=0).

Truco rápido: si el primer byte de la MAC es 00 → después de EUI-64 es 02. Si es 02 → queda 00.

🔵 Link-Local — La dirección siempre presente

Toda interfaz IPv6 activa genera automáticamente una dirección Link-Local con prefijo FE80::/10. Esta dirección existe incluso si no hay GUA configurada. Es imprescindible para el funcionamiento de NDP, protocolos de routing y SLAAC.

⚠️ La Link-Local no se enruta entre subredes

Una Link-Local solo es válida en el enlace donde se generó. Cuando configuras una ruta IPv6 estática con next-hop Link-Local, siempre debes especificar la interfaz de salida — el router no puede saber de qué interfaz es ese FE80:: sin esa información.

📡 NDP — Neighbor Discovery Protocol

NDP (RFC 4861) es uno de los protocolos más importantes de IPv6. Reemplaza a ARP (resolución de MACs), ICMP Router Discovery (descubrimiento de routers) y parte de ICMP Redirect de IPv4. Opera sobre ICMPv6 — no existe como protocolo separado.

🔄 Sustituye a ARP

NDP resuelve IPv6→MAC usando Neighbor Solicitation (NS) y Neighbor Advertisement (NA) — mucho más eficiente que el broadcast ARP de IPv4.

🌐 Descubrimiento de routers

Los routers anuncian su presencia con Router Advertisements (RA). Los hosts los solicitan con Router Solicitations (RS). SLAAC depende completamente de este mecanismo.

🔒 DAD

Duplicate Address Detection — antes de usar una dirección IPv6, el host verifica que no hay duplicados en el segmento enviando un NS especial.

↩️ Redirect

El router informa a los hosts de un camino mejor hacia un destino, similar al ICMP Redirect de IPv4.

🔎 Resolución de direcciones — NDP vs ARP

NDP resolución (equivalente a ARP Request/Reply)

📌 PC-A quiere la MAC de PC-B (2001:DB8::B)

  1. PC-A calcula el Solicited-Node Multicast de PC-B: FF02::1:FF00:000B
  2. PC-A envía un Neighbor Solicitation (NS) a FF02::1:FF00:000B preguntando por la MAC de 2001:DB8::B
  3. Solo PC-B escucha en esa dirección multicast (es única para su IP)
  4. PC-B responde con un Neighbor Advertisement (NA) unicast a PC-A con su MAC
  5. PC-A guarda la relación IPv6↔MAC en su Neighbor Cache (equivalente a la tabla ARP)

DAD — Duplicate Address Detection

📌 Verificar que una IPv6 no está duplicada antes de usarla

  1. Antes de configurar definitivamente su dirección IPv6, el host la pone en estado tentative
  2. Envía un Neighbor Solicitation con IP origen :: (unspecified) preguntando por su propia IP tentativa
  3. Si nadie responde → la dirección es única → el host la configura como preferred
  4. Si alguien responde con un NA → hay un duplicado → la dirección queda en estado duplicate y no se usa

⚙️ SLAAC — Stateless Address Autoconfiguration

SLAAC (RFC 4862) es el mecanismo que permite a un host IPv6 configurar su propia dirección global sin necesidad de ningún servidor. Solo necesita recibir el prefijo de red del router mediante un Router Advertisement (RA). Es una de las características más elegantes de IPv6.

📌 El proceso SLAAC completo — 5 pasos

  1. La interfaz se activa → el host genera automáticamente su Link-Local FE80:: con EUI-64
  2. El host envía un Router Solicitation (RS) a FF02::2 (all-routers)
  3. El router responde con un Router Advertisement (RA) con el prefijo de red (ej: 2001:DB8:1::/64) y los flags M=0, O=0
  4. El host combina el prefijo /64 del RA con su Interface ID EUI-64 para construir la GUA completa
  5. El host ejecuta DAD para verificar unicidad antes de usar la dirección

🚦 Los 3 modos de autoconfiguración IPv6

ModoFlag MFlag OIPDNS/otrosServidor

SLAAC + DHCPv6 stateless01EUI-64 + prefijo RA✅ Via DHCPv6 (solo DNS)✅ DHCPv6 solo para DNS

⚠️ SLAAC no proporciona DNS — Una limitación importante

El RA original (sin extensiones) no incluye información de servidores DNS. Si los hosts necesitan DNS y usas SLAAC puro, hay dos soluciones: 1) Configurar RDNSS (RFC 8106) en el RA para incluir DNS — soportado en IOS moderno. 2) Usar SLAAC + DHCPv6 stateless (M=0, O=1) donde el host obtiene la IP por SLAAC pero consulta a un servidor DHCPv6 solo para DNS.

⚙️ Configuración SLAAC y RA en Cisco IOS

Habilitar SLAAC — El router anuncia el prefijo

! Paso 1: Habilitar routing IPv6 (OBLIGATORIO)
R1(config)# ipv6 unicast-routing
! Sin este comando el router NO envía RAs y no hace routing IPv6

! Paso 2: Configurar la interfaz LAN con GUA
R1(config)# interface GigabitEthernet0/0
R1(config-if)# ipv6 address 2001:DB8:1::1/64
! Automáticamente el router envía RA con el prefijo 2001:DB8:1::/64
! Los hosts en este segmento recibirán el prefijo y construirán su GUA
R1(config-if)# ipv6 address FE80::1 link-local
! Link-local manual — más fácil de identificar que la autogenerada
R1(config-if)# no shutdown

! Paso 3 (opcional): Configurar RDNSS en el RA para proporcionar DNS
R1(config-if)# ipv6 nd ra dns server 2001:DB8::53 infinite
! Los hosts aprenden el servidor DNS desde el RA (no necesitan DHCPv6)

! Verificar que el RA se envía correctamente
R1# show ipv6 interface GigabitEthernet0/0

  ND router advertisements live for 1800 seconds
  ND advertised reachable time is 0 milliseconds
  ND advertised retransmit interval is 0 milliseconds

Configurar flags M y O en el RA

! ─── SLAAC puro (default: M=0, O=0) ─────────────────
! No requiere configuración adicional

! ─── SLAAC + DHCPv6 stateless (M=0, O=1) ────────────
R1(config-if)# ipv6 nd other-config-flag
! Activa O=1 → los hosts usan DHCPv6 para obtener DNS
! Los hosts siguen usando SLAAC para la IP

! ─── DHCPv6 stateful (M=1, O=1) ─────────────────────
R1(config-if)# ipv6 nd managed-config-flag
! Activa M=1 → los hosts usan DHCPv6 para obtener IP Y DNS
! Deshabilitar SLAAC implícitamente (los hosts no construyen IP por EUI-64)

! ─── Suprimir el envío de RAs (disable SLAAC en interfaz)
R1(config-if)# ipv6 nd ra suppress
! El router no envía RAs por esta interfaz
! Útil en interfaces WAN o hacia otras subredes

🗺️ Routing IPv6 — Comandos y protocolos

El routing IPv6 funciona de forma análoga al IPv4, pero con nuevos comandos, consideraciones para las link-local como next-hop, y versiones actualizadas de los protocolos de routing. El primer paso siempre es habilitar ipv6 unicast-routing.

⚠️ ipv6 unicast-routing — Obligatorio en TODOS los routers

Sin ipv6 unicast-routing, el router actúa como host IPv6, no como router. No reenvía paquetes IPv6 entre interfaces, no envía RA, y no construye tabla de routing IPv6. Es el error de configuración IPv6 más frecuente.

📋 Rutas estáticas IPv6

Las tres variantes de rutas estáticas IPv6

! Habilitar routing IPv6
R1(config)# ipv6 unicast-routing

! ─── Variante 1: next-hop Global Unicast ─────────────
R1(config)# ipv6 route 2001:DB8:2::/48 2001:DB8:12::2
! Hacia la red 2001:DB8:2::/48 → enviar al GUA 2001:DB8:12::2

! ─── Variante 2: next-hop Link-Local + interfaz ──────
R1(config)# ipv6 route 2001:DB8:3::/48 GigabitEthernet0/0 FE80::2
! ⚠️ Link-local SIEMPRE requiere especificar la interfaz de salida
! Las FE80:: son ambiguas sin saber a qué enlace pertenecen

! ─── Variante 3: solo interfaz (P2P serial) ──────────
R1(config)# ipv6 route 2001:DB8:4::/48 Serial0/0/0

! ─── Default route IPv6 ──────────────────────────────
R1(config)# ipv6 route ::/0 2001:DB8:ISP::1
! ::/0 = equivalente de 0.0.0.0/0 en IPv4

! ─── Ruta flotante (backup, AD mayor) ────────────────
R1(config)# ipv6 route 2001:DB8:2::/48 Serial0/0/1 130
! AD=130 → backup de la ruta principal OSPFv3 (AD=110)

⭕ OSPFv3 — OSPF para IPv6

OSPFv3 (RFC 5340) es la versión de OSPF diseñada para IPv6. Mantiene los mismos conceptos de OSPFv2 (áreas, DR/BDR, LSA types, SPF) pero con cambios importantes en la configuración y los tipos de LSA.

R1(config)# ipv6 unicast-routing
R1(config)# ipv6 router ospf 1
R1(config-rtr)# router-id 1.1.1.1
! Router ID en formato IPv4 — OBLIGATORIO en OSPFv3
! Si no hay loopback IPv4 ni router-id manual → OSPFv3 NO arranca

! Activar OSPFv3 en cada interfaz
R1(config)# interface GigabitEthernet0/0
R1(config-if)# ipv6 ospf 1 area 0
! Esta interfaz participa en OSPFv3 proceso 1, área 0
R1(config)# interface Serial0/0/0
R1(config-if)# ipv6 ospf 1 area 0

! passive-interface en OSPFv3
R1(config)# ipv6 router ospf 1
R1(config-rtr)# passive-interface GigabitEthernet0/0

📋 Resumen Maestro — IPv6 CCNA 200-301

✅ Los 30 puntos más críticos del examen

  1. IPv6 usa 128 bits (vs 32 bits IPv4) → 2¹²⁸ = 340 undecillones de direcciones.
  2. Notación: 8 grupos de 4 dígitos hexadecimales separados por ":" (dos puntos).
  3. Regla 1: omitir ceros iniciales de cada grupo (0DB8 → DB8).
  4. Regla 2: reemplazar grupos consecutivos de 0000 con :: — solo una vez por dirección.
  5. IPv6 elimina el broadcast — todo lo que era broadcast en IPv4 es multicast en IPv6.
  6. Cabecera IPv6 es fija en 40 bytes — más simple y rápida de procesar que IPv4.
  7. Global Unicast (GUA): prefijo 2000::/3 — IPs públicas IPv6, enrutables en Internet.
  8. Link-Local: prefijo FE80::/10 — autogenerada, solo válida en el enlace local, siempre presente.
  9. Unique Local (ULA): prefijo FC00::/7 (uso real FD00::/8) — equivalente a IPs privadas RFC 1918.
  10. Loopback: ::1/128 — equivalente a 127.0.0.1.
  11. Unspecified: ::/128 — equivalente a 0.0.0.0. Usado antes de tener IP (DAD, SLAAC).
  12. Multicast FF02::1 = All-Nodes (todos los dispositivos). FF02::2 = All-Routers.
  13. Solicited-Node Multicast: FF02::1:FF + últimos 24 bits de la GUA. Reemplaza ARP.
  14. EUI-64: MAC 48 bits → Interface ID 64 bits. Proceso: dividir MAC + insertar FF:FE + invertir bit U/L (bit 7 del primer octeto).
  15. Si la MAC comienza en 00 → después de EUI-64 comienza en 02 (el bit U/L se invierte).
  16. NDP (RFC 4861): reemplaza ARP y Router Discovery. Opera sobre ICMPv6.
  17. 5 mensajes NDP: RS (tipo 133), RA (134), NS (135), NA (136), Redirect (137).
  18. NS/NA = Neighbor Solicitation/Advertisement = equivalente de ARP Request/Reply en IPv4.
  19. DAD (Duplicate Address Detection): NS con src=:: antes de usar una IP. Si hay respuesta → duplicado.
  20. Neighbor Cache = equivalente de la tabla ARP en IPv4. Comando: show ipv6 neighbors.
  21. SLAAC (RFC 4862): host genera su GUA = prefijo del RA + Interface ID EUI-64. Sin servidor.
  22. Flags RA: M=0,O=0 → SLAAC puro. M=0,O=1 → SLAAC + DHCPv6 stateless. M=1,O=1 → DHCPv6 stateful.
  23. SLAAC puro no proporciona DNS. Solución: RDNSS en el RA o DHCPv6 stateless.
  24. ipv6 unicast-routing es OBLIGATORIO — sin él el router actúa como host IPv6.
  25. Ruta estática con next-hop Link-Local: siempre especificar la interfaz de salida.
  26. Default route IPv6: ipv6 route ::/0 NEXT-HOP — equivalente a 0.0.0.0/0 en IPv4.
  27. OSPFv3: se activa por interfaz con ipv6 ospf PROC area ÁREA. Router ID en formato IPv4.
  28. EIGRPv6: necesita no shutdown en el proceso — está administrativamente apagado por defecto.
  29. show ipv6 route: tabla routing IPv6. show ipv6 interface X: IPs, grupos multicast, NDP info.
  30. Dual Stack: el dispositivo tiene IPv4 e IPv6 simultáneamente — el mecanismo de transición más recomendado.

⚠️ Los 12 errores más comunes en el examen IPv6

  1. Usar "::" más de una vez en la misma dirección → INVÁLIDO
  2. Olvidar ipv6 unicast-routing → el router no hace routing IPv6
  3. Configurar ruta estática con next-hop FE80:: sin especificar la interfaz de salida
  4. Confundir GUA (2000::/3) con ULA (FC00::/7) — GUA es pública, ULA es privada
  5. Pensar que el broadcast existe en IPv6 — no existe, todo es multicast
  6. Creer que ARP funciona en IPv6 — lo reemplaza NDP (NS/NA)
  7. Creer que SLAAC proporciona DNS — SLAAC puro no incluye DNS
  8. En EUI-64: olvidar invertir el bit U/L del primer octeto de la MAC
  9. En OSPFv3: no configurar el Router ID (en formato IPv4) → OSPFv3 no arranca si no hay loopback IPv4
  10. En EIGRPv6: olvidar no shutdown → el proceso queda administrativamente apagado
  11. Confundir FF02::1 (all-nodes) con FF02::2 (all-routers)
  12. Creer que la Link-Local FE80:: es enrutable entre subredes — solo es válida en su enlace local
