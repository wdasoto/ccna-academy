¿Por qué IPv6? — El agotamiento de IPv4
IPv4 utiliza direcciones de 32 bits → solo ~4.300 millones de direcciones posibles. Con el explosivo crecimiento de Internet, IoT y dispositivos móviles, el espacio de direcciones IPv4 se agotó oficialmente en 2011 (IANA) y en 2015 en Europa (RIPE NCC).

IPv6 usa direcciones de 128 bits → 340 undecillones de direcciones (3.4 × 10³⁸). Suficiente para asignar miles de millones de IPs a cada átomo de la Tierra.
📏
Espacio enorme
128 bits = 340 undecillones de direcciones. Nunca se agotará en la práctica humana.
⚡
Cabecera simplificada
IPv6 tiene cabecera fija de 40 bytes, más eficiente que IPv4. Sin checksum en cabecera → procesamiento más rápido.
🔒
Seguridad integrada
IPsec fue diseñado nativamente para IPv6. Autenticación e integridad son parte del protocolo.
🚫
Sin NAT necesario
Cada dispositivo puede tener una IP pública única. End-to-end connectivity restaurada.
🤖
Autoconfiguración
SLAAC permite que los dispositivos configuren su IP automáticamente sin servidor DHCP.
📡
Multicast nativo
IPv6 reemplaza broadcast con multicast eficiente. No existe broadcast en IPv6.
📐
Estructura de una dirección IPv6
Una dirección IPv6 tiene 128 bits divididos en 8 grupos de 16 bits (hextetos), separados por dos puntos (:). Cada grupo se escribe en hexadecimal.
Dirección completa (full):
2001:0DB8:0000:0001:0000:0000:0000:0001
2001: 0DB8: 0000: 0001: 0000: 0000: 0000: 0001
■ Prefijo global (48 bits) ■ Subred (16 bits) ■ Interface ID (64 bits)
✂️
Reglas de compresión IPv6
ReglaDescripciónEjemplo
Regla 1 Omitir ceros a la izquierda dentro de un grupo 0DB8 → DB8  |  0001 → 1
Regla 2 Reemplazar uno o más grupos de solo ceros consecutivos con :: — solo UNA vez por dirección :0000:0000:0000: → ::
Paso a paso — Compresión de IPv6
🎯
EXAMEN CCNA: El :: solo puede aparecer una vez en una dirección. Si hay dos grupos de ceros separados, se comprime el más largo. En caso de empate, se comprime el primero. Ejemplo: 2001:0:0:1:0:0:0:1 → 2001:0:0:1::1 (no 2001::1:0:0:0:1).

Tipos de direcciones IPv6
A diferencia de IPv4, IPv6 tiene 3 tipos principales de direcciones. No existe broadcast en IPv6 — fue reemplazado por multicast eficiente.
📍
UNICAST
Identifica una única interfaz. El paquete se entrega a un solo destino.
📢
MULTICAST
Identifica un grupo de interfaces. El paquete se entrega a todos los miembros del grupo. Prefijo FF00::/8
🎯
ANYCAST
Asignada a múltiples interfaces. El paquete se entrega solo al más cercano (routing).
🌐
Tipos de direcciones Unicast
TipoPrefijoDescripciónEquivalente IPv4
Global Unicast (GUA) 2000::/3
(2000:: a 3FFF::)		 Direcciones públicas enrutables en Internet. Asignadas por ISPs. IP pública IPv4
Link-Local (LLA) FE80::/10 Solo válida en el enlace local (segmento). Obligatoria en toda interfaz IPv6. No enrutable. Usada por NDP, routing protocols. 169.254.x.x (APIPA)
Unique Local (ULA) FC00::/7
(FC00:: a FDFF::)		 IPs privadas IPv6. No enrutables en Internet. Para redes internas. 192.168.x.x / 10.x.x.x
Loopback ::1/128 Interfaz loopback. Equivale a 127.0.0.1 en IPv4. 127.0.0.1
Unspecified ::/128 Dirección no especificada. Usada en el origen de paquetes sin IP aún configurada. 0.0.0.0
IPv4 Mapped ::FFFF:0:0/96 Permite representar una IPv4 dentro de IPv6. Usada en mecanismos de transición. ::FFFF:192.168.1.1
🎯
EXAMEN CCNA: Toda interfaz IPv6 habilitada SIEMPRE tiene una Link-Local (FE80::/10) configurada automáticamente, incluso si no se configura ninguna GUA. La Link-Local es requerida para NDP y muchos protocolos de routing.
📢
Direcciones Multicast más importantes
DirecciónNombreDestinatarios
FF02::1All-nodesTodos los nodos IPv6 en el enlace (≈ broadcast)
FF02::2All-routersTodos los routers IPv6 en el enlace
FF02::5All-OSPFv3 routersTodos los routers OSPF (DR/BDR)
FF02::6OSPF DR routersSolo el Designated Router OSPF
FF02::1:FFxx:xxxxSolicited-NodeGenerada automáticamente por cada dirección unicast. Usada en NDP para Neighbor Discovery.
💜
Solicited-Node Multicast: Se genera tomando los últimos 24 bits de la dirección unicast y añadiéndolos al prefijo FF02::1:FF. Reemplaza al ARP de IPv4 con eficiencia multicast en lugar de broadcast.
🔗
EUI-64 — Generación automática del Interface ID
EUI-64 es el método para generar automáticamente los 64 bits del Interface ID a partir de la dirección MAC de 48 bits. Usado en SLAAC y Link-Local automáticas.
Proceso EUI-64 — De MAC a Interface ID
FF:FE:DD:EE:FF 64 bits = 8 grupos de 8 bits ↓ Paso 2: invertir bit U/L (bit 7 del primer octeto) RESULT EUI-64: A8BB:CCFF:FEDD:EEFF AA → A8 (0xAA = 10101010 → bit U/L invertido → 10101000 = 0xA8) Interface ID final de 64 bits para usar en SLAAC
⚠️
Privacidad: EUI-64 expone la MAC del dispositivo en la dirección IPv6, lo que puede ser un problema de privacidad. Por eso los sistemas operativos modernos (Windows, Linux, Android) usan extensiones de privacidad que generan Interface IDs aleatorios en lugar de EUI-64.
⚙️
SLAAC — Stateless Address Autoconfiguration
SLAAC (RFC 4862) permite que un host configure automáticamente su dirección IPv6 sin necesidad de un servidor DHCP. El host obtiene el prefijo de red del router y genera su propio Interface ID (EUI-64 o aleatorio).
Proceso SLAAC — 4 pasos
🔍
NDP — Neighbor Discovery Protocol
NDP (RFC 4861) es el protocolo que reemplaza a ARP en IPv6. Usa mensajes ICMPv6 para descubrir routers, vecinos y detectar direcciones duplicadas. Opera en el enlace local usando multicast en lugar de broadcast.
Mensaje ICMPv6TipoFunción
Router Solicitation (RS) Type 133 Host solicita info de configuración a los routers del enlace → FF02::2
Router Advertisement (RA) Type 134 Router anuncia prefijo, gateway y flags de configuración → FF02::1 o unicast
Neighbor Solicitation (NS) Type 135 Equivale al ARP Request. Resuelve IPv6 → MAC. También usado en DAD.
Neighbor Advertisement (NA) Type 136 Equivale al ARP Reply. Responde con la MAC solicitada.
Redirect Type 137 Router informa al host de un mejor next-hop para un destino.
📌
DAD (Duplicate Address Detection): Antes de usar una dirección IPv6, el host envía un Neighbor Solicitation a la dirección Solicited-Node multicast correspondiente. Si recibe respuesta (Neighbor Advertisement), la dirección ya existe → conflicto. Si no hay respuesta en cierto tiempo, la dirección es única y puede usarse.
🚦
Flags del Router Advertisement — Modos de configuración
FlagNombreSignificadoResultado
A=1, M=0, O=0 SLAAC puro Autoconfigure con prefijo del RA, sin DHCP IP y gateway del RA. Sin DNS automático.
A=1, M=0, O=1 SLAAC + DHCPv6 Stateless IP de SLAAC, info adicional (DNS) de DHCPv6 IP del RA + DNS del servidor DHCPv6
A=0, M=1, O=1 DHCPv6 Stateful Toda la configuración viene de DHCPv6 Como DHCP en IPv4. El servidor asigna todo.

Configuración IPv6 en Cisco IOS
💜
Habilitar IPv6 globalmente: En Cisco IOS, primero debes ejecutar ipv6 unicast-routing en modo global para que el router enrute paquetes IPv6. Sin este comando, las interfaces solo tienen IPs locales pero no enrutan.
PASO 1 — Habilitar IPv6 Unicast Routing
GLOBAL CONFIG Router> enable Router# configure terminal ! Habilitar routing IPv6 (obligatorio) Router(config)# ipv6 unicast-routing
PASO 2 — Configurar dirección GUA en interfaz
INTERFAZ IPv6 Router(config)# interface GigabitEthernet0/0 ! Asignar dirección IPv6 manualmente Router(config-if)# ipv6 address 2001:DB8:ACAD:1::1/64 Router(config-if)# no shutdown Router(config-if)# exit ! Segunda interfaz Router(config)# interface GigabitEthernet0/1 Router(config-if)# ipv6 address 2001:DB8:ACAD:2::1/64 Router(config-if)# no shutdown
PASO 3 — Configurar Link-Local manualmente (opcional)
LINK-LOCAL ! Por defecto se genera automáticamente, pero se puede configurar manualmente Router(config-if)# ipv6 address FE80::1 link-local ! Usar FE80::1 es buena práctica — fácil de recordar como gateway
PASO 4 — Configurar EUI-64 (Interface ID automático)
EUI-64 ! EUI-64: el IOS genera el Interface ID desde la MAC de la interfaz Router(config-if)# ipv6 address 2001:DB8:ACAD:1::/64 eui-64 ! Resultado: 2001:DB8:ACAD:1:[Interface-ID-generado-de-MAC]/64
PASO 5 — Configurar SLAAC en el router (RA)
SLAAC / RA ! El router enviará RA automáticamente al habilitar ipv6 unicast-routing ! Para forzar envío de RA inmediato: Router(config-if)# ipv6 nd ra-interval 10 ! Cada 10 seg ! Para deshabilitar RA (en puertos hacia usuarios finales): Router(config-if)# ipv6 nd ra suppress ! Deshabilitar RA ! Configurar flags para DHCPv6 stateful (M=1, O=1): Router(config-if)# ipv6 nd managed-config-flag ! Flag M=1 Router(config-if)# ipv6 nd other-config-flag ! Flag O=1
PASO 6 — Ruta estática IPv6
STATIC ROUTE ! Ruta estática IPv6 con next-hop Router(config)# ipv6 route 2001:DB8:ACAD:3::/64 2001:DB8:ACAD:2::2 ! Ruta estática con interfaz de salida Router(config)# ipv6 route 2001:DB8:ACAD:3::/64 GigabitEthernet0/1 ! Ruta por defecto IPv6 Router(config)# ipv6 route ::/0 2001:DB8:ACAD:2::2
VERIFICACIÓN COMPLETA
SHOW COMMANDS Router# show ipv6 interface brief ! Ver IPs en todas las interfaces Router# show ipv6 interface Gi0/0 ! Detalle de una interfaz Router# show ipv6 route ! Tabla de routing IPv6 Router# show ipv6 neighbors ! Tabla NDP (equivale a show arp) Router# show ipv6 routers ! Ver RAs recibidos Router# ping ipv6 2001:DB8:ACAD:1::10 ! Ping IPv6 Router# traceroute ipv6 2001:DB8:ACAD:3::1 ! Traceroute IPv6

Lab Completo — Red Dual-Stack IPv4/IPv6
🔬 Escenario: Empresa DualNet Corp

R1 con dos interfaces: G0/0 hacia LAN1 y G0/1 hacia LAN2.
• LAN1: 2001:DB8:ACAD:1::/64 — usuarios finales con SLAAC
• LAN2: 2001:DB8:ACAD:2::/64 — servidores con IPs manuales
• Link-Local: FE80::1 en ambas interfaces (fácil de recordar)
• Ruta por defecto hacia ISP: ::/0 via 2001:DB8:1::1

  1. Habilitar IPv6 routing y configurar G0/0
    ipv6 unicast-routing → interface G0/0 → ipv6 address 2001:DB8:ACAD:1::1/64 → ipv6 address FE80::1 link-local → no shutdown
  2. Configurar G0/1 para servidores
    interface G0/1 → ipv6 address 2001:DB8:ACAD:2::1/64 → ipv6 address FE80::1 link-local → no shutdown
  3. Configurar ruta por defecto IPv6
    ipv6 route ::/0 2001:DB8:1::1 — Todo tráfico sin ruta específica va al ISP.
  4. Verificar con show ipv6 interface brief y ping ipv6
    Confirmar que las interfaces muestran estado UP/UP y que los PCs reciben su IP por SLAAC. Verificar show ipv6 neighbors para la tabla NDP.

Mecanismos de transición IPv4 → IPv6
La transición de IPv4 a IPv6 no es inmediata — Internet necesita años para completarla. Existen 3 categorías de mecanismos para permitir coexistencia e interoperabilidad durante la transición.
🥞
Dual Stack
El dispositivo ejecuta IPv4 e IPv6 simultáneamente. Usa IPv6 cuando está disponible, IPv4 como fallback. La solución más limpia y recomendada.
🚇
Tunneling
Encapsula paquetes IPv6 dentro de IPv4 para cruzar redes IPv4. 6to4, 6in4, ISATAP, Teredo, GRE tunnels.
🔀
Translation (NAT64)
Traduce entre IPv4 e IPv6. Permite que hosts IPv6-only se comuniquen con servidores IPv4-only y viceversa.
🥞
Dual Stack — La solución preferida
En Dual Stack, un dispositivo tiene configuradas ambas direcciones — IPv4 e IPv6 — en la misma interfaz. El sistema operativo y las aplicaciones pueden usar cualquiera de los dos protocolos según disponibilidad y preferencia (normalmente IPv6 tiene preferencia).
Configuración Dual Stack en Cisco IOS
DUAL STACK Router(config)# ipv6 unicast-routing Router(config)# interface GigabitEthernet0/0 ! IPv4 en la misma interfaz Router(config-if)# ip address 192.168.1.1 255.255.255.0 ! IPv6 en la misma interfaz Router(config-if)# ipv6 address 2001:DB8:ACAD:1::1/64 Router(config-if)# ipv6 address FE80::1 link-local Router(config-if)# no shutdown ! Ambos protocolos activos en la misma interfaz física
✅
Buena práctica: Dual Stack es la estrategia de transición recomendada por Cisco y el IETF. Permite migrar gradualmente sin interrupciones. Las empresas pueden mantener IPv4 para aplicaciones legadas mientras adoptan IPv6 para servicios nuevos.
🚇
Tunneling IPv6 — Tipos principales
TecnologíaDescripciónUso
6in4 (GRE) Encapsula IPv6 en GRE/IPv4. Requiere IPs IPv4 estáticas en ambos extremos. Manual. Conexión punto a punto entre islas IPv6 sobre backbone IPv4
6to4 Automático. Embebe la IP IPv4 dentro del prefijo IPv6 (2002::/16). No requiere configuración manual del túnel. Redes medianas. Deprecated en muchos sistemas.
ISATAP Intra-Site Automatic Tunnel Addressing Protocol. Para conectar hosts IPv6 aislados dentro de una red IPv4. Empresas en transición. Windows lo soporta nativamente.
Teredo Tunneling IPv6 sobre UDP/IPv4 cruzando NAT. Diseñado para hosts detrás de NAT IPv4. Hosts de usuario final sin IPv6 nativo en su ISP.
Configuración túnel GRE IPv6-over-IPv4
GRE TUNNEL ! Crear túnel GRE para transportar IPv6 sobre red IPv4 Router(config)# interface Tunnel0 Router(config-if)# ipv6 address 2001:DB8:TUNNEL::1/64 Router(config-if)# tunnel source 192.168.1.1 ! IP IPv4 local Router(config-if)# tunnel destination 10.0.0.2 ! IP IPv4 remota Router(config-if)# tunnel mode ipv6ip ! IPv6 sobre IPv4 Router(config-if)# no shutdown

Tarea Práctica — Lab IPv6 en Packet Tracer
🔬 Lab: Red Corporativa IPv6 — FutureNet S.A.

2 routers (R1, R2) conectados entre sí. Cada uno con una LAN.
• R1 LAN: 2001:DB8:1::/64 — SLAAC para PCs
• R2 LAN: 2001:DB8:2::/64 — SLAAC para PCs
• Enlace R1-R2: 2001:DB8:FF::/64
• Link-Locals: FE80::1 en R1, FE80::2 en R2
• Rutas estáticas en ambos routers para alcanzar la LAN remota
• Verificar con ping ipv6 entre PCs de distintas LANs

  1. Configurar R1
    ipv6 unicast-routing → G0/0: 2001:DB8:1::1/64 + FE80::1 → G0/1: 2001:DB8:FF::1/64 + FE80::1
  2. Configurar R2
    ipv6 unicast-routing → G0/0: 2001:DB8:2::1/64 + FE80::2 → G0/1: 2001:DB8:FF::2/64 + FE80::2
  3. Rutas estáticas
    R1: ipv6 route 2001:DB8:2::/64 2001:DB8:FF::2 | R2: ipv6 route 2001:DB8:1::/64 2001:DB8:FF::1
  4. Verificar SLAAC en PCs
    Las PCs deben recibir automáticamente una IP del prefijo de su red via RA. Verificar show ipv6 neighbors en los routers.
  5. Ping y traceroute IPv6
    ping ipv6 desde PC de LAN1 a PC de LAN2. Usar traceroute ipv6 para ver el path.

Resumen de Comandos Esenciales — IPv6 CCNA

  • ipv6 unicast-routing (global)
