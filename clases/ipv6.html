IPv6

Direccionamiento · Tipos · EUI-64 · SLAAC · NDP · DHCPv6 · Transición · Configuración IOS

CCNA 200-301 Módulo 1 · Fundamentos 🌐 El futuro de Internet

🌐 ¿Por qué IPv6? — El agotamiento de IPv4

IPv4 usa direcciones de 32 bits → máximo 4,294,967,296 direcciones (~4.3 mil millones). En 2011, la IANA agotó los últimos bloques. IPv6 soluciona esto con 128 bits.

IPv4

32 bits · ~4.3 mil millones de direcciones · Espacio agotado desde 2011 · Requiere NAT para escalar

IPv6

128 bits · 340 undecillones de direcciones (3.4×10³⁸) · Suficiente para cada grano de arena en la Tierra

✅ Ventajas clave de IPv6 vs IPv4

  • Espacio masivo: elimina la necesidad de NAT
  • Autoconfiguración: SLAAC sin servidor DHCP
  • IPsec nativo: seguridad integrada en el protocolo
  • Sin broadcast: usa multicast en su lugar (más eficiente)
  • Cabecera simplificada: procesamiento más rápido en routers
  • Movilidad mejorada: mejor soporte para dispositivos móviles

🔢 Estructura de una dirección IPv6

Una dirección IPv6 tiene 128 bits, escritos como 8 grupos de 4 dígitos hexadecimales separados por dos puntos.

📌 Ejemplo completo

2001:0DB8:AC10:FE01:0000:0000:0000:0001

= 8 grupos × 16 bits = 128 bits totales

Cada dígito hex = 4 bits · Cada grupo = 16 bits = 4 dígitos hex

✂️ Reglas de Abreviación IPv6

IPv6 permite dos simplificaciones para escribir menos:

Regla 1 — Omitir ceros iniciales en cada grupo

2001:0DB8:0000:0001 → 2001:DB8:0:1

0000 → 0  |  00AB → AB  |  0001 → 1

Regla 2 — Comprimir grupos consecutivos de ceros con :: (solo UNA vez)

2001:DB8:0:0:0:0:0:1 → 2001:DB8::1

FE80:0:0:0:200:39FF:FE36:1 → FE80::200:39FF:FE36:1

⚠️ El :: solo puede usarse UNA vez en una dirección

Dirección completaAbreviada correctamente
2001:0DB8:0000:0000:0000:0000:0000:00012001:DB8::1
FE80:0000:0000:0000:0200:39FF:FE36:0001FE80::200:39FF:FE36:1
0000:0000:0000:0000:0000:0000:0000:0001::1 (loopback)
0000:0000:0000:0000:0000:0000:0000:0000:: (todo ceros)
FF02:0000:0000:0000:0000:0000:0000:0001FF02::1

⚠️ Errores comunes de abreviación

  • 2001::DB8::1 — ❌ INCORRECTO: dos :: en la misma dirección
  • 2001:DB8:0:0:1 — ❌ INCORRECTO: faltan grupos (deben ser 8)
  • FE80::1::2 — ❌ INCORRECTO: :: solo puede aparecer una vez

🏗️ Prefijo y longitud de prefijo

IPv6 usa notación CIDR igual que IPv4. El prefijo indica la parte de red.

📌 Ejemplo

2001:DB8:ACAD:1::/64

Primeros 64 bits = prefijo de red (parte de red)

Últimos 64 bits = Interface ID (parte de host)

El prefijo /64 es el más común en LANs IPv6.

🏷️ Tipos de Direcciones IPv6

IPv6 tiene tres categorías principales — no existe broadcast.

🦄 Unicast

Identifica una sola interfaz. El paquete va a un único destino. Es el tipo más común.

📢 Multicast

Identifica un grupo de interfaces. El paquete llega a todos los miembros del grupo. Prefijo FF00::/8.

🎯 Anycast

Asignada a múltiples interfaces, el paquete va al más cercano topológicamente. Usado en DNS y CDN.

🔍 Tipos de Unicast en detalle

TipoPrefijoDescripciónUso
Global Unicast (GUA) 2000::/3 Equivalente a IPv4 pública. Enrutable en Internet. Comunicación global
Link-Local FE80::/10 Solo válida en el segmento local. NO enrutable. Se auto-genera. Comunicación entre vecinos (NDP, routing)
Unique Local FC00::/7 Equivalente a IPv4 privada (RFC 1918). No enrutable en Internet. Redes internas corporativas
Loopback ::1/128 Equivalente a 127.0.0.1 de IPv4. Pruebas locales
Unspecified ::/128 Dirección no asignada, usada durante autoconfiguración. Origen antes de tener IP

🔗 Link-Local — La más importante para el CCNA

Toda interfaz IPv6 siempre tiene una dirección link-local automática, incluso si no se configura ninguna global.

Formato: FE80::[Interface-ID]

Se usa como gateway por defecto y para protocolos de routing (OSPF, EIGRP).

Solo es válida dentro del enlace — no puede cruzar un router.

📢 Multicast IPv6 — Reemplaza al Broadcast

IPv6 no tiene broadcast. En su lugar usa multicast para comunicarse con grupos de dispositivos.

⚙️ EUI-64 — Generar Interface ID desde MAC

EUI-64 es el proceso mediante el cual un dispositivo genera su Interface ID de 64 bits a partir de su dirección MAC de 48 bits.

🔧 Proceso EUI-64 paso a paso

MAC de ejemplo: 00:1A:2B:3C:4D:5E

Paso 1 — Dividir la MAC en dos mitades

00:1A:2B  |  3C:4D:5E

Paso 2 — Insertar FFFE en el centro

00:1A:2B:FF:FE:3C:4D:5E

Paso 3 — Invertir el bit U/L (bit 7 del primer octeto)

El bit 7 del primer octeto 00 = 0000 0000

Invertir bit 7 (el que vale 2): 0000 0010 = 02

Interface ID resultante: 02:1A:2B:FF:FE:3C:4D:5E

Resultado — Dirección completa con prefijo /64

Si el prefijo es 2001:DB8:ACAD:1::/64:

→ 2001:DB8:ACAD:1:021A:2BFF:FE3C:4D5E

⚠️ Truco para identificar EUI-64 en el examen

Si ves FF:FE en el medio del Interface ID → es EUI-64.

Ejemplo: FE80::021A:2BFF:FE3C:4D5E → Interface ID generado con EUI-64.

🚀 SLAAC — Stateless Address Autoconfiguration

SLAAC permite a un dispositivo configurarse automáticamente sin servidor DHCP. Solo necesita un router que envíe Router Advertisements (RA).

📋 Proceso SLAAC completo

  1. El host genera su link-local (FE80::[EUI-64])
  2. El host envía RS (Router Solicitation) a FF02::2 (todos los routers)
  3. El router responde con RA (Router Advertisement) con el prefijo /64 de la red
  4. El host combina el prefijo del RA + su Interface ID EUI-64 → dirección GUA
  5. El host realiza DAD (Duplicate Address Detection) para verificar que la IP es única

🔵 SLAAC (Stateless)

Solo prefijo del RA. El host genera su propio Interface ID. Sin servidor.

Bandera A=1, O=0, M=0 en el RA.

🟡 SLAAC + DHCPv6 (Stateless)

El host obtiene el prefijo del RA y parámetros adicionales (DNS) de un servidor DHCPv6 sin estado.

Bandera A=1, O=1, M=0 en el RA.

🔴 DHCPv6 Stateful

El host obtiene IP completa del servidor DHCPv6, igual que DHCP en IPv4.

Bandera A=0, M=1 en el RA.

🔍 NDP — Neighbor Discovery Protocol

NDP reemplaza a ARP de IPv4. Usa mensajes ICMPv6 para descubrir vecinos, routers y detectar duplicados.

Mensaje NDPICMPv6 TipoFunciónEquivalente IPv4
Router Solicitation (RS)133Host pide RA a los routers → FF02::2—
Router Advertisement (RA)134Router anuncia prefijo, gateway, flags—
Neighbor Solicitation (NS)135Resolver IPv6 → MAC (como ARP Request)ARP Request
Neighbor Advertisement (NA)136Responder con la MAC (como ARP Reply)ARP Reply
Redirect137Router informa mejor ruta al hostICMP Redirect

🔒 DAD — Duplicate Address Detection

Antes de usar una dirección IPv6, el host envía un Neighbor Solicitation a la Solicited-Node multicast de su propia IP. Si alguien responde, la dirección está duplicada.

Esto ocurre automáticamente — no requiere configuración.

💻 Configuración IPv6 en IOS Cisco

La configuración de IPv6 en Cisco IOS requiere habilitar el routing IPv6 globalmente y luego configurar cada interfaz.

Paso 1 — Habilitar routing IPv6 (obligatorio en routers)

! Habilitar forwarding de paquetes IPv6
Router(config)# ipv6 unicast-routing

Paso 2 — Asignar dirección Global Unicast (GUA) estática

Router(config)# interface GigabitEthernet0/0
Router(config-if)# ipv6 address 2001:DB8:ACAD:1::1/64
Router(config-if)# no shutdown

Paso 3 — Asignar GUA con EUI-64 (generada desde MAC)

Router(config)# interface GigabitEthernet0/0
Router(config-if)# ipv6 address 2001:DB8:ACAD:1::/64 eui-64
Router(config-if)# no shutdown
! IOS calcula automáticamente el Interface ID desde la MAC

Paso 4 — Configurar link-local estática (opcional)

! Por defecto se auto-genera, pero se puede forzar una específica
Router(config-if)# ipv6 address FE80::1 link-local

Paso 5 — Habilitar SLAAC (el router anuncia el prefijo)

! El router enviará RA con el prefijo configurado
! No se requiere comando adicional si ipv6 unicast-routing está activo
! Para ajustar los flags del RA:
Router(config-if)# ipv6 nd managed-config-flag
! (activa el flag M=1 → DHCPv6 stateful)

Paso 6 — Ruta estática IPv6

! Ruta estática: destino, prefijo, next-hop
Router(config)# ipv6 route 2001:DB8:ACAD:2::/64 2001:DB8:ACAD:1::2
! Ruta por defecto IPv6
Router(config)# ipv6 route ::/0 2001:DB8:ACAD:1::2

🔄 Mecanismos de Transición IPv4 → IPv6

La transición de IPv4 a IPv6 es gradual. Cisco y el CCNA cubren tres mecanismos principales.

🔀 Pila Dual (Dual Stack)

El dispositivo ejecuta IPv4 e IPv6 simultáneamente. Cada protocolo funciona de forma independiente.

El método más simple y recomendado hoy en día.

🚇 Tunneling

Encapsula paquetes IPv6 dentro de paquetes IPv4 para cruzar redes que no soportan IPv6 nativamente.

Tipos: 6to4, Teredo, ISATAP, GRE.

🔁 Traducción (NAT64)

Traduce paquetes IPv6 a IPv4 y viceversa. Permite que hosts IPv6 se comuniquen con servidores IPv4 legacy.

Requiere un dispositivo NAT64.

📋 Pila Dual — Configuración IOS

! Configurar IPv4 e IPv6 en la misma interfaz
Router(config)# interface GigabitEthernet0/0
Router(config-if)# ip address 192.168.1.1 255.255.255.0
Router(config-if)# ipv6 address 2001:DB8:ACAD:1::1/64
Router(config-if)# no shutdown
! El router ahora habla IPv4 e IPv6 en la misma interfaz

🚇 Tunnel 6to4 — Configuración básica

! Crear un túnel GRE para transportar IPv6 sobre IPv4
Router(config)# interface Tunnel0
Router(config-if)# ipv6 address 2001:DB8:ACAD:100::1/64
Router(config-if)# tunnel source GigabitEthernet0/0
Router(config-if)# tunnel destination 203.0.113.2
Router(config-if)# tunnel mode gre ip

📋 Resumen — Todo IPv6 para el examen CCNA

✅ Puntos críticos del examen

  • IPv6 = 128 bits, 8 grupos de 4 hex, separados por :
  • Abreviación: omitir ceros iniciales + :: para grupos cero consecutivos (solo 1 vez)
  • Prefijo /64 = parte de red | Interface ID (últimos 64 bits) = parte de host
  • GUA: 2000::/3 — enrutable en Internet
  • Link-Local: FE80::/10 — siempre presente, no enrutable
  • Loopback: ::1 — equivalente a 127.0.0.1
  • Multicast All-Nodes: FF02::1 | All-Routers: FF02::2
  • EUI-64: MAC dividida + FFFE en centro + bit U/L invertido
  • SLAAC: prefijo del RA + Interface ID EUI-64, sin servidor
  • NDP: reemplaza ARP → NS (135) y NA (136)
  • DAD: verifica unicidad antes de usar la dirección
  • ipv6 unicast-routing = obligatorio para que el router reenvíe IPv6
  • Ruta por defecto IPv6 = ::/0

⚠️ Errores comunes en el examen

  • Usar :: más de una vez en la misma dirección
  • Olvidar ipv6 unicast-routing en el router
  • Confundir Link-Local con Unique Local
  • Creer que IPv6 tiene broadcast (no tiene, solo multicast)
  • No invertir el bit U/L en EUI-64
  • Confundir NS/NA (NDP) con RS/RA (SLAAC) — son procesos distintos
