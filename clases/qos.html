📊 ¿Qué es QoS?

QoS (Quality of Service) es un conjunto de técnicas que permiten a la red priorizar ciertos tipos de tráfico sobre otros cuando los recursos (ancho de banda, buffers) son limitados.

⚠️ QoS no crea ancho de banda

QoS administra la congestión dando preferencia al tráfico crítico. Si no hay congestión, QoS no tiene efecto.

ProblemaDescripciónImpacto sin QoSTráfico sensible
BandwidthAncho de banda insuficiente para todo el tráficoTransferencias lentas, buffering en videoVideo streaming, backups
Delay/LatenciaTiempo en llegar del origen al destinoEco en llamadas, lag en videollamadasVoIP (<150ms)
JitterVariación del retardo entre paquetes consecutivosVoz entrecortada, video pixeladoVoIP (<30ms)
Packet LossPaquetes descartados por buffers llenosCortes en la voz, retransmisiones TCPVoIP (<1%)

🎨 Marcado de tráfico — DSCP y CoS

QoS funciona marcando los paquetes con una clase de prioridad en la cabecera, permitiendo que los dispositivos intermedios tomen decisiones de forwarding.

DSCP (Capa 3)

Differentiated Services Code Point. Campo de 6 bits en la cabecera IP. Rango: 0-63. Estándar RFC 2474.

CoS (Capa 2)

Class of Service. Campo de 3 bits en la cabecera 802.1Q (Ethernet). Rango: 0-7. Solo en tramas etiquetadas.

Clase DSCPValor DSCPNombreUso típico
EF46Expedited ForwardingVoIP, videoconferencia (máxima prioridad)
AF4134Assured Forwarding 4Video streaming, aplicaciones críticas
AF3126Assured Forwarding 3Call signaling, voz de señalización
AF2118Assured Forwarding 2Datos transaccionales importantes
AF1110Assured Forwarding 1Datos de menor prioridad
CS0/BE0Best EffortTráfico normal, sin prioridad

⚙️ Mecanismos QoS

MecanismoFunciónCuándo se usa
ClassificationIdentifica el tipo de tráfico (ACL, NBAR, DSCP)Primer paso siempre
MarkingMarca el paquete con DSCP/CoS para que otros dispositivos lo traten igualEn el borde de la red
Queuing (LLQ)Low Latency Queue: cola de prioridad estricta para VoIP + WFQ para el restoVoIP + datos
Congestion Avoidance (WRED)Descarta selectivamente paquetes ANTES de que el buffer se lleneTráfico TCP
PolicingDescarta o re-marca tráfico que excede el límiteEdge hacia ISP
ShapingRetrasa (no descarta) tráfico que excede el límite. Suaviza el tráfico.Hacia el ISP

📌 Policing vs Shaping

  • Policing: descarta o re-marca al instante → picos de tráfico = pérdida
  • Shaping: almacena en buffer y envía después → no pierde, pero añade delay

💻 MQC — Modular QoS CLI

El framework MQC usa 3 bloques para configurar QoS en Cisco IOS:

📌 Los 3 bloques MQC

  1. class-map: identifica/clasifica el tráfico
  2. policy-map: define qué hacer con cada clase (mark, queue, police)
  3. service-policy: aplica la policy-map a una interfaz

Configuración QoS para VoIP

! 1. Clasificar tráfico VoIP
R1(config)# class-map match-any VOIP-TRAFFIC
R1(config-cmap)# match protocol rtp audio
R1(config-cmap)# match dscp ef
! Clasificar tráfico de datos críticos
R1(config)# class-map match-any DATA-CRITICA
R1(config-cmap)# match dscp af41
! 2. Policy-map con LLQ
R1(config)# policy-map POLITICA-QOS
R1(config-pmap)# class VOIP-TRAFFIC
R1(config-pmap-c)# priority 512
! priority = LLQ (Low Latency Queue) en Kbps — prioridad estricta
R1(config-pmap)# class DATA-CRITICA
R1(config-pmap-c)# bandwidth percent 30
R1(config-pmap)# class class-default
R1(config-pmap-c)# fair-queue
! 3. Aplicar a la interfaz de salida (WAN)
R1(config)# interface GigabitEthernet0/1
R1(config-if)# service-policy output POLITICA-QOS

Verificación QoS

R1# show policy-map interface GigabitEthernet0/1

🧪 Laboratorio QoS — Marcado y Colas

Escenario: Priorizar VoIP y marcar tráfico web

! Clasificar y marcar tráfico
R1(config)# class-map match-all MARCAR-VOIP
R1(config-cmap)# match protocol rtp audio
R1(config)# class-map match-all MARCAR-HTTP
R1(config-cmap)# match protocol http
R1(config)# policy-map MARCAR-TRAFICO
R1(config-pmap)# class MARCAR-VOIP
R1(config-pmap-c)# set dscp ef
R1(config-pmap)# class MARCAR-HTTP
R1(config-pmap-c)# set dscp af21
! Aplicar en la interfaz LAN de entrada
R1(config)# interface GigabitEthernet0/0
R1(config-if)# service-policy input MARCAR-TRAFICO
R1# show policy-map interface GigabitEthernet0/0

📋 Resumen QoS

✅ Puntos críticos

  • QoS administra congestión — no crea ancho de banda
  • 4 problemas: Bandwidth, Delay, Jitter, Packet Loss
  • VoIP: delay <150ms · jitter <30ms · pérdida <1%
  • DSCP = 6 bits en IP (Capa 3) · CoS = 3 bits en 802.1Q (Capa 2)
  • EF (46) = VoIP · AF41 (34) = video · BE (0) = best effort
  • MQC: class-map → policy-map → service-policy
  • LLQ (priority): cola de prioridad estricta para VoIP
  • Policing: descarta el exceso · Shaping: retrasa el exceso
