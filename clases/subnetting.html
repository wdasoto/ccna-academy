Subnetting IPv4

Binario · Clases IPv4 · CIDR · Máscaras · Cálculo de Subredes · VLSM · Ejercicios

🔢 Fundamento — Sistema Binario

Una dirección IPv4 es un número de 32 bits dividido en 4 grupos de 8 bits (octetos), representados en decimal separados por puntos. Para dominar el subnetting es imprescindible entender la conversión entre binario y decimal.
VALOR DE CADA BIT EN UN OCTETO Bit: 128 64 32 16 8 4 2 1 Pos: 7 6 5 4 3 2 1 0   Ejemplo: 192 → 1 1 0 0 0 0 0 0 128 + 64 = 192 ✅   Ejemplo: 168 → 1 0 1 0 1 0 0 0 128 + 32 + 8 = 168 ✅   Ejemplo: 255 → 1 1 1 1 1 1 1 1 128+64+32+16+8+4+2+1 = 255 ✅

🔄 Conversión Rápida Decimal ↔ Binario

Método de las potencias: Para convertir decimal a binario, pregúntate de izquierda a derecha si el número restante es ≥ al valor del bit. Si sí → pon 1 y resta. Si no → pon 0.
CONVERTIR 172 A BINARIO Bits: 128 64 32 16 8 4 2 1   172 ≥ 128? Sí → 1 172-128 = 44 44 ≥ 64? No → 0 44 ≥ 32? Sí → 1 44-32 = 12 12 ≥ 16? No → 0 12 ≥ 8? Sí → 1 12-8 = 4 4 ≥ 4? Sí → 1 4-4 = 0 0 ≥ 2? No → 0 0 ≥ 1? No → 0   172 = 1010 1100
DIRECCIÓN IPv4 COMPLETA EN BINARIO 192 . 168 . 1 . 10 11000000.10101000.00000001.00001010   192 = 11000000 → 128+64 168 = 10101000 → 128+32+8 1 = 00000001 → 1 10 = 00001010 → 8+2

El sistema de clases fue el esquema original de IPv4 (classful). Aunque hoy se usa CIDR (classless), las clases siguen siendo fundamentales para el examen CCNA y para entender los rangos privados y las reglas básicas de direccionamiento.

⚠️
127.x.x.x — Loopback: El rango 127.0.0.0/8 está reservado para pruebas locales (loopback). La más famosa es 127.0.0.1. No pertenece a la Clase A aunque su primer octeto sea 127. El examen CCNA pregunta esto frecuentemente.

🏠 Rangos de Direcciones Privadas (RFC 1918)

Las IPs privadas son rangos reservados para uso interno de redes. No son enrutables en Internet — para salir a Internet necesitan NAT. Son gratuitas y se pueden reutilizar en cualquier red interna.

🔑 AND Lógico — Cómo el router halla la red

El router usa la operación AND lógico entre la IP y la máscara para obtener la dirección de red. Es la operación fundamental del subnetting. Regla: 1 AND 1 = 1 · cualquier AND con 0 = 0.
AND LÓGICO — ENCONTRAR DIRECCIÓN DE RED IP: 192.168.1.130 → 11000000.10101000.00000001.10000010 Máscara: 255.255.255.192 → 11111111.11111111.11111111.11000000 AND: ──────────────────────────────────── Red: 192.168.1.128 → 11000000.10101000.00000001.10000000   Conclusión: 192.168.1.130 pertenece a la red 192.168.1.128/26

🧮 Método de Cálculo Rápido — Magic Number

El Magic Number (número mágico) es la técnica más rápida para subnetting sin hacer cálculos binarios completos. Se calcula como 256 − octeto interesante de la máscara. El resultado es el incremento de bloque — el tamaño de cada subred.
MÉTODO MAGIC NUMBER Fórmula: Magic Number = 256 − [octeto interesante de la máscara]   Máscara /26 = 255.255.255.192 Octeto interesante = 192 → Magic Number = 256 - 192 = 64   Subredes de la red 192.168.1.0/26: 192.168.1.0 — 192.168.1.63 (red .0, broadcast .63) 192.168.1.64 — 192.168.1.127 (red .64, broadcast .127) 192.168.1.128 — 192.168.1.191 (red .128, broadcast .191) 192.168.1.192 — 192.168.1.255 (red .192, broadcast .255)
✅
Pasos del Magic Number: 1) Calcular 256 − máscara = bloque. 2) Listar bloques: 0, bloque, 2×bloque, 3×bloque... 3) Para cada bloque: Red = inicio, Broadcast = siguiente inicio − 1, Hosts útiles = Red+1 hasta Broadcast−1.

📋 Ejercicio Completo — Analizar una Red

Problema: Dada la dirección 172.16.45.14/20, determinar: dirección de red, broadcast, primera IP útil, última IP útil, número de hosts y número de subredes posibles.
RESOLUCIÓN PASO A PASO — 172.16.45.14/20 Paso 1: Identificar la máscara /20 = 255.255.240.0 → octeto interesante = tercer octeto (240)   Paso 2: Magic Number 256 - 240 = 16 (incremento de bloque)   Paso 3: Bloques en el tercer octeto 172.16.0.0 ... 172.16.15.255 172.16.16.0 ... 172.16.31.255 172.16.32.0 ... 172.16.47.255 ← 45 está aquí   Paso 4: Resultados Dirección de red: 172.16.32.0 Primera IP útil: 172.16.32.1 Última IP útil: 172.16.47.254 Broadcast: 172.16.47.255 Hosts útiles: 2^12 - 2 = 4.094 hosts Subredes posibles: 2^4 = 16 subredes en 172.16.0.0/16

🔀 VLSM — Variable Length Subnet Masking

VLSM permite usar máscaras de diferente longitud en distintas subredes dentro del mismo espacio de direcciones. Es el subnetting moderno y eficiente — evita el desperdicio de IPs asignando a cada segmento exactamente el tamaño que necesita.
✅
Regla de oro VLSM: Siempre empieza asignando la subred más grande primero, luego la siguiente más grande, y así sucesivamente. Esto maximiza el uso eficiente del espacio de direcciones.
🎯
CCNA Examen: VLSM es obligatorio para CCNA. Los protocolos que soportan VLSM: RIPv2, OSPF, EIGRP, IS-IS. El que NO soporta VLSM es RIPv1 (classful, no envía máscara en las actualizaciones).

📝 Ejercicios Resueltos — Nivel CCNA

Ejercicio 1: Dada la IP 10.1.200.35/21, determina todos los parámetros de la subred.
RESOLUCIÓN EJERCICIO 1 — 10.1.200.35/21 /21 → 255.255.248.0 → octeto interesante: tercer octeto (248) Magic Number: 256 - 248 = 8   Bloques en tercer octeto: 0, 8, 16 ... 192, 200, 208 ... 200 es inicio de bloque → siguiente bloque = 208   Dirección de red: 10.1.200.0/21 Primera IP útil: 10.1.200.1 Última IP útil: 10.1.207.254 Broadcast: 10.1.207.255 Hosts útiles: 2^11 - 2 = 2.046 hosts
Ejercicio 2: Necesitas una subred para exactamente 60 hosts. ¿Qué prefijo usas y cuál es la máscara?
RESOLUCIÓN EJERCICIO 2 — 60 HOSTS ¿Qué prefijo da ≥ 60 hosts útiles? /27 → 2^5 - 2 = 30 ← NO alcanza /26 → 2^6 - 2 = 62 ← ✅ primer prefijo que alcanza 60 /25 → 2^7 - 2 = 126 ← desperdicia mucho   Respuesta: /26 = 255.255.255.192
Ejercicio 3: ¿A qué subred /27 pertenece el host 192.168.5.200?
RESOLUCIÓN EJERCICIO 3 — SUBRED /27 /27 → 255.255.255.224 → Magic Number: 256 - 224 = 32   Bloques: 0, 32, 64, 96, 128, 160, 192, 224 ... 200 está entre 192 y 224 → pertenece al bloque que empieza en 192   Subred: 192.168.5.192/27 Broadcast: 192.168.5.223 Hosts: 192.168.5.193 → 192.168.5.222 (30 hosts) ¿.200 es válido? Sí ✅ (está entre .193 y .222)
Ejercicio 4: Divide la red 172.20.0.0/16 en subredes de al menos 500 hosts cada una. ¿Cuántas subredes obtienes?
RESOLUCIÓN EJERCICIO 4 — 500 HOSTS ¿Qué prefijo da ≥ 500 hosts? /23 → 2^9 - 2 = 510 ← ✅ primer prefijo que alcanza 500 /24 → 2^8 - 2 = 254 ← NO alcanza   ¿Cuántas subredes /23 caben en un /16? Bits prestados = 23 - 16 = 7 bits Subredes: 2^7 = 128 subredes /23 Hosts útiles: 510 por subred
