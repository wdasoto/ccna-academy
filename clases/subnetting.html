Subnetting IPv4

Binario · Clases IPv4 · CIDR · Máscaras · Cálculo · VLSM · Laboratorio · Quiz

CCNA 200-301 Módulo 1 · Fundamentos ⭐ Base de todo CCNA

🔢 ¿Qué es una dirección IPv4?

Una dirección IPv4 es un número de 32 bits dividido en 4 grupos de 8 bits llamados octetos, representados en decimal separados por puntos.

📌 Estructura básica

192.168.1.10 en binario → 11000000.10101000.00000001.00001010

Cada octeto vale entre 0 y 255 (2⁸ = 256 combinaciones posibles).

🔄 Sistema Binario — Fundamento crítico

Para dominar el subnetting es imprescindible entender la conversión decimal ↔ binario. El router trabaja solo en binario.

⚡ Valor de cada bit en un octeto (de mayor a menor)

Posición: 7 — 6 — 5 — 4 — 3 — 2 — 1 — 0

Valor: 128 — 64 — 32 — 16 — 8 — 4 — 2 — 1

Suma de todos los bits activos (en 1): 128+64+32+16+8+4+2+1 = 255

🎯 Método de conversión rápida: Potencias de 2

De izquierda a derecha: ¿el número restante ≥ valor del bit? Si SÍ → pon 1 y resta. Si NO → pon 0.

Ejemplo: 172 → binario

172≥128? Sí→1, resto=44 | 44≥64? No→0 | 44≥32? Sí→1, resto=12

12≥16? No→0 | 12≥8? Sí→1, resto=4 | 4≥4? Sí→1, resto=0 | 0≥2? No→0 | 0≥1? No→0

Resultado: 172 = 1010 1100

📦 Clases de Direcciones IPv4 (Classful)

Antes de CIDR, las IPs se asignaban por clases. El CCNA requiere conocerlas para identificar rangos privados y el comportamiento del routing classful.

ClaseRango 1er OctetoMáscara DefaultHosts útiles máx.Uso

🔒 Rangos Privados RFC 1918 — No enrutables en Internet

  • Clase A: 10.0.0.0 – 10.255.255.255  (10.0.0.0/8)
  • Clase B: 172.16.0.0 – 172.31.255.255  (172.16.0.0/12)
  • Clase C: 192.168.0.0 – 192.168.255.255  (192.168.0.0/16)
  • Loopback: 127.0.0.0/8  (127.0.0.1 = localhost)

⚠️ Trampas del examen CCNA

  • 127.x.x.x parece Clase A pero está reservado para loopback
  • 172.32.x.x NO es privado — el rango B privado termina en 172.31.x.x
  • 169.254.x.x = APIPA (autoconfiguration), no es privado RFC 1918

🎯 CIDR — Classless Inter-Domain Routing

CIDR elimina las limitaciones del sistema de clases. La máscara se expresa como prefijo /X indicando cuántos bits son de red. Permite asignar bloques exactos.

📌 Notación CIDR vs Decimal Punteada

192.168.1.0/24 = 192.168.1.0 mask 255.255.255.0

10.0.0.0/8 = 10.0.0.0 mask 255.0.0.0

172.16.0.0/12 = 172.16.0.0 mask 255.240.0.0

Wildcard mask = 255.255.255.255 − máscara  (se usa en ACLs y OSPF)

Ejemplo: /26 → máscara .192 → wildcard = 255 − 192 = 0.0.0.63

🎭 Máscara de Subred — Función y Tipos

La máscara de subred indica qué parte de una dirección IP identifica la red y qué parte identifica el host.

📌 Regla fundamental de los bits

Bits en 1 = parte de red (fija dentro de la subred)

Bits en 0 = parte de host (cambia para cada dispositivo)

📐 Fórmulas que debes memorizar

  • Hosts útiles = 2n − 2   (n = bits de host, −2 por dirección de red y broadcast)
  • Subredes = 2s   (s = bits tomados prestados del campo de host)
  • Magic Number / Incremento = 256 − valor del octeto de subred en la máscara

🔑 AND lógico — Cómo el router encuentra la red

Para saber a qué red pertenece una IP, el router aplica AND bit a bit entre la IP y la máscara.

💡 Ejemplo: 192.168.1.65 con máscara /26 (255.255.255.192)

IP: 1100 0000 . 1010 1000 . 0000 0001 . 0100 0001

Máscara: 1111 1111 . 1111 1111 . 1111 1111 . 1100 0000

AND: 1100 0000 . 1010 1000 . 0000 0001 . 0100 0000

→ Red: 192.168.1.64

🏠 Dirección de Red

Todos los bits de host en 0. No se asigna a ningún host. Identifica la subred en la tabla de routing.

Ejemplo: 192.168.1.64

📢 Broadcast

Todos los bits de host en 1. No se asigna a hosts. Llega a todos los dispositivos de la subred.

Ejemplo: 192.168.1.127

💻 Rango de Hosts

De (Red+1) hasta (Broadcast−1). Son las únicas IPs asignables a dispositivos.

Ejemplo: .65 – .126 (62 hosts)

🧮 Método del Magic Number

El Magic Number (incremento de bloque) es el método más rápido para calcular subredes sin aritmética binaria completa. Es el método preferido en el examen CCNA.

⚡ Magic Number = 256 − octeto_de_subred_en_la_máscara

  • /25 → máscara .128 → Magic = 256 − 128 = 128
  • /26 → máscara .192 → Magic = 256 − 192 = 64
  • /27 → máscara .224 → Magic = 256 − 224 = 32
  • /28 → máscara .240 → Magic = 256 − 240 = 16
  • /29 → máscara .248 → Magic = 256 − 248 = 8
  • /30 → máscara .252 → Magic = 256 − 252 = 4

Las subredes "saltan" de Magic en Magic: .0, .Magic, .Magic×2, .Magic×3...

📋 Ejercicio Completo Paso a Paso

🎯 Dado: 192.168.10.200/27 — Determinar todo sobre esta subred

Paso 1 — Identificar la máscara

/27 = 255.255.255.224

El octeto "interesante" (donde están los bits de subred) es el 4to: 224

Paso 2 — Calcular el Magic Number

Magic = 256 − 224 = 32

Las subredes del 4to octeto: 0, 32, 64, 96, 128, 160, 192, 224...

Paso 3 — Identificar en qué subred cae .200

192 ≤ 200 < 224 → La dirección de red es 192.168.10.192

Paso 4 — Calcular el Broadcast

Broadcast = próxima subred − 1 = 224 − 1 = 192.168.10.223

Paso 5 — Rango de hosts y resumen

Primer host: 192.168.10.193

Último host: 192.168.10.222

Hosts útiles: 2⁵ − 2 = 30 hosts

🔀 VLSM — Variable Length Subnet Masking

VLSM permite usar máscaras de longitud variable dentro de un mismo espacio de red, asignando el tamaño exacto que necesita cada subred. Es el estándar moderno.

⚠️ Classful vs Classless — Soporte de VLSM

NO soportan VLSM: RIPv1 (classful — no envía máscara en actualizaciones)

SÍ soportan VLSM: RIPv2, OSPF, EIGRP, IS-IS, BGP (classless — envían máscara)

📌 Regla de oro del diseño VLSM

Asignar siempre de mayor a menor: primero las subredes más grandes, luego las pequeñas. Así se evita el solapamiento y el desperdicio de espacio contiguo.

🏗️ Ejercicio VLSM Completo — Empresa 4 departamentos

📋 Requerimiento: espacio de red 192.168.1.0/24

  • Ventas: 50 hosts
  • Marketing: 25 hosts
  • IT: 10 hosts
  • Enlace WAN Router↔Router: 2 hosts

Paso 1 — Ordenar los requerimientos de mayor a menor

50 → 25 → 10 → 2

Luego elegir el prefijo mínimo que cubra cada uno:

50 hosts → /26 (62 útiles) | 25 → /27 (30) | 10 → /28 (14) | 2 → /30 (2)

Paso 2 — Ventas (50 hosts) → /26

Red: 192.168.1.0/26

Hosts: 192.168.1.1 – 192.168.1.62

Broadcast: 192.168.1.63

Siguiente bloque disponible: 192.168.1.64

Paso 3 — Marketing (25 hosts) → /27

Red: 192.168.1.64/27

Hosts: 192.168.1.65 – 192.168.1.94

Broadcast: 192.168.1.95

Siguiente bloque disponible: 192.168.1.96

Paso 4 — IT (10 hosts) → /28

Red: 192.168.1.96/28

Hosts: 192.168.1.97 – 192.168.1.110

Broadcast: 192.168.1.111

Siguiente bloque disponible: 192.168.1.112

Paso 5 — Enlace WAN → /30

Red: 192.168.1.112/30

Hosts: 192.168.1.113 – 192.168.1.114

Broadcast: 192.168.1.115

✅ Resultado final del diseño VLSM

Espacio utilizado: .0 – .115  |  Espacio libre: .116 – .255 (disponible para crecimiento)

Con máscara fija /26 en todas: hubieran sobrado 62 IPs en las redes de 25, 10 y 2 hosts = 186 IPs desperdiciadas.

🧪 Laboratorio Packet Tracer — VLSM en IOS Cisco

Escenario: Empresa con 3 departamentos. Red base: 172.16.0.0/24. Dos routers interconectados por enlace serial.

Diseño previo — Subredes VLSM

Dpto A (50 hosts): 172.16.0.0/26 → .1–.62

Dpto B (25 hosts): 172.16.0.64/27 → .65–.94

Dpto C (10 hosts): 172.16.0.96/28 → .97–.110

Enlace serial: 172.16.0.112/30 → .113 y .114

Configurar R1 — Gateway Dpto A + enlace serial

R1# configure terminal
! LAN hacia Dpto A
R1(config)# interface GigabitEthernet0/0
R1(config-if)# description LAN-DPTO-A-172.16.0.0/26
R1(config-if)# ip address 172.16.0.1 255.255.255.192
R1(config-if)# no shutdown
! Enlace serial hacia R2
R1(config)# interface Serial0/0/0
R1(config-if)# ip address 172.16.0.113 255.255.255.252
R1(config-if)# no shutdown
R1(config-if)# end
R1# write memory

Configurar R2 — Gateways Dpto B y C

R2# configure terminal
! Enlace serial hacia R1
R2(config)# interface Serial0/0/0
R2(config-if)# ip address 172.16.0.114 255.255.255.252
R2(config-if)# no shutdown
! LAN hacia Dpto B
R2(config)# interface GigabitEthernet0/0
R2(config-if)# ip address 172.16.0.65 255.255.255.224
R2(config-if)# description LAN-DPTO-B-172.16.0.64/27
R2(config-if)# no shutdown
! LAN hacia Dpto C
R2(config)# interface GigabitEthernet0/1
R2(config-if)# ip address 172.16.0.97 255.255.255.240
R2(config-if)# description LAN-DPTO-C-172.16.0.96/28
R2(config-if)# no shutdown
R2(config-if)# end

Rutas estáticas — Para que se comuniquen todas las subredes

! R1 necesita conocer los Dptos B y C (están detrás de R2)
R1(config)# ip route 172.16.0.64 255.255.255.224 172.16.0.114
R1(config)# ip route 172.16.0.96 255.255.255.240 172.16.0.114
! R2 necesita conocer el Dpto A (está detrás de R1)
R2(config)# ip route 172.16.0.0 255.255.255.192 172.16.0.113

📋 Resumen Ejecutivo — Todo lo que necesitas para el examen

✅ Fórmulas obligatorias

  • Hosts útiles = 2n − 2   (n = bits de host)
  • Subredes = 2s   (s = bits prestados de host)
  • Magic Number = 256 − octeto_de_subred_en_la_máscara
  • Wildcard = 255.255.255.255 − máscara
  • Broadcast = siguiente_red − 1 (o todos bits host en 1)

⚠️ Errores comunes en el examen CCNA

  • Olvidar restar 2 (red + broadcast) al calcular hosts
  • Confundir máscara de subred con wildcard mask
  • Creer que 172.32.x.x es privado — el rango B privado termina en 172.31.x.x
  • 127.x.x.x parece Clase A pero es loopback reservado
  • Usar RIPv1 con VLSM (solo funciona con classless)
  • No identificar correctamente el octeto "interesante" de la máscara
  • Confundir la dirección de red con el primer host asignable
