AC Ls

Estándar · Extendida · Nombrada · Wildcard · Implicit Deny · Filtrado de Tráfico

CCNA 200-301Módulo 6 · Seguridad🛡️ Control de Acceso

🛡️ ¿Qué es una ACL?

Una ACL (Access Control List) es una lista de instrucciones que el router evalúa en orden secuencial para permitir (permit) o denegar (deny) tráfico basándose en criterios definidos.

🔒 Filtrado de paquetes

Controla qué tráfico puede entrar o salir de una interfaz del router.

🎯 Doble uso

Las ACLs también se usan para identificar tráfico en NAT, QoS, VPNs y distribución de rutas.

📏 Regla fundamental

El router evalúa las ACEs en orden. Al primer match se ejecuta la acción y se detiene. Al final hay un implicit deny all.

⚠️ Implicit Deny All

Toda ACL tiene al final un deny any any implícito que no aparece en show running-config. Si no hay ningún permit, todo el tráfico es bloqueado.

📌 Regla de ubicación

  • ACL Estándar: ubicar lo más cercano al destino (filtra solo por IP origen)
  • ACL Extendida: ubicar lo más cercano al origen (filtra origen, destino, protocolo, puerto)

📌 Máximo 1 ACL por interfaz, dirección y protocolo

Se puede aplicar máximo 1 ACL IPv4 inbound Y 1 ACL IPv4 outbound por interfaz. No se pueden tener 2 ACLs en la misma dirección.

🔵 ACL Estándar — Solo filtra por IP origen

Las ACLs estándar solo pueden usar la dirección IP de origen como criterio de filtrado. Números: 1-99 y 1300-1999.

Wildcard especialSignificado
host 192.168.1.1Solo esa IP específica (equivale a 0.0.0.0 wildcard)
anyCualquier IP (equivale a 0.0.0.0 255.255.255.255)
192.168.1.0 0.0.0.255Toda la subred /24
! ACL 10: permite la red 192.168.1.0/24, deniega el resto
R1(config)# access-list 10 permit 192.168.1.0 0.0.0.255
R1(config)# access-list 10 deny any
! Aplicar en interfaz de SALIDA hacia el destino
R1(config)# interface GigabitEthernet0/1
R1(config-if)# ip access-group 10 out
! Verificar
R1# show access-lists
Standard IP access list 10
    10 permit 192.168.1.0, wildcard bits 0.0.0.255 (45 matches)
    20 deny   any (3 matches)

🔴 ACL Extendida — Control granular

Las ACLs extendidas filtran por IP origen, IP destino, protocolo y puerto. Números: 100-199 y 2000-2699.

! Sintaxis: access-list NUM [permit|deny] PROTO SRC [PORT] DST [PORT]
! Permitir HTTP desde red Ventas hacia cualquier servidor
R1(config)# access-list 110 permit tcp 192.168.1.0 0.0.0.255 any eq 80
! Permitir HTTPS
R1(config)# access-list 110 permit tcp 192.168.1.0 0.0.0.255 any eq 443
! Permitir DNS (UDP 53)
R1(config)# access-list 110 permit udp 192.168.1.0 0.0.0.255 any eq 53
! Bloquear todo el resto (explícito — ya existe implicit deny)
R1(config)# access-list 110 deny ip any any
! Aplicar INBOUND en la interfaz hacia el origen
R1(config)# interface GigabitEthernet0/0
R1(config-if)# ip access-group 110 in
Operador de puertoSignificado
eq 80Igual al puerto 80
neq 23No igual al puerto 23
lt 1024Menor que 1024
gt 1023Mayor que 1023
range 80 443Entre 80 y 443 inclusive

📝 ACL Nombrada

Las ACLs nombradas permiten usar nombres descriptivos y editar entradas individuales sin reescribir toda la ACL.

Crear ACL nombrada extendida

R1(config)# ip access-list extended BLOCK-TELNET
R1(config-ext-nacl)# 10 deny tcp any any eq 23
R1(config-ext-nacl)# 20 permit ip any any
R1(config)# interface GigabitEthernet0/0
R1(config-if)# ip access-group BLOCK-TELNET in

Editar una ACL nombrada — sin borrar toda la lista

! Insertar nueva ACE entre la 10 y la 20
R1(config)# ip access-list extended BLOCK-TELNET
R1(config-ext-nacl)# 15 deny tcp any any eq 22
! Eliminar una ACE específica
R1(config-ext-nacl)# no 15
! Verificar la ACL con números de secuencia
R1# show ip access-lists BLOCK-TELNET
Extended IP access list BLOCK-TELNET
    10 deny tcp any any eq telnet
    20 permit ip any any

🧪 Laboratorio ACLs

Escenario: Red Ventas no debe acceder a Red IT, pero sí a Internet

! Topología: Ventas(192.168.1.0/24)─R1─IT(10.10.10.0/24) e Internet
! ACL extendida: bloquea acceso de Ventas a IT, permite Internet
R1(config)# ip access-list extended VENTAS-POLICY
R1(config-ext-nacl)# 10 deny ip 192.168.1.0 0.0.0.255 10.10.10.0 0.0.0.255
R1(config-ext-nacl)# 20 permit ip 192.168.1.0 0.0.0.255 any
! Aplicar en la interfaz de entrada (desde Ventas)
R1(config)# interface GigabitEthernet0/0
R1(config-if)# ip access-group VENTAS-POLICY in
! Verificar
R1# show ip access-lists
! Probar desde PC Ventas
! ping 10.10.10.1 → U.U.U (denegado)
! ping 8.8.8.8 → !!!!! (permitido)

📋 Resumen ACLs

✅ Puntos críticos

  • ACL = lista de ACEs evaluadas en orden → primer match gana
  • Implicit deny all: al final de toda ACL (no visible en config)
  • Máximo 1 ACL por interfaz, dirección y protocolo
  • ACL Estándar (1-99): solo IP origen → ubicar cerca del destino
  • ACL Extendida (100-199): origen+destino+protocolo+puerto → cerca del origen
  • ACL Nombrada: permite editar ACEs individuales con número de secuencia
  • Wildcard: host = /32 | any = todo | subred + wildcard
  • show ip access-lists = ver ACL con contador de matches
