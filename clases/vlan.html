VLANs

Virtual LAN · Segmentación Lógica · Trunking 802.1Q · Inter-VLAN Routing

💡 ¿Qué es una VLAN?

Una VLAN (Virtual Local Area Network) es una red lógica creada dentro de uno o varios switches físicos que permite agrupar dispositivos en dominios de broadcast separados, independientemente de su ubicación física en la red.
🎯
Regla fundamental: Los dispositivos en diferentes VLANs NO pueden comunicarse entre sí directamente. Necesitan un dispositivo de Capa 3 (router o switch multilayer). Las VLANs operan en la Capa 2 del modelo OSI.

✅ Beneficios de las VLANs

BeneficioDescripción
SeguridadAísla departamentos sensibles. Un ataque en VLAN Ventas no llega a VLAN Contabilidad.
RendimientoReduce el dominio de broadcast. Menos tráfico innecesario para todos los dispositivos.
FlexibilidadReorganiza la red lógicamente sin mover cables físicos.
AhorroUn switch físico actúa como múltiples switches lógicos.
🎯
Rangos CCNA: Normal: 1–1005 · Extendido: 1006–4094 · Reservadas: 1002–1005 (Token Ring/FDDI, no usar) · Default: VLAN 1 (no eliminar, no usar para datos)

🏷️ Tipos de VLANs

TipoDescripciónRecomendación
Default (VLAN 1)Todos los puertos la usan por defecto. No se puede borrar.❌ Nunca usar para datos de usuario
Data VLANTransporta tráfico de usuarios finales. Ej: VLAN 10=Ventas, VLAN 20=TI✅ Configurar para cada departamento
Native VLANTráfico sin etiquetar 802.1Q en trunk. Default VLAN 1.⚠️ Cambiar a VLAN sin uso (ej: 99)
Management VLANAcceso SSH/Telnet al switch. Asignada a la SVI.✅ Usar VLAN dedicada (ej: 99)
Voice VLANSepara tráfico VoIP del de datos. QoS automático.✅ Siempre separar VoIP

🔗 Trunk Links — IEEE 802.1Q

Un trunk es un enlace que transporta múltiples VLANs simultáneamente. Usa el estándar IEEE 802.1Q que inserta una etiqueta de 4 bytes en la trama Ethernet con el VLAN ID (12 bits → valores 1–4094).
🎯
Native VLAN: El tráfico de la Native VLAN viaja SIN etiqueta en el trunk. Debe coincidir en ambos extremos o causará un Native VLAN mismatch (error CDP y posible VLAN hopping).
⚠️
DTP (Dynamic Trunking Protocol): Protocolo Cisco propietario que negocia trunks automáticamente. Por seguridad, desactívalo con switchport nonegotiate en puertos de acceso y trunks hacia no-Cisco.

⌨️ Configuración Completa de VLANs

CREAR VLANs Switch(config)# vlan 10 Switch(config-vlan)# name VENTAS Switch(config-vlan)# exit Switch(config)# vlan 20 Switch(config-vlan)# name TECNOLOGIA
PUERTOS ACCESS Switch(config)# interface fastEthernet 0/1 Switch(config-if)# switchport mode access Switch(config-if)# switchport access vlan 10 Switch(config-if)# no shutdown
PUERTO TRUNK Switch(config)# interface gigabitEthernet 0/1 Switch(config-if)# switchport mode trunk Switch(config-if)# switchport trunk native vlan 99 Switch(config-if)# switchport trunk allowed vlan 10,20,30,99 Switch(config-if)# switchport nonegotiate
VERIFICACIÓN Switch# show vlan brief Switch# show interfaces trunk Switch# show interfaces fa0/1 switchport

🔀 Inter-VLAN Routing

Para que dos VLANs se comuniquen necesitas un dispositivo de Capa 3. Los métodos son: Router-on-a-Stick (ROAS) usando subinterfaces en un router, o un Switch Multilayer (L3) con SVIs y ip routing.
ROUTER-ON-A-STICK Router(config)# interface GigabitEthernet0/0.10 Router(config-subif)# encapsulation dot1Q 10 Router(config-subif)# ip address 192.168.10.1 255.255.255.0
SWITCH MULTILAYER (L3) MLS(config)# ip routing MLS(config)# interface vlan 10 MLS(config-if)# ip address 192.168.10.1 255.255.255.0 MLS(config-if)# no shutdown
