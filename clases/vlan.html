VLAN s

Virtual LAN · Access · Trunk · 802.1Q · VLAN Nativa · Inter-VLAN

CCNA 200-301Módulo 3 · Switching🏷️ Segmentación

🏷️ ¿Qué es una VLAN?

Una VLAN (Virtual Local Area Network) es una segmentación lógica de una red LAN que permite dividir un switch físico en múltiples redes virtuales independientes, cada una con su propio dominio de broadcast.

🔒 Seguridad

Dispositivos en VLANs diferentes no se pueden comunicar directamente sin pasar por un router (o switch capa 3).

⚡ Rendimiento

Reduce el tamaño del dominio de broadcast, disminuyendo el tráfico innecesario en cada segmento.

🏗️ Flexibilidad

Agrupa usuarios por función (Ventas, IT, RRHH) sin importar su ubicación física en la red.

📌 VLANs reservadas por defecto en Cisco

  • VLAN 1 — VLAN nativa por defecto. Todos los puertos la usan por defecto. No eliminar.
  • VLAN 1002–1005 — Reservadas para Token Ring y FDDI. No se pueden usar ni eliminar.
  • VLAN 1006–4094 — VLANs extendidas (requieren VTP transparent o off).

🔌 Tipos de Puerto en un Switch

TipoDescripciónTráficoUso típico

Trunk PortTransporta tráfico de múltiples VLANs simultáneamente usando etiquetas 802.1Q.Con etiqueta 802.1QUplinks switch↔switch, switch↔router

⚠️ 802.1Q — Encapsulación de trunk

El estándar 802.1Q agrega un campo de 4 bytes al frame Ethernet con el ID de VLAN (12 bits → hasta 4094 VLANs). ISL es el protocolo propietario de Cisco (obsoleto). El CCNA solo cubre 802.1Q.

⚙️ Configuración de VLANs

Crear VLANs y asignar puertos access

! Crear VLANs
SW1(config)# vlan 10
SW1(config-vlan)# name VENTAS
SW1(config-vlan)# exit
SW1(config)# vlan 20
SW1(config-vlan)# name IT
SW1(config-vlan)# exit
SW1(config)# vlan 30
SW1(config-vlan)# name RRHH
SW1(config-vlan)# exit
! Asignar puertos a VLANs (modo access)
SW1(config)# interface FastEthernet0/1
SW1(config-if)# switchport mode access
SW1(config-if)# switchport access vlan 10
SW1(config)# interface range FastEthernet0/2-5
SW1(config-if-range)# switchport mode access
SW1(config-if-range)# switchport access vlan 20

Verificar VLANs

🔗 Configuración Trunk 802.1Q

Configurar enlace trunk entre switches

! En el puerto de uplink hacia otro switch o router
SW1(config)# interface GigabitEthernet0/1
SW1(config-if)# switchport mode trunk
SW1(config-if)# switchport trunk encapsulation dot1q
! Especificar qué VLANs se permiten en el trunk
SW1(config-if)# switchport trunk allowed vlan 10,20,30
! Cambiar la VLAN nativa (por seguridad, no usar VLAN 1)
SW1(config-if)# switchport trunk native vlan 99

Verificar trunk

SW1# show interfaces trunk
Port      Mode  Encapsulation  Status    Native vlan
Gi0/1     on    802.1q         trunking  99

Port      Vlans allowed on trunk
Gi0/1     10,20,30

Port      Vlans in spanning tree forwarding state
Gi0/1     10,20,30

⚠️ VLAN nativa — Riesgo de seguridad

Si dos switches tienen distinta VLAN nativa en un trunk, los frames se envían sin etiqueta a VLANs distintas (VLAN hopping attack). Siempre sincronizar la VLAN nativa y cambiarla de la VLAN 1.

🧪 Laboratorio VLANs completo

📋 Topología: SW1 ─── trunk ─── SW2, cada uno con Ventas (10) e IT (20)

SW1 y SW2 — Configuración completa

! En ambos switches — crear las VLANs
SW(config)# vlan 10
SW(config-vlan)# name VENTAS
SW(config)# vlan 20
SW(config-vlan)# name IT
! Puertos de acceso
SW(config)# interface FastEthernet0/1
SW(config-if)# switchport mode access
SW(config-if)# switchport access vlan 10
SW(config)# interface FastEthernet0/2
SW(config-if)# switchport mode access
SW(config-if)# switchport access vlan 20
! Puerto trunk entre SW1 y SW2
SW(config)# interface GigabitEthernet0/1
SW(config-if)# switchport mode trunk
SW(config-if)# switchport trunk allowed vlan 10,20
SW(config-if)# switchport trunk native vlan 99
SW# copy running-config startup-config

📋 Resumen — VLANs para el CCNA

✅ Puntos críticos

  • VLAN = segmentación lógica → cada VLAN es un dominio de broadcast separado
  • Access port = 1 VLAN, sin etiqueta | Trunk = múltiples VLANs con etiqueta 802.1Q
  • VLAN 1 = nativa y por defecto. No eliminar pero sí cambiar para seguridad
  • VLANs 1002-1005 = reservadas, no usar
  • Trunk nativa debe coincidir en ambos lados → si no, hay VLAN hopping
  • switchport trunk allowed vlan controla qué VLANs pasan por el trunk
  • show vlan brief = resumen de VLANs y puertos | show interfaces trunk = ver trunks

⚠️ Errores comunes

  • Crear una VLAN sin asignarla a ningún puerto (existe pero no tiene tráfico)
  • No agregar la VLAN al trunk → el tráfico de esa VLAN no pasa
  • VLAN nativa diferente en cada extremo del trunk
