Python Esencial para Redes

Domina los fundamentos de Python orientados 100% a networking. Variables, strings, listas, diccionarios, loops y funciones — todos con ejemplos reales de IPs, configuraciones Cisco y automatización de red.

🐍 Python 3.x 🎯 CCNA Domain 6.1 ⏱ ~75 min lectura 💻 Ejercicios interactivos 🧪 Proyecto incluido

¿Por qué Python para Redes?

Los ingenieros de redes que no saben programar están siendo reemplazados por los que sí saben. No es exageración — es lo que está pasando en la industria ahora mismo. Python es el lenguaje elegido por Cisco, por sus equipos de automatización y por el 85% de los scripts de red en producción.

La razón es simple: Python tiene una sintaxis tan clara que se lee casi como inglés, tiene librerías especializadas para cada protocolo de red (netmiko, napalm, ncclient, requests) y puedes aprender lo suficiente para automatizar tareas reales en una semana.

💡 Lo que aprenderás en esta clase

Todos los conceptos fundamentales de Python orientados al trabajo real en redes: variables para guardar IPs y hostnames, strings para construir comandos IOS, listas para inventarios de dispositivos, diccionarios para guardar configuraciones completas, loops para iterar sobre todos los routers, y funciones para encapsular tareas reutilizables. Cada concepto viene con código de red real.

Tu primer script de red

Antes de entrar en detalles, mira cómo se ve Python para redes en la práctica. Este script conecta a un router y obtiene la tabla de routing:

primer_script.py — El resultado de esta clase
# Lo que podrás hacer al terminar esta clase
from netmiko import ConnectHandler

# Variables — guardan datos del dispositivo
hostname  = "R1-CORE"
ip_mgmt   = "192.168.1.1"
usuario   = "admin"
password  = "Cisco123"

# Diccionario — describe el dispositivo completo
dispositivo = {
    "device_type": "cisco_ios",
    "host":        ip_mgmt,
    "username":    usuario,
    "password":    password,
}

# Lista de routers a verificar
routers = ["192.168.1.1", "192.168.1.2", "192.168.1.3"]

# Función — tarea reutilizable
def verificar_router(ip):
    dispositivo["host"] = ip
    with ConnectHandler(**dispositivo) as conn:
        output = conn.send_command("show ip route")
        return output

# Loop — repite para cada router
for router_ip in routers:
    resultado = verificar_router(router_ip)
    print(f"--- Router {router_ip} ---")
    print(resultado)

💡 Convención de nombres en Python para redes
  • Variables: snake_case → ip_gateway, vlan_datos

Strings — Construir comandos y parsear outputs

Los strings son el tipo más usado en automatización de redes. Todo lo que recibes de un dispositivo (output de show commands) es un string. Todo lo que envías (comandos IOS) es un string. Dominar los strings es dominar la comunicación con los dispositivos.

Parsear output de dispositivos Cisco

La tarea más frecuente en automatización: recibir el texto de un show command y extraer la información que necesitas. Python tiene herramientas poderosas para esto:

parsear_output.py — Extraer datos de show commands
# ── ESCENARIO REAL: parsear "show ip interface brief" ──
output_show = """
Interface              IP-Address      OK? Method Status                Protocol
GigabitEthernet0/0    192.168.1.1     YES manual up                    up
GigabitEthernet0/1    10.0.0.1        YES manual up                    up
GigabitEthernet0/2    unassigned      YES unset  administratively down down
Loopback0             1.1.1.1         YES manual up                    up
"""

# Método 1: split() y procesamiento línea a línea
lineas = output_show.strip().split("\n")
print(f"Total de líneas: {len(lineas)}")  # 5

for linea in lineas:
    if "up" in linea and "down" not in linea:
        campos = linea.split()
        interfaz = campos[0]
        ip       = campos[1]
        print(f"✅ {interfaz:25} → {ip}")

# Output:
# ✅ GigabitEthernet0/0      → 192.168.1.1
# ✅ GigabitEthernet0/1      → 10.0.0.1
# ✅ Loopback0               → 1.1.1.1

# ── Método 2: "in" para buscar keywords ──────────────
if "administratively down" in output_show:
    print("⚠️  Hay interfaces administrativamente apagadas")

# ── Método 3: find() para encontrar posición ─────────
pos = output_show.find("Loopback0")
print(f"Loopback0 encontrado en posición: {pos}")

# ── Contar interfaces up ─────────────────────────────
interfaces_up   = output_show.count(" up                    up")
interfaces_down = output_show.count(" down")
print(f"Interfaces UP: {interfaces_up} | DOWN: {interfaces_down}")
💡 Truco pro: format spec en f-strings para tablas alineadas

Usa f"{variable:<ancho}" para alinear texto y crear reportes con columnas perfectas. Ejemplo: f"{hostname:<20} {ip:<16} {estado:>10}" — ideal para reportes de inventario.

Listas — Inventarios de dispositivos y comandos

Una lista es una colección ordenada y mutable de elementos. En networking, las listas son fundamentales: una lista de IPs de routers, una lista de comandos a ejecutar, una lista de VLANs a configurar. Con una lista puedes iterar sobre todos los dispositivos de tu red en segundos.

Diccionarios — Configuraciones completas de dispositivos

Un diccionario almacena pares clave: valor. Es la estructura de datos más importante para networking — prácticamente todas las APIs de red (Netmiko, DNA Center, Meraki) usan diccionarios para describir dispositivos. Un router es un diccionario con claves como "host", "username", "device_type".

Loops — Iterar sobre toda tu infraestructura

El superpoder real de Python para redes: un loop for que recorre 500 routers ejecuta la misma tarea 500 veces mientras tú tomas café. Lo que manualmente tomaría días, con un loop tarda segundos. Este es el concepto más transformador de la automatización.

loops_red.py — Iterar sobre dispositivos e infraestructura
# ── FOR LOOP — El más usado en networking ─────────────

# Caso 1: Iterar sobre lista de IPs
routers = ["192.168.1.1", "192.168.1.2", "192.168.1.3"]
for ip in routers:
    print(f"Conectando a {ip}...")
    # aquí iría: conn = ConnectHandler(host=ip, ...)

# Caso 2: enumerate() — obtener índice y valor
for num, ip in enumerate(routers, start=1):
    print(f"  [{num}/{len(routers)}] Procesando {ip}")
# [1/3] Procesando 192.168.1.1
# [2/3] Procesando 192.168.1.2
# [3/3] Procesando 192.168.1.3

# Caso 3: range() — generar secuencias
for vlan in range(10, 51, 10):   # 10, 20, 30, 40, 50
    print(f"vlan {vlan}")

# Caso 4: Iterar sobre diccionario
red = {"R1": "10.0.0.1", "R2": "10.0.0.2", "R3": "10.0.0.3"}
for hostname, ip in red.items():
    print(f"{hostname} → {ip}")

# Caso 5: zip() — combinar dos listas
hostnames = ["R1", "R2", "R3"]
ips       = ["10.0.0.1", "10.0.0.2", "10.0.0.3"]
for host, ip in zip(hostnames, ips):
    print(f"{host:<5} → {ip}")

# ── WHILE LOOP — Para reintentos y esperas ────────────
MAX_REINTENTOS = 3
intento = 0
conectado = False

while intento < MAX_REINTENTOS and not conectado:
    intento += 1
    print(f"Intento {intento} de {MAX_REINTENTOS}...")
    # simulación — en código real sería ConnectHandler
    if intento == 2:
        conectado = True
        print("✅ Conexión establecida")

if not conectado:
    print("❌ Fallo tras máximos reintentos")

# ── CONTROL DE FLUJO: break, continue, else ──────────
for ip in routers:
    if ip == "192.168.1.2":
        print(f"⏭️  Saltando {ip} (en mantenimiento)")
        continue    # Ir al siguiente elemento
    if ip == "192.168.1.99":
        break       # Salir del loop completamente
    print(f"Procesando {ip}")

# ── FUNCIÓN DE VALIDACIÓN DE IP ───────────────────────
def es_ip_valida(ip):
    """Valida si una cadena es una dirección IPv4 válida"""
    try:
        partes = ip.split(".")
        if len(partes) != 4:
            return False
        return all(0 <= int(p) <= 255 for p in partes)
    except ValueError:
        return False

print(es_ip_valida("192.168.1.1"))    # True
print(es_ip_valida("999.0.0.1"))     # False
print(es_ip_valida("abc"))           # False

# ── FUNCIÓN: CONVERTIR MÁSCARA A PREFIJO ─────────────
def mascara_a_prefijo(mascara):
    """Convierte 255.255.255.0 → /24"""
    octetos = [int(o) for o in mascara.split(".")]
    bits = sum(bin(o).count("1") for o in octetos)
    return bits

print(mascara_a_prefijo("255.255.255.0"))    # 24
print(mascara_a_prefijo("255.255.255.192"))  # 26
print(mascara_a_prefijo("255.255.255.252"))  # 30

#!/usr/bin/env python3
"""
inventario_red.py
─────────────────────────────────────────────────
Genera un reporte completo del inventario de red:
  - Valida direcciones IP del inventario
  - Calcula hosts disponibles por subred
  - Genera comandos de configuración base
  - Produce reporte formateado en consola
─────────────────────────────────────────────────
"""

# ══════════════════════════════════════════════════════
# SECCIÓN 1 — INVENTARIO DE DISPOSITIVOS (DICCIONARIOS)
# ══════════════════════════════════════════════════════
INVENTARIO = [
    {"hostname": "R1-CORE",  "ip": "10.0.0.1",  "tipo": "router",  "red": "10.0.0.0/30"},
    {"hostname": "R2-DIST",  "ip": "10.0.0.5",  "tipo": "router",  "red": "10.0.0.4/30"},
    {"hostname": "SW1-ACC",  "ip": "172.16.0.1", "tipo": "switch",  "red": "172.16.0.0/24"},
    {"hostname": "SW2-ACC",  "ip": "172.16.0.2", "tipo": "switch",  "red": "172.16.0.0/24"},
    {"hostname": "FW1-BORDE", "ip": "999.x.y.z", "tipo": "firewall", "red": "192.168.0.0/30"},
]

# VLANs de la organización
VLANS_ORG = {10: "DATOS", 20: "VOZ", 30: "MGMT", 40: "SERVERS"}

# ══════════════════════════════════════════════════════
# SECCIÓN 2 — FUNCIONES UTILITARIAS
# ══════════════════════════════════════════════════════

def validar_ip(ip):
    """Retorna True si la IP es válida"""
    try:
        partes = ip.split(".")
        return len(partes) == 4 and all(0 <= int(p) <= 255 for p in partes)
    except:
        return False

def calcular_hosts(prefijo):
    """Hosts útiles dado un prefijo CIDR"""
    return 2**(32 - prefijo) - 2

def generar_config_base(hostname, ip, vlans):
    """Genera lista de comandos de configuración base IOS"""
    config = [
        f"hostname {hostname}",
        f"enable secret Cisco@{hostname[-2:]}",
        "service password-encryption",
        "ip ssh version 2",
        "no cdp run",
        "banner motd ^*** ACCESO AUTORIZADO SOLO ***^",
    ]
    for vlan, nombre in vlans.items():
        config.append(f"vlan {vlan}")
        config.append(f" name {nombre}")
    return config

def imprimir_separador(titulo="", ancho=60):
    """Imprime separador visual para el reporte"""
    if titulo:
        pad = (ancho - len(titulo) - 2) // 2
        print(f"\n{'─'*pad} {titulo} {'─'*pad}")
    else:
        print("─" * ancho)

# ══════════════════════════════════════════════════════
# SECCIÓN 3 — LÓGICA PRINCIPAL
# ══════════════════════════════════════════════════════

def main():
    validos   = []
    invalidos = []

    imprimir_separador("CCNA Academy — Reporte de Inventario de Red")

    # ── PASO 1: Validar IPs (loop + condicional) ─────────
    imprimir_separador("PASO 1: Validación de Direcciones IP")
    print(f"{'Hostname':<12} {'IP':<14} {'Estado':<8} {'Tipo'}")
    imprimir_separador()

    for disp in INVENTARIO:
        ok = validar_ip(disp["ip"])
        estado = "✅ OK" if ok else "❌ INVÁLIDA"
        print(f"{disp['hostname']:<12} {disp['ip']:<14} {estado:<12} {disp['tipo']}")
        (validos if ok else invalidos).append(disp)

    print(f"\n  ✅ Válidos: {len(validos)}  ❌ Inválidos: {len(invalidos)}")

    # ── PASO 2: Calcular información de subredes ─────────
    imprimir_separador("PASO 2: Análisis de Subredes")
    redes_vistas = []

    for disp in validos:
        red = disp["red"]
        if red not in redes_vistas:
            redes_vistas.append(red)
            prefijo = int(red.split("/")[1])
            hosts   = calcular_hosts(prefijo)
            print(f"  Red: {red:<20} → {hosts} hosts útiles")

    # ── PASO 3: Generar configuración del primer router ──
    imprimir_separador("PASO 3: Config Base Generada (R1-CORE)")
    r1 = validos[0]
    config = generar_config_base(r1["hostname"], r1["ip"], VLANS_ORG)
    for i, cmd in enumerate(config, 1):
        print(f"  {i:2}. {cmd}")

    # ── PASO 4: Resumen ejecutivo ────────────────────────
    imprimir_separador("RESUMEN EJECUTIVO")
    print(f"  Total dispositivos : {len(INVENTARIO)}")
    print(f"  IPs válidas        : {len(validos)}")
    print(f"  IPs inválidas      : {len(invalidos)}")
    print(f"  Redes detectadas   : {len(redes_vistas)}")
    print(f"  VLANs organizacion : {len(VLANS_ORG)}")
    print(f"  Comandos generados : {len(config)}")
    imprimir_separador()

# Punto de entrada del script
if __name__ == "__main__":
    main()
