DH CP

DORA · Pool · excluded-address · ip helper-address · DHCP Snooping · Relay Agent

CCNA 200-301Módulo 5 · Servicios IP📋 Asignación dinámica

📋 ¿Qué es DHCP?

DHCP (Dynamic Host Configuration Protocol) asigna automáticamente parámetros de red (IP, máscara, gateway, DNS) a los clientes, eliminando la configuración manual.

🔑 Parámetros asignados

  • Dirección IP
  • Máscara de subred
  • Default gateway
  • Servidores DNS
  • Lease time (tiempo de arrendamiento)

📌 Características

Protocolo UDP: cliente usa puerto 68, servidor usa puerto 67. Funciona por broadcast en la red local.

🌐 Funcionamiento

El cliente no tiene IP → usa broadcast para encontrar un servidor DHCP → servidor asigna IP del pool configurado.

🔄 Proceso DORA — 4 mensajes

PasoMensajeTipoDescripción
1DISCOVERBroadcastCliente busca servidores DHCP disponibles en la red
2OFFERBroadcast/UnicastServidor ofrece una IP disponible del pool + parámetros
3REQUESTBroadcastCliente acepta la oferta y solicita formalmente la IP
4ACKBroadcast/UnicastServidor confirma el arrendamiento → cliente configura la IP

♻️ Renovación del lease

  • Al 50% del lease: cliente envía REQUEST unicast al servidor para renovar
  • Al 87.5% del lease: si no responde, broadcast REQUEST a cualquier servidor
  • Al 100%: IP expirada → reinicia proceso DORA

⚙️ DHCP Server en Cisco IOS

Configurar pool DHCP

! Excluir IPs reservadas (gateways, servidores, impresoras)
R1(config)# ip dhcp excluded-address 192.168.1.1 192.168.1.20
! Crear pool DHCP
R1(config)# ip dhcp pool LAN-VENTAS
R1(dhcp-config)# network 192.168.1.0 255.255.255.0
R1(dhcp-config)# default-router 192.168.1.1
R1(dhcp-config)# dns-server 8.8.8.8 8.8.4.4
R1(dhcp-config)# domain-name empresa.com
R1(dhcp-config)# lease 7
! lease días [horas] [minutos]

📡 DHCP Relay Agent

DHCP usa broadcast, que los routers no reenvían. El DHCP Relay Agent permite que clientes en una subred diferente usen un servidor DHCP centralizado convirtiendo el broadcast en unicast.

🔄 Funcionamiento del Relay

Cliente broadcast → Router (relay) intercepta el DISCOVER → lo reenvía como unicast al servidor DHCP remoto → el servidor responde al relay → el relay reenvía al cliente.

Configurar DHCP Relay

! Topología: Cliente(192.168.2.x)─R1─Servidor DHCP(10.0.0.100)
! En la interfaz LAN del router (hacia los clientes)
R1(config)# interface GigabitEthernet0/1
R1(config-if)# ip helper-address 10.0.0.100
! ip helper-address convierte broadcast DHCP en unicast
! hacia la IP del servidor DHCP

Snooping DHCP — Seguridad en switches

! Previene rogue DHCP servers
SW(config)# ip dhcp snooping
SW(config)# ip dhcp snooping vlan 10
! Puerto trusted: hacia el servidor DHCP legítimo
SW(config)# interface GigabitEthernet0/1
SW(config-if)# ip dhcp snooping trust
! Puertos untrusted (default): hacia clientes
! El switch descarta DHCP Offers/Acks de puertos untrusted

🧪 Laboratorio DHCP completo

R1 como servidor DHCP para 2 LANs

! Pool para VLAN 10 — Ventas
R1(config)# ip dhcp excluded-address 192.168.10.1 192.168.10.10
R1(config)# ip dhcp pool VLAN10-VENTAS
R1(dhcp-config)# network 192.168.10.0 255.255.255.0
R1(dhcp-config)# default-router 192.168.10.1
R1(dhcp-config)# dns-server 8.8.8.8
R1(dhcp-config)# lease 1
! Pool para VLAN 20 — IT
R1(config)# ip dhcp excluded-address 192.168.20.1 192.168.20.10
R1(config)# ip dhcp pool VLAN20-IT
R1(dhcp-config)# network 192.168.20.0 255.255.255.0
R1(dhcp-config)# default-router 192.168.20.1
R1(dhcp-config)# dns-server 8.8.8.8
! Verificar asignaciones
R1# show ip dhcp binding
R1# show ip dhcp pool

📋 Resumen DHCP

✅ Puntos críticos

  • DORA: Discover(broadcast) → Offer → Request(broadcast) → ACK
  • Puertos: servidor UDP 67, cliente UDP 68
  • ip dhcp excluded-address = reservar IPs antes de crear el pool
  • ip helper-address = DHCP relay para subredes remotas
  • DHCP Snooping: puertos trusted=servidor legítimo | untrusted=clientes
  • show ip dhcp binding = ver IPs asignadas y clientes
  • show ip dhcp conflict = IPs en conflicto
