📐 El Modelo OSI — 7 Capas

El Modelo OSI (Open Systems Interconnection) es un marco de referencia que divide las funciones de red en 7 capas independientes. Cada capa tiene una función específica y se comunica solo con las capas adyacentes. Fue creado por la ISO en 1984 y es la base conceptual para entender cualquier protocolo de red.

# Capa Función Protocolos / Dispositivos PDU
7 Aplicación Interfaz con el usuario final. Servicios de red a aplicaciones. HTTP · HTTPS · FTP · DNS · DHCP · SMTP · SSH · Telnet Datos
6 Presentación Formato, cifrado/descifrado, compresión de datos. SSL/TLS · JPEG · MPEG · ASCII · UTF-8 Datos
5 Sesión Establece, gestiona y cierra sesiones entre aplicaciones. NetBIOS · RPC · SQL Datos
4 Transporte Transporte extremo a extremo, control de flujo, corrección de errores. TCP · UDP Segmento
3 Red Direccionamiento lógico (IP), routing entre redes. IPv4 · IPv6 · ICMP · OSPF · EIGRP · Routers Paquete
2 Enlace de Datos Direccionamiento físico (MAC), detección de errores, frame switching. Ethernet · 802.11 Wi-Fi · VLANs · Switches Trama
1 Física Transmisión de bits por el medio físico. Voltajes, señales, cables. Cables UTP/STP · Fibra óptica · Wi-Fi · Hubs · Repetidores Bit

🎯 Mnemotecnia CCNA

De 7 a 1 (bajando): "All People Seem To Need Data Processing"
(Aplicación, Presentación, Sesión, Transporte, Red, Datos, Física)

De 1 a 7 (subiendo): "Please Do Not Throw Sausage Pizza Away"

🔄 Encapsulación y Desencapsulación

Cuando un dato viaja de una PC hacia la red, cada capa del modelo OSI agrega su propio encabezado (header) — esto se llama encapsulación. En el destino, el proceso inverso elimina los headers capa por capa — esto es la desencapsulación.

L7 – Datos
📦 Datos de la aplicación (HTTP, DNS, FTP…)
L4 – Segmento
[TCP/UDP Header]Datos
L3 – Paquete
[IP Header: src/dst IP][TCP Header]Datos
L2 – Trama
[ETH Header: src/dst MAC][IP][TCP]Datos[FCS]
L1 – Bits
01001101 00110101 10110010 … (señal eléctrica / luz)

💡 Regla de oro de encapsulación

Dato → Segmento (L4 agrega puerto TCP/UDP) → Paquete (L3 agrega IP origen/destino) → Trama (L2 agrega MAC origen/destino + FCS) → Bits (L1 convierte a señal)

🌐 Modelo TCP/IP — 4 Capas

El modelo TCP/IP (también llamado modelo de Internet o DoD) es el modelo práctico que realmente se implementa en las redes actuales. Tiene 4 capas y es el estándar de facto de Internet.

Capa TCP/IP Capas OSI equivalentes Protocolos principales PDU
4 · Aplicación OSI 5,6,7 (Sesión, Presentación, Aplicación) HTTP · HTTPS · FTP · SFTP · DNS · DHCP · SMTP · SSH · Telnet · SNMP · NTP Datos
3 · Transporte OSI 4 (Transporte) TCP · UDP Segmento
2 · Internet OSI 3 (Red) IPv4 · IPv6 · ICMP · ARP · OSPF · EIGRP · BGP Paquete
1 · Acceso a Red OSI 1,2 (Física + Enlace) Ethernet · 802.11 Wi-Fi · PPP · HDLC · Frame Relay Trama / Bits

🔵 TCP vs UDP

  • TCP: orientado a conexión, ACKs, confiable (HTTP, SSH, FTP)
  • UDP: sin conexión, rápido, sin garantía (DNS, DHCP, VoIP)
  • TCP usa handshake de 3 vías: SYN → SYN-ACK → ACK

📌 Puertos bien conocidos

  • HTTP: 80 · HTTPS: 443
  • SSH: 22 · Telnet: 23
  • FTP: 20/21 · SFTP: 22
  • DNS: 53 · DHCP: 67/68
  • SMTP: 25 · SNMP: 161
  • NTP: 123 · BGP: 179

🌐 ARP — Address Resolution

ARP traduce una dirección IP a su MAC correspondiente en la misma red local.

  • ARP Request: broadcast → "¿Quién tiene 192.168.1.1?"
  • ARP Reply: unicast → "Yo soy AA:BB:CC:DD:EE:FF"
  • show ip arp → ver tabla ARP

📡 ICMP — Mensajes de control

ICMP (Internet Control Message Protocol) envía mensajes de error y diagnóstico:

  • Echo Request/Reply → ping
  • Destination Unreachable
  • Time Exceeded (TTL=0)
  • traceroute utiliza ICMP TTL

📡 Topologías de Red

⭐ Estrella

Todos los dispositivos conectados a un switch/hub central. La más común en redes LAN actuales. Si el central falla, toda la red cae.

🔘 Bus

Todos comparten un solo cable (coaxial). Obsoleta. Una falla en el cable corta toda la red. Colisiones frecuentes.

⭕ Anillo

Cada dispositivo conectado al siguiente formando un anillo. Token Ring (IBM). Una falla puede cortar todo el tráfico.

🕸️ Malla Completa

Cada dispositivo conectado a todos los demás. Alta redundancia. Usada en backbone WAN. Costo elevado: n(n-1)/2 enlaces.

🌿 Malla Parcial

Solo algunos nodos tienen conexiones múltiples. Balance entre costo y redundancia. Común en redes empresariales WAN.

🌲 Árbol (Jerárquica)

Modelo de 3 capas: Core → Distribution → Access. El diseño estándar de redes empresariales Cisco.

🏢 Tipos de Red

LAN — Local Area Network

Red de área local. Cubre un edificio o campus. Alta velocidad (1–10 Gbps). Tecnología: Ethernet 802.3, Wi-Fi 802.11.

WAN — Wide Area Network

Red de área amplia. Cubre ciudades, países o el planeta. Velocidades variables. Tecnologías: MPLS, SD-WAN, leased lines.

MAN — Metropolitan Area Network

Red metropolitana. Cubre una ciudad. Conecta múltiples LANs. Tecnologías: Metro Ethernet, fibra óptica.

PAN — Personal Area Network

Red de área personal. Rango de metros. Bluetooth, USB, NFC, Zigbee. Dispositivos personales.
