Fundamentos de Redes

Modelos OSI · TCP/IP · Protocolos · Topologías · Direccionamiento

📐 El Modelo OSI — 7 Capas

El Modelo OSI (Open Systems Interconnection) es un marco de referencia que divide las funciones de red en 7 capas independientes. Cada capa tiene una función específica y se comunica solo con las capas adyacentes. Fue creado por la ISO en 1984 y es la base conceptual para entender cualquier protocolo de red.
#CapaFunciónProtocolos / DispositivosPDU
7AplicaciónInterfaz con el usuario final. Servicios de red a aplicaciones.HTTP, HTTPS, FTP, DNS, DHCP, SMTP, SSH, TelnetDatos
6PresentaciónFormato, cifrado/descifrado, compresión de datos.SSL/TLS, JPEG, MPEG, ASCII, UTF-8Datos
5SesiónEstablece, gestiona y cierra sesiones entre aplicaciones.NetBIOS, RPC, SQLDatos
4TransporteTransporte extremo a extremo, control de flujo, corrección de errores.TCP, UDP. SegmentosSegmento
3RedDireccionamiento lógico (IP), routing entre redes.IPv4, IPv6, ICMP, OSPF, EIGRP. Routers.Paquete
2Enlace de DatosDireccionamiento físico (MAC), detección de errores, frame switching.Ethernet, 802.11 Wi-Fi, VLANs. Switches.Trama
1FísicaTransmisión de bits por el medio físico. Voltajes, señales, cables.Cables UTP/STP, fibra óptica, Wi-Fi, Hubs, Repetidores.Bit
🎯
Mnemotecnia CCNA: "All People Seem To Need Data Processing" (Aplicación, Presentación, Sesión, Transporte, Red, Datos, Física) — de capa 7 a 1. O al revés: "Please Do Not Throw Sausage Pizza Away"

🔄 Encapsulación y Desencapsulación

Cuando un dato viaja de una PC hacia la red, cada capa del modelo OSI agrega su propio encabezado (header) — esto se llama encapsulación. En el destino, el proceso inverso elimina los headers capa por capa — esto es la desencapsulación.
✅
Dato → Segmento (Capa 4 agrega puerto TCP/UDP) → Paquete (Capa 3 agrega IP origen/destino) → Trama (Capa 2 agrega MAC origen/destino) → Bits (Capa 1 convierte a señal eléctrica/óptica)

🌐 Modelo TCP/IP — 4 Capas

El Modelo TCP/IP (también llamado DoD o Internet Model) es el modelo práctico que usa Internet. Tiene 4 capas y es el que realmente implementan los sistemas operativos y dispositivos de red.
Capa TCP/IPEquivalente OSIProtocolos clave
AplicaciónCapas 5, 6, 7 OSIHTTP, FTP, DNS, DHCP, SSH, Telnet, SNMP, NTP
TransporteCapa 4 OSITCP (confiable, orientado a conexión) · UDP (no confiable, rápido)
InternetCapa 3 OSIIPv4, IPv6, ICMP, ARP (debate: algunos lo ubican en capa 2)
Acceso a RedCapas 1 y 2 OSIEthernet, 802.11 Wi-Fi, PPP, Frame Relay

⚖️ TCP vs UDP

CaracterísticaTCPUDP
Orientado a conexión✓ Sí (3-way handshake)✗ No
Confiabilidad✓ ACKs, retransmisión✗ Best-effort
Control de flujo✓ Ventana deslizante✗ No
VelocidadMás lentoMás rápido
Uso típicoHTTP, FTP, SSH, SMTPDNS, DHCP, VoIP, video streaming, TFTP
Puerto conocido ej.HTTP:80, HTTPS:443, SSH:22DNS:53, DHCP:67/68, TFTP:69
🎯
CCNA Examen: TCP usa el 3-Way Handshake: SYN → SYN-ACK → ACK. DNS usa UDP puerto 53 para consultas normales y TCP para transferencias de zona (>512 bytes). DHCP usa UDP puertos 67 (servidor) y 68 (cliente).
