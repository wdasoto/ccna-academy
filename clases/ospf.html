OSPF

Open Shortest Path First · Áreas · DR/BDR · LSA · Costo · OSPFv2 · Configuración · Troubleshooting

⭕ ¿Qué es OSPF?

OSPF (Open Shortest Path First) es un protocolo de routing dinámico de estado de enlace (Link-State), estándar abierto IEEE definido en el RFC 2328. Cada router construye un mapa completo de la topología de la red (LSDB) y calcula las mejores rutas usando el algoritmo SPF de Dijkstra.
CaracterísticaValor
Tipo de protocoloIGP — Interior Gateway Protocol
AlgoritmoSPF — Dijkstra (Shortest Path First)
MétricaCosto (Cost) = 100 Mbps / Ancho de banda
Distancia Administrativa110
Protocolo de transporteIP Protocolo 89 (no usa TCP ni UDP)
Multicast Hello224.0.0.5 (todos los routers OSPF) · 224.0.0.6 (DR/BDR)
VersionesOSPFv2 (IPv4) · OSPFv3 (IPv6)
EstándarRFC 2328 (OSPFv2) · RFC 5340 (OSPFv3)
🎯
CCNA Examen: OSPF tiene AD = 110. Memoriza: Connected=0, Static=1, EIGRP=90, OSPF=110, RIP=120. El protocolo usa IP protocolo 89 — no TCP ni UDP.

📐 Cálculo del Costo OSPF

La métrica de OSPF es el Costo. Se calcula como: Costo = Ancho de banda de referencia / Ancho de banda de la interfaz. El ancho de banda de referencia por defecto es 100 Mbps (100,000,000 bps). OSPF elige siempre el camino con menor costo acumulado.
InterfazAncho de BandaCosto (ref=100Mbps)Costo (ref=1Gbps)
Serial1.544 Mbps (T1)64647
FastEthernet100 Mbps110
GigabitEthernet1000 Mbps11
10GigabitEthernet10,000 Mbps11
⚠️
Problema clásico: Con referencia de 100 Mbps, FastEthernet y GigabitEthernet tienen el mismo costo (1). Para redes modernas se recomienda cambiar la referencia a 1 Gbps o 10 Gbps con el comando auto-cost reference-bandwidth.
AJUSTAR REFERENCIA DE COSTO ! Cambiar referencia a 1 Gbps (recomendado en redes modernas) Router(config-router)# auto-cost reference-bandwidth 1000   ! Ver el costo de una interfaz Router# show ip ospf interface GigabitEthernet0/0   ! Configurar costo manualmente en una interfaz Router(config-if)# ip ospf cost 10

🤝 Proceso de Vecindad OSPF — Hello Protocol

OSPF usa paquetes Hello para descubrir vecinos, establecer adyacencias y mantener la relación activa. Los Hellos se envían al multicast 224.0.0.5 cada cierto intervalo. Si no se recibe un Hello en el tiempo Dead interval, el vecino se declara caído.
Parámetro HelloRedes Broadcast/P2PRedes NBMARequerimiento
Hello Interval10 segundos30 segundosDeben coincidir en ambos vecinos
Dead Interval40 segundos120 segundos
✅
Para formar adyacencia OSPF los routers vecinos deben coincidir en: mismo Area ID, mismo Hello/Dead interval, mismo Subnet, misma MTU, mismo tipo de red y misma contraseña (si se usa autenticación).
ESTADOS DE VECINDAD OSPF Down → No se han recibido Hellos Init → Hello recibido, pero no hay relación bidireccional 2-Way → Comunicación bidireccional. Se elige DR/BDR aquí ExStart → Se negocia master/slave para el intercambio de DBD Exchange → Se intercambian paquetes DBD (Database Description) Loading → Se solicitan LSAs faltantes con LSR/LSU Full → ✅ LSDB sincronizada — adyacencia completa
🎯
CCNA Examen: El estado final deseado es FULL. En redes broadcast, los routers DROther solo llegan a estado 2-WAY entre sí — solo alcanzan FULL con el DR y BDR.

🗂️ Diseño Jerárquico — Áreas OSPF

OSPF organiza los routers en áreas para limitar la propagación de LSAs y reducir el tamaño de la LSDB. Toda red OSPF con múltiples áreas debe tener un Área 0 (Backbone). Todas las demás áreas deben conectarse directamente al Área 0.
ESTRUCTURA DE ÁREAS [ Área 0 — Backbone ] / | \ [Área 1] [Área 2] [Área 3] / \ Routers internos Routers internos   Regla: Toda área debe conectarse directamente al Área 0 Si el Área 0 no es accesible directamente → usar virtual-link
Tipo de ÁreaDescripciónLSAs permitidos
Área 0 — BackboneÁrea central obligatoria. Todas las áreas deben conectarse aquíTodos (LSA 1,2,3,4,5)
Standard AreaÁrea normal que acepta todos los tipos de LSA incluyendo rutas externasLSA 1,2,3,4,5
Stub AreaNo acepta LSA tipo 5 (rutas externas). Usa ruta default para salirLSA 1,2,3
Totally StubbyNo acepta LSA 3,4,5. Solo ruta default. Cisco propietarioLSA 1,2
NSSANot-So-Stubby Area: stub que puede recibir rutas externas via LSA tipo 7LSA 1,2,3,7

🖥️ Tipos de Routers OSPF

TipoSiglaDescripción
Internal Router IR Todas sus interfaces están en la misma área. Solo tiene LSDB de esa área
Backbone Router BR Al menos una interfaz en el Área 0
Area Border Router ABR Conecta dos o más áreas. Tiene LSDB de cada área. Genera LSA tipo 3
Autonomous System Boundary Router ASBR Conecta OSPF con otro protocolo de routing (RIP, EIGRP, BGP). Redistribuye rutas externas
🎯
CCNA Examen: El ABR conecta áreas OSPF entre sí y genera LSA tipo 3. El ASBR conecta OSPF con protocolos externos y genera LSA tipo 5. Un router puede ser ABR y ASBR simultáneamente.

📦 Tipos de LSA

Los LSA (Link-State Advertisements) son los mensajes que los routers OSPF intercambian para construir la LSDB. Cada tipo describe una parte diferente de la topología.
TipoNombreGenerado porDescripción
1Router LSATodos los routersDescribe los enlaces del router y sus estados. Solo se propaga dentro del área
2Network LSADRDescribe la red multi-acceso y los routers conectados. Solo dentro del área
3Summary LSAABRAnuncia redes de otras áreas. Viaja entre áreas a través del Backbone
4ASBR Summary LSAABRInforma la ubicación del ASBR. Necesario para alcanzar rutas externas
5AS External LSAASBRRutas redistribuidas desde otros protocolos. Se propaga por todo el AS OSPF
7NSSA External LSAASBR en NSSARutas externas en áreas NSSA. El ABR convierte tipo 7 a tipo 5

👑 DR y BDR — Designated Router

En redes multi-acceso (Ethernet, Frame Relay), OSPF elige un DR (Designated Router) y un BDR (Backup Designated Router) para reducir el tráfico de LSAs. Sin DR/BDR, cada router formaría adyacencia FULL con todos los demás — en una red de N routers: N×(N-1)/2 adyacencias. Con DR/BDR: solo N-1 adyacencias con el DR.
SIN DR vs CON DR — RED DE 4 ROUTERS Sin DR: 4×3/2 = 6 adyacencias FULL necesarias Con DR: Solo 3 adyacencias FULL (cada router con el DR)   R1 ←──FULL──→ DR ←──FULL──→ R3 ↑ FULL ↓ R2 (también FULL con BDR)   R1, R2, R3 entre sí: solo estado 2-WAY (DROther)

🗳️ Proceso de Elección DR/BDR

La elección se realiza durante el intercambio de Hellos en el estado 2-Way. Se basa en dos criterios en orden de prioridad:
CriterioOrdenDescripción
OSPF Priority 1° Mayor prioridad gana. Rango: 0-255. Default: 1. Prioridad 0 = nunca será DR/BDR
Router ID 2° (desempate) Mayor Router ID gana si la prioridad es igual
⚠️
Importante: La elección de DR/BDR es no preemptiva. Si llega un router con mayor prioridad después de la elección, NO desplaza al DR actual. Solo cambia cuando el DR actual falla.
🎯
Router ID en OSPF — se determina en este orden: (1) router-id configurado manualmente, (2) IP más alta de loopback activa, (3) IP más alta de interfaz física activa.

📡 Roles en la Red Multi-acceso

RolDescripciónMulticastEstado con vecinos
DR Designated Router — recolecta y redistribuye todos los LSAs del segmento Escucha 224.0.0.5 · Envía a 224.0.0.5 FULL con todos
BDR Backup DR — monitorea al DR, toma el relevo si el DR falla Escucha 224.0.0.5 y 224.0.0.6 FULL con DR y DROther
DROther Router que no es DR ni BDR. Envía LSAs solo al DR y BDR Envía a 224.0.0.6 2-WAY con otros DROther · FULL con DR y BDR
CONFIGURAR PRIORIDAD DR ! Asegurar que R1 sea siempre el DR (prioridad máxima) R1(config)# interface GigabitEthernet0/0 R1(config-if)# ip ospf priority 255   ! Excluir un router de la elección DR/BDR (prioridad 0) R3(config-if)# ip ospf priority 0   ! Verificar quién es el DR/BDR Router# show ip ospf neighbor

🏗️ Topología de Laboratorio

Laboratorio con 3 routers (R1, R2, R3) en una sola área OSPF (Área 0). Todas las redes LAN deben ser alcanzables mediante OSPFv2 — Single Area.
TOPOLOGÍA PC1 (192.168.1.10) | [R1] Gi0/0: 192.168.1.1/24 ← LAN1 Gi0/1: 10.0.12.1/30 ← WAN R1-R2 | [R2] Gi0/0: 10.0.12.2/30 ← WAN R1-R2 Gi0/1: 10.0.23.1/30 ← WAN R2-R3 Gi0/2: 192.168.2.1/24 ← LAN2 | [R3] Gi0/0: 10.0.23.2/30 ← WAN R2-R3 Gi0/1: 192.168.3.1/24 ← LAN3 PC3 (192.168.3.10)   Todos en Área 0 — OSPFv2 proceso 1

⌨️ Configuración R1

R1 — OSPF CONFIG ! Paso 1: Configurar Router ID estable (recomendado: loopback) R1(config)# interface loopback 0 R1(config-if)# ip address 1.1.1.1 255.255.255.255 R1(config-if)# exit   ! Paso 2: Activar proceso OSPF R1(config)# router ospf 1   ! Paso 3: Definir Router ID explícitamente R1(config-router)# router-id 1.1.1.1   ! Paso 4: Anunciar redes con comando network ! Sintaxis: network [IP-red] [wildcard-mask] area [área] R1(config-router)# network 192.168.1.0 0.0.0.255 area 0 R1(config-router)# network 10.0.12.0 0.0.0.3 area 0   ! Paso 5: Evitar Hellos hacia la LAN (pasiva — no hay routers) R1(config-router)# passive-interface GigabitEthernet0/0   ! Ajustar referencia de costo para redes Gigabit R1(config-router)# auto-cost reference-bandwidth 1000

⌨️ Configuración R2 y R3

R2 — OSPF CONFIG R2(config)# interface loopback 0 R2(config-if)# ip address 2.2.2.2 255.255.255.255 R2(config-if)# exit R2(config)# router ospf 1 R2(config-router)# router-id 2.2.2.2 R2(config-router)# network 10.0.12.0 0.0.0.3 area 0 R2(config-router)# network 10.0.23.0 0.0.0.3 area 0 R2(config-router)# network 192.168.2.0 0.0.0.255 area 0 R2(config-router)# passive-interface GigabitEthernet0/2 R2(config-router)# auto-cost reference-bandwidth 1000
R3 — OSPF CONFIG R3(config)# interface loopback 0 R3(config-if)# ip address 3.3.3.3 255.255.255.255 R3(config-if)# exit R3(config)# router ospf 1 R3(config-router)# router-id 3.3.3.3 R3(config-router)# network 10.0.23.0 0.0.0.3 area 0 R3(config-router)# network 192.168.3.0 0.0.0.255 area 0 R3(config-router)# passive-interface GigabitEthernet0/1 R3(config-router)# auto-cost reference-bandwidth 1000
✅
passive-interface evita que OSPF envíe Hellos por interfaces donde no hay routers OSPF (LANs con PCs). Mejora la seguridad y reduce el tráfico innecesario.

📐 Wildcard Mask — La inversa de la máscara

El comando network usa wildcard mask, que es la inversa de la máscara de subred. Un 0 significa "debe coincidir" y un 1 significa "no importa".
Máscara de subredWildcard maskEjemplo de uso
/8 — 255.0.0.00.255.255.255network 10.0.0.0 0.255.255.255 area 0
/16 — 255.255.0.00.0.255.255network 172.16.0.0 0.0.255.255 area 0
/24 — 255.255.255.00.0.0.255network 192.168.1.0 0.0.0.255 area 0
/30 — 255.255.255.2520.0.0.3network 10.0.12.0 0.0.0.3 area 0
/32 — 255.255.255.2550.0.0.0network 1.1.1.1 0.0.0.0 area 0 (host exacto)
✅
Alternativa moderna: En lugar de network, puedes activar OSPF directamente en la interfaz:
Router(config-if)# ip ospf 1 area 0
Es más preciso y recomendado en entornos actuales.
ACTIVAR OSPF POR INTERFAZ (MÉTODO MODERNO) R1(config)# interface GigabitEthernet0/0 R1(config-if)# ip ospf 1 area 0 R1(config)# interface GigabitEthernet0/1 R1(config-if)# ip ospf 1 area 0

🚨 Causas comunes de fallo en adyacencias

❌
Hello/Dead interval distintos — Si R1 tiene Hello=10 y R2 tiene Hello=30, nunca forman adyacencia.
Verificar con: show ip ospf interface — ambos lados deben coincidir exactamente.
❌
Area ID diferente — R1 en área 0 y R2 en área 1 en el mismo enlace.
Verificar con: show ip ospf — deben estar en la misma área para ser vecinos directos.
❌
MTU mismatch — Las interfaces tienen MTU diferente. El intercambio de DBD falla y la adyacencia queda en EXSTART/EXCHANGE.
Solución: igualar MTU con ip mtu 1500 o usar ip ospf mtu-ignore.
❌
Passive-interface incorrecta — Se configuró passive-interface en una interfaz que debería tener vecinos.
Solo usar passive en interfaces de LAN donde no hay routers OSPF.
❌
Subnet diferente — Las interfaces del enlace están en subnets distintas.
Verificar con: show ip interface brief — los dos lados deben estar en la misma subred.
🎯
Método sistemático: 1) show ip ospf neighbor → 2) show ip ospf interface → 3) show ip route ospf → 4) Verificar passive-interface → 5) debug ip ospf adj
