⭕ ¿Qué es OSPF?

OSPF (Open Shortest Path First) es un protocolo de routing dinámico de estado de enlace (Link-State), estándar abierto (RFC 2328), que usa el algoritmo SPF de Dijkstra para calcular la mejor ruta.

CaracterísticaValor
TipoLink-State · IGP (Interior Gateway Protocol)
AlgoritmoSPF — Dijkstra (Shortest Path First)
MétricaCosto = 100 Mbps / Ancho de banda de la interfaz
Distancia Administrativa110
Protocolo de transporteIP Protocolo 89 (no usa TCP ni UDP)
Multicast Hello224.0.0.5 (todos los routers OSPF) · 224.0.0.6 (DR/BDR)
VersionesOSPFv2 (IPv4) · OSPFv3 (IPv6)
ConvergenciaRápida (segundos)

🗺️ Áreas OSPF — Escalabilidad

OSPF usa áreas para dividir la red en grupos jerárquicos, reduciendo el tamaño de la LSDB y los cálculos SPF.

Area 0 — Backbone

El área backbone es obligatoria. Todas las demás áreas deben conectarse a ella. Es el núcleo del dominio OSPF.

Área regular

Área no-backbone. Contiene los routers y redes de un segmento específico. Se conecta al area 0 a través de un ABR.

Tipos de Router

IR — Internal Router (solo en un área)
ABR — Area Border Router (conecta dos áreas)
ASBR — AS Boundary Router (redistribuye externas)
BR — Backbone Router (tiene interfaz en area 0)

🔑 Router ID OSPF

El Router ID es la identidad de cada router en OSPF. Se selecciona en este orden:

  1. Manual: router-id X.X.X.X
  2. La IP más alta de una interfaz Loopback activa
  3. La IP más alta de cualquier interfaz activa

⚙️ Configuración OSPFv2

Configuración básica OSPF single-area

! Habilitar OSPF proceso 1 (local, no se intercambia)
R1(config)# router ospf 1
! Configurar Router ID manualmente (recomendado)
R1(config-router)# router-id 1.1.1.1
! Anunciar redes (network + wildcard + area)
R1(config-router)# network 192.168.1.0 0.0.0.255 area 0
R1(config-router)# network 10.0.0.0 0.0.0.3 area 0
! Interfaz pasiva: no envía Hellos (redes LAN de usuarios)
R1(config-router)# passive-interface GigabitEthernet0/0
! Alternativa: usar ip ospf directamente en la interfaz
R1(config)# interface GigabitEthernet0/1
R1(config-if)# ip ospf 1 area 0

Modificar el costo OSPF

! Fórmula: Costo = 100 Mbps (referencia) / BW de la interfaz
! GigabitEthernet (1000 Mbps): costo = 100/1000 = 1
! FastEthernet (100 Mbps): costo = 100/100 = 1
! Serial (1.544 Mbps): costo = 100/1.544 ≈ 64
! Cambiar referencia global (recomendado para Gbps)
R1(config)# router ospf 1
R1(config-router)# auto-cost reference-bandwidth 1000
! O cambiar manualmente en la interfaz
R1(config)# interface GigabitEthernet0/0
R1(config-if)# ip ospf cost 10

Verificación OSPF

R1# show ip ospf neighbor
Neighbor ID  Pri  State       Dead Time  Address      Interface
2.2.2.2       1   FULL/DR     00:00:35   10.0.0.2     GigEth0/1
3.3.3.3       1   FULL/BDR    00:00:37   10.0.0.3     GigEth0/1

R1# show ip ospf database
R1# show ip route ospf
O    192.168.2.0/24 [110/2] via 10.0.0.2, 00:05:10, GigEth0/1
R1# show ip ospf interface GigabitEthernet0/0

🔗 DR y BDR — Redes Multiaccess

En redes Ethernet multiaccess (bus), OSPF elige un DR (Designated Router) y un BDR para reducir el número de adyacencias.

📌 Por qué DR/BDR

Sin DR/BDR en una red con N routers habría N×(N-1)/2 adyacencias. Con DR, todos forman adyacencia solo con el DR y BDR.

RolDescripciónEstado con vecinos
DR (Designated Router)Centraliza LSA updates. Escucha en 224.0.0.6.FULL con todos
BDR (Backup DR)Monitorea al DR, toma su lugar si falla.FULL con DR
DROtherLos demás routers. Solo forman FULL con DR y BDR.2-WAY entre sí

🏆 Elección de DR/BDR

  1. Mayor OSPF priority (default=1, rango 0-255). Priority 0 = no puede ser DR/BDR.
  2. En caso de empate: mayor Router ID
! Forzar que R1 sea siempre DR
R1(config)# interface GigabitEthernet0/0
R1(config-if)# ip ospf priority 255
! Evitar que un router sea DR/BDR
R3(config-if)# ip ospf priority 0

🧪 Laboratorio OSPF Single-Area

📋 Topología: R1─R2─R3 en área 0, cada uno con una LAN

R1 — OSPF completo

R1(config)# interface GigabitEthernet0/0
R1(config-if)# ip address 192.168.1.1 255.255.255.0
R1(config-if)# no shutdown
R1(config)# interface GigabitEthernet0/1
R1(config-if)# ip address 10.0.0.1 255.255.255.252
R1(config-if)# no shutdown
R1(config)# router ospf 1
R1(config-router)# router-id 1.1.1.1
R1(config-router)# network 192.168.1.0 0.0.0.255 area 0
R1(config-router)# network 10.0.0.0 0.0.0.3 area 0
R1(config-router)# passive-interface GigabitEthernet0/0

Verificación final

R1# show ip ospf neighbor
Neighbor ID  State    Address    Interface

R1# show ip route ospf

📋 Resumen OSPF

✅ Puntos críticos

  • OSPF AD = 110 · Protocolo IP número 89 (no TCP/UDP)
  • Multicast: 224.0.0.5 (todos) · 224.0.0.6 (DR/BDR)
  • Métrica: Costo = 100 Mbps / BW. Menor costo = mejor ruta
  • Router ID: manual > IP loopback más alta > IP interfaz más alta
  • Todas las áreas deben conectarse al Area 0 (backbone)
  • DR/BDR: mayor priority gana (default=1) · empate: mayor Router ID
  • passive-interface: no envía Hellos (redes LAN de hosts)
  • Estado FULL = adyacencia completa · 2-WAY = DROther entre sí
