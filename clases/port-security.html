Port Security

Shutdown · Restrict · Protect · Sticky MAC · Err-Disabled · CAM Flooding

CCNA 200-301Módulo 3 · Switching🔒 Seguridad Capa 2

🔒 ¿Qué es Port Security?

Port Security es una función de seguridad de Capa 2 que restringe qué dispositivos (MACs) pueden conectarse a un puerto del switch, limitando el número de MACs permitidas y definiendo acciones ante violaciones.

🎯 Control de acceso

Define cuántas y cuáles MACs pueden conectarse a un puerto. Bloquea MACs no autorizadas.

🌊 Protección CAM flooding

Previene ataques de inundación de MACs que buscan llenar la tabla CAM y forzar flooding.

📍 Asignación estática

Permite fijar MACs específicas permanentemente (sticky) para que persistan tras reinicios.

⚠️ Requisito obligatorio

Port Security solo funciona en puertos configurados como access o trunk estático. No funciona en puertos dinámicos (dynamic auto o dynamic desirable).

🛡️ Modos de Violación

ModoQué hace al detectar violaciónContador violacionesLog/SNMPPuerto activo
Shutdown (default)Pone el puerto en estado err-disabledSíSíNo — requiere recuperación manual
RestrictDescarta frames de MACs no permitidasSíSíSí — sigue funcionando para MACs permitidas
ProtectDescarta frames de MACs no permitidasNoNoSí — modo silencioso

💡 Recuperar un puerto err-disabled

! Manual: apagar y encender la interfaz
SW(config)# interface FastEthernet0/1
SW(config-if)# shutdown
SW(config-if)# no shutdown
! Automático con timer (en segundos)
SW(config)# errdisable recovery cause psecure-violation
SW(config)# errdisable recovery interval 300

⚙️ Configuración Port Security

Configuración básica Port Security

SW(config)# interface FastEthernet0/1
SW(config-if)# switchport mode access
SW(config-if)# switchport access vlan 10
! Habilitar Port Security
SW(config-if)# switchport port-security
! Máximo de MACs permitidas (default = 1)
SW(config-if)# switchport port-security maximum 2
! MAC estática permitida
SW(config-if)# switchport port-security mac-address 0030.F2A0.0001
! Modo de violación (shutdown es el default)
SW(config-if)# switchport port-security violation restrict

📌 Sticky MAC — Aprendizaje dinámico persistente

Sticky MAC permite al switch aprender automáticamente las MACs de los dispositivos conectados y guardarlas como entradas estáticas que persisten tras un reinicio (si se guarda la config).

💡 Ventaja de Sticky

No necesitas conocer las MACs de antemano. El switch las aprende y las "pega" (sticky) automáticamente.

SW(config)# interface FastEthernet0/2
SW(config-if)# switchport mode access
SW(config-if)# switchport port-security
SW(config-if)# switchport port-security maximum 1
! Activar sticky learning
SW(config-if)# switchport port-security mac-address sticky
SW(config-if)# switchport port-security violation shutdown
! La primera MAC que se conecte quedará registrada automáticamente
! Se puede ver en running-config y en show port-security address
SW# show running-config interface FastEthernet0/2
interface FastEthernet0/2
 switchport mode access
 switchport port-security
 switchport port-security mac-address sticky
 switchport port-security mac-address sticky 0050.7966.6800

🧪 Laboratorio Port Security completo

Escenario: Fa0/1 solo para PC-RRHH, máx 1 MAC, shutdown en violación

SW(config)# interface FastEthernet0/1
SW(config-if)# switchport mode access
SW(config-if)# switchport access vlan 10
SW(config-if)# switchport port-security
SW(config-if)# switchport port-security maximum 1
SW(config-if)# switchport port-security mac-address sticky
SW(config-if)# switchport port-security violation shutdown
SW(config-if)# end
! Guardar para que las sticky MACs persistan
SW# copy running-config startup-config
! Simular violación: conectar PC diferente
! El puerto entra en err-disabled
SW# show interfaces FastEthernet0/1
FastEthernet0/1 is down, line protocol is down (err-disabled)
! Recuperar el puerto
SW(config)# interface FastEthernet0/1
SW(config-if)# shutdown
SW(config-if)# no shutdown

📋 Resumen Port Security

✅ Puntos críticos

  • Solo funciona en puertos access o trunk estáticos
  • Default: máximo 1 MAC, modo violación = shutdown
  • Shutdown = err-disabled (requiere manual o errdisable recovery)
  • Restrict = descarta + incrementa contador + envía syslog
  • Protect = descarta silenciosamente (sin contador ni log)
  • Sticky: aprende MACs dinámicamente y las guarda como estáticas
  • Para que sticky persista hay que hacer copy run start
  • show port-security = resumen | show port-security interface = detalle
