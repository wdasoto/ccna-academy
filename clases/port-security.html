¿Qué es Port Security?
Port Security es una característica de seguridad de Capa 2 de los switches Cisco que permite restringir el acceso a un puerto basándose en las direcciones MAC de los dispositivos conectados.

Permite definir cuántos y cuáles dispositivos pueden conectarse a cada puerto del switch. Si un dispositivo no autorizado intenta conectarse, el switch ejecuta una acción de violación configurada por el administrador.
📌
Capa de acción: Port Security opera en la Capa 2 del modelo OSI (Enlace de datos). Solo funciona en puertos de acceso (switchport mode access). No puede configurarse en puertos trunk ni en EtherChannel sin antes configurar los parámetros en cada interfaz física.
🏢
¿Por qué es necesario Port Security?
🚨 Escenario de Ataque — MAC Flooding

Un atacante conecta su laptop al switch de la empresa y ejecuta una herramienta como macof que genera miles de direcciones MAC falsas por segundo. La tabla CAM del switch (que normalmente almacena ~8.000 a 16.000 MACs) se llena completamente. El switch ya no puede aprender nuevas MACs y comienza a comportarse como un hub, enviando todos los frames a todos los puertos → el atacante captura TODO el tráfico de la red.

Ataque MAC Flooding vs. Red protegida con Port Security
🛡️
Previene MAC Flooding
Limita el número de MACs por puerto, evitando que un atacante llene la tabla CAM del switch.
🚫
Bloquea dispositivos no autorizados
Solo los dispositivos con MAC conocida o aprendida pueden usar el puerto. Cualquier otro es rechazado.
📋
Control de acceso físico
Evita que empleados conecten hubs, switches no autorizados o laptops externas a la red corporativa.
📊
Parámetros configurables de Port Security
ParámetroDescripciónValor por defectoRango
maximum Número máximo de direcciones MAC permitidas en el puerto 1 1 – 8192
violation Acción que toma el switch cuando se detecta una violación shutdown protect / restrict / shutdown
mac-address Modo de aprendizaje de direcciones MAC dynamic static / dynamic / sticky
aging Tiempo de envejecimiento de MACs aprendidas dinámicamente 0 (desactivado) 0 – 1440 minutos
🎯
EXAMEN CCNA 200-301: Port Security solo funciona en puertos de acceso. Si intentas activarlo en un puerto trunk o en un puerto en estado err-disabled, el IOS retorna error. El valor por defecto de maximum es 1 y el modo de violación por defecto es shutdown.
⚠️
Modos de Violación — Los 3 comportamientos ante una infracción
Una violación ocurre cuando el switch detecta que en un puerto protegido aparece una MAC desconocida (no autorizada) o se supera el número máximo de MACs configuradas. El switch reacciona según el modo de violación elegido.
🔵 PROTECT
Tráfico infractor:Descartado silenciosamente
Tráfico legítimo:Sigue pasando ✓
Puerto apagado:No ✓
Log / Syslog:No ✗
Contador viola.:No incrementa ✗
SNMP trap:No ✗
⚠️ El modo más silencioso. El atacante no es detectado ni notificado. Útil solo cuando el administrador monitorea activamente por otros medios.
🟡 RESTRICT
Tráfico infractor:Descartado
Tráfico legítimo:Sigue pasando ✓
Puerto apagado:No ✓
Log / Syslog:Sí ✓
Contador viola.:Incrementa ✓
SNMP trap:Sí ✓
✅ Balance entre seguridad y disponibilidad. El puerto sigue operando para dispositivos autorizados, pero registra y alerta sobre los intentos no autorizados.
🔴 SHUTDOWN
Tráfico infractor:Bloqueado total
Tráfico legítimo:También bloqueado ✗
Puerto apagado:Sí → err-disabled ✗
Log / Syslog:Sí ✓
Contador viola.:No (puerto off)
SNMP trap:Sí ✓
🔴 Modo por defecto y más estricto. El puerto entra en estado err-disabled. Requiere intervención manual del administrador para reactivarse.
CaracterísticaProtectRestrictShutdown (default)
Descarta tráfico no autorizado ✅ Sí ✅ Sí ✅ Sí
El puerto sigue operativo ✅ Sí ✅ Sí ❌ No (err-disabled)
Envía mensaje Syslog ❌ No ✅ Sí ✅ Sí
Incrementa contador de violaciones ❌ No ✅ Sí ❌ No (puerto off)
Envía SNMP trap ❌ No ✅ Sí ✅ Sí
Requiere intervención manual ❌ No ❌ No ✅ Sí (shutdown/no shutdown)
🚨
Estado err-disabled — ¿Qué pasa y cómo recuperar el puerto?
Cuando el modo de violación es shutdown y se detecta una violación, el puerto entra en estado err-disabled. Esto significa que el puerto está administrativamente apagado por el sistema IOS y no puede pasar tráfico hasta que el administrador lo recupere manualmente.
Cómo identificar un puerto en err-disabled
VERIFICACIÓN Switch# show interfaces fa0/1 ! Salida típica de puerto err-disabled: FastEthernet0/1 is down, line protocol is down (err-disabled) Switch# show port-security interface fa0/1 Port Security : Enabled Port Status : Secure-shutdown Violation Mode : Shutdown Aging Time : 0 mins Maximum MAC Addresses : 1 Total MAC Addresses : 1 Configured MAC Addresses : 0 Sticky MAC Addresses : 0 Last Source Address:Vlan : 00:0A:BC:12:34:56:1 Security Violation Count : 1
Recuperar un puerto err-disabled (manual)
RECUPERACIÓN ! Paso 1: Ir a la interfaz Switch(config)# interface fastEthernet 0/1 ! Paso 2: Apagar primero Switch(config-if)# shutdown ! Paso 3: Volver a encender Switch(config-if)# no shutdown ! Alternativa: auto-recovery (no recomendado en producción) Switch(config)# errdisable recovery cause psecure-violation Switch(config)# errdisable recovery interval 300 ! 300 segundos = 5 min
⚠️
Importante: Antes de reactivar el puerto, desconecta el dispositivo no autorizado. Si el dispositivo sigue conectado al reactivar, el puerto entrará en err-disabled nuevamente de forma inmediata.
🔑
Tipos de Aprendizaje de MAC — Static, Dynamic y Sticky
Port Security puede aprender las direcciones MAC autorizadas de tres maneras distintas. La elección del método determina la flexibilidad vs. seguridad de la configuración.
📝
Static (Manual)
El administrador configura manualmente cada dirección MAC autorizada. Máxima seguridad, máximo trabajo administrativo. Sobrevive a reinicios sin necesidad de guardar.
🔄
Dynamic (Dinámico)
El switch aprende automáticamente las primeras MACs que aparecen en el puerto hasta el límite máximo. Se pierden al reiniciar el switch. Modo más flexible pero menos seguro.
🗝️
Sticky (Recomendado)
Combina lo mejor de ambos mundos: el switch aprende las MACs automáticamente pero las guarda en la running-config como si fueran estáticas. Persisten con write memory.
Flujo de aprendizaje — Modo STICKY
📊
Comparativa de modos de aprendizaje MAC
CaracterísticaStaticDynamicSticky ⭐
Configuración manual ✅ Requiere ❌ No ❌ No (automático)
Persiste tras reinicio ✅ Siempre ❌ No ⚠️ Solo con write memory
Aparece en running-config ✅ Sí ❌ No ✅ Sí (como sticky)
Escalabilidad Baja (trabajo manual) Alta Alta (aprendizaje auto)
Nivel de seguridad Muy alto Bajo Alto
Uso recomendado Servidores/infraestructura crítica Rara vez en producción ✅ Ambientes de usuario final
✅
Mejor práctica: Usa sticky para puertos de usuarios finales — el switch aprende automáticamente y las MACs sobreviven reinicios. Usa static para puertos de servidores y equipos de infraestructura donde la MAC nunca debe cambiar.

Configuración completa de Port Security en Cisco IOS
📌
Requisito previo: El puerto DEBE estar en modo access antes de habilitar Port Security.
Secuencia obligatoria: switchport mode access → switchport port-security
CONFIGURACIÓN BÁSICA — Un solo dispositivo por puerto
CONFIG BÁSICA Switch(config)# interface fastEthernet 0/1 ! Paso 1: Modo acceso (obligatorio) Switch(config-if)# switchport mode access ! Paso 2: Habilitar Port Security Switch(config-if)# switchport port-security ! Con esto ya está activo con valores por defecto: ! maximum = 1, violation = shutdown, aprendizaje = dynamic
CONFIGURACIÓN CON STICKY — Recomendada para usuarios finales
STICKY CONFIG Switch(config)# interface fastEthernet 0/2 Switch(config-if)# switchport mode access Switch(config-if)# switchport port-security ! Máximo 1 dispositivo Switch(config-if)# switchport port-security maximum 1 ! Modo de violación: restrict (no cierra el puerto, pero registra) Switch(config-if)# switchport port-security violation restrict ! Sticky: aprende la MAC y la guarda en running-config Switch(config-if)# switchport port-security mac-address sticky Switch(config-if)# no shutdown Switch(config-if)# exit ! Guardar la MAC aprendida para que persista tras reinicio Switch# write memory
CONFIGURACIÓN CON MAC ESTÁTICA — Para servidores/infraestructura
STATIC CONFIG Switch(config)# interface gigabitEthernet 0/1 Switch(config-if)# switchport mode access Switch(config-if)# switchport access vlan 10 Switch(config-if)# switchport port-security Switch(config-if)# switchport port-security maximum 1 Switch(config-if)# switchport port-security violation shutdown ! Configurar la MAC estática del servidor Switch(config-if)# switchport port-security mac-address 00A1.B2C3.D4E5 Switch(config-if)# no shutdown
CONFIGURACIÓN EN RANGO DE PUERTOS — Para múltiples puertos
RANGE CONFIG ! Aplicar port security a puertos Fa0/1 al Fa0/24 de una sola vez Switch(config)# interface range fastEthernet 0/1 - 24 Switch(config-if-range)# switchport mode access Switch(config-if-range)# switchport port-security Switch(config-if-range)# switchport port-security maximum 2 Switch(config-if-range)# switchport port-security violation restrict Switch(config-if-range)# switchport port-security mac-address sticky Switch(config-if-range)# no shutdown
AGING — Tiempo de envejecimiento de MACs
AGING CONFIG ! Configurar aging time de 60 minutos (las MACs se borran si no se ven en 60 min) Switch(config-if)# switchport port-security aging time 60 ! Tipo de aging: absolute (desde que se aprendió) o inactivity (desde última trama) Switch(config-if)# switchport port-security aging type inactivity ! Para que las MACs estáticas también tengan aging: Switch(config-if)# switchport port-security aging static
DESHABILITAR Port Security en un puerto
DESACTIVAR Switch(config-if)# no switchport port-security ! Eliminar MACs sticky específicas Switch(config-if)# no switchport port-security mac-address sticky 00A1.B2C3.D4E5 ! Eliminar TODAS las MACs sticky del puerto Switch(config-if)# no switchport port-security mac-address sticky

Lab Completo — Escenario Empresa TechSec
🏢 Escenario Real de Configuración

La empresa TechSec tiene un switch Catalyst 2960 con:
• Fa0/1 – Fa0/8: Puertos de usuarios (1 PC por puerto, sticky)
• Fa0/9: Servidor (MAC conocida, violation shutdown)
• Fa0/10 – Fa0/12: Sala de reuniones (máx 3 dispositivos, protect)
• Gi0/1: Uplink al core (trunk, sin port security)

  1. Puertos de usuarios — Sticky + Restrict
    interface range fa0/1 - 8 → switchport mode access → port-security → maximum 1 → violation restrict → mac-address sticky → no shutdown
  2. Servidor — Static MAC + Shutdown
    interface fa0/9 → switchport mode access → port-security → maximum 1 → violation shutdown → mac-address 00A1.B2C3.0009 → no shutdown
  3. Sala de reuniones — Múltiples MACs + Protect
    interface range fa0/10 - 12 → switchport mode access → port-security → maximum 3 → violation protect → mac-address sticky → no shutdown
  4. Guardar la configuración
    Switch# write memory — Las MACs sticky aprendidas quedan grabadas en startup-config y sobreviven reinicios.

Estados posibles del puerto (Port Status):
• Secure-up → Puerto activo, port security funcionando correctamente
• Secure-down → Puerto desconectado físicamente pero con port security configurado
• Secure-shutdown → Puerto en err-disabled por violación de seguridad

Troubleshooting — Errores Comunes
Síntoma / ErrorCausa probableSolución
Command rejected: not an access port Puerto no está en modo access cuando se intenta activar port-security switchport mode access antes de switchport port-security
Puerto entra en err-disabled al conectar una PC El switch ya aprendió la MAC de otro dispositivo anterior con sticky Limpiar MACs sticky: no switchport port-security mac-address sticky, luego reconectar
MACs sticky se pierden tras reinicio No se ejecutó write memory después del aprendizaje Ejecutar write memory (o copy run start) para persistir la configuración
Port Security no aparece en show port-security Port security no fue habilitado con switchport port-security Verificar con show run interface fa0/X y habilitar el comando base

Tarea Práctica — Lab Port Security en Packet Tracer
🔬 Lab: Proteger la red de SecureBank S.A.

Tienes un switch Cisco 2960 con 24 puertos. Configura Port Security según:
• Fa0/1 – Fa0/10: Usuarios finales — sticky, máximo 1, violation restrict
• Fa0/11 – Fa0/15: Sala VIP — sticky, máximo 2, violation shutdown
• Fa0/16: Servidor de archivos — static MAC AA:BB:CC:DD:EE:FF, violation shutdown
• Fa0/17 – Fa0/24: Deshabilitados (shutdown) — zona no utilizada
• Verificar con show port-security y show port-security address

  1. Configura los puertos de usuarios (Fa0/1-10)
    Usa interface range. Aplica mode access, habilita port-security, maximum 1, violation restrict, sticky. Verifica que los puertos quedan en Secure-up.
  2. Configura la Sala VIP (Fa0/11-15)
    Maximum 2 permite conectar PC + teléfono IP por puesto. Violation shutdown para máxima seguridad en sala ejecutiva.
  3. Configura el servidor con MAC estática (Fa0/16)
    Usa switchport port-security mac-address AABB.CCDD.EEFF. Solo ese servidor físico puede conectarse, aunque lo muevan de rack.
  4. Simula un ataque y verifica la respuesta
    Conecta un dispositivo "no autorizado" a Fa0/1. Verifica el estado del puerto y el mensaje de log. Practica la recuperación del puerto err-disabled.
  5. Guarda y verifica
    Ejecuta write memory para persistir las MACs sticky. Verifica con show port-security, show port-security address y show running-config.
