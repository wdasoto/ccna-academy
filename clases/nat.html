NAT / PAT

Static NAT · Dynamic NAT · PAT Overload · Inside/Outside · Terminología · Laboratorio IOS

CCNA 200-301Módulo 5 · Servicios IP🔄 Traducción de Direcciones

🔄 ¿Qué es NAT?

NAT (Network Address Translation) traduce direcciones IP privadas a públicas (y viceversa) cuando el tráfico cruza el router hacia Internet. Permite que miles de dispositivos privados compartan una sola IP pública.

🏠 Inside Local

IP privada del dispositivo interno tal como la ve la red interna. Ej: 192.168.1.10

🌐 Inside Global

IP pública que representa al dispositivo interno en Internet. Ej: 203.0.113.5

🖥️ Outside Global

IP del servidor externo tal como la ve Internet. Ej: 8.8.8.8

📍 Outside Local

IP del servidor externo tal como la ve la red interna. Generalmente igual a Outside Global.

🔑 Interfaces NAT: Inside y Outside

  • inside: interfaz hacia la red privada (LAN)
  • outside: interfaz hacia Internet/ISP
  • NAT solo traduce cuando el paquete cruza de inside a outside (o viceversa)

🔄 Tipos de NAT

TipoMapeoIPs públicasUso típico
Static NAT1 IP privada ↔ 1 IP pública fija1 por cada host internoServidores web, DMZ
Dynamic NATIP privada → IP pública del pool (dinámica)Pool de IPs públicasAcceso saliente controlado
PAT / NAT OverloadMuchas IPs privadas → 1 IP pública + puertos1 sola IP públicaHogares, pymes (lo más común)

📌 Operación NAT en Capa 3

NAT opera en Capa 3 del modelo OSI. El router NAT se llama NAT router o border router. Modifica las cabeceras IP de los paquetes.

🔢 PAT — Port Address Translation

PAT (también llamado NAT Overload) es la forma más común de NAT. Permite que miles de hosts privados compartan una sola IP pública usando números de puerto únicos para distinguir cada sesión.

📌 Cómo funciona PAT

192.168.1.10:1024 → 203.0.113.5:10001

192.168.1.11:1024 → 203.0.113.5:10002

192.168.1.12:3000 → 203.0.113.5:10003

La misma IP pública (203.0.113.5) con puertos diferentes identifica cada sesión.

⚠️ Limitaciones de NAT

  • Rompe el modelo end-to-end de Internet
  • Dificulta protocolos que incluyen IPs en el payload (FTP, SIP, H.323)
  • Añade latencia (procesamiento adicional en el router)
  • No es compatible con IPsec ESP en algunos casos

⚙️ Configuración NAT/PAT

Static NAT — 1 a 1

! Mapeo estático: servidor interno 192.168.1.100 → IP pública 203.0.113.10
R1(config)# ip nat inside source static 192.168.1.100 203.0.113.10
! Marcar interfaces
R1(config)# interface GigabitEthernet0/0
R1(config-if)# ip nat inside
R1(config)# interface GigabitEthernet0/1
R1(config-if)# ip nat outside

PAT / NAT Overload — usando la IP de la interfaz WAN

! 1. ACL que define qué IPs privadas se traducen
R1(config)# access-list 1 permit 192.168.1.0 0.0.0.255
! 2. Configurar PAT usando la interfaz outside
R1(config)# ip nat inside source list 1 interface GigabitEthernet0/1 overload
! 3. Marcar interfaces
R1(config)# interface GigabitEthernet0/0
R1(config-if)# ip nat inside
R1(config)# interface GigabitEthernet0/1
R1(config-if)# ip nat outside

Verificación NAT

R1# show ip nat translations

🧪 Laboratorio NAT/PAT completo

Escenario: LAN 192.168.1.0/24 → Internet vía PAT

! R1: Gi0/0=LAN(192.168.1.1/24), Gi0/1=ISP(203.0.113.2/30)
R1(config)# access-list 10 permit 192.168.1.0 0.0.0.255
R1(config)# ip nat inside source list 10 interface GigabitEthernet0/1 overload
R1(config)# interface GigabitEthernet0/0
R1(config-if)# ip nat inside
R1(config)# interface GigabitEthernet0/1
R1(config-if)# ip nat outside
! Ruta por defecto hacia el ISP
R1(config)# ip route 0.0.0.0 0.0.0.0 203.0.113.1
! Verificar desde PC interno
! PC: ping 8.8.8.8 → R1 traduce 192.168.1.10 → 203.0.113.2
R1# show ip nat translations

📋 Resumen NAT/PAT

✅ Puntos críticos

  • NAT opera en Capa 3, traduce cabeceras IP
  • Inside Local = IP privada | Inside Global = IP pública del host
  • Outside Global = IP del servidor externo
  • Static NAT: 1:1 (servidores en DMZ)
  • Dynamic NAT: pool de IPs públicas
  • PAT/Overload: N privadas → 1 pública + puertos (más común)
  • Marcar interfaces: ip nat inside (LAN) y ip nat outside (WAN)
  • show ip nat translations = ver tabla de traducciones activas
