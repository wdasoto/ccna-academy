NAT / PAT

Network Address Translation · Static NAT · Dynamic NAT · PAT/Overload · Verificación · Troubleshooting

🔄 ¿Qué es NAT?

NAT (Network Address Translation) es un proceso que traduce direcciones IP privadas a públicas (y viceversa) cuando el tráfico atraviesa el router hacia Internet. Permite que múltiples dispositivos con IPs privadas compartan una o pocas IPs públicas, siendo la solución principal al agotamiento de IPv4.
✅
¿Por qué existe NAT? IPv4 solo tiene ~4.300 millones de direcciones. Con NAT, toda una empresa con miles de PCs puede conectarse a Internet usando una sola IP pública. Sin NAT, cada dispositivo necesitaría su propia IP pública.
🎯
CCNA Examen: NAT opera en Capa 3 del modelo OSI. El router que realiza NAT se llama NAT router o border router. NAT modifica las cabeceras IP de los paquetes.

🗺️ Terminología Clave NAT

NAT usa cuatro términos para describir las direcciones — es esencial entender la diferencia entre Inside/Outside y Local/Global.
TérminoDescripciónEjemplo típico
Inside Local IP privada del dispositivo interno — como la ve la red interna 192.168.1.10 (PC en la LAN)
Inside Global IP pública que representa al dispositivo interno hacia Internet 203.0.113.5 (IP del ISP)
Outside Global IP pública del servidor en Internet — como la ven todos 8.8.8.8 (DNS de Google)
Outside Local IP con la que el router interno ve al servidor externo (usualmente = Outside Global) 8.8.8.8 (sin NAT en destino)
FLUJO DE TRADUCCIÓN NAT [PC: 192.168.1.10] → Router → [Internet: 8.8.8.8]   Salida (inside → outside): IP origen: 192.168.1.10 (Inside Local) → 203.0.113.5 (Inside Global) IP destino: 8.8.8.8 (Outside Global) → 8.8.8.8 (Outside Local, sin cambio)   Retorno (outside → inside): IP origen: 8.8.8.8 → 8.8.8.8 (sin cambio) IP destino: 203.0.113.5 (Inside Global) → 192.168.1.10 (Inside Local)

📊 Tipos de NAT

TipoMapeoIPs públicasUso típico
Static NAT 1 IP privada ↔ 1 IP pública (fija) Una por host Servidores web, DMZ — acceso desde Internet
Dynamic NAT Pool de IPs privadas → Pool de IPs públicas Un pool definido Empresas con varias IPs públicas disponibles
PAT / Overload Múltiples IPs privadas → 1 sola IP pública (usando puertos) Solo una Hogares, empresas — lo más común en el mundo real
🎯
CCNA Examen: PAT (Port Address Translation) también se llama NAT Overload. Es el tipo más común y el que permite que toda una red hogareña acceda a Internet con una sola IP del ISP. PAT usa los puertos TCP/UDP para distinguir conexiones.

🏷️ Interfaces Inside y Outside

Para que NAT funcione, el router debe saber qué interfaz es inside (hacia la red privada) y cuál es outside (hacia Internet/red pública). Esta configuración es obligatoria — sin ella NAT no traduce nada.
DEFINIR INSIDE Y OUTSIDE ! Interfaz hacia la LAN privada → inside Router(config)# interface GigabitEthernet0/0 ← LAN 192.168.1.0/24 Router(config-if)# ip nat inside   ! Interfaz hacia Internet/ISP → outside Router(config)# interface GigabitEthernet0/1 ← ISP / Internet Router(config-if)# ip nat outside
⚠️
Error más común: Olvidar configurar ip nat inside y ip nat outside en las interfaces. Sin esto, el router no realiza ninguna traducción aunque el NAT esté configurado correctamente.

📌 NAT Estático — 1 a 1 Permanente

El Static NAT crea un mapeo permanente y fijo entre una IP privada y una IP pública. A diferencia del dinámico, la traducción siempre es la misma. Se usa principalmente para servidores que deben ser accesibles desde Internet (web, correo, FTP).
✅
Bidireccional: Con Static NAT, el tráfico puede iniciar desde Internet hacia el servidor interno. Esto es imposible con Dynamic NAT o PAT (donde la conexión siempre debe iniciar desde dentro).
Sintaxis:
ip nat inside source static [IP-local] [IP-global]

⌨️ Configuración NAT Estático

STATIC NAT — SERVIDOR WEB EN DMZ ! Escenario: Servidor web interno 10.0.0.10 accesible desde Internet ! IP pública asignada: 203.0.113.10   ! Paso 1: Definir la traducción estática R1(config)# ip nat inside source static 10.0.0.10 203.0.113.10   ! Paso 2: Configurar interfaces R1(config)# interface GigabitEthernet0/0 ← hacia el servidor R1(config-if)# ip nat inside R1(config)# interface GigabitEthernet0/1 ← hacia Internet R1(config-if)# ip nat outside   ! Verificar R1# show ip nat translations Pro Inside global Inside local Outside local Outside global --- 203.0.113.10 10.0.0.10 --- ---
STATIC NAT CON PUERTO — PORT FORWARDING ! Redirigir solo el puerto 80 de la IP pública al servidor interno ! (Port Forwarding — variante del Static NAT) R1(config)# ip nat inside source static tcp 10.0.0.10 80 203.0.113.5 80   ! También SSH (puerto 22) a otro servidor interno R1(config)# ip nat inside source static tcp 10.0.0.20 22 203.0.113.5 2222 ! ↑ Conexión a 203.0.113.5:2222 → redirige a 10.0.0.20:22
🎯
CCNA Examen: Static NAT con puerto específico se llama Port Forwarding. Permite redirigir un puerto de la IP pública a un servidor interno específico. Es muy común en laboratorios domésticos y pequeñas empresas.

🔀 NAT Dinámico — Pool de IPs Públicas

El Dynamic NAT mapea IPs privadas a un pool (grupo) de IPs públicas de forma dinámica. Cuando un host interno quiere salir, el router le asigna la primera IP pública disponible del pool. Cuando termina la sesión, la IP pública vuelve al pool para ser usada por otro host.
⚠️
Limitación crítica: Si el pool tiene 10 IPs y hay 11 hosts intentando salir simultáneamente, el host número 11 no podrá conectarse hasta que una IP quede libre. Esto es un problema real que PAT resuelve.
ComponenteComandoDescripción
ACLaccess-list [n] permit [red] [wildcard]Define qué IPs internas pueden usar NAT
Pool NATip nat pool [nombre] [IP-inicio] [IP-fin] netmask [máscara]Define el rango de IPs públicas disponibles
Bindingip nat inside source list [ACL] pool [nombre]Vincula la ACL con el pool de IPs públicas

⌨️ Configuración NAT Dinámico

DYNAMIC NAT — POOL DE IPs PÚBLICAS ! Escenario: LAN 192.168.1.0/24 → Pool público 203.0.113.10 a 203.0.113.20   ! Paso 1: ACL — definir quién puede usar NAT R1(config)# access-list 1 permit 192.168.1.0 0.0.0.255   ! Paso 2: Crear el pool de IPs públicas R1(config)# ip nat pool POOL-PUBLICO 203.0.113.10 203.0.113.20 netmask 255.255.255.0   ! Paso 3: Vincular ACL con pool R1(config)# ip nat inside source list 1 pool POOL-PUBLICO   ! Paso 4: Interfaces inside/outside R1(config)# interface GigabitEthernet0/0 R1(config-if)# ip nat inside R1(config)# interface GigabitEthernet0/1 R1(config-if)# ip nat outside   ! Verificar traducciones activas R1# show ip nat translations Pro Inside global Inside local Outside local Outside global tcp 203.0.113.10:1024 192.168.1.10:1024 8.8.8.8:53 8.8.8.8:53 tcp 203.0.113.11:1025 192.168.1.20:1025 1.1.1.1:80 1.1.1.1:80

⚡ PAT — Port Address Translation

PAT (también llamado NAT Overload) es la forma más usada en el mundo real. Permite que miles de hosts internos compartan una sola IP pública, diferenciándolos por el número de puerto TCP/UDP origen. Tu router doméstico usa PAT.
CÓMO PAT DISTINGUE LAS CONEXIONES PC1 (192.168.1.10:50001) → 203.0.113.5:50001 → Google 8.8.8.8:53 PC2 (192.168.1.20:50002) → 203.0.113.5:50002 → YouTube 1.1.1.1:443 PC3 (192.168.1.30:50003) → 203.0.113.5:50003 → Netflix 52.1.1.1:443   Misma IP pública: 203.0.113.5 Cada sesión identificada por un puerto único El router mantiene la tabla de traducción para el retorno
✅
Capacidad teórica: Con una sola IP pública, PAT puede manejar hasta 65.535 conexiones simultáneas (una por número de puerto disponible). En la práctica, los routers Cisco soportan miles de traducciones concurrentes.

⌨️ Configuración PAT — Usando IP de Interfaz

El método más común: usar la IP de la interfaz outside del router como IP pública. Ideal cuando el ISP asigna una sola IP dinámica o estática. Se agrega la palabra overload al final del comando.
PAT CON IP DE INTERFAZ (MÁS COMÚN) ! Paso 1: ACL — definir la red interna que usará PAT R1(config)# access-list 1 permit 192.168.1.0 0.0.0.255   ! Paso 2: Habilitar PAT usando la IP de la interfaz outside ! "interface" en lugar de un pool + "overload" al final R1(config)# ip nat inside source list 1 interface GigabitEthernet0/1 overload   ! Paso 3: Interfaces inside / outside R1(config)# interface GigabitEthernet0/0 ← LAN privada R1(config-if)# ip nat inside R1(config)# interface GigabitEthernet0/1 ← Internet/ISP R1(config-if)# ip nat outside

⌨️ Configuración PAT — Usando Pool de IPs

Variante de PAT con un pool de IPs públicas. Útil cuando el ISP proporciona más de una IP pero aun así se quiere usar PAT para maximizar el número de usuarios.
PAT CON POOL R1(config)# access-list 1 permit 192.168.0.0 0.0.255.255 R1(config)# ip nat pool PAT-POOL 203.0.113.1 203.0.113.3 netmask 255.255.255.0 ! "overload" al final → activa PAT en el pool R1(config)# ip nat inside source list 1 pool PAT-POOL overload
🎯
Diferencia clave CCNA: Dynamic NAT sin overload → una IP pública por host (limitado al tamaño del pool). Dynamic NAT con overload → PAT, múltiples hosts por IP pública usando puertos. La palabra overload es lo que activa PAT.

🏗️ Topología de Laboratorio Completo

Red empresarial con LAN interna, servidor web en DMZ y acceso a Internet. Se implementan los tres tipos de NAT: Static para el servidor web, PAT para los usuarios de la LAN.
TOPOLOGÍA LAN usuarios: 192.168.1.0/24 DMZ servidores: 10.0.0.0/30 \ / [R1] Gi0/0: 192.168.1.1/24 Gi0/2: 10.0.0.1/30 Gi0/1: 203.0.113.2/30 ← hacia ISP | [ISP] 203.0.113.1/30 | Internet   Servidor web: 10.0.0.2 → IP pública: 203.0.113.10 (Static NAT) Usuarios LAN: 192.168.1.x → salen por 203.0.113.2 (PAT/Overload)

⌨️ Configuración Completa — Paso a Paso

PASO 1 — INTERFACES R1(config)# interface GigabitEthernet0/0 R1(config-if)# ip address 192.168.1.1 255.255.255.0 R1(config-if)# ip nat inside R1(config-if)# no shutdown   R1(config)# interface GigabitEthernet0/1 R1(config-if)# ip address 203.0.113.2 255.255.255.252 R1(config-if)# ip nat outside R1(config-if)# no shutdown   R1(config)# interface GigabitEthernet0/2 R1(config-if)# ip address 10.0.0.1 255.255.255.252 R1(config-if)# ip nat inside ← DMZ también es inside R1(config-if)# no shutdown
PASO 2 — STATIC NAT PARA SERVIDOR WEB ! Servidor 10.0.0.2 visible desde Internet como 203.0.113.10 R1(config)# ip nat inside source static 10.0.0.2 203.0.113.10
PASO 3 — PAT PARA USUARIOS LAN ! ACL para identificar los usuarios de la LAN R1(config)# access-list 10 permit 192.168.1.0 0.0.0.255   ! PAT usando la IP de la interfaz outside (203.0.113.2) R1(config)# ip nat inside source list 10 interface GigabitEthernet0/1 overload
PASO 4 — RUTA DEFAULT HACIA ISP ! Sin ruta default el router no sabe hacia dónde enviar el tráfico a Internet R1(config)# ip route 0.0.0.0 0.0.0.0 203.0.113.1 ← gateway ISP
✅
Orden correcto de implementación: 1) Interfaces + inside/outside → 2) Static NAT (servidores) → 3) ACL para usuarios → 4) PAT/Overload → 5) Ruta default → 6) Verificar con show ip nat translations.

🚨 Errores Comunes — Troubleshooting

❌
Error 1: Falta ip nat inside / outside en interfaces
NAT configurado correctamente pero no traduce nada.
Verificar con show ip nat statistics → si Inside/Outside interfaces están vacíos, falta la configuración en las interfaces.
❌
Error 2: ACL incorrecta en Dynamic NAT / PAT
La ACL no coincide con la red interna — los hosts no pueden salir.
Verificar con show access-lists y comprobar que la wildcard mask cubre los hosts de la LAN.
❌
Error 3: Pool NAT con máscara incorrecta
La máscara del pool no coincide con las IPs públicas del rango definido.
Ejemplo: pool 203.0.113.10 a 203.0.113.20 necesita netmask 255.255.255.0, no /30.
❌
Error 4: Falta ruta default hacia el ISP
NAT traduce correctamente pero el tráfico no llega a Internet.
Verificar con show ip route — debe existir una ruta 0.0.0.0/0 hacia el gateway del ISP.
❌
Error 5: Olvidar "overload" en PAT
Sin overload, el router usa Dynamic NAT (una IP por host) y se agota el pool rápidamente.
El comando correcto es: ip nat inside source list 1 interface Gi0/1 overload
✅
Método sistemático: 1) show ip nat statistics → interfaces ok 2) show ip nat translations → hay entradas 3) show access-lists → ACL correcta 4) show ip route → ruta default existe 5) debug ip nat solo si todo lo anterior es correcto.
