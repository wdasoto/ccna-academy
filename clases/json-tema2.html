Parsing JSON con Python

Convierte JSON a diccionarios Python y viceversa. Domina json.loads(), json.dumps() y json.load(). Accede a datos anidados, extrae información de arrays y maneja errores profesionalmente en respuestas de APIs de red.

🐍 Python json 🎯 CCNA Domain 6.7 ⏱ ~65 min lectura 🔄 Bidireccional 🛡️ Manejo de errores

El Módulo json de Python

Python incluye soporte nativo para JSON mediante el módulo estándar json. No necesitas instalar nada (pip install). Este módulo proporciona dos funciones esenciales para el networking: parsear respuestas de APIs (de JSON a Python) y generar configuraciones (de Python a JSON).

json.loads()
string → dict
Parsea un string JSON (de una API) y lo convierte en diccionario Python.
json.dumps()
dict → string
Convierte un diccionario Python a string JSON formateado, listo para enviar.
json.load()
file → dict
Lee directamente desde un archivo .json y lo convierte a estructura Python.
json.dump()
dict → file
Guarda un diccionario Python directamente a un archivo .json.

Importación y Conceptos Clave

Importación estándar
import json

# Verificar versión (útil para debugging)
print(json.__version__)  # 2.0.9 (ejemplo)
💡 Analogía: JSON ↔ Python
  • JSON {"a": "b"} se convierte en Python {"a": "b"} (dict)
  • JSON [1, 2, 3] se convierte en Python [1, 2, 3] (list)
  • JSON "texto" se convierte en Python "texto" (str)
  • JSON true/false se convierte en Python True/False (bool)
  • JSON null se convierte en Python None (NoneType)

json.loads() — De API a Python

Cuando haces una petición HTTP a una API REST (RESTCONF, DNA Center, Meraki), recibes un string. json.loads() convierte ese string en estructuras Python nativas que puedes manipular.

Respuesta de API → Diccionario Python
import json

# Simulación de respuesta de API (string JSON)
respuesta_api = '''
{
  "hostname": "R1-CORE",
  "ip_mgmt": "192.168.1.1",
  "modelo": "ISR4321",
  "uptime_seconds": 452340,
  "interfaces_up": 4,
  "os_version": "16.9.4",
  "configurado": true,
  "ultimo_error": null
}
'''

# Convertir string JSON a diccionario Python
router = json.loads(respuesta_api)

# Ahora es un diccionario Python normal
print(router["hostname"])       # R1-CORE
print(router["ip_mgmt"])        # 192.168.1.1
print(router["uptime_seconds"]) # 452340 (int, no string)
print(router["configurado"])     # True (bool Python, no string)

💡 Tip: Validación implícita

Si json.loads() no lanza excepción, el JSON es sintácticamente válido (RFC 7159). Esto es útil para validar datos antes de procesarlos.

json.dumps() — De Python a JSON

Cuando necesitas enviar datos a una API o guardar configuraciones, conviertes diccionarios Python a strings JSON. El parámetro indent es esencial para debugging y legibilidad.

Diccionario → String JSON formateado
import json

# Configuración de red en Python
config_router = {
    "hostname": "R2-DIST",
    "interfaces": [
        {"name": "Gi0/0", "ip": "10.0.0.2/30", "ospf": True},
        {"name": "Gi0/1", "ip": "192.168.2.1/24", "ospf": False}
    ],
    "vlans": [10, 20, 30],
    "ssh_enabled": True,
    "ultimo_backup": None
}

# Convertir a JSON string (sin formato)
json_compacto = json.dumps(config_router)
print(json_compacto)  # Una línea, difícil de leer

# Convertir con indentación (formato "pretty print")
json_formateado = json.dumps(config_router, indent=2)
print(json_formateado)

Parámetros clave de dumps()

Opciones avanzadas de serialización
# Ordenar claves alfabéticamente (útil para comparar configs)
json.dumps(config, indent=2, sort_keys=True)

# Separadores personalizados (más compacto)
json.dumps(config, separators=(',', ':'))  # Sin espacios

# Asegurar ASCII (escapar caracteres especiales)
json.dumps(data, ensure_ascii=True)

# Permitir tipos no estándar (fechas, sets) con default
def serializar_fechas(obj):
    if isinstance(obj, datetime):
        return obj.isoformat()
    raise TypeError(f"Tipo no serializable: {type(obj)}")

json.dumps(datos_con_fecha, default=serializar_fechas)
⚡ Diferencias en la salida
  • Python True → JSON true (minúscula)
  • Python None → JSON null
  • Python tuples (1,2) → JSON arrays [1,2] (pérdida de inmutabilidad)
  • Python sets {1,2} → Error (no serializable directamente)

Lectura y Escritura de Archivos

Para inventarios persistentes y configuraciones guardadas, usa json.load() (lee archivo) y json.dump() (escribe archivo).

Leer inventario desde archivo JSON
import json
from pathlib import Path

# Leer archivo inventory.json
with open('inventory/network_inventory.json', 'r', encoding='utf-8') as f:
    inventario = json.load(f)  # Nota: load() no loads()

# Procesar
for device in inventario['devices']:
    print(f"Conectando a {device['hostname']}...")
Guardar reporte a archivo JSON
from datetime import datetime

reporte = {

Patrón: Configuración por Defecto

Cargar config con valores default
def cargar_config(ruta='config/settings.json'):
    """Carga JSON o retorna config por defecto si no existe"""
    config_default = {
        "timeout_ssh": 30,
        "max_retries": 3,
        "logging_level": "INFO"
    }
    
    try:
        with open(ruta) as f:
            return json.load(f)
    except FileNotFoundError:
        print(f"⚠️  {ruta} no encontrado. Usando defaults.")
        return config_default

Navegación de Estructuras Anidadas

Las respuestas de APIs de red (RESTCONF, DNA Center) son profundamente anidadas: objetos dentro de arrays dentro de objetos. Aprende a navegar estas estructuras sin perder la cordura.

Ejemplo complejo: Respuesta RESTCONF Cisco
restconf_response = '''
{
  "ietf-interfaces:interfaces": {
    "interface": [
      {
        "name": "GigabitEthernet1",
        "ietf-ip:ipv4": {
          "address": [
            {"ip": "192.168.1.1", "netmask": "255.255.255.0"},
            {"ip": "10.0.0.1", "netmask": "255.255.255.252"}
          ]
        },
        "statistics": {
          "in-octets": 1543200,
          "out-octets": 890000
        }
      }
    ]
  }
}
'''

data = json.loads(restconf_response)

# Acceso seguro a datos anidados
interfaces = data.get("ietf-interfaces:interfaces", {}).get("interface", [])
if interfaces:
    primer_intf = interfaces[0]
    nombre = primer_intf["name"]
    ips = primer_intf["ietf-ip:ipv4"]["address"]
    
    for ip_config in ips:
        print(f"{nombre}: {ip_config['ip']}/{ip_config['netmask']}")

💡 Patrón defensivo: .get() con defaults

Usa .get("clave", {}) en lugar de ["clave"] cuando no estés seguro de que la clave exista. Evita KeyError en estructuras variables de APIs.

Manejo de Errores en Parsing

Las APIs pueden devolver HTML de error en lugar de JSON, o JSON malformado. Tu código debe ser resiliente.

Manejo profesional de errores
import json
from json import JSONDecodeError

def parsear_respuesta_api(respuesta_texto):
    """
    Parsea JSON de API con manejo robusto de errores.
    Retorna (exito: bool, datos: dict|None, error: str|None)
    """
    try:
        # Intento 1: Parsear directamente
        datos = json.loads(respuesta_texto)
        return True, datos, None
        
    except JSONDecodeError as e:
        # Error de sintaxis JSON
        error_msg = f"JSON inválido: {e.msg} en línea {e.lineno}"
        return False, None, error_msg
        
    except Exception as e:
        # Otros errores (encoding, memory, etc)
        return False, None, str(e)

# Uso
exito, datos, error = parsear_respuesta_api(respuesta_api)
if exito:
    procesar_datos(datos)
else:
    loggear_error(error)

Errores HTTP vs JSON

Detectar HTML de error (404, 500)
import requests

def obtener_json_api(url):
    try:
        response = requests.get(url, timeout=10)
        response.raise_for_status()  # Lanza excepción en 4xx/5xx
        
        # Verificar que sea JSON, no HTML
        content_type = response.headers.get('content-type', '')
        if 'application/json' not in content_type:
            return None, f"Respuesta no es JSON: {content_type}"
            
        return response.json(), None  # requests tiene .json() integrado
        
    except requests.RequestException as e:
        return None, f"Error de red: {e}"
🛡️ Seguridad: Nunca uses eval()

Nunca conviertas JSON con eval() o ast.literal_eval() sin validación. json.loads() es seguro contra código malicioso. eval() ejecutaría cualquier código Python.

Ejemplo Completo: API Real

Flujo completo: Autenticación → GET → Parseo → Extracción → Acción.

Consultar Meraki API y procesar JSON
import json
import requests

def obtener_organizaciones_meraki(api_key):
    url = "https://api.meraki.com/api/v1/organizations"
    headers = {
        "X-Cisco-Meraki-API-Key": api_key,
        "Content-Type": "application/json"
    }
    
    try:
        response = requests.get(url, headers=headers, timeout=30)
        response.raise_for_status()
        
        # Parseo automático (requests internamente usa json.loads)
        orgs = response.json()
        
        # Procesar datos
        resultado = []
        for org in orgs:
            resultado.append({
                "id": org["id"],
                "nombre": org["name"],
                "url": org.get("url", "N/A")
            })
        
        # Guardar como JSON formateado
        with open("meraki_orgs.json", "w") as f:
            json.dump(resultado, f, indent=2)
            
        return resultado
        
    except json.JSONDecodeError:
        print("❌ La API no devolvió JSON válido")
    except requests.RequestException as e:
        print(f"❌ Error de conexión: {e}")

Proyecto Práctico: Inventario desde API

Guardar estado de red en JSON
def backup_configuracion(dispositivos, filename="backup.json"):
    """
    Recibe lista de dispositivos, obtiene config vía API y guarda todo.
    """
    backup_completo = {
        "timestamp": datetime.now().isoformat(),
        "total_devices": len(dispositivos),
        "configs": []
    }
    
    for dev in dispositivos:
        try:
            config = obtener_config_via_api(dev)
            backup_completo["configs"].append({
                "hostname": dev["hostname"],
                "config": config,
                "status": "success"
            })
        except Exception as e:
            backup_completo["configs"].append({
                "hostname": dev["hostname"],
                "error": str(e),
                "status": "failed"
            })
    
    # Guardar todo como JSON indentado
    with open(filename, "w") as f:
        json.dump(backup_completo, f, indent=2, sort_keys=True)
    
    return backup_completo

Parámetros Importantes

Parámetro Función Descripción Ejemplo
indent dumps/dump Espacios de indentación para pretty-print indent=2, indent=4
sort_keys dumps/dump Ordenar claves alfabéticamente sort_keys=True
ensure_ascii dumps/dump Escapar caracteres Unicode vs dejar como UTF-8 ensure_ascii=False (recomendado)
separators dumps Separadores compactos (item, key) (',', ':') para mínimo tamaño

Excepciones del módulo json

Excepción Cuándo ocurre Cómo manejar
json.JSONDecodeError String no es JSON válido try/except, loggear, usar default
TypeError Objeto no serializable (datetime, set) Convertir antes o usar default=func
FileNotFoundError Archivo no existe (load) Verificar con Path.exists() o try/except
UnicodeDecodeError Archivo con encoding incorrecto Especificar encoding='utf-8'
