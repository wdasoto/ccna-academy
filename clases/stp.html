Spanning Tree Protocol

Root Bridge · Elección · Roles · Estados · RSTP · PVST+ · PortFast · BPDU Guard

CCNA 200-301 Módulo 3 · Switching 🌳 Sin bucles Capa 2

⚠️ El problema — Bucles de Capa 2

En redes con switches redundantes, los bucles de Capa 2 destruyen la red en segundos. A diferencia de IP (que tiene el TTL para evitar bucles de routing), Ethernet no tiene ningún mecanismo nativo para detener un frame que circula infinitamente.

🌪️ Tormenta de Broadcast

Un frame broadcast (como ARP) llega al switch, que lo inunda por todos sus puertos. Al haber un bucle, el frame regresa y vuelve a inundarse infinitamente. En segundos la red alcanza el 100% de utilización y colapsa.

📋 Inestabilidad de Tabla MAC

El switch ve la misma dirección MAC llegando simultáneamente por múltiples puertos. Actualiza la tabla MAC constantemente sin poder converger. Ninguna comunicación unicast funciona correctamente.

♻️ Frames Duplicados

Los dispositivos reciben múltiples copias del mismo frame. Protocolos de capa superior como TCP se confunden y el rendimiento cae drásticamente.

🌳 STP — La solución: IEEE 802.1D

Spanning Tree Protocol crea un árbol lógico libre de bucles bloqueando selectivamente los puertos redundantes. El proceso es automático: STP detecta la topología, elige un punto raíz y calcula el mejor camino desde cada switch hacia esa raíz, bloqueando los caminos alternativos.

Los puertos bloqueados permanecen en escucha pasiva: si el enlace activo falla, STP reactiva automáticamente el puerto bloqueado para mantener la conectividad.

📨 BPDUs — Bridge Protocol Data Units

Los switches intercambian mensajes BPDU para construir y mantener el árbol STP. Son la base de todo el proceso de STP.

Campo BPDUDescripciónFunción
Root Bridge IDBridge ID del Root Bridge actualIdentifica la raíz del árbol
Root Path CostCosto acumulado hasta el Root BridgeDetermina el mejor camino

📡 Hello Timers — El latido de STP

  • Hello Time: 2 segundos — frecuencia de envío de BPDUs
  • Max Age: 20 segundos — si no llega BPDU, la topología se considera cambiada
  • Forward Delay: 15 segundos — tiempo en cada estado transitorio (Listening + Learning)

🏆 Proceso de Elección STP — 4 decisiones

STP toma 4 decisiones en orden para construir el árbol libre de bucles. Cada decisión elimina puertos candidatos hasta determinar el rol de cada puerto.

Decisión 1 — Elegir el Root Bridge

El switch con el menor Bridge ID se convierte en Root Bridge. Todos sus puertos quedan como Designated Ports (Forwarding).

🔑 Bridge ID = Priority (2 bytes) + MAC Address (6 bytes)

La prioridad se configura en múltiplos de 4096 (0, 4096, 8192... hasta 61440). El default es 32768.

Si dos switches tienen igual priority, gana el de menor MAC address.

En realidad el campo de prioridad tiene 2 partes: Priority (4 bits) + System ID Extension (12 bits = VLAN ID en PVST+)

SwitchPriorityMACBridge ID¿Root?

Decisión 2 — Elegir el Root Port de cada switch

Cada switch no-Root elige UN Root Port: el puerto con el menor costo acumulado hacia el Root Bridge.

🔢 Desempate de Root Port (en caso de igual costo)

  1. Menor Sender Bridge ID del vecino upstream
  2. Menor Sender Port ID del vecino upstream
  3. Menor Port ID local (priority.número de puerto)

Decisión 3 — Elegir Designated Port en cada segmento

En cada segmento de red, el puerto con el menor costo acumulado al Root Bridge se convierte en Designated Port. Si hay empate, gana el menor Bridge ID.

Todos los puertos del Root Bridge son Designated Ports automáticamente (costo = 0).

Decisión 4 — Bloquear los Non-Designated Ports

Cualquier puerto que no sea Root Port ni Designated Port queda como Non-Designated Port (Blocking). Esto elimina el bucle.

🗺️ Ejemplo completo — Red triangular

📋 Topología: SW1─FA0/1─SW2─FA0/1─SW3─FA0/2─SW1

SW1: priority=4096, MAC=AA | SW2: priority=32768, MAC=BB | SW3: priority=32768, MAC=CC

Resultado de las 4 decisiones

! Decisión 1: Root Bridge = SW1 (priority 4096, el menor)
! Todos los puertos de SW1 → Designated Port (Forwarding)

! Decisión 2: Root Ports
! SW2: Fa0/1 conecta a SW1 (costo=19 Fa) → Root Port
! SW3: Fa0/2 conecta a SW1 (costo=19 Fa) → Root Port

! Decisión 3: Designated Port en segmento SW2─SW3
! SW2 Fa0/2: costo acum. hacia Root = 19 (ya tiene 1 salto a Root)
! SW3 Fa0/1: costo acum. hacia Root = 19 (igual)
! Empate → menor Bridge ID gana → SW2 (BB < CC)
! SW2 Fa0/2 → Designated Port (Forwarding)

! Decisión 4: Non-Designated
! SW3 Fa0/1 → Non-Designated → BLOCKING ← elimina el bucle

🔌 Roles de Puerto STP

RolDescripciónEstadoQuién lo tiene
Root Port (RP) El puerto con el menor costo acumulado hacia el Root Bridge Forwarding Uno por cada switch no-Root
Designated Port (DP) El mejor puerto en cada segmento para enviar tráfico hacia el Root Forwarding Todos los puertos del Root Bridge + 1 por cada segmento
Non-Designated Port Puerto bloqueado para prevenir bucles Blocking Los puertos "perdedores" en cada segmento
Disabled Puerto administrativamente apagado (shutdown) Disabled Puertos con shutdown

🔄 Estados de Puerto STP 802.1D

Cuando un puerto STP pasa de Blocking a Forwarding, debe atravesar estados intermedios para evitar crear bucles temporales durante la convergencia.

EstadoDuraciónRecibe BPDUsEnvía BPDUsAprende MACsReenvía frames
Blocking 20 seg (Max Age) ✅ Sí❌ No❌ No❌ No
Listening 15 seg (Forward Delay) ✅ Sí✅ Sí❌ No❌ No
Learning 15 seg (Forward Delay) ✅ Sí✅ Sí✅ Sí❌ No
Forwarding Indefinido (normal operation) ✅ Sí✅ Sí✅ Sí✅ Sí
Disabled Hasta no shutdown ❌ No❌ No❌ No❌ No

⏱️ Tiempo total de convergencia 802.1D = 50 segundos

Blocking (20s) → Listening (15s) → Learning (15s) → Forwarding

Esto es un problema crítico en redes modernas donde se espera convergencia en segundos. Por eso se usa RSTP.

⚡ PortFast — Acceso instantáneo para hosts

PortFast hace que un puerto salte directamente a Forwarding (omitiendo Listening y Learning). Solo debe usarse en puertos conectados a hosts finales.

✅ Cuándo usar PortFast

Puertos conectados a PCs, servidores, impresoras, IP phones. Cualquier dispositivo que no sea un switch.

❌ Cuándo NO usar PortFast

Puertos conectados a otros switches, APs con múltiples clientes, hubs. Usarlo ahí puede crear bucles.

! Habilitar PortFast en un puerto específico
SW(config)# interface FastEthernet0/1
SW(config-if)# spanning-tree portfast

! Habilitar PortFast globalmente en todos los puertos de acceso
SW(config)# spanning-tree portfast default

! BPDU Guard: si un puerto PortFast recibe un BPDU → err-disabled
SW(config-if)# spanning-tree bpduguard enable

! BPDU Guard global (en todos los puertos PortFast)
SW(config)# spanning-tree portfast bpduguard default

! Recuperar puerto err-disabled
SW(config)# errdisable recovery cause bpduguard
SW(config)# errdisable recovery interval 300

⚡ RSTP — Rapid Spanning Tree Protocol (802.1w)

RSTP mejora drásticamente la convergencia de STP usando un mecanismo de negociación propuesta/acuerdo (Proposal/Agreement) en lugar de esperar timers fijos.

⏱️ Convergencia

RSTP converge en menos de 1 segundo en la mayoría de los casos, frente a los 50 segundos de 802.1D.

🔄 Retrocompatible

RSTP es retrocompatible con 802.1D. Si detecta un switch 802.1D en el segmento, vuelve automáticamente al comportamiento lento.

🔌 Tipos de enlace

P2P (full-duplex): negociación rápida. Shared (half-duplex): usa timers como 802.1D. Edge (PortFast): va directamente a Forwarding.

🔌 Nuevos roles de puerto RSTP

Rol RSTPEquivalente 802.1DEstadoDescripción
Root PortRoot PortForwardingIgual que 802.1D
Designated PortDesignated PortForwardingIgual que 802.1D
Alternate PortNon-Designated (Blocking)DiscardingBackup del Root Port. Se activa inmediatamente si el Root Port falla.
Backup PortNon-Designated (Blocking)DiscardingBackup del Designated Port. Solo en redes compartidas (half-duplex).
Edge PortPortFastForwarding inmediatoPuerto conectado a host. No participa en RSTP.

Estados RSTP — Simplificados

Estado RSTPEquivalente 802.1DReenvía frames
DiscardingBlocking + Listening + DisabledNo
LearningLearningNo (aprende MACs)
ForwardingForwardingSí

📌 RSTP reduce 5 estados a 3

RSTP fusiona Blocking, Listening y Disabled en un único estado Discarding. El proceso de convergencia ya no depende de timers fijos sino de la negociación entre switches.

🔀 PVST+ y Rapid PVST+

Cisco extiende STP para tener una instancia por VLAN, permitiendo balanceo de carga entre VLANs.

ProtocoloEstándarConvergenciaInstanciasEstado en Cisco
STP 802.1DIEEE~50 segundos1 para todas las VLANsObsoleto
PVST+Cisco propietario~50 segundos1 por VLANDisponible
RSTP 802.1wIEEE<1 segundo1 para todas las VLANsEstándar
Rapid PVST+Cisco propietario<1 segundo1 por VLAN✅ Default en Cisco IOS
MST 802.1sIEEE<1 segundoVLANs agrupadas en instanciasGrandes redes

🔀 Balanceo de carga con PVST+

Con PVST+ se puede tener:

  • SW1 = Root Bridge para VLAN 10 → el tráfico VLAN 10 usa el enlace de SW1
  • SW2 = Root Bridge para VLAN 20 → el tráfico VLAN 20 usa el enlace de SW2

Esto distribuye la carga entre los enlaces físicos, aprovechando mejor la redundancia.

Configurar Root Bridge primario y secundario

! MÉTODO 1 — Macro (ajusta priority automáticamente)
SW1(config)# spanning-tree vlan 1 root primary
! Cisco ajusta a 24576 (o menos si hay competidor con priority ≤ 24576)

SW2(config)# spanning-tree vlan 1 root secondary
! Cisco ajusta a 28672

! MÉTODO 2 — Manual (control total)
SW1(config)# spanning-tree vlan 1 priority 4096
SW2(config)# spanning-tree vlan 1 priority 8192

! Balanceo de carga: SW1=Root para VLAN10, SW2=Root para VLAN20
SW1(config)# spanning-tree vlan 10 root primary
SW1(config)# spanning-tree vlan 20 root secondary
SW2(config)# spanning-tree vlan 20 root primary
SW2(config)# spanning-tree vlan 10 root secondary

Modificar costo de puerto

! Forzar que un puerto sea Root Port reduciendo su costo
SW3(config)# interface FastEthernet0/1
SW3(config-if)# spanning-tree cost 10
! El puerto con menor costo hacia el Root Bridge será RP

! Modificar prioridad de puerto (desempate)
SW3(config-if)# spanning-tree port-priority 64
! Default = 128, rango 0-240 en múltiplos de 16

Configurar modo STP

! Ver modo actual
SW1# show spanning-tree summary | include mode
Switch is in rapid-pvst mode

! Cambiar a MST (redes grandes)
SW1(config)# spanning-tree mode mst

! Volver a Rapid PVST+ (recomendado)
SW1(config)# spanning-tree mode rapid-pvst

! Habilitar PortFast en puertos de acceso individuales

SW1(config-if-range)# spanning-tree bpduguard enable

! O globalmente (aplica a todos los puertos en modo access)
SW1(config)# spanning-tree portfast default
SW1(config)# spanning-tree portfast bpduguard default

BPDU Filter — Silenciar BPDUs en puertos de acceso

! BPDU Filter: el puerto no envía NI recibe BPDUs
! En interfaz: el puerto se convierte en edge (no participa en STP)
SW1(config-if)# spanning-tree bpdufilter enable
! Global: el puerto envía algunos BPDUs al inicio; si recibe uno, desactiva el filtro
SW1(config)# spanning-tree portfast bpdufilter default

📋 Resumen — STP para el examen CCNA

✅ Puntos críticos del examen

  • STP evita bucles Capa 2 bloqueando selectivamente puertos redundantes
  • Root Bridge = menor Bridge ID = menor Priority → menor MAC en empate
  • Priority default = 32768, configurable en múltiplos de 4096
  • Root Port = puerto con menor costo hacia el Root Bridge (uno por switch no-Root)
  • Designated Port = mejor puerto en cada segmento (Forwarding)
  • Non-Designated Port = bloqueado (previene el bucle)
  • Costos 802.1D: 10M=100, 100M=19, 1G=4, 10G=2
  • Convergencia 802.1D = 50 segundos (20+15+15)
  • RSTP = <1 segundo, 3 estados (Discarding/Learning/Forwarding)
  • Default en Cisco IOS moderno = Rapid PVST+
  • PortFast = salta directamente a Forwarding (solo en puertos de hosts)
  • BPDU Guard = err-disabled si PortFast recibe BPDU
  • spanning-tree vlan X root primary = ajusta priority a 24576
  • spanning-tree vlan X root secondary = ajusta priority a 28672

⚠️ Errores comunes en el examen

  • Configurar priority con valores que no son múltiplo de 4096 (error de sintaxis)
  • Habilitar PortFast en puertos hacia otros switches → crea bucles
  • Confundir Root Port (mejor camino al Root) con Designated Port (mejor en el segmento)
  • Olvidar que el Root Bridge tiene TODOS sus puertos como Designated
  • Confundir PVST+ (lento, Cisco) con Rapid PVST+ (rápido, Cisco default)
  • Creer que BPDU Guard solo funciona con PortFast (también funciona sin él)
