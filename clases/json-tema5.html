JSON en Respuestas de APIs Cisco

Analiza respuestas JSON reales de Cisco DNA Center API, IOS-XE RESTCONF y Meraki Dashboard API. Aprende a extraer información útil: interfaces, rutas, dispositivos y clientes usando Python.

🌐 APIs Reales 🎯 CCNA Domain 6.2 ⏱ ~90 min lectura 🧬 DNA/IOS-XE/Meraki 📊 Parseo Avanzado

El Lenguaje Común de las APIs Cisco

En la era de la red programable (Intent-Based Networking), los dispositivos Cisco ya no se configuran únicamente vía CLI. Las APIs REST devuelven datos estructurados en JSON, y tu trabajo como ingeniero de redes evolucionó: debes parsear estos JSON para extraer estados de interfaces, rutas, clientes wireless y topologías de red.

Cada plataforma Cisco habla JSON pero con dialectos diferentes. DNA Center devuelve estructuras profundas con metadados de intención. IOS-XE RESTCONF usa modelos YANG estandarizados. Meraki utiliza un formato plano y directo. Dominar los tres es dominar la visibilidad total de tu infraestructura.

🧬 Cisco DNA Center

Plataforma de control centralizada. APIs de intención, topología, clientes e inventario. Devuelve JSON profundo con arrays anidados.

/dna/intent/api/v1/network-device

⚙️ IOS-XE RESTCONF

API nativa en routers/switches Catalyst 9000. Acceso directo a modelos YANG (IETF y Cisco nativos). JSON estructurado según RFC 7951.

/restconf/data/Cisco-IOS-XE-native:native

☁️ Meraki Dashboard

API cloud-first. JSON limpio y plano. Ideal para automatización de redes SD-WAN y wireless. Rate limiting estricto.

/api/v1/organizations/{orgId}/networks

Anatomía de una Respuesta JSON Cisco

Todas las APIs Cisco comparten patrones comunes: un código de respuesta HTTP, headers de autenticación (token Bearer), y un cuerpo JSON. La diferencia radica en la estructura de datos:

Patrones de respuesta por plataforma
# DNA Center — Respuesta envuelta en "response"
{
  "response": [
    {
      "managementIpAddress": "10.10.10.1",
      "hostname": "c9300-switch-01",
      "platformId": "C9300-48UXM"
    }
  ],
  "version": "1.0"
}

# IOS-XE RESTCONF — Modelo YANG directo
{
  "Cisco-IOS-XE-native:interfaces": {
    "interface": [
      {
        "name": "GigabitEthernet1",
        "ip": {"address": {"primary": {"address": "10.1.1.1"}}}
      }
    ]
  }
}

# Meraki — Array plano de objetos
[
  {
    "name": "Sede Central",
    "id": "N_1234567890",
    "timeZone": "America/Mexico_City"
  }
]
💡 Clave para el éxito

No memorices las estructuras. Usa print(json.dumps(data, indent=2)) para visualizar el JSON devuelto, identifica el path de claves hasta tu dato objetivo (ej: ["response"][0]["managementIpAddress"]), y luego accede programáticamente.

Cisco DNA Center API

DNA Center es el cerebro de la red programable Cisco. Sus APIs devuelven JSON rico en metadatos. La clave está en entender que casi todas las respuestas vienen envueltas en un objeto response que contiene arrays de dispositivos, clientes o eventos.

🧬 Endpoints críticos para CCNA
  • /dna/intent/api/v1/network-device — Inventario completo
  • /dna/intent/api/v1/interface — Detalles de interfaces
  • /dna/intent/api/v1/client-health — Estado de clientes wireless
  • /dna/intent/api/v1/topology/physical-topology — Topología L1
Ejemplo real: Inventario de dispositivos
import requests
import json

# Token obtenido previamente via /dna/system/api/v1/auth/token
headers = {
    "X-Auth-Token": "eyJ0eXAiOiJKV1Q...",
    "Content-Type": "application/json"
}

# Obtener todos los dispositivos gestionados
url = "https://dnac.example.com/dna/intent/api/v1/network-device"
response = requests.get(url, headers=headers, verify=False)
data = response.json()

# Extracción de datos críticos
dispositivos = data["response"]  # ¡Siempre está envuelto en "response"!

for device in dispositivos:
    hostname = device["hostname"]
    ip = device["managementIpAddress"]
    plataforma = device.get("platformId", "Desconocida")
    estado = device["reachabilityStatus"]  # Reachable / Unreachable
    
    print(f"{hostname:<20} {ip:<15} {plataforma:<15} {estado}")

⚠️ Manejo de paginación

DNA Center pagina resultados grandes usando offset y limit. Siempre verifica si existe response.pagination o maneja el loop hasta que el array esté vacío para obtener el inventario completo.

IOS-XE RESTCONF API

RESTCONF (RFC 8040) proporciona una interfaz HTTP para acceder a modelos de datos YANG en dispositivos IOS-XE (Catalyst 9000, ISR 4000, ASR). El JSON devuelto sigue estrictamente la estructura del modelo YANG, con namespaces largos como Cisco-IOS-XE-native.

⚙️ Configuración previa en el dispositivo
ip http server
ip http secure-server
restconf
ip http authentication local
Consumo RESTCONF con Python requests
import requests
import json
from requests.auth import HTTPBasicAuth

# Datos del dispositivo
device = {
    "host": "192.168.1.1",
    "username": "admin",
    "password": "Cisco123",
    "port": 443
}

# Headers obligatorios para RESTCONF
headers = {
    "Accept": "application/yang-data+json",
    "Content-Type": "application/yang-data+json"
}

# Obtener interfaces nativas
url = f"https://{device['host']}:{device['port']}/restconf/data/Cisco-IOS-XE-native:native/interface"

resp = requests.get(
    url,
    headers=headers,
    auth=HTTPBasicAuth(device["username"], device["password"]),
    verify=False
)

interfaces = resp.json()
print(json.dumps(interfaces, indent=2))  # Inspeccionar estructura

Parseo de Interfaces y Estados

El modelo YANG de interfaces devuelve arrays por tipo de interfaz (GigabitEthernet, Loopback, etc.), cada uno con su propio namespace:

Extracción de datos de interfaces
# Estructura típica de respuesta RESTCONF interfaces
data = {
    "Cisco-IOS-XE-native:interface": {
        "GigabitEthernet": [
            {
                "name": "0/0/0",
                "description": "Uplink-Core",
                "ip": {
                    "address": {
                        "primary": {
                            "address": "10.1.1.1",
                            "mask": "255.255.255.0"
                        }
                    }
                },
                "shutdown": False  # Ausente si está activa
            },
            {
                "name": "0/0/1",
                "description": "Unused",
                "shutdown": True
            }
        ],
        "Loopback": [
            {"name": "0", "ip": {...}}
        ]
    }
}

# Normalizar datos de diferentes tipos de interfaz
interfaces_flat = []

for intf_type, intf_list in data["Cisco-IOS-XE-native:interface"].items():
    for intf in intf_list:
        info = {
            "tipo": intf_type,
            "nombre": f"{intf_type}{intf['name']}",
            "descripcion": intf.get("description", "Sin descripción"),
            "activa": not intf.get("shutdown", False),
            "ip": "N/A"
        }
        
        # Extracción segura de IP (puede no existir)
        if "ip" in intf and "address" in intf["ip"]:
            ip_data = intf["ip"]["address"]["primary"]
            info["ip"] = f"{ip_data['address']}/{ip_data['mask']}"
        
        interfaces_flat.append(info)

# Reporte formateado
print(f"{'Interface':<25} {'Estado':<10} {'IP Address'}")
for i in interfaces_flat:
    estado = "UP" if i["activa"] else "DOWN"
    print(f"{i['nombre']:<25} {estado:<10} {i['ip']}")
💡 Patrón: Manejo de claves opcionales

En YANG/JSON muchas claves son opcionales (como shutdown o description). Usa siempre .get() con valores por defecto para evitar KeyError cuando parsees configuraciones variables.

Meraki Dashboard API

La API de Meraki (cloud-managed) es la más simple de las tres. Devuelve arrays planos de objetos JSON sin envolturas adicionales. Es ideal para inventarios rápidos y automatización de redes SD-WAN.

☁️ Autenticación y Rate Limiting
  • Header X-Cisco-Meraki-API-Key en todas las peticiones
  • Rate limit: 5 llamadas por segundo (throttle automático necesario)
  • No requiere certificados locales (cloud proxy)
Inventario de organización y redes
import requests
import time

API_KEY = "tu-api-key-aqui"
BASE_URL = "https://api.meraki.com/api/v1"

headers = {
    "X-Cisco-Meraki-API-Key": API_KEY,
    "Content-Type": "application/json"
}

def get_organizations():
    """Obtiene todas las organizaciones accesibles"""
    url = f"{BASE_URL}/organizations"
    resp = requests.get(url, headers=headers)
    return resp.json()  # Devuelve array directo [{}, {}]

def get_networks(org_id):
    """Obtiene redes de una organización"""
    url = f"{BASE_URL}/organizations/{org_id}/networks"
    resp = requests.get(url, headers=headers)
    return resp.json()

# Ejecución
orgs = get_organizations()
for org in orgs:
    print(f"\n🏢 Organización: {org['name']} (ID: {org['id']})")
    
    networks = get_networks(org["id"])
    for net in networks:
        print(f"   └─ 🌐 {net['name']:<20} [{net['productTypes']}]")
    
    time.sleep(0.2)  # Respetar rate limiting

Dispositivos y Clientes

Extracción de datos de switches, APs y MX (security appliances), incluyendo clientes conectados:

Parseo de dispositivos y topología clientes
def get_devices(network_id):
    """Obtiene equipos físicos (switches, APs, firewalls)"""
    url = f"{BASE_URL}/networks/{network_id}/devices"
    return requests.get(url, headers=headers).json()

def get_clients(network_id, timespan=300):
    """Clientes activos en los últimos X segundos"""
    url = f"{BASE_URL}/networks/{network_id}/clients"
    params = {"timespan": timespan}
    return requests.get(url, headers=headers, params=params).json()

# Procesamiento de datos
network_id = "N_1234567890"

# Dispositivos
devices = get_devices(network_id)
inventory = []

for dev in devices:
    inventory.append({
        "nombre": dev.get("name", "Sin nombre"),
        "modelo": dev["model"],
        "serial": dev["serial"],
        "mac": dev["mac"],
        "ip_lan": dev.get("lanIp", "Desconocida"),
        "estado": "Online" if dev.get("status") == "online" else "Offline"
    })

# Clientes conectados (filtrar solo los con uso)
clients = get_clients(network_id)
clientes_activos = [c for c in clients if c.get("usage", {}).get("sent", 0) > 0]

print(f"Dispositivos: {len(inventory)} | Clientes activos: {len(clientes_activos)}")
⚠️ Datos potencialmente ausentes

Meraki omite claves completas si el valor es nulo (ej: lanIp para dispositivos offline). Nunca asumas que una clave existe; usa .get() siempre.

Técnicas de Extracción Avanzada

Cuando automatizas redes heterogéneas (DNA + IOS-XE + Meraki), necesitas normalizar datos de estructuras JSON radicalmente diferentes. Aquí técnicas profesionales de parseo defensivo.

Funciones utilitarias para extracción segura
def safe_get(data, path, default=None):
    """
    Extrae valor de un path anidado en dict sin lanzar KeyError.
    Ej: safe_get(data, ["response", 0, "managementIpAddress"])
    """
    current = data
    for key in path:
        if isinstance(current, dict) and key in current:
            current = current[key]
        elif isinstance(current, list) and isinstance(key, int) and 0 <= key < len(current):
            current = current[key]
        else:
            return default
    return current

def normalizar_dispositivo(api_type, raw_data):
    """Convierte cualquier formato Cisco a diccionario estándar"""
    
    if api_type == "dnac":
        return {
            "id": raw_data.get("id"),
            "hostname": raw_data.get("hostname"),
            "ip": raw_data.get("managementIpAddress"),
            "plataforma": raw_data.get("platformId"),
            "estado": raw_data.get("reachabilityStatus"),
            "fuente": "DNAC"
        }
    
    elif api_type == "meraki":
        return {
            "id": raw_data.get("serial"),
            "hostname": raw_data.get("name"),
            "ip": raw_data.get("lanIp"),
            "plataforma": raw_data.get("model"),
            "estado": raw_data.get("status"),
            "fuente": "Meraki"
        }
    
    elif api_type == "iosxe":
        # Para RESTCONF necesitas parsear el nombre completo
        nombre = f"{raw_data.get('type','Intf')}{raw_data.get('name','')}"
        ip_obj = safe_get(raw_data, ["ip", "address", "primary", "address"])
        
        return {
            "id": "N/A",
            "hostname": nombre,
            "ip": ip_obj or "N/A",
            "plataforma": "IOS-XE",
            "estado": "UP" if not raw_data.get("shutdown") else "DOWN",
            "fuente": "RESTCONF"
        }
    
    return {}

# Uso unificado
dnac_device = {"hostname": "sw01", "managementIpAddress": "10.0.0.5"}
standard = normalizar_dispositivo("dnac", dnac_device)
print(standard["ip"])  # 10.0.0.5

Manejo de Errores en JSON

Las APIs pueden devolver HTML en lugar de JSON (errores 500), JSON malformado, o estructuras inesperadas. Tu código debe ser defensivo:

Parseo defensivo con manejo de excepciones
import json
from json import JSONDecodeError

def parse_api_response(response_text):
    """Parsea respuesta HTTP con múltiples salvaguardas"""
    
    # 1. Verificar si hay contenido
    if not response_text or response_text.strip() == "":
        return {"error": "Respuesta vacía", "data": None}
    
    # 2. Intentar parsear JSON
    try:
        data = json.loads(response_text)
    except JSONDecodeError as e:
        return {
            "error": f"JSON inválido: {e}",
            "raw": response_text[:200]  # Primeros 200 chars para debug
        }
    except Exception as e:
        return {"error": f"Error inesperado: {e}"}
    
    # 3. Verificar estructura esperada (ej: DNA Center siempre tiene "response")
    if isinstance(data, dict) and "response" in data:
        return {"error": None, "data": data["response"], "type": "dnac"}
    
    # 4. Si es array (Meraki), devolver directo
    if isinstance(data, list):
        return {"error": None, "data": data, "type": "meraki"}
    
    # 5. Si es dict plano (RESTCONF)
    return {"error": None, "data": data, "type": "iosxe"}

🚀 Proyecto Final — Dashboard Unificado Multi-Plataforma

Integra datos reales (o simulados) de DNA Center, IOS-XE y Meraki en un único reporte CSV/JSON unificado. Este es el patrón profesional usado en NetDevOps para obtener visibilidad 360°.

💡 Arquitectura del Proyecto

Tres módulos de extracción (uno por plataforma) → normalización a esquema común → exportación a CSV o envío a base de datos. Manejo centralizado de errores y logging.

dashboard_unificado.py — Extractor multi-API completo
#!/usr/bin/env python3
"""
dashboard_unificado.py
─────────────────────────────────────────────────
Consolida inventario de DNA Center, IOS-XE y Meraki
en formato estándar para análisis unificado.
─────────────────────────────────────────────────
"""

import json
import csv
from datetime import datetime

# ══════════════════════════════════════════════════════
# SECCIÓN 1 — DATOS SIMULADOS (reemplazar con llamadas reales)
# ══════════════════════════════════════════════════════

def get_dnac_devices():
    """Simula respuesta de /dna/intent/api/v1/network-device"""
    return {
        "response": [
            {
                "hostname": "C9300-Core-01",
                "managementIpAddress": "10.1.1.10",
                "platformId": "C9300-48P",
                "reachabilityStatus": "Reachable",
                "serialNumber": "FCW12345678"
            },
            {
                "hostname": "C9300-Core-02",
                "managementIpAddress": "10.1.1.11",
                "platformId": "C9300-48P",
                "reachabilityStatus": "Unreachable",
                "serialNumber": "FCW87654321"
            }
        ]
    }

def get_iosxe_interfaces():
    """Simula respuesta RESTCONF de interfaces"""
    return {
        "Cisco-IOS-XE-native:interface": {
            "GigabitEthernet": [
                {"name": "0/0/0", "ip": {"address": {"primary": {"address": "10.1.1.1"}}}, "shutdown": False},
                {"name": "0/0/1", "shutdown": True}
            ],
            "Loopback": [
                {"name": "0", "ip": {"address": {"primary": {"address": "1.1.1.1"}}}, "shutdown": False}
            ]
        }
    }

def get_meraki_devices():
    """Simula respuesta de /organizations/{id}/devices"""
    return [
        {"name": "MX-Border-01", "model": "MX100", "serial": "Q2XX-XXXX-XXXX", "lanIp": "10.2.1.1", "status": "online"},
        {"name": "MR-AP-Floor2", "model": "MR52", "serial": "Q2XX-YYYY-YYYY", "lanIp": None, "status": "offline"}
    ]

# ══════════════════════════════════════════════════════
# SECCIÓN 2 — NORMALIZACIÓN
# ══════════════════════════════════════════════════════

def normalize_dnac(data):
    devices = []
    for d in data.get("response", []):
        devices.append({
            "fuente": "DNAC",
            "nombre": d.get("hostname", "N/A"),
            "ip": d.get("managementIpAddress", "N/A"),
            "modelo": d.get("platformId", "N/A"),
            "estado": d.get("reachabilityStatus", "Unknown"),
            "id": d.get("serialNumber", "N/A")
        })
    return devices

def normalize_iosxe(data):
    devices = []
    interfaces = data.get("Cisco-IOS-XE-native:interface", {})
    
    for intf_type, intf_list in interfaces.items():
        if not isinstance(intf_list, list):
            continue
        for intf in intf_list:
            ip = safe_get(intf, ["ip", "address", "primary", "address"], "N/A")
            devices.append({
                "fuente": "IOS-XE",
                "nombre": f"{intf_type}{intf['name']}",
                "ip": ip,
                "modelo": intf_type,
                "estado": "UP" if not intf.get("shutdown") else "DOWN",
                "id": "N/A"
            })
    return devices

def normalize_meraki(data):
    return [{
        "fuente": "Meraki",
        "nombre": d.get("name", "Unnamed"),
        "ip": d.get("lanIp") or "DHCP/Unknown",
        "modelo": d["model"],
        "estado": d.get("status", "unknown").capitalize(),
        "id": d["serial"]
    } for d in data]

def safe_get(data, path, default=None):
    current = data
    for key in path:
        if isinstance(current, dict) and key in current:
            current = current[key]
        else:
            return default
    return current

# ══════════════════════════════════════════════════════
# SECCIÓN 3 — ORQUESTACIÓN Y EXPORTACIÓN
# ══════════════════════════════════════════════════════

def main():
    print("🔍 Recolectando datos de APIs Cisco...")
    
    # Extracción
    raw_dnac = get_dnac_devices()
    raw_iosxe = get_iosxe_interfaces()
    raw_meraki = get_meraki_devices()
    
    # Normalización
    todos_dispositivos = []
    todos_dispositivos.extend(normalize_dnac(raw_dnac))
    todos_dispositivos.extend(normalize_iosxe(raw_iosxe))
    todos_dispositivos.extend(normalize_meraki(raw_meraki))
    
    # Reporte en consola
    print(f"\n📊 Total de entidades descubiertas: {len(todos_dispositivos)}")
    print(f"{'Fuente':<12} {'Nombre':<20} {'IP':<16} {'Estado':<12} {'Modelo'}")
    print("-" * 70)
    
    for dev in sorted(todos_dispositivos, key=lambda x: x["fuente"]):
        print(f"{dev['fuente']:<12} {dev['nombre']:<20} {dev['ip']:<16} {dev['estado']:<12} {dev['modelo']}")
    
    # Exportación a CSV
    timestamp = datetime.now().strftime("%Y%m%d_%H%M%S")
    filename = f"inventario_unificado_{timestamp}.csv"
    
    with open(filename, 'w', newline='') as f:
        writer = csv.DictWriter(f, fieldnames=["fuente", "nombre", "ip", "modelo", "estado", "id"])
        writer.writeheader()
        writer.writerows(todos_dispositivos)
    
    print(f"\n✅ Exportado a: {filename}")

if __name__ == "__main__":
    main()
▶️ Ejecución y Resultado Esperado

Ejecuta python dashboard_unificado.py. El script generará un CSV con columnas estandarizadas aunque los datos vengan de fuentes heterogéneas. Este patrón es el fundamento de cualquier CMDB (Configuration Management Database) automatizado.
