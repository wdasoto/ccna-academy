Routers Cisco

Arquitectura · IOS CLI · Modos · Interfaces · Seguridad · SSH · Verificación

CCNA 200-301 Módulo 2 · Dispositivos Cisco 📡 Capa 3 — OSI

📡 ¿Qué es un Router?

Un router es un dispositivo de Capa 3 del modelo OSI que interconecta redes diferentes y toma decisiones de reenvío basadas en direcciones IP. Es el dispositivo central de Internet.

🔀 Función principal

Determina la mejor ruta para cada paquete consultando su tabla de routing (RIB — Routing Information Base).

🛡️ Segmentación

Cada interfaz del router es un dominio de broadcast separado. Separa subredes y controla el tráfico entre ellas.

🌐 Interconexión

Conecta LANs con WANs, redes empresariales con Internet, y diferentes tecnologías de enlace entre sí.

🔧 Componentes internos del Router

ComponenteFunciónAnalogíaPersistencia
RAM Almacena la running-config, tablas de routing, ARP, buffers de paquetes Memoria de trabajo Volátil — se pierde al apagar
NVRAM Almacena la startup-config (configuración que carga al encender) Memoria de arranque No volátil — persiste
Flash Almacena el IOS (sistema operativo). Puede tener múltiples versiones. Disco duro del router No volátil — persiste
ROM Bootstrap, POST (Power-On Self Test), ROMMON (modo recuperación) BIOS del router Solo lectura
CPU Procesa instrucciones del IOS, algoritmos de routing, switching de paquetes Procesador —

🔄 Proceso de arranque (Boot Sequence)

  1. POST — Verifica hardware (ROM)
  2. Bootstrap — Carga el programa de arranque (ROM)
  3. Busca IOS — Primero en Flash, luego TFTP, luego ROM
  4. Carga IOS — El IOS se descomprime en RAM
  5. Busca startup-config — En NVRAM; si no existe, inicia Setup Dialog
  6. Carga running-config — La config activa queda en RAM

💻 Modos de Operación del IOS

El IOS Cisco usa una jerarquía de modos de configuración. Cada modo tiene un prompt diferente y acceso a distintos comandos.

ModoPromptAccesoQué puedes hacer
User EXEC Router> Al conectar Comandos básicos de monitoreo limitados
Privileged EXEC Router# enable Todos los show, debug, copy, reload
Global Config Router(config)# configure terminal Configuración global del dispositivo
Interface Config Router(config-if)# interface [tipo/número] Configurar IPs, velocidad, descripción
Line Config Router(config-line)# line [console|vty|aux] [número] Configurar acceso por consola/Telnet/SSH
Router Config Router(config-router)# router [ospf|eigrp|rip] Configurar protocolos de routing

Router>
Router> enable
Router#

⚙️ Configuración Inicial del Router

Paso 1 — Nombre del dispositivo (hostname)

Router(config)# hostname R1
R1(config)#
! El prompt cambia inmediatamente al nuevo nombre

Paso 2 — Banner MOTD (Message of the Day)

R1(config)# banner motd #
Enter TEXT message. End with the character '#'.
ADVERTENCIA: Acceso autorizado solamente.
Todo acceso no autorizado es monitoreado y reportado.
#

Paso 3 — Configurar interfaces LAN

R1(config)# interface GigabitEthernet0/0
R1(config-if)# description LAN-VENTAS-192.168.1.0/24
R1(config-if)# ip address 192.168.1.1 255.255.255.0
R1(config-if)# no shutdown
%LINK-5-CHANGED: Interface GigabitEthernet0/0, changed state to up
R1(config-if)# exit

R1(config)# interface GigabitEthernet0/1
R1(config-if)# description LAN-IT-10.10.10.0/24
R1(config-if)# ip address 10.10.10.1 255.255.255.0
R1(config-if)# no shutdown

Paso 4 — Ruta por defecto (gateway de último recurso)

! Enviar todo el tráfico sin ruta específica hacia el ISP
R1(config)# ip route 0.0.0.0 0.0.0.0 203.0.113.1
! Verificar que aparece como S* en show ip route

Paso 5 — Guardar la configuración

R1# copy running-config startup-config
Destination filename [startup-config]?
Building configuration...
[OK]
! Alternativa corta:
R1# write memory
Building configuration... [OK]

📁 Gestión de configuración y archivos

ComandoFunción
copy running-config startup-configGuardar RAM → NVRAM (persistir config)
copy startup-config running-configRestaurar NVRAM → RAM
erase startup-configBorrar la startup-config de NVRAM
reloadReiniciar el router
copy running-config tftp:Hacer backup de la config en servidor TFTP
copy tftp: running-configRestaurar config desde servidor TFTP
show flash:Ver archivos en la memoria Flash (IOS)
show versionVer versión IOS, hardware, uptime, registros

⚠️ Diferencia crítica — running-config vs startup-config

  • running-config → en RAM → cambios inmediatos → se PIERDE al apagar
  • startup-config → en NVRAM → carga al encender → persiste sin energía
  • Si no ejecutas copy run start y el router se reinicia, pierdes todos los cambios

🔒 Seguridad del Router — Passwords y Acceso

Un router sin contraseñas es un vector de ataque crítico. El CCNA exige configurar todas las líneas de acceso.

Enable Secret — Protege el modo Privileged EXEC

! enable secret usa MD5 hash — más seguro que enable password
R1(config)# enable secret Cisco@2024!
! NUNCA usar: enable password (texto plano, obsoleto)

Console Password — Protege el acceso físico local

R1(config)# line console 0
R1(config-line)# password Consola@123
R1(config-line)# login
R1(config-line)# exec-timeout 5 0
! exec-timeout 5 0 = cierra sesión inactiva a los 5 min
R1(config-line)# logging synchronous
! Evita que mensajes del sistema interrumpan la escritura

VTY Lines — Protege el acceso Telnet/SSH (0-15)

R1(config)# line vty 0 4
R1(config-line)# password VTY@Segura2024
R1(config-line)# login
R1(config-line)# exec-timeout 10 0
R1(config-line)# transport input ssh
! transport input ssh → solo permite SSH, bloquea Telnet

Cifrar TODAS las contraseñas en running-config

! Sin este comando, las passwords de consola y VTY son texto plano
R1(config)# service password-encryption
! Usa cifrado tipo 7 (débil pero mejor que texto plano)
! enable secret ya usa MD5 (tipo 5) — no se ve afectado

🔐 SSH — Configuración Completa

SSH reemplaza a Telnet. Es el único método de acceso remoto aceptado en producción porque cifra toda la comunicación.

Requisitos previos para SSH

1. Hostname diferente del default (Router) → ya configurado

2. Domain name configurado → obligatorio para generar el par de claves RSA

3. Usuario local con privilegio → obligatorio con login local

Configuración SSH completa

! 1. Dominio (obligatorio para generar claves RSA)
R1(config)# ip domain-name empresa.com

! 2. Crear usuario local con contraseña cifrada
R1(config)# username admin privilege 15 secret Admin@Seguro!

! 3. Generar par de claves RSA (mínimo 1024 bits, recomendado 2048)
R1(config)# crypto key generate rsa modulus 2048
The name for the keys will be: R1.empresa.com
% The key modulus size is 2048 bits
% Generating 2048 bit RSA keys, keys will be non-exportable...
[OK] (elapsed time was 3 seconds)

! 4. Forzar SSH versión 2 (más seguro que v1)
R1(config)# ip ssh version 2

! 5. Configurar VTY para usar login local + solo SSH
R1(config)# line vty 0 4
R1(config-line)# login local
R1(config-line)# transport input ssh
R1(config-line)# exec-timeout 10 0

Verificar SSH

R1# show ip ssh
SSH Enabled - version 2.0
Authentication timeout: 120 secs; Authentication retries: 3
Minimum expected Diffie Hellman key size : 1024 bits
IOS Keys in SECSH format(ssh-rsa, base64 encoded): R1.empresa.com

R1# show ssh
Connection Version Mode  Encryption  Hmac      State        Username
0          2.0    IN    aes128-cbc  hmac-md5  Session started  admin

⚠️ Telnet vs SSH — Examen CCNA

  • Telnet: Puerto 23, texto plano, NUNCA usar en producción
  • SSH: Puerto 22, cifrado, autenticación fuerte
  • transport input telnet ssh — permite ambos (no recomendado)
  • transport input ssh — solo SSH (lo correcto)
  • transport input none — deshabilita todo acceso remoto

🧪 Laboratorio — Router R1 desde cero

Escenario completo: Empresa con 2 LANs conectadas por R1. Configurar desde cero con seguridad, SSH y verificación.

📋 Topología del laboratorio

PC-VENTAS (192.168.1.10/24) ─── Gi0/0 [R1] Gi0/1 ─── PC-IT (10.10.10.10/24)

Gateway Ventas: 192.168.1.1 | Gateway IT: 10.10.10.1

Paso 1 — Configuración inicial y seguridad básica

Router> enable
Router# configure terminal
Router(config)# hostname R1
R1(config)# no ip domain-lookup
! no ip domain-lookup evita delays al escribir comandos con errores
R1(config)# enable secret Cisco@2024!
R1(config)# service password-encryption
R1(config)# banner motd #
ADVERTENCIA: Sistema de uso exclusivo autorizado.
#

Paso 2 — Configurar interfaces LAN

R1(config)# interface GigabitEthernet0/0
R1(config-if)# description LAN-VENTAS-192.168.1.0/24
R1(config-if)# ip address 192.168.1.1 255.255.255.0
R1(config-if)# no shutdown
R1(config-if)# exit

R1(config)# interface GigabitEthernet0/1
R1(config-if)# description LAN-IT-10.10.10.0/24
R1(config-if)# ip address 10.10.10.1 255.255.255.0
R1(config-if)# no shutdown
R1(config-if)# exit

Paso 3 — Consola y SSH

R1(config)# line console 0
R1(config-line)# password Consola@123
R1(config-line)# login
R1(config-line)# logging synchronous
R1(config-line)# exit

R1(config)# ip domain-name empresa.com
R1(config)# username admin privilege 15 secret Admin@2024!
R1(config)# crypto key generate rsa modulus 2048
R1(config)# ip ssh version 2
R1(config)# line vty 0 4
R1(config-line)# login local
R1(config-line)# transport input ssh
R1(config-line)# exec-timeout 10 0
R1(config-line)# end

Paso 4 — Guardar y verificar

R1# copy running-config startup-config
Building configuration... [OK]

R1# show ip interface brief
Interface         IP-Address      OK? Method Status   Protocol
GigabitEthernet0/0  192.168.1.1  YES manual up       up
GigabitEthernet0/1  10.10.10.1   YES manual up       up

R1# ping 192.168.1.10
!!!!!   (Success rate is 100 percent)

R1# ping 10.10.10.10
!!!!!   (Success rate is 100 percent)

! Verificar conectividad entre LANs (PC-VENTAS pinga PC-IT)
! Desde PC-VENTAS: ping 10.10.10.10 → R1 enruta el paquete

📋 Resumen — Routers Cisco para el examen CCNA

✅ Puntos críticos del examen

  • Router = dispositivo de Capa 3, separa dominios de broadcast
  • RAM = running-config (volátil) | NVRAM = startup-config (no volátil)
  • Flash = IOS | ROM = Bootstrap + ROMMON
  • Modos: > User → # Privileged → (config)# Global → (config-if)# Interface
  • enable secret usa MD5 — SIEMPRE preferir sobre enable password
  • service password-encryption cifra con tipo 7 (débil pero mejor que texto plano)
  • no ip domain-lookup evita delays al escribir comandos mal escritos
  • SSH requiere: hostname + domain-name + usuario local + crypto key RSA
  • ip ssh version 2 — siempre usar SSHv2, no v1
  • transport input ssh — bloquea Telnet, solo permite SSH
  • copy run start = guardar; sin esto los cambios se pierden al reiniciar
  • no shutdown — las interfaces de router vienen apagadas por default
  • Las interfaces de switch vienen activas por default (diferencia importante)

⚠️ Errores comunes en el examen

  • Usar enable password en lugar de enable secret
  • Olvidar no shutdown en las interfaces (permanecen apagadas)
  • No guardar con copy run start antes de reiniciar
  • Usar login en vez de login local cuando SSH requiere usuario local
  • Olvidar ip domain-name antes de generar las claves RSA
  • No especificar transport input ssh (deja Telnet activo)
