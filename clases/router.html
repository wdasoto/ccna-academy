🌐 ¿Qué es un Router?
Un Router es un dispositivo de red de Capa 3 (Red) del modelo OSI que interconecta redes diferentes y toma decisiones de forwarding de paquetes basándose en direcciones IP. Es el dispositivo que permite la comunicación entre redes distintas — sin router, solo hay comunicación dentro de la misma red.
🎯
CCNA Clave: Switch = Capa 2 (MACs). Router = Capa 3 (IPs). Un router decapsula la trama Ethernet, lee la IP destino, consulta su tabla de routing y reencapsula en una nueva trama con las MACs correctas para el siguiente salto.

🔄 Proceso de Forwarding de Paquetes
1
Recibe la trama — Verifica MAC destino
El router recibe la trama Ethernet. Verifica que la MAC destino sea la suya (la de su interfaz). Si no coincide, descarta la trama.
2
Desencapsula — Lee el paquete IP
Remueve el header Ethernet (Capa 2) y accede al paquete IP (Capa 3). Lee la IP destino del paquete.
3
Consulta la Tabla de Routing
Busca la coincidencia más larga (Longest Match) en su tabla de routing. Si no hay coincidencia → descarta el paquete y envía ICMP "Destination Unreachable".
4
Decrementa TTL — Verifica validez
Resta 1 al campo TTL (Time To Live) del paquete IP. Si TTL llega a 0, descarta el paquete y envía ICMP "Time Exceeded" al origen.
5
Reencapsula — Envía por la interfaz de salida
Crea una nueva trama Ethernet con su MAC como origen y la MAC del siguiente salto (next-hop) como destino. Envía el paquete por la interfaz correcta.
🎯
CCNA Examen: El router cambia las MACs en cada salto pero la IP origen e IP destino NO cambian a lo largo del camino (excepto con NAT). Esto es fundamental — pregunta común en el examen.
🗺️ La Tabla de Routing — El Cerebro del Router
La Tabla de Routing (también llamada RIB - Routing Information Base) es la base de datos del router donde almacena todas las rutas conocidas hacia las redes destino. Cada entrada contiene: red destino, máscara, protocolo origen, métrica, next-hop e interfaz de salida.

📏 Administrative Distance (AD) — Confiabilidad de Fuentes
La Administrative Distance (AD) es un valor del 0 al 255 que indica la confiabilidad de la fuente de la ruta. Cuando el router aprende la misma red por múltiples protocolos, instala en la tabla de routing la ruta con menor AD. Una AD de 255 significa "no confiable — nunca usar".
🎯
CCNA Examen — Memoriza estos valores: Connected=0, Static=1, EIGRP=90, OSPF=110, RIP=120, External EIGRP=170, Unknown=255. Si hay dos rutas a la misma red, gana la de menor AD.

📍 Routing Estático
Una ruta estática es una ruta configurada manualmente por el administrador. No cambia automáticamente cuando la topología cambia. Es simple, predecible y tiene la menor AD (1) después de las rutas conectadas. Ideal para redes pequeñas o para definir la ruta por defecto.
✅
Ventajas
Simple de configurar. No consume ancho de banda ni CPU. Predecible y seguro. AD=1 (alta prioridad). Ideal para redes pequeñas o stub.
❌
Desventajas
No se adapta a cambios de topología. Administración manual compleja en redes grandes. No escala bien. Requiere conocimiento completo de la red.
ROUTING ESTÁTICO ! Sintaxis básica: Router(config)# ip route [red-destino] [máscara] [next-hop | interfaz] ! Ejemplo 1 — Via next-hop IP (recomendado en WAN): Router(config)# ip route 192.168.3.0 255.255.255.0 10.0.0.2 ! Ejemplo 2 — Via interfaz de salida (recomendado en LAN): Router(config)# ip route 192.168.3.0 255.255.255.0 GigabitEthernet0/1 ! Ejemplo 3 — Via next-hop + interfaz (más específico): Router(config)# ip route 192.168.3.0 255.255.255.0 GigabitEthernet0/1 10.0.0.2 ! Ruta Default — hacia Internet (0.0.0.0/0): Router(config)# ip route 0.0.0.0 0.0.0.0 203.0.113.1 ! IP del ISP ! Ruta flotante (AD mayor = respaldo): Router(config)# ip route 192.168.3.0 255.255.255.0 10.0.0.5 5 ! AD=5
✅
Ruta Flotante: Es una ruta estática con AD mayor que la ruta principal. Solo se activa si la ruta principal falla. Por ejemplo: ruta primaria AD=1, ruta flotante AD=5. Mientras la primaria esté activa, la flotante no aparece en la tabla.
⚠️
Ruta Recursiva: Cuando el next-hop es una IP (no una interfaz), el router hace una segunda búsqueda en la tabla para resolver esa IP. Esto se llama recursive lookup. Para evitarlo, especifica la interfaz O usa next-hop + interfaz juntos.
⚡ Routing Dinámico — Visión General
Los protocolos de routing dinámico permiten a los routers aprender automáticamente las rutas de la red y adaptarse a cambios de topología sin intervención manual. Intercambian información de routing entre routers vecinos usando mensajes propios de cada protocolo.

🎯
CCNA Enfoque: El examen CCNA 200-301 se enfoca principalmente en OSPF de área única (Single Area OSPF). RIP y EIGRP son conceptuales. BGP es de nivel CCNP/CCIE.

Interfaz WAN (hacia el ISP / otra red): R1-CORE(config)# interface GigabitEthernet0/1 R1-CORE(config-if)# description Enlace WAN a ISP R1-CORE(config-if)# ip address 203.0.113.2 255.255.255.252 ! /30 en WAN R1-CORE(config-if)# no shutdown R1-CORE(config-if)# exit ! Subinterfaces para Router-on-a-Stick (Inter-VLAN Routing): R1-CORE(config)# interface GigabitEthernet0/0.10 R1-CORE(config-subif)# encapsulation dot1Q 10 R1-CORE(config-subif)# ip address 192.168.10.1 255.255.255.0 R1-CORE(config-subif)# exit
✅
Router-on-a-Stick: Técnica para hacer Inter-VLAN Routing con un solo enlace físico entre el router y el switch. Se crean subinterfaces (Gi0/0.10, Gi0/0.20) con encapsulación 802.1Q. El switch debe configurar ese puerto como trunk.

⚠️
Debug en Producción: Nunca uses debug ip packet en un router de producción con tráfico real — puede colapsar la CPU al 100%. Siempre usa undebug all al terminar.
🔍 Metodología de Troubleshooting — Capa por Capa
1
Capa 1 — ¿Hay señal física?
show interfaces Gi0/0 → Busca "line protocol is up". Si dice "down/down" → cable desconectado o puerto apagado. Verifica con no shutdown.
2
Capa 3 — ¿Está la IP configurada correctamente?
show ip interface brief → Verifica IP, máscara correcta y estado "up/up". Haz ping a la IP del gateway desde el router.
3
Tabla de Routing — ¿Existe la ruta?
show ip route → Verifica que existe ruta hacia el destino. Si no hay ruta → configura ruta estática o verifica el protocolo de routing.
4
Conectividad end-to-end
ping y traceroute para verificar la ruta completa. Si el ping llega hasta cierto hop y falla, el problema está en ese router o en el router siguiente.
