Pregunta 1

¿Qué métrica utiliza EIGRP por defecto para calcular la mejor ruta?

✓ Respuesta Correcta: B

EIGRP utiliza por defecto ancho de banda (bandwidth) y retardo (delay) para calcular su métrica compuesta. Aunque puede usar MTU, carga y confiabilidad, en la configuración predeterminada solo considera bandwidth y delay con valores K: K1=1, K2=0, K3=1, K4=0, K5=0.
Pregunta 2

En OSPF, ¿qué tipo de LSA describe rutas externas redistribuidas en el dominio OSPF por un ASBR?

✓ Respuesta Correcta: C

Los LSA Tipo 5 son generados por routers ASBR (Autonomous System Boundary Router) y describen rutas externas al dominio OSPF. Se propagan por todas las áreas excepto áreas stub. Los LSA Tipo 7 se usan solo en áreas NSSA como alternativa.
Pregunta 3

¿Cuál es la distancia administrativa por defecto de EIGRP para rutas internas?

✓ Respuesta Correcta: A

EIGRP tiene AD=90 para rutas internas, 170 para rutas externas (redistribuidas), y 5 para rutas sumario. OSPF por su parte tiene AD=110.
Pregunta 4

¿Qué algoritmo utiliza EIGRP para calcular rutas?

✓ Respuesta Correcta: C

EIGRP utiliza el algoritmo DUAL (Diffusing Update Algorithm) que garantiza rutas libres de loops y permite convergencia rápida. Dijkstra es usado por OSPF. EIGRP es clasificado como protocolo de "distancia-vector avanzado".
Pregunta 5

En OSPF, ¿cuál es la función del DR (Designated Router) en una red broadcast?

✓ Respuesta Correcta: B

El DR y BDR se eligen en redes broadcast/multi-access para reducir el número de adyacencias. Sin DR, en una red con N routers se necesitarían N(N-1)/2 adyacencias. Con DR, cada router forma adyacencia solo con DR/BDR, reduciendo la complejidad a 2N-3.
Pregunta 6

¿Cuál es el propósito del Type 2 LSA (Network LSA) en OSPF?

✓ Respuesta Correcta: B

El Type 2 LSA (Network LSA) es generado por el DR (Designated Router) en redes multi-acceso (Ethernet, FDDI).

Contenido del Type 2 LSA:
• Dirección IP de la red
• Máscara de subred
• Lista de todos los routers conectados al segmento

Dónde se genera:
• Solo en redes multi-acceso (Broadcast/NBMA)
• NO se genera en enlaces point-to-point
• El DR es el único que lo origina

Flujo de información:
1. DR recopila info de todos los routers del segmento
2. Genera Type 2 LSA
3. Lo inunda dentro del área
Pregunta 7

¿Cuál es la función del comando 'variance' en EIGRP?

✓ Respuesta Correcta: B

El comando variance permite a EIGRP realizar load balancing desigual (unequal-cost load balancing) entre múltiples rutas.

Sintaxis:
router eigrp 100
variance 2

Funcionamiento:
• Valor por defecto: 1 (solo equal-cost load balancing)
• Rango: 1-128
• Una ruta se usa si: Métrica ≤ (Mejor_Métrica × Variance)

Ejemplo:
Si la mejor ruta tiene métrica 1000 y variance=2:
• Ruta con métrica 1500: ✅ Se usa (1500 ≤ 2000)
• Ruta con métrica 2500: ❌ No se usa (2500 > 2000)

Requisito: La ruta debe cumplir la feasibility condition (ser libre de loops).
Pregunta 8

En OSPF, ¿qué significa el estado 'EXSTART' durante la formación de adyacencia?

✓ Respuesta Correcta: B

El estado EXSTART es el tercer estado en la formación de adyacencia OSPF.

Proceso de formación completo:
1. DOWN: No se han recibido Hellos
2. INIT: Se recibió Hello del vecino
3. 2-WAY: Comunicación bidireccional establecida
4. EXSTART: Negociación master/slave (router con RID mayor = master)
5. EXCHANGE: Intercambio de DBDs (Database Descriptions)
6. LOADING: Solicitud de LSAs faltantes vía LSR/LSU
7. FULL: LSDBs completamente sincronizadas

En EXSTART:
• Se determina el master (inicia el intercambio)
• Se establece el número de secuencia inicial
• Se envían DBDs vacíos para negociación
Pregunta 9

¿Qué condición debe cumplirse para que una ruta sea considerada 'Feasible Successor' en EIGRP?

✓ Respuesta Correcta: A

La Feasibility Condition garantiza rutas libres de loops en EIGRP.

Términos clave:
• FD (Feasible Distance): Métrica de la mejor ruta actual
• RD (Reported Distance): Métrica que anuncia el vecino hacia el destino
• Successor: Mejor ruta (menor FD)
• Feasible Successor (FS): Ruta de respaldo que cumple: RD < FD

Ejemplo:
Ruta A (Successor): FD = 1000
Ruta B: RD = 800 ✅ Es FS (800 < 1000)
Ruta C: RD = 1200 ❌ No es FS (1200 > 1000)

Beneficio:
Si el Successor falla, EIGRP usa el FS inmediatamente sin ejecutar DUAL (convergencia instantánea).
Pregunta 10

¿Cuál es el intervalo de Hello por defecto en OSPF para redes Ethernet?

✓ Respuesta Correcta: B

OSPF utiliza diferentes timers según el tipo de red:

Redes Broadcast y Point-to-Point (Ethernet, Serial):
• Hello interval: 10 segundos
• Dead interval: 40 segundos (4× Hello)

Redes NBMA (Frame Relay, ATM):
• Hello interval: 30 segundos
• Dead interval: 120 segundos (4× Hello)

Comandos de verificación:
show ip ospf interface
show ip ospf neighbor

Modificar timers:
interface GigabitEthernet0/0
ip ospf hello-interval 5
ip ospf dead-interval 20

IMPORTANTE: Los timers deben coincidir en ambos extremos para formar adyacencia.
Pregunta 11

¿Qué tipo de área OSPF permite redistribución de rutas externas pero bloquea Type 5 LSAs?

✓ Respuesta Correcta: C

Un NSSA es un área especial que permite redistribución de rutas externas de manera limitada.

Características de NSSA:
• Bloquea Type 5 LSAs (rutas externas normales)
• Permite Type 7 LSAs (generados por ASBR en el NSSA)
• El ABR convierte Type 7 → Type 5 al salir del NSSA
• Permite ASBRs dentro del área stub

Configuración:
router ospf 1
area 10 nssa

Caso de uso:
Oficinas remotas que necesitan redistribuir rutas de otro protocolo (ej. RIP) pero que también se benefician de las ventajas de un área stub (menor LSDB).

Variante: area 10 nssa no-summary crea un Totally NSSA.
Pregunta 12

En EIGRP, ¿qué comando muestra las rutas almacenadas en la tabla de topología pero que no están en la tabla de routing?

✓ Respuesta Correcta: C

EIGRP mantiene diferentes estructuras de datos:

Tablas EIGRP:
• Neighbor Table: Vecinos activos
• Topology Table: Todas las rutas conocidas
• Routing Table: Solo Successors (mejores rutas)

Comandos de visualización:
• show ip eigrp topology: Solo Successors y Feasible Successors
• show ip eigrp topology all-links: TODAS las rutas, incluyendo las que no cumplen feasibility condition
• show ip route eigrp: Solo rutas en tabla de routing

Información revelada por all-links:
• Rutas con RD > FD (no son FS)
• Métricas de todas las rutas posibles
• Útil para troubleshooting y entender por qué una ruta no se eligió
Pregunta 13

¿Cuál es el propósito del Type 3 LSA (Summary LSA) en OSPF?

✓ Respuesta Correcta: B

El Type 3 LSA (Summary LSA) es generado por el ABR (Area Border Router) para anunciar rutas de un área a otra.

Funcionamiento:
1. ABR recibe Type 1 y Type 2 LSAs en un área
2. Calcula las rutas óptimas
3. Genera Type 3 LSAs para anunciar esas rutas a otras áreas
4. El Type 3 LSA cruza el área backbone (Area 0)

Contenido del Type 3 LSA:
• Prefijo de red y máscara
• Métrica hacia el destino
• Router ID del ABR que lo generó

IMPORTANTE:
• Type 3 NO contiene información topológica detallada
• Cada área mantiene su propia topología completa
• Areas stub bloquean Type 5, totally stub bloquean Type 3 y Type 5
Pregunta 14

En EIGRP, ¿qué sucede cuando se pierde el Successor y no existe un Feasible Successor?

✓ Respuesta Correcta: B

Cuando se pierde el Successor y no hay Feasible Successor, EIGRP entra en estado Active y ejecuta DUAL (Diffusing Update Algorithm).

Proceso DUAL:
1. Router marca la ruta como Active
2. Envía Query a todos los vecinos preguntando por la red
3. Los vecinos responden con Reply (tienen ruta o no)
4. Router espera respuestas de TODOS los vecinos
5. Calcula nueva ruta óptima
6. Marca la ruta como Passive (estable)

Problema potencial: SIA (Stuck-In-Active)
• Si un vecino no responde en 3 minutos (180 seg)
• El router declara el vecino como SIA
• Rompe la adyacencia con ese vecino

Con Feasible Successor:
✅ Convergencia instantánea (sin DUAL)
Pregunta 15

¿Qué comando OSPF configura autenticación MD5 en una interfaz?

✓ Respuesta Correcta: A

OSPF soporta tres tipos de autenticación:

1. Null (sin autenticación)
• Por defecto
• No requiere configuración

2. Clear Text (tipo 1)
interface GigabitEthernet0/0
ip ospf authentication
ip ospf authentication-key CISCO123

3. MD5 (tipo 2) - RECOMENDADO
interface GigabitEthernet0/0
ip ospf authentication message-digest
ip ospf message-digest-key 1 md5 SECRET_KEY

Configuración por área (alternativa):
router ospf 1
area 0 authentication message-digest

interface GigabitEthernet0/0
ip ospf message-digest-key 1 md5 SECRET_KEY

Verificación:
show ip ospf interface
Pregunta 16

¿Cuál es la dirección multicast utilizada por EIGRP para enviar actualizaciones?

✓ Respuesta Correcta: D

EIGRP utiliza 224.0.0.10 como dirección multicast para comunicarse con vecinos.

Direcciones multicast de protocolos de routing:
• EIGRP: 224.0.0.10
• OSPF (All Routers): 224.0.0.5
• OSPF (DR/BDR): 224.0.0.6
• RIPv2: 224.0.0.9
• HSRP: 224.0.0.2
• VRRP: 224.0.0.18

Ventajas del multicast en EIGRP:
• Reduce carga de CPU en routers que no ejecutan EIGRP
• Más eficiente que broadcast
• Permite comunicación uno-a-muchos

Protocolo IP:
EIGRP usa protocolo número 88 (no TCP ni UDP)

Dirección MAC multicast correspondiente:
01:00:5E:00:00:0A
Pregunta 17

En OSPF, ¿qué router se elige como DR (Designated Router) en un segmento multi-acceso?

✓ Respuesta Correcta: C

Proceso de elección DR/BDR:
1. Se elige el router con prioridad OSPF más alta
2. Si hay empate, gana el Router ID más alto
3. Prioridad por defecto: 1
4. Rango de prioridad: 0-255
5. Prioridad 0 = router NO puede ser DR/BDR

Configurar prioridad:
interface GigabitEthernet0/0
ip ospf priority 100

Importante - Elección NO es preemptiva:
• Si llega un router con mayor prioridad/RID después de la elección
• NO desplaza al DR/BDR actual
• Solo se re-elige si DR/BDR falla

Verificación:
show ip ospf neighbor
show ip ospf interface
Pregunta 18

¿Qué tipo de paquete EIGRP se utiliza para confirmar la recepción de actualizaciones confiables?

✓ Respuesta Correcta: C

EIGRP utiliza 5 tipos de paquetes:

1. Hello
• Descubrimiento y mantenimiento de vecinos
• Multicast 224.0.0.10
• No requiere ACK (no confiable)

2. Update
• Envía información de rutas
• Requiere ACK (confiable)
• Puede ser multicast o unicast

3. Query
• Solicita información sobre una ruta (DUAL activo)
• Requiere ACK (confiable)
• Multicast

4. Reply
• Responde a un Query
• Requiere ACK (confiable)
• Unicast

5. ACK (Acknowledgment)
• Confirma recepción de Update, Query o Reply
• Es un Hello vacío enviado unicast
• No requiere ACK (no confiable)

Verificación de paquetes:
show ip eigrp traffic
Pregunta 19

¿Cuál es el máximo número de rutas de equal-cost que OSPF puede instalar en la tabla de routing por defecto?

✓ Respuesta Correcta: B

Por defecto, OSPF permite 4 rutas de equal-cost (mismo costo) para load balancing.

Modificar el límite:
router ospf 1
maximum-paths 8

Rango permitido:
• Mínimo: 1 (desactiva load balancing)
• Máximo: 32 (IOS moderno)
• Default: 4

Comparación con EIGRP:
• EIGRP también default 4, máximo 32
• EIGRP permite unequal-cost load balancing con variance
• OSPF solo equal-cost load balancing

Verificación:
show ip route ospf
(Las rutas con load balancing se muestran con múltiples next-hops)

Algoritmo de reparto:
• Per-destination (basado en IP destino)
• CEF (Cisco Express Forwarding) por defecto
Pregunta 20

En EIGRP, ¿qué significa el término 'Split Horizon'?

✓ Respuesta Correcta: A

Split Horizon es una técnica anti-loop fundamental en protocolos de vector distancia (RIP, EIGRP).

Regla:
Si un router aprende una ruta por la interfaz X, NO la anuncia de vuelta por la interfaz X.

Ejemplo:
R1 -- [Fa0/0] -- R2 -- [Fa0/1] -- R3

• R2 aprende 10.0.0.0/8 desde R1 vía Fa0/0
• R2 anuncia 10.0.0.0/8 a R3 vía Fa0/1 ✅
• R2 NO anuncia 10.0.0.0/8 de vuelta a R1 vía Fa0/0 ❌

Variante: Split Horizon with Poison Reverse
• Anuncia la ruta de vuelta, pero con métrica infinita
• En EIGRP: métrica = 4,294,967,295 (2^32 - 1)

Desactivar (solo si es necesario):
interface Serial0/0
no ip split-horizon eigrp 100
Pregunta 21

¿Qué comando OSPF suma rutas en un ABR para reducir el tamaño de la LSDB?

✓ Respuesta Correcta: B

OSPF permite dos tipos de sumarización:

1. Sumarización Inter-Área (en ABR)
router ospf 1
area 10 range 192.168.0.0 255.255.252.0

• Resume Type 1/2 LSAs de un área en Type 3 LSAs
• Reduce LSAs propagados a otras áreas
• Se aplica en el ABR

2. Sumarización de Rutas Externas (en ASBR)
router ospf 1
summary-address 10.0.0.0 255.0.0.0

• Resume Type 5 LSAs (rutas redistribuidas)
• Se aplica en el ASBR

Beneficios:
• Reduce tamaño de LSDB
• Reduce CPU y memoria
• Mejora estabilidad (los flaps internos no se propagan)

Auto-generación de ruta null:
El router crea automáticamente Null0 para la ruta resumen (prevenir loops).
Pregunta 22

¿Cuál es el rango de números AS (Autonomous System) válidos para EIGRP?

✓ Respuesta Correcta: A

EIGRP utiliza números de Autonomous System (AS) para identificar procesos de routing.

Rango válido:
• Mínimo: 1
• Máximo: 65535 (16 bits)

Configuración:
router eigrp 100

• El número AS debe coincidir entre vecinos
• Routers con diferentes AS no formarán adyacencia
• Un router puede ejecutar múltiples procesos EIGRP

Comparación con BGP:
• BGP AS públicos: 1-64511
• BGP AS privados: 64512-65535
• BGP AS extendidos: 1-4294967295 (32 bits)

Named EIGRP (IOS 15+):
router eigrp CAMPUS
address-family ipv4 autonomous-system 100
network 10.0.0.0

Permite configuración más modular.
Pregunta 23

En OSPF, ¿qué campo del paquete Hello debe coincidir para formar adyacencia (además del área)?

✓ Respuesta Correcta: B

Para formar adyacencia OSPF, los siguientes parámetros DEBEN coincidir:

1. Area ID
• Deben estar en la misma área

2. Hello y Dead intervals
• Default: 10s Hello, 40s Dead (broadcast/p2p)
• Default: 30s Hello, 120s Dead (NBMA)

3. Subnet y máscara
• Deben estar en la misma subred
• Excepto en enlaces point-to-point

4. Autenticación
• Tipo (null, clear text, MD5)
• Clave (password o MD5 key)

5. Stub area flag
• Ambos stub o ambos no-stub

NO necesitan coincidir:
• Router ID (debe ser único)
• Priority (solo afecta elección DR/BDR)
• Cost de interfaz

Troubleshooting:
debug ip ospf adj
Pregunta 24

¿Qué comando EIGRP configura una ruta por defecto y la propaga a todos los vecinos?

✓ Respuesta Correcta: B

EIGRP puede propagar una ruta por defecto de varias formas:

Método 1: Ruta estática + Redistribución
ip route 0.0.0.0 0.0.0.0 10.1.1.1
router eigrp 100
redistribute static

Método 2: Network statement (menos recomendado)
ip route 0.0.0.0 0.0.0.0 10.1.1.1
router eigrp 100
network 0.0.0.0

Método 3: ip default-network (legacy)
ip default-network 192.168.1.0
router eigrp 100
network 192.168.1.0

IMPORTANTE:
• default-information originate es para OSPF, NO EIGRP
• La ruta por defecto se marca como EX (externa) en vecinos
• AD = 170 (ruta externa EIGRP)

Verificación:
show ip route 0.0.0.0
Pregunta 25

¿Cuál es el propósito del Type 4 LSA en OSPF?

✓ Respuesta Correcta: B

El Type 4 LSA (ASBR Summary LSA) es generado por el ABR para informar a otras áreas cómo alcanzar un ASBR (Autonomous System Boundary Router).

Flujo de información:
1. ASBR redistribuye rutas externas → genera Type 5 LSAs
2. ABR recibe Type 5 LSAs en su área
3. ABR genera Type 4 LSA para anunciar la ubicación del ASBR
4. Type 4 LSA se propaga a otras áreas vía backbone
5. Routers en otras áreas usan Type 4 para calcular ruta al ASBR
6. Luego combinan esa ruta con Type 5 para calcular costo total

Contenido del Type 4 LSA:
• Router ID del ASBR
• Métrica para alcanzar el ASBR

Visualización:
show ip ospf database asbr-summary

Nota: Type 4 NO contiene rutas, solo dice dónde está el ASBR.
Pregunta 26

En EIGRP, ¿qué tabla almacena todas las rutas conocidas, incluyendo Successors y Feasible Successors?

✓ Respuesta Correcta: C

EIGRP mantiene tres tablas de datos:

1. Neighbor Table
• Lista de vecinos EIGRP directamente conectados
• Dirección IP del vecino
• Interfaz local
• Hold time (tiempo antes de declarar vecino caído)
• Comando: show ip eigrp neighbors

2. Topology Table
• TODAS las rutas conocidas hacia cada red
• Successor (mejor ruta)
• Feasible Successors (rutas de respaldo)
• Rutas que no cumplen feasibility condition
• FD, RD, métricas
• Comando: show ip eigrp topology
• Comando completo: show ip eigrp topology all-links

3. Routing Table (IP)
• Solo las mejores rutas (Successors)
• Compartida con otros protocolos
• Comando: show ip route eigrp
Pregunta 27

¿Qué tipo de red OSPF NO requiere la elección de DR/BDR?

✓ Respuesta Correcta: B

OSPF clasifica las redes en diferentes tipos según su topología:

1. Broadcast (Ethernet)
• Requiere DR/BDR ✅
• Multicast habilitado
• Hello: 10s, Dead: 40s

2. Point-to-Point (Serial PPP/HDLC)
• NO requiere DR/BDR ❌
• Solo 2 routers en el segmento
• Hello: 10s, Dead: 40s
• Forma adyacencia directa (estado FULL)

3. NBMA (Frame Relay, ATM)
• Requiere DR/BDR ✅
• Sin multicast nativo
• Hello: 30s, Dead: 120s
• Requiere configuración de vecinos estáticos

4. Point-to-Multipoint
• NO requiere DR/BDR ❌
• Trata NBMA como colección de enlaces P2P

Configurar tipo de red:
interface Serial0/0
ip ospf network point-to-point

Pregunta 29

En EIGRP, ¿qué valor K por defecto está habilitado en la fórmula de métrica compuesta?

✓ Respuesta Correcta: B

EIGRP utiliza la siguiente fórmula de métrica compuesta:

Metric = 256 × [(K1×BW + (K2×BW)/(256-Load) + K3×Delay) × (K5/(Reliability+K4))]

Valores K por defecto:
• K1 = 1 (Bandwidth habilitado)
• K2 = 0 (Bandwidth/Load deshabilitado)
• K3 = 1 (Delay habilitado)
• K4 = 0 (Reliability deshabilitado)
• K5 = 0 (Reliability deshabilitado)

Esto simplifica la fórmula a:
Metric = 256 × (Bandwidth + Delay)

Modificar valores K (NO RECOMENDADO):
router eigrp 100
metric weights 0 1 0 1 0 0
(tos k1 k2 k3 k4 k5)

IMPORTANTE:
• Los valores K DEBEN coincidir entre vecinos
• Si no coinciden, no se forma adyacencia
• Cambiar valores K puede desestabilizar la red
Pregunta 30

¿Qué comando OSPF permite ajustar el costo (métrica) de una interfaz manualmente?

✓ Respuesta Correcta: A

Cálculo de costo OSPF:
Cost = Reference_Bandwidth / Interface_Bandwidth

Reference Bandwidth por defecto:
• 100 Mbps (100,000,000 bps)
• Problema: FastEthernet (100Mbps) y GigabitEthernet (1Gbps) tienen cost = 1

Modificar Reference Bandwidth (recomendado):
router ospf 1
auto-cost reference-bandwidth 10000 ! 10 Gbps

Modificar Cost manualmente (por interfaz):
interface GigabitEthernet0/0
ip ospf cost 50

Ejemplos de costos con ref-bw default (100 Mbps):
• 10 Mbps: cost = 10
• 100 Mbps (FastEthernet): cost = 1
• 1 Gbps (GigabitEthernet): cost = 1 ❌
• 10 Gbps: cost = 1 ❌

Verificación:
show ip ospf interface
