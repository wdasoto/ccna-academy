💡 ¿Qué es un Switch?
Un Switch es un dispositivo de red de Capa 2 (Enlace de Datos) del modelo OSI que conecta múltiples dispositivos en una red local. A diferencia de un Hub, el switch toma decisiones inteligentes de reenvío basadas en direcciones MAC, enviando las tramas únicamente al puerto de destino correcto — no a todos los puertos.
🎯
CCNA Clave: Un switch opera en Capa 2 usando MACs. Un router opera en Capa 3 usando IPs. Un switch multilayer (L3) puede hacer ambas cosas con el comando ip routing.

🗂️ La Tabla CAM — El Cerebro del Switch
La Tabla CAM (Content Addressable Memory), también llamada Tabla MAC, es la base de datos del switch donde almacena las direcciones MAC aprendidas, el puerto por donde las aprendió y el ID de VLAN. Con esta información toma decisiones de forwarding en microsegundos.
1
Llega una trama — Switch revisa MAC origen
El switch lee la MAC de origen de la trama. Si no está en la tabla CAM, la aprende y la asocia al puerto de entrada + VLAN. Esto se llama MAC Address Learning.
2
Busca la MAC destino en la tabla
El switch busca la MAC de destino en su tabla CAM. Si la encuentra → Forwarding directo al puerto. Si NO la encuentra → Unknown Unicast Flood (envía por todos los puertos menos el de entrada).
3
¿MAC destino es Broadcast o Multicast?
Si la MAC destino es FF:FF:FF:FF:FF:FF (broadcast) → se envía por todos los puertos de la misma VLAN. El broadcast no cruza dominios de broadcast (VLANs o routers).
4
Aging — Limpieza automática
Las entradas en la tabla CAM tienen un timer de 300 segundos (5 minutos) por defecto. Si no se recibe tráfico de esa MAC en ese tiempo, la entrada se elimina automáticamente para mantener la tabla actualizada.
🎯
CCNA Examen: El proceso se llama Store-and-Forward en switches modernos: el switch recibe la trama completa, verifica el FCS (checksum), y luego decide el forwarding. Otros métodos: Cut-Through (reenvía apenas lee el destino, sin verificar errores) y Fragment-Free (espera 64 bytes).

🔌 Half Duplex vs Full Duplex
Half Duplex: El dispositivo puede enviar O recibir, pero no ambos simultáneamente. Usa CSMA/CD para detectar colisiones. Solo en conexiones a Hubs o dispositivos muy antiguos.

Full Duplex: El dispositivo puede enviar Y recibir simultáneamente. No hay colisiones. Duplica el throughput efectivo. Estándar en todas las conexiones modernas switch-switch y switch-PC.
VelocidadHalf DuplexFull Duplex
FastEthernet (100 Mbps)100 Mbps compartido200 Mbps efectivos ✅
GigabitEthernet (1 Gbps)Raro / no recomendado2 Gbps efectivos ✅
10GigabitEthernetNo soportado20 Gbps efectivos ✅
⚠️
Duplex Mismatch: Si un extremo está en Full y el otro en Half, habrá colisiones y rendimiento degradado. Síntoma: muchos late collisions en show interfaces. Solución: configurar manualmente ambos extremos o dejar autonegociación en ambos.
⚙️ Autonegociación de Velocidad y Dúplex
CONFIGURACIÓN Switch(config)# interface fastEthernet 0/1 Switch(config-if)# duplex full ! full | half | auto Switch(config-if)# speed 100 ! 10 | 100 | 1000 | auto Switch(config-if)# no shutdown ! Verificar velocidad/dúplex negociados: Switch# show interfaces fa0/1 ! Busca la línea: "Full-duplex, 100Mb/s"
✅
Best Practice: En enlaces switch-switch o switch-router de producción, configura velocidad y dúplex manualmente en ambos extremos (no dejes autonegociación). En puertos de acceso a PCs, autonegociación está bien.

🔒 Port Security — Seguridad en Puertos de Acceso
Port Security es una función de seguridad de Capa 2 que permite limitar qué MACs pueden acceder a un puerto de switch. Si se conecta un dispositivo no autorizado, el switch puede restringir, apagar o descartar el tráfico automáticamente.
🎯
CCNA Examen: Port Security solo funciona en puertos de acceso (access), NO en puertos trunk. Debes hacer switchport mode access antes de habilitarlo.
⚖️ Modos de Violación
ModoAcción al violarEnvía SyslogIncrementa contadorPuerto sigue activo
Protect Descarta tramas de MACs no autorizadas ✗ No ✗ No ✓ Sí
Restrict Descarta tramas + notifica al administrador ✓ Sí ✓ Sí ✓ Sí
Shutdown Pone el puerto en err-disabled ✓ Sí ✓ Sí ✗ No — err-disabled
⚠️
El modo Shutdown es el default en Cisco IOS. Si no especificas el modo, se usará Shutdown. Para recuperar un puerto en err-disabled: shutdown seguido de no shutdown.
⌨️ Configuración Completa de Port Security
PORT SECURITY Switch(config)# interface fastEthernet 0/1 Switch(config-if)# switchport mode access ! OBLIGATORIO primero Switch(config-if)# switchport port-security ! Habilitar Port Security Switch(config-if)# switchport port-security maximum 2 ! Máx 2 MACs permitidas Switch(config-if)# switchport port-security violation restrict ! Modo violación ! Opción A — MAC estática (manual): Switch(config-if)# switchport port-security mac-address aaaa.bbbb.cccc ! Opción B — MAC sticky (aprende y guarda automáticamente): Switch(config-if)# switchport port-security mac-address sticky
VERIFICACIÓN Switch# show port-security ! Resumen de todos los puertos Switch# show port-security interface fa0/1 ! Detalle de un puerto Switch# show port-security address ! MACs seguras configuradas ! Recuperar puerto en err-disabled: Switch(config-if)# shutdown Switch(config-if)# no shutdown
✅
MAC Sticky: Es la opción más práctica. El switch aprende la primera MAC que se conecta y la "pega" como MAC segura automáticamente. Si haces copy run start, esa MAC queda guardada permanentemente.

🌐 SVI — Switch Virtual Interface (Management)
Una SVI (Switch Virtual Interface) es una interfaz lógica asociada a una VLAN que permite asignarle una dirección IP al switch para administración remota vía SSH/Telnet. La VLAN de management es generalmente la VLAN 99 (nunca usar VLAN 1 por seguridad).
SVI MANAGEMENT SW-CORE-01(config)# vlan 99 SW-CORE-01(config-vlan)# name MANAGEMENT SW-CORE-01(config-vlan)# exit SW-CORE-01(config)# interface vlan 99 SW-CORE-01(config-if)# ip address 192.168.99.10 255.255.255.0 SW-CORE-01(config-if)# no shutdown SW-CORE-01(config-if)# exit ! Default gateway para que el switch alcance otras redes: SW-CORE-01(config)# ip default-gateway 192.168.99.1 ! Guardar configuración: SW-CORE-01# copy running-config startup-config
📡 CDP — Cisco Discovery Protocol
CDP (Cisco Discovery Protocol) es un protocolo propietario de Cisco que opera en Capa 2 y permite descubrir automáticamente los dispositivos Cisco directamente conectados. Envía anuncios cada 60 segundos hacia la dirección multicast 01:00:0C:CC:CC:CC. Esencial para troubleshooting y documentación de red.
⚠️
Seguridad: CDP revela información sensible del dispositivo (modelo, versión IOS, IPs). Desactívalo en puertos que dan a Internet o a usuarios no confiables con no cdp enable en la interfaz.
CDP COMANDOS Switch# show cdp neighbors ! Ver vecinos: nombre, interfaz, modelo Switch# show cdp neighbors detail ! Ver IPs, versión IOS, capacidades Switch# show cdp interface ! Ver interfaces donde CDP está activo ! Deshabilitar CDP globalmente: Switch(config)# no cdp run ! Deshabilitar CDP en un puerto específico: Switch(config-if)# no cdp enable
🌐 LLDP — Link Layer Discovery Protocol
LLDP (IEEE 802.1AB) es el estándar abierto equivalente a CDP. Funciona con dispositivos de cualquier fabricante (Cisco, Juniper, HP, etc.). Por defecto está deshabilitado en Cisco IOS — debes activarlo manualmente.
