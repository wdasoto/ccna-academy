Switches Cisco

Arquitectura · Tabla MAC · Forwarding · VLANs · SVI · Seguridad · Verificación

CCNA 200-301 Módulo 2 · Dispositivos Cisco 🔀 Capa 2 — OSI

🔀 ¿Qué es un Switch?

Un switch es un dispositivo de Capa 2 que interconecta dispositivos dentro de una misma red LAN. Toma decisiones de reenvío basadas en direcciones MAC, no en IPs.

🧠 Inteligencia

Aprende la ubicación de cada dispositivo consultando las MACs de origen de los frames recibidos → construye la tabla CAM/MAC.

⚡ Microsegmentación

Cada puerto es un dominio de colisión separado (full-duplex). Elimina colisiones CSMA/CD del hub.

📢 Broadcast

El switch inunda los frames broadcast/multicast y unicast desconocidos a todos los puertos excepto el de origen.

📊 Métodos de Forwarding

MétodoCuándo reenvíaLatenciaDetección errores

🎯 Para el examen CCNA

Los switches Cisco modernos usan Store-and-Forward por defecto. Es el único que garantiza que los frames corruptos no se propagan por la red.

📋 Tabla MAC — Cómo aprende el Switch

La tabla MAC (también llamada tabla CAM) es la base del switching. El switch la construye dinámicamente observando el campo MAC origen de cada frame recibido.

🔄 Proceso de aprendizaje paso a paso

  1. Llegada del frame: el switch lee la MAC origen y el puerto de entrada
  2. Aprende: registra [MAC origen → Puerto → VLAN] en la tabla MAC
  3. Busca destino: consulta la tabla con la MAC destino
  4. Si encuentra: reenvía solo al puerto correspondiente (unicast)
  5. Si NO encuentra: inunda a todos los puertos excepto el de origen (flooding)
  6. Aging: las entradas se eliminan si no hay actividad en 300 seg (default)

Diferencias clave Switch vs Router

  • Las interfaces del switch están activas por default (no requieren no shutdown)
  • El switch tiene 16 líneas VTY (0-15) vs 5 en el router (0-4)
  • El switch NO necesita ip routing para funcionar en Capa 2
  • Para acceso remoto el switch usa una SVI (Switch Virtual Interface)

🌐 SVI — Switch Virtual Interface

Una SVI es una interfaz virtual que le asigna una dirección IP al switch para gestión remota (SSH/Telnet). No procesa tráfico de datos — es solo para administración.

📌 La SVI por defecto es VLAN 1

Cisco recomienda cambiarla a otra VLAN por seguridad (la VLAN 1 no se puede eliminar pero sí se puede dejar sin IP).

Configurar SVI para gestión

! Crear VLAN de gestión
SW1(config)# vlan 99
SW1(config-vlan)# name GESTION
SW1(config-vlan)# exit
! Asignar IP a la SVI de gestión
SW1(config)# interface vlan 99
SW1(config-if)# ip address 192.168.99.2 255.255.255.0
SW1(config-if)# no shutdown
SW1(config-if)# exit
! Gateway por defecto para que el switch alcance otras redes
SW1(config)# ip default-gateway 192.168.99.1
! Asignar el puerto de uplink a VLAN 99
SW1(config)# interface FastEthernet0/24
SW1(config-if)# switchport mode access
SW1(config-if)# switchport access vlan 99

📋 Resumen — Switches Cisco para el CCNA

✅ Puntos críticos del examen

  • Switch = Capa 2, decisiones basadas en MAC, no IP
  • Tabla MAC: aprende MAC origen → almacena [MAC, Puerto, VLAN]
  • Frame desconocido → flooding a todos excepto el puerto origen
  • Store-and-Forward: verifica FCS antes de reenviar (default en Cisco)
  • Interfaces de switch: activas por defecto (distinto al router)
  • SVI VLAN 1 = interfaz de gestión por defecto
  • El switch necesita ip default-gateway para alcanzar otras redes
  • Aging de tabla MAC: 300 segundos por defecto
  • Switch tiene 16 líneas VTY (0-15), router tiene 5 (0-4)

⚠️ Errores comunes

  • Olvidar no shutdown en la SVI (sí la necesita, las interfaces físicas no)
  • No configurar ip default-gateway → switch no alcanza otras redes
  • Confundir duplex mismatch: medio duplex + full duplex = muchas colisiones
