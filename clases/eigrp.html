EIG RP

DUAL · Successor · Feasible Successor · K-values · Variance · Advanced Distance Vector

CCNA 200-301Módulo 4 · Routing⚡ DUAL Algorithm

⚡ ¿Qué es EIGRP?

EIGRP (Enhanced Interior Gateway Routing Protocol) es un protocolo de routing avanzado de distancia vector mejorado, propietario de Cisco, que usa el algoritmo DUAL para garantizar rutas libres de bucles con convergencia rápida.

CaracterísticaValor
TipoAdvanced Distance Vector · IGP · Classless
AlgoritmoDUAL (Diffusing Update Algorithm)
Métrica compuestaBandwidth + Delay (por defecto) · + Reliability, Load, MTU (k-values)
Distancia AdministrativaInternal = 90 · External = 170 · Summary = 5
ProtocoloIP protocolo 88
Multicast Hello224.0.0.10 (todos los routers EIGRP)
Soporta VLSMSí (classless, envía máscara)
ConvergenciaMuy rápida (usa rutas de backup — Feasible Successor)

📊 Ventajas de EIGRP

  • Convergencia rápida: tiene rutas backup (Feasible Successor) precomputadas
  • Eficiencia: solo envía actualizaciones cuando hay cambios (no periódicas como RIP)
  • Balanceo de carga desigual: puede balancear en rutas con diferente métrica (variance)
  • Sumarización automática: puede resumir rutas en límites de classful (desactivado en IOS moderno)

⚙️ Algoritmo DUAL — Rutas y Distancias

📌 Conceptos clave

  • FD (Feasible Distance): costo total desde el router local hasta la red destino
  • RD / AD (Reported/Advertised Distance): costo que el vecino reporta hasta el destino
  • Successor: la ruta con menor FD → es la ruta instalada en la tabla de routing
  • Feasible Successor (FS): ruta backup · su RD debe ser < FD del Successor actual
  • Feasibility Condition (FC): RD del candidato < FD del Successor = evita bucles

💻 Configuración EIGRP

Configuración básica EIGRP

! AS number debe ser el mismo en todos los routers del dominio
R1(config)# router eigrp 100
! Anunciar redes (acepta classful, pero wildcard es más preciso)
R1(config-router)# network 192.168.1.0
R1(config-router)# network 10.0.0.0 0.0.0.3
! Router ID (recomendado configurar manualmente)
R1(config-router)# eigrp router-id 1.1.1.1
! Deshabilitar auto-summary (IMPORTANTE en IOS < 15.x)
R1(config-router)# no auto-summary
! Interfaz pasiva: no envía Hellos (LANs de usuarios)
R1(config-router)# passive-interface GigabitEthernet0/0

K-values — Pesos de la métrica compuesta

! Métrica = [K1×BW + (K2×BW)/(256-Load) + K3×Delay] × [K5/(Reliability+K4)]
! Por defecto K1=1, K2=0, K3=1, K4=0, K5=0 → solo BW y Delay
! IMPORTANTE: K-values deben ser idénticos en todos los vecinos
R1# show ip protocols
Routing Protocol is "eigrp 100"
  Outgoing update filter list for all interfaces is not set
  EIGRP metric weight K1=1, K2=0, K3=1, K4=0, K5=0

Balanceo de carga desigual con variance

! variance X: balancea rutas cuya FD sea < X × FD_del_Successor
! Ejemplo: Successor FD=100, variance 2 → balancea rutas con FD<200
R1(config-router)# variance 2

🔗 Vecinos y Adyacencias EIGRP

📌 Requisitos para formar vecindad

  • AS number igual en ambos extremos
  • K-values idénticos
  • Autenticación si está configurada — misma contraseña
  • No requiere misma subnet (a diferencia de OSPF)
  • Hello/Hold timers pueden ser diferentes (EIGRP los tolera)

🧪 Laboratorio EIGRP completo

📋 Topología: R1─R2─R3 AS 100, cada uno con LAN y enlace punto a punto

Configuración en los 3 routers

! R1 — AS 100
R1(config)# router eigrp 100
R1(config-router)# eigrp router-id 1.1.1.1
R1(config-router)# network 192.168.1.0 0.0.0.255
R1(config-router)# network 10.0.0.0 0.0.0.3
R1(config-router)# no auto-summary
R1(config-router)# passive-interface GigabitEthernet0/0

! R2 — conecta con R1 y R3
R2(config)# router eigrp 100
R2(config-router)# eigrp router-id 2.2.2.2
R2(config-router)# network 10.0.0.0 0.0.0.3
R2(config-router)# network 10.0.0.4 0.0.0.3
R2(config-router)# no auto-summary

📋 Resumen EIGRP

✅ Puntos críticos

  • EIGRP AD internal = 90 · external = 170 · summary = 5
  • IP protocolo 88 · Multicast 224.0.0.10
  • Métrica default: Bandwidth + Delay (K1=1, K3=1)
  • Mismo AS number = obligatorio para formar vecindad
  • K-values deben ser idénticos en ambos vecinos
  • Successor = mejor ruta (en tabla de routing)
  • Feasible Successor = backup instantáneo (RD < FD del Successor)
  • no auto-summary = obligatorio en topologías discontiguas
  • variance = permite balanceo de carga desigual
