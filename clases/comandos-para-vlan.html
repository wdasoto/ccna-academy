Comandos para Virtual LAN (VLAN)

Domina la segmentación lógica de redes con Cisco IOS. Creación de VLANs, asignación de puertos, trunking 802.1Q, DTP, VTP, verificación avanzada y troubleshooting. Desde cero hasta configuraciones enterprise-grade para el examen CCNA 200-301.

📋 IOS 15.x / IOS XE 🎯 CCNA 200-301 Exam Topic 4.1, 4.2 ⏱ ~120 min lectura 🧪 Lab: 3 Switches + Router

Fundamentos de VLANs en Cisco IOS

Las VLANs (Virtual Local Area Networks) son la tecnología fundamental de segmentación lógica en switches Cisco. Permiten dividir un switch físico en múltiples redes de broadcast independientes, mejorando seguridad, rendimiento y gestión.

Concepto VLAN: Un Switch, Múltiples Redes Lógicas

¿Qué es una VLAN y por qué usarla?

🔒

Segmentación de Seguridad

Los dispositivos en VLANs diferentes no se comunican directamente en Capa 2. Requieren un router o switch L3 (Inter-VLAN Routing) para comunicarse.

📡

Control de Broadcast

Cada VLAN es un dominio de broadcast independiente. El tráfico broadcast de una VLAN no inunda las demás, mejorando rendimiento.

🏢

Flexibilidad Organizativa

Un usuario puede estar en cualquier VLAN independientemente de su ubicación física. El departamento de Ventas en el piso 3 y el piso 7 comparten la misma VLAN.

⚡

Reducción de Costos

Un solo switch físico puede reemplazar múltiples switches independientes, reduciendo hardware y administración.

🎯

Calidad de Servicio (QoS)

Políticas de QoS pueden aplicarse por VLAN, priorizando tráfico de voz (VLAN de voz) sobre datos.

🔧

Gestión Simplificada

Cambios de red lógica (mover usuarios entre VLANs) se hacen por software sin mover cables físicos.

⚠️ VLAN 1 — Consideraciones Críticas de Seguridad

La VLAN 1 es la VLAN por defecto en todos los switches Cisco y tiene características especiales:

  • No puede ser renombrada ni eliminada
  • Es la VLAN nativa por defecto en trunks (cambiar por seguridad)
  • El tráfico de control (CDP, VTP, DTP, PAgP) viaja por VLAN 1
  • Es el target principal de ataques VLAN Hopping
  • Mejor práctica: No asignar usuarios a VLAN 1, usar VLAN 99 o similar como native

Creación y Eliminación de VLANs

Existen dos métodos para crear VLANs en Cisco IOS: el modo de configuración global (más común) y el modo de base de datos VLAN (obsoleto, solo mencionado por compatibilidad).

Método 1: Configuración Global (Recomendado)

Creación de VLANs paso a paso
! === CREAR VLAN INDIVIDUAL ===
SW1# configure terminal
SW1(config)# vlan 10
SW1(config-vlan)# name VENTAS
SW1(config-vlan)# state active              ! Opcional: active (default) o suspend
SW1(config-vlan)# exit

! === CREAR MÚLTIPLES VLANs ===
SW1(config)# vlan 20
SW1(config-vlan)# name IT
SW1(config-vlan)# exit
SW1(config)# vlan 30
SW1(config-vlan)# name MANAGEMENT
SW1(config-vlan)# exit

! === CREAR RANGO DE VLANs (IOS moderno) ===
SW1(config)# vlan 40,50,60                  ! VLANs individuales
SW1(config)# vlan 70-80                    ! Rango consecutivo
SW1(config-vlan)# exit

! === VERIFICAR CREACIÓN ===
SW1# show vlan brief
SW1# show vlan id 10
SW1# show vlan name VENTAS

! === ELIMINAR VLAN ===
SW1(config)# no vlan 10                      ! Elimina VLAN 10
! ⚠️ ADVERTENCIA: Los puertos asignados a VLAN eliminada
!    vuelven a VLAN 1 (default). NO se pierden, pero quedan
!    en VLAN 1 inesperadamente.

! === ELIMINAR RANGO DE VLANs ===
SW1(config)# no vlan 70-80
SW1(config)# no vlan 40,50,60

Método 2: Base de Datos VLAN (Obsoleto)

Comando antiguo — No usar en producción
! === VLAN DATABASE MODE (OBSOLETO) ===
! Solo para switches antiguos o referencia histórica
! No disponible en IOS moderno (15.x, IOS-XE)

SW1# vlan database
SW1(vlan)# vlan 10 name VENTAS
SW1(vlan)# vlan 20 name IT
SW1(vlan)# apply                            ! Aplicar cambios
SW1(vlan)# exit                             ! Guardar y salir

! ⚠️ Este modo está DEPRECATED. Usar siempre configure terminal.
💡 Tip del Examen CCNA

Al crear una VLAN, el switch automáticamente crea una SVI (Switch Virtual Interface) asociada si tienes configurado ip routing (switch L3). Sin embargo, la SVI no aparece hasta que configuras interface vlan X explícitamente.

Configuración de VLANs Extendidas (1006-4094)

Requisitos especiales para VLANs extendidas
! === VLANs EXTENDIDAS ===
! Requieren VTP mode transparent o VTPv3
! No se almacenan en vlan.dat, solo en running-config

SW1(config)# vtp mode transparent           ! Desactivar VTP
SW1(config)# vlan 2000
SW1(config-vlan)# name DATACENTER-EXT
SW1(config-vlan)# exit

! Verificación:
SW1# show vlan id 2000
VLAN Name                             Status    Ports

!    no se pueden crear. Error: "VTP VLAN configuration not allowed"

Asignación de Puertos a VLANs

Los puertos de acceso (access ports) conectan dispositivos finales (PCs, impresoras, servidores) a una VLAN específica. Cada puerto de acceso pertenece a una única VLAN.

Configuración de Puerto de Acceso

Paso a paso: Asignar puerto a VLAN
! === CONFIGURACIÓN BÁSICA DE PUERTO DE ACCESO ===

SW1(config)# interface FastEthernet0/1
SW1(config-if)# description PC-VENTAS-01
SW1(config-if)# switchport mode access       ! Fijar modo access
SW1(config-if)# switchport access vlan 10    ! Asignar a VLAN 10
SW1(config-if)# no shutdown                  ! Activar interfaz

! === CONFIGURACIÓN MÚLTIPLE (RANGO DE PUERTOS) ===
SW1(config)# interface range FastEthernet0/1 - 12
SW1(config-if-range)# switchport mode access
SW1(config-if-range)# switchport access vlan 10
SW1(config-if-range)# no shutdown

! === RANGO DISCONTINUO ===
SW1(config)# interface range FastEthernet0/1 - 6, FastEthernet0/13 - 18
SW1(config-if-range)# switchport mode access
SW1(config-if-range)# switchport access vlan 20

! === MACROS DE INTERFAZ (Interface Range Macro) ===
SW1(config)# define interface-range VENTAS Fa0/1-12
SW1(config)# define interface-range IT Fa0/13-24

SW1(config)# interface range macro VENTAS
SW1(config-if-range)# switchport access vlan 10

! === VERIFICAR ASIGNACIÓN ===
SW1# show vlan brief
SW1# show interfaces FastEthernet0/1 switchport
SW1# show mac address-table interface FastEthernet0/1

⚠️ Error Común del Examen: VLAN Inexistente

Si intentas asignar un puerto a una VLAN que no existe, el switch la crea automáticamente en algunas versiones de IOS, pero en otras muestra error. Mejor práctica: Crear la VLAN explícitamente antes de asignar puertos.

SW1(config-if)# switchport access vlan 99
% Access VLAN does not exist. Creating vlan 99

Restablecer Puerto a Valores Default

Comandos para resetear configuración de puerto
! === RESETEAR PUERTO A DEFAULT ===
SW1(config)# interface FastEthernet0/1
SW1(config-if)# no switchport access vlan       ! Quitar VLAN asignada
SW1(config-if)# no switchport mode             ! Volver a dynamic auto
SW1(config-if)# no description                 ! Quitar descripción

! === O USAR DEFAULT INTERFACE (MÁS LIMPIO) ===

Interface FastEthernet0/1 reset to default configuration

! Verificar:
SW1# show interfaces FastEthernet0/1 switchport
Access Mode VLAN: 1 (default)

Trunking 802.1Q: Comandos y Configuración

Los puertos trunk transportan tráfico de múltiples VLANs entre switches. Cisco utiliza el estándar IEEE 802.1Q para etiquetar (tag) las tramas Ethernet con el ID de VLAN.

Encapsulación 802.1Q en Trunk

Configuración de Puerto Trunk

Comandos esenciales de trunking
! === CONFIGURACIÓN BÁSICA DE TRUNK ===

SW1(config)# interface GigabitEthernet0/1
SW1(config-if)# description TRUNK-TO-SW2
SW1(config-if)# switchport mode trunk        ! Forzar modo trunk
SW1(config-if)# switchport trunk encapsulation dot1q   ! 802.1Q (estándar)
! Nota: En switches nuevos, dot1q es el único método disponible
! ISL (Inter-Switch Link) está obsoleto desde IOS 12.2(25)

! === CONFIGURAR VLAN NATIVA (SEGURIDAD) ===
SW1(config-if)# switchport trunk native vlan 99
! ⚠️ La VLAN nativa debe coincidir EN AMBOS LADOS del trunk
!    Mismatch de native VLAN = vulnerabilidad de seguridad

! === LIMITAR VLANs EN EL TRUNK (SEGURIDAD) ===
SW1(config-if)# switchport trunk allowed vlan 10,20,30,99
! Solo estas VLANs pueden pasar por el trunk
! VLANs no listadas están bloqueadas

! === AGREGAR VLAN A TRUNK EXISTENTE ===
SW1(config-if)# switchport trunk allowed vlan add 40
! Usar "add" para no sobrescribir la lista existente

! === REMOVER VLAN DE TRUNK ===
SW1(config-if)# switchport trunk allowed vlan remove 20

! === EXCEPTO (Método alternativo) ===
SW1(config-if)# switchport trunk allowed vlan except 1,1002-1005
! Permite todas EXCEPTO las listadas

! === VERIFICAR CONFIGURACIÓN ===
SW1# show interfaces GigabitEthernet0/1 trunk
SW1# show interfaces GigabitEthernet0/1 switchport
SW1# show interfaces trunk

Configuración de Trunk en Router (Router-on-a-Stick)

Inter-VLAN Routing básico
! === ROUTER-ON-A-STICK ===
! Un solo interfaz física del router maneja múltiples VLANs
! mediante subinterfaces lógicas

R1(config)# interface GigabitEthernet0/0
R1(config-if)# no shutdown                   ! Activar interfaz física

! Subinterfaz para VLAN 10:
R1(config)# interface GigabitEthernet0/0.10
R1(config-subif)# description VLAN-10-VENTAS
R1(config-subif)# encapsulation dot1Q 10      ! Tag 802.1Q VLAN 10
R1(config-subif)# ip address 192.168.10.1 255.255.255.0
R1(config-subif)# exit

! Subinterfaz para VLAN 20:
R1(config)# interface GigabitEthernet0/0.20
R1(config-subif)# description VLAN-20-IT
R1(config-subif)# encapsulation dot1Q 20
R1(config-subif)# ip address 192.168.20.1 255.255.255.0
R1(config-subif)# exit

! Subinterfaz para VLAN 99 (Native):
R1(config)# interface GigabitEthernet0/0.99
R1(config-subif)# description NATIVE-VLAN
R1(config-subif)# encapsulation dot1Q 99 native  ! Keyword "native"
R1(config-subif)# ip address 192.168.99.1 255.255.255.0

! Verificación:
R1# show ip interface brief
R1# show vlans
⚠️ Trampa del Examen: Native VLAN Mismatch

Si un switch tiene switchport trunk native vlan 99 y el otro lado tiene el default (VLAN 1) o un valor diferente:

  • El trunk se forma (estado up/up)
  • Pero el tráfico untagged se "filtra" a la VLAN incorrecta
  • Esto crea una vulnerabilidad de VLAN hopping
  • El switch genera mensajes de error: %CDP-4-NATIVE_VLAN_MISMATCH

Solución: Configurar explícitamente la misma native VLAN en ambos lados, o deshabilitar CDP si no es necesario (no recomendado).

DTP (Dynamic Trunking Protocol) y VTP (VLAN Trunking Protocol)

DTP negocia automáticamente el estado trunk entre switches Cisco. VTP distribuye información de VLANs entre switches en un dominio. Ambos son propietarios de Cisco y requieren cuidado en configuración.

Modos DTP y Comportamiento de Negociación

Modo LocalModo Vecino AccessModo Vecino TrunkModo Vecino Dynamic AutoModo Vecino Dynamic Desirable
accessAccessAccessAccessAccess
trunk (nonegotiate)AccessTrunkTrunkTrunk
dynamic autoAccessTrunkAccessTrunk
dynamic desirableAccessTrunkTrunkTrunk
Configuración DTP y deshabilitación
! === CONFIGURAR MODO DTP ===

! Forzar trunk sin negociación (MÁS SEGURO):
SW1(config)# interface GigabitEthernet0/1
SW1(config-if)# switchport mode trunk
SW1(config-if)# switchport nonegotiate       ! Desactivar DTP
! El puerto NO envía tramas DTP, no negocia

! Modo dynamic auto (default en algunos switches):
SW1(config-if)# switchport mode dynamic auto
! Espera que el vecino inicie trunk. Si vecino es trunk/desirable → trunk
! Si vecino es auto → access

! Modo dynamic desirable:
SW1(config-if)# switchport mode dynamic desirable
! Activamente intenta formar trunk. Más proactivo que auto.

! === VERIFICAR ESTADO DTP ===
SW1# show dtp interface GigabitEthernet0/1

    TOS/TAS/TNS:                  TRUNK/ACCESS/TRUNK

    Neighbor port:                  GigabitEthernet0/1

! === DESHABILITAR DTP GLOBALMENTE ===
! No existe comando global, se hace por interfaz:
SW1(config)# interface range GigabitEthernet0/1-2
SW1(config-if-range)# switchport mode trunk
SW1(config-if-range)# switchport nonegotiate
⚠️ Advertencia de Seguridad: Deshabilitar DTP

Dejar DTP activado permite que un atacante conecte un switch falso y negocie trunk, obteniendo acceso a todas las VLANs. Mejor práctica de seguridad:

  1. Configurar puertos de acceso explícitamente con switchport mode access
  2. Configurar puertos trunk explícitamente con switchport mode trunk
  3. Deshabilitar DTP con switchport nonegotiate en trunks
  4. Configurar puertos no utilizados en modo access VLAN dummy y shutdown

VTP (VLAN Trunking Protocol)

Configuración VTP para gestión centralizada
! === VTP MODES ===
! Server:   Puede crear, modificar, eliminar VLANs. Anuncia a clientes.
! Client:   No puede modificar VLANs. Sincroniza con server.
! Transparent: No participa en VTP pero reenvía anuncios. Mantiene VLANs locales.
! Off:      Desactivado completamente (IOS moderno).

! === CONFIGURAR VTP SERVER ===
SW1(config)# vtp domain CCNA-LAB
SW1(config)# vtp mode server
SW1(config)# vtp version 2                    ! o version 3 para IPv6/Extended
SW1(config)# vtp password SecureVTP123       ! Opcional: autenticación MD5

! === CONFIGURAR VTP CLIENT ===
SW2(config)# vtp domain CCNA-LAB
SW2(config)# vtp mode client
SW2(config)# vtp password SecureVTP123

! === CONFIGURAR VTP TRANSPARENT ===
SW3(config)# vtp domain CCNA-LAB
SW3(config)# vtp mode transparent
! Transparent reenvía anuncios VTP pero no sincroniza su base de datos

! === VTP PRUNING (Optimización) ===
SW1(config)# vtp pruning                      ! Habilitar pruning global
! Evita enviar tráfico broadcast por trunks donde no hay puertos
! de esa VLAN. Reduce tráfico innecesario.

SW1(config)# vtp version 3

! VTPv3 soporta VLANs extendidas, private VLANs, y es más seguro
⚠️ VTP VULNERABILIDAD: El "VTP Bomb"
1 Un atacante conecta un switch con VTP server y configuration revision más alto que el server legítimo.
2 Los clientes y servers con menor revision sobrescriben su base de datos VLAN con la del atacante.
3 Resultado: Todas las VLANs pueden ser eliminadas o reemplazadas, causando denegación de servicio total.
4 Protección: Usar VTP password, VTP transparent en switches de acceso, o desactivar VTP completamente.

Troubleshooting de VLANs — Problemas Comunes y Soluciones

El troubleshooting de VLANs requiere un enfoque metodológico: verificación física, configuración lógica, y análisis de protocolos. A continuación se presentan los problemas más frecuentes y sus soluciones paso a paso.

1. Native VLAN Mismatch — El Problema Más Crítico

⚠️ SEVERIDAD: ALTA — Seguridad y Conectividad

Un Native VLAN Mismatch ocurre cuando los puertos trunk en ambos extremos de un enlace tienen configuraciones de VLAN nativa diferentes. Esto causa que el tráfico untagged sea interpretado en VLANs distintas, creando fugas de tráfico, problemas de STP y potencialmente habilitando ataques VLAN hopping.

Síntomas del Problema

  • Mensaje de error en consola: %CDP-4-NATIVE_VLAN_MISMATCH
  • Tráfico de VLAN nativa no llega al destino esperado
  • Problemas de conectividad intermitente entre switches
  • BPDUs de STP procesados incorrectamente

Comandos de Diagnóstico

Identificar el mismatch
! === Detectar Native VLAN Mismatch ===
SW1# show cdp neighbors detail
SW1# show interfaces trunk
SW1# show interfaces GigabitEthernet0/1 switchport | include Native

! === Salida típica del error ===
%CDP-4-NATIVE_VLAN_MISMATCH: Native VLAN mismatch discovered on Gi0/1 (99), with SW2 Gi0/1 (1)

Solución Paso a Paso

Corregir Native VLAN Mismatch
! === Verificar configuración actual en AMBOS switches ===
SW1# show interfaces GigabitEthernet0/1 switchport | include Native
SW2# show interfaces GigabitEthernet0/1 switchport | include Native

! === Configurar la misma VLAN nativa en ambos lados ===
SW1(config)# interface GigabitEthernet0/1
SW1(config-if)# switchport trunk native vlan 99
SW2(config)# interface GigabitEthernet0/1
SW2(config-if)# switchport trunk native vlan 99

! === Verificación post-cambio ===
SW1# show interfaces trunk
Port        Mode             Encapsulation  Status        Native vlan
Gi0/1       on               802.1q         trunking      99

2. VLAN Inexistente — Asignación Fallida

Cuando se asigna un puerto a una VLAN que no existe en el switch, el comportamiento varía según la versión de IOS:

  • IOS moderno: El switch crea la VLAN automáticamente (pero sin SVI configurada)
  • IOS antiguo: Genera error: % Access VLAN does not exist

Comandos de Diagnóstico y Solución

Verificar existencia de VLAN
SW1# show vlan brief | include 50
! Si no aparece, la VLAN no existe

! === Solución: Crear la VLAN explícitamente ===
SW1(config)# vlan 50
SW1(config-vlan)# name SALES_TEMP
SW1(config-vlan)# exit

! === O verificar VLAN correcta ===
SW1(config)# interface FastEthernet0/5
SW1(config-if)# switchport access vlan 50

3. Trunk No Forma — Problemas de Negociación

Cuando un trunk no forma correctamente, los hosts en diferentes VLANs no pueden comunicarse:

Causas Comunes

  • Modo Dynamic Auto en ambos lados: Ningún lado inicia negociación DTP
  • Modo Access vs Trunk: Un lado en access, otro en trunk
  • DTP Deshabilitado (nonegotiate) vs modo dynamic: Incompatibilidad de negociación
  • Encapsulación ISL vs 802.1Q: Métodos incompatibles (ISL es obsoleto)
Forzar Trunk Estático
! === Configuración recomendada para evitar problemas ===
SW1(config)# interface GigabitEthernet0/1
SW1(config-if)# switchport mode trunk
SW1(config-if)# switchport nonegotiate      ! Deshabilitar DTP
SW2(config)# interface GigabitEthernet0/1
SW2(config-if)# switchport mode trunk
SW2(config-if)# switchport nonegotiate

! === Verificar formación del trunk ===
SW1# show interfaces trunk
Port        Mode             Encapsulation  Status        Native vlan
Gi0/1       on               802.1q         trunking      99

4. VTP Synchronization Issues

Problemas de sincronización de VLANs entre switches cuando VTP está mal configurado:

Síntomas

  • VLANs creadas en un switch no aparecen en otro
  • Mensaje: Configuration revision mismatch
  • VLANs "desaparecen" repentinamente

Solución

Verificar dominio VTP
SW1# show vtp status
! Verificar: VTP Domain Name, Operating Mode, Configuration Revision

! === Soluciones comunes ===
! 1. Asegurar mismo VTP Domain en todos los switches:
SW1(config)# vtp domain CCNA-LAB
SW2(config)# vtp domain CCNA-LAB

! 2. Verificar modo VTP compatible:
!    - Server puede crear/modificar VLANs
!    - Client sincroniza con Server
!    - Transparent: No sincroniza, mantiene VLANs locales

! 3. Si hay inconsistencia de revision:
SW1# show vtp counters
! Buscar "Configuration revision errors"

5. Problemas de Asignación de Puertos

Port Security Violation

Cuando port security está habilitado y se violan las restricciones (MAC address no permitida):

SW1# show port-security interface FastEthernet0/1
Port Security : Enabled
Violation Mode : Shutdown
MAC Address    : 0000.0000.0000 (None)
Security Violation Count : 1

Solución

  • Verificar MAC address permitida: switchport port-security mac-address XXXX.XXXX.XXXX
  • Cambiar modo de violación: switchport port-security violation restrict (en lugar de shutdown)
  • Clear de violación: clear port-security sticky interface FastEthernet0/1

6. Problemas de Inter-VLAN Routing

Después de configurar VLANs y trunks, el routing entre VLANs puede fallar:

ip address 192.168.10.1 255.255.255.0

Quiz de Evaluación: VLANs

15 preguntas para evaluar tu conocimiento sobre VLANs, trunking y VTP.

Preguntas de Evaluación

1. ¿Qué comando se usa para crear una VLAN 50 llamada VENTAS?
A. vlan 50 luego name VENTAS
B. create vlan 50 VENTAS
C. vlan database luego vlan 50 name VENTAS
D. switchport vlan 50 VENTAS
✓ Correcto: El modo global vlan 50 entra al modo de configuración de VLAN donde se puede asignar el nombre con name VENTAS. La opción C usa el modo obsoleto de VLAN database. La D es incorrecta sintaxis.
2. ¿Qué modo de VTP permite crear VLANs pero no sincroniza con otros switches?
A. Server
B. Transparent
C. Client
D. Off
✓ Correcto: El modo transparent permite crear VLANs locales pero no participa en el dominio VTP ni sincroniza con otros switches (solo reenvía anuncios VTP).
3. ¿Cuál es el rango de VLANs extendidas en Cisco IOS?
A. 1-1005
B. 1006-4094
C. 1-4094
D. 1002-1005
✓ Correcto: Las VLANs extendidas son 1006-4094. Requieren VTP transparent o VTPv3.
4. ¿Qué comando verifica si un puerto está configurado como trunk?
A. show vlan brief
B. show interfaces trunk
C. show trunk ports
D. show switchport trunk
✓ Correcto: show interfaces trunk muestra todos los puertos trunk, sus VLANs permitidas y la VLAN nativa.
5. ¿Cuál es la mejor práctica de seguridad para la VLAN nativa?
A. Cambiarla a una VLAN no utilizada (ej: 999) y no usar VLAN 1
B. Dejarla en VLAN 1 (default)
C. Eliminar la VLAN nativa
D. Usar VLAN 100 como nativa
✓ Correcto: La mejor práctica es cambiar la VLAN nativa de 1 a un número no utilizado (ej: 999) y evitar usar VLAN 1 para datos de usuarios.
6. ¿Qué significa el error %CDP-4-NATIVE_VLAN_MISMATCH?
A. Las VLANs no coinciden en ambos lados
B. La VLAN nativa no coincide en los puertos trunk conectados
C. El trunk no está formado
D. VTP no está sincronizando
✓ Correcto: Este error específico indica que los puertos trunk conectados tienen configuraciones de VLAN nativa diferentes.
7. ¿Qué comando configura un puerto para que negocie trunk activamente?
A. switchport mode dynamic auto
B. switchport mode dynamic desirable
C. switchport mode trunk
D. switchport negotiate trunk
✓ Correcto: dynamic desirable activamente intenta formar un trunk enviando tramas DTP, a diferencia de dynamic auto que solo espera.
8. ¿En qué modo VTP un switch puede crear y eliminar VLANs?
A. Server
B. Client
C. Transparent
D. Off
✓ Correcto: Solo el modo server puede crear, modificar y eliminar VLANs en un dominio VTP.
9. ¿Qué comando muestra la tabla MAC filtrada por una VLAN específica?
A. show mac vlan 10
B. show mac-address vlan 10
C. show mac address-table vlan 10
D. show vlan mac 10
✓ Correcto: El comando correcto es show mac address-table vlan 10.
10. ¿Cuál es el propósito principal del switchport nonegotiate?
A. Habilitar la negociación DTP
B. Deshabilitar DTP para seguridad
C. Forzar el modo trunk
D. Configurar VLAN nativa
✓ Correcto: switchport nonegotiate deshabilita el protocolo DTP para prevenir que un atacante negocie un trunk falso.
11. ¿Qué sucede cuando se elimina una VLAN que tiene puertos asignados?
A. Los puertos se apagan automáticamente
B. Los puertos vuelven a VLAN 1 (default)
C. Los puertos mantienen la VLAN eliminada
D. Los puertos se convierten en trunk
✓ Correcto: Cuando se elimina una VLAN, los puertos asignados a ella vuelven automáticamente a VLAN 1 (default). Esto puede causar problemas de seguridad si VLAN 1 tiene acceso no autorizado.
12. ¿Qué comando configura una VLAN de voz en un puerto de acceso?
A. switchport voice 100
B. switchport voice vlan 100
C. voice vlan 100
D. switchport vlan voice 100
✓ Correcto: El comando correcto es switchport voice vlan 100. Esto permite que un puerto de acceso maneje tanto la VLAN de datos (PC) como la VLAN de voz (IP Phone) simultáneamente.
13. ¿Dónde se almacenan las VLANs creadas en un switch Cisco?
A. En startup-config únicamente
B. En flash:vlan.dat separado de la configuración
C. En NVRAM junto con startup-config
D. En RAM únicamente
✓ Correcto: Las VLANs se almacenan en el archivo flash:vlan.dat, separado de la configuración de inicio. Por eso erase startup-config NO elimina las VLANs. Para resetearlas se debe usar delete flash:vlan.dat.
14. ¿Qué comando agrega una VLAN a un trunk existente sin sobrescribir las VLANs permitidas?
A. switchport trunk allowed vlan 40
B. switchport trunk allowed vlan add 40
C. switchport trunk add vlan 40
D. switchport trunk vlan add 40
✓ Correcto: La palabra clave add agrega la VLAN a la lista existente sin sobrescribirla. Sin add, el comando reemplazaría toda la lista de VLANs permitidas.
15. ¿Qué protocolo de trunking es el estándar IEEE y el único soportado en IOS moderno?
A. ISL (Inter-Switch Link)
B. 802.1Q (dot1q)
C. DTP (Dynamic Trunking Protocol)
D. VTP (VLAN Trunking Protocol)
✓ Correcto: 802.1Q (dot1q) es el estándar IEEE para trunking. ISL era propietario de Cisco pero está obsoleto desde IOS 12.2(25). DTP es para negociación de trunk, no encapsulación. VTP es para distribución de información VLAN.
