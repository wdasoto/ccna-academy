CDP / LLDP

Cisco Discovery Protocol · LLDP 802.1AB · Vecinos · Topología · Seguridad

CCNA 200-301Módulo 3 · Switching🔍 Descubrimiento

🔍 Protocolos de Descubrimiento de Vecinos

CDP y LLDP son protocolos de Capa 2 que permiten a los dispositivos descubrir automáticamente sus vecinos directamente conectados sin necesidad de configurar IPs o routing.

🔵 CDP (Cisco Discovery Protocol)

Propietario de Cisco. Funciona en cualquier dispositivo Cisco: routers, switches, IP phones, APs. Activo por defecto.

🟢 LLDP (Link Layer Discovery Protocol)

Estándar abierto IEEE 802.1AB. Funciona entre dispositivos de diferentes fabricantes. Inactivo por defecto en Cisco.

📌 Información compartida

  • Device ID (hostname del vecino)
  • IP address del vecino
  • Plataforma / modelo del dispositivo
  • Versión del IOS/firmware
  • Puerto local y puerto del vecino
  • Capacidades (router, switch, phone)
  • VLAN nativa (CDP) | VLAN ID (LLDP)

⚠️ Riesgo de seguridad

CDP/LLDP revelan información sensible (modelo, versión IOS, IPs) a cualquier dispositivo conectado. Deshabilitar en puertos hacia usuarios o Internet.

⚙️ Configuración CDP y LLDP

Habilitar / Deshabilitar CDP

! CDP está activo por defecto — desactivar globalmente
R1(config)# no cdp run
! Desactivar en una interfaz específica (más selectivo)
R1(config)# interface GigabitEthernet0/0
R1(config-if)# no cdp enable
! Reactivar CDP
R1(config)# cdp run
! Cambiar timers
R1(config)# cdp timer 30
R1(config)# cdp holdtime 90

Habilitar LLDP (inactivo por defecto)

! Habilitar LLDP globalmente
R1(config)# lldp run
! Control granular por interfaz
R1(config)# interface GigabitEthernet0/0
R1(config-if)# lldp transmit
R1(config-if)# lldp receive
! Solo recibir (no revelar info)
R1(config-if)# no lldp transmit
R1(config-if)# lldp receive

🧪 Laboratorio CDP / LLDP

Escenario: R1─SW1─SW2 — Descubrir topología sin diagrama

! Desde R1 — ¿Qué hay conectado directamente?
R1# show cdp neighbors
SW1     Gi0/0     156    S    WS-C2960    Fa0/24
! Desde SW1 — ¿Qué hay más allá?
SW1# show cdp neighbors detail
Device ID: R1
  IP: 192.168.1.1, Port: Gi0/0 → Fa0/24
Device ID: SW2
  IP: 192.168.2.1, Port: Gi0/1 → Fa0/24
! Ahora sabemos: R1─Fa0/24─SW1─Gi0/1─SW2
! Esto es "CDP hopping" — mapear la red sin acceso físico
! Habilitar LLDP para interoperabilidad con equipos no-Cisco
R1(config)# lldp run
SW1(config)# lldp run
R1# show lldp neighbors

📋 Resumen CDP / LLDP

✅ Puntos críticos del examen

  • CDP = Cisco propietario, activo por defecto, Capa 2
  • LLDP = IEEE 802.1AB, abierto, inactivo por defecto en Cisco
  • CDP Hello: 60 seg | Holdtime: 180 seg
  • LLDP Hello: 30 seg | Holdtime: 120 seg
  • CDP se deshabilita con no cdp run (global) o no cdp enable (interfaz)
  • LLDP se habilita con lldp run y se controla por interfaz (transmit/receive)
  • Ambos revelan info sensible → deshabilitar en puertos hacia usuarios/Internet
  • show cdp neighbors detail = IP, plataforma, IOS, puerto del vecino
