Routing Estático

Tabla de Routing · Rutas Estáticas · Ruta Default · Rutas Flotantes · Troubleshooting

📋 ¿Qué es la Tabla de Routing?

La tabla de routing (RIB — Routing Information Base) es la base de datos principal de un router. Contiene todas las rutas conocidas hacia redes destino. Cuando llega un paquete, el router consulta esta tabla y elige la mejor ruta según la red destino, la distancia administrativa y la métrica.
✅
El comando show ip route muestra la tabla de routing completa. Es el comando más importante para verificar la conectividad en un router Cisco.
🎯
CCNA Examen: Debes saber leer e interpretar la salida de show ip route. Aparece en casi todos los exámenes de simulación.

🔤 Códigos de la Tabla de Routing

Cada ruta en la tabla tiene un código que indica su origen — cómo el router aprendió esa ruta:

CódigoSignificadoDistancia AdministrativaDescripción
CConnected0Red directamente conectada a una interfaz del router
LLocal0Dirección IP específica de la interfaz del router (/32)
SStatic1Ruta configurada manualmente por el administrador
S*Static Default1Ruta estática por defecto (gateway of last resort)
RRIP120Aprendida por el protocolo RIP
OOSPF110Aprendida por el protocolo OSPF
DEIGRP90Aprendida por el protocolo EIGRP
BBGP20 (eBGP) / 200 (iBGP)Aprendida por el protocolo BGP
⚠️
Distancia Administrativa (AD): Cuando un router conoce la misma red por varios protocolos, elige la ruta con menor AD. Connected (0) siempre gana sobre todo.

🗺️ ¿Qué es una Ruta Estática?

Una ruta estática es una ruta configurada manualmente por el administrador en la tabla de routing. No cambia automáticamente si la topología cambia — el administrador debe actualizarla. Son ideales para redes pequeñas, rutas de respaldo o conexiones a ISP.
VentajasDesventajas
✅ No consume CPU ni ancho de banda❌ No se adapta a cambios de topología
✅ Más seguro — control total del administrador❌ No escala en redes grandes
✅ AD = 1 (muy confiable)❌ Requiere configuración manual en cada router
✅ Fácil de troubleshoot❌ Propenso a errores humanos

📝 Sintaxis del Comando ip route

ip route [red-destino] [máscara] {next-hop-IP | interfaz-salida} [distancia-administrativa]
ParámetroDescripciónEjemplo
red-destinoDirección de red a la que quieres llegar192.168.2.0
máscaraMáscara de subred de la red destino255.255.255.0
next-hopIP del siguiente router (recomendado)10.0.0.1
interfazInterfaz de salida (alternativo al next-hop)GigabitEthernet0/0
ADDistancia administrativa opcional (default: 1)5, 10, 200...
✅
Recomendación: Usar siempre next-hop IP en lugar de interfaz de salida. Con interfaz de salida en redes multi-acceso (Ethernet), el router debe resolver ARP por cada destino, lo que es ineficiente.

🏗️ Tipos de Rutas Estáticas

TipoDescripciónCuándo usar
Estándar Ruta hacia una red específica con next-hop o interfaz Redes remotas conocidas y estables
Default (0.0.0.0/0) Captura todo el tráfico sin ruta específica Conexión a Internet / ISP
Flotante Ruta de respaldo con AD mayor que la ruta principal Redundancia y failover
Host (/32) Ruta hacia un host específico con máscara /32 Acceso a servidores o dispositivos específicos
Summarizada Una ruta resume varias redes contiguas Reducir entradas en la tabla de routing

🔗 Ruta Estática Flotante

Una ruta flotante es una ruta estática con una distancia administrativa mayor que la ruta principal. Permanece inactiva en la tabla mientras la ruta principal esté activa. Si la principal falla, la flotante aparece automáticamente.
RUTA FLOTANTE ! Ruta principal via OSPF (AD=110) Router(config)# router ospf 1   ! Ruta flotante — solo activa si OSPF falla (AD=200 > 110) Router(config)# ip route 192.168.2.0 255.255.255.0 10.0.0.2 200
🎯
CCNA Examen: La ruta flotante con AD=200 NO aparece en show ip route mientras la ruta OSPF (AD=110) esté activa. Solo aparece cuando la ruta con menor AD desaparece.

🌐 Ruta por Defecto — Gateway of Last Resort

La ruta por defecto (default route) es la ruta 0.0.0.0/0. Es la ruta de "último recurso" — si el router no encuentra ninguna ruta más específica para un destino, usa esta ruta. Es esencial para la conexión a Internet o a un ISP.
✅
En la tabla de routing, cuando está configurada una ruta default verás: Gateway of last resort is X.X.X.X to network 0.0.0.0
🎯
CCNA Examen: La ruta 0.0.0.0/0 usa el principio de longest prefix match — siempre pierde frente a cualquier ruta más específica. Es la ruta con el prefijo más corto posible.

📐 Longest Prefix Match

Cuando un router tiene múltiples rutas que coinciden con un destino, siempre elige la ruta con el prefijo más largo (más específico). Una /24 gana a una /16, que gana a una /8, que gana a la /0 (default).
EJEMPLO ! Tabla de routing tiene estas rutas: S 10.0.0.0/8 via 10.1.1.1 ! /8 — menos específica O 10.10.0.0/16 via 10.1.1.2 ! /16 — más específica S 10.10.10.0/24 via 10.1.1.3 ! /24 — aún más específica S* 0.0.0.0/0 via 10.1.1.4 ! default — último recurso   ! Para destino 10.10.10.5 → usa 10.10.10.0/24 (via 10.1.1.3) ✅ ! Para destino 10.10.20.5 → usa 10.10.0.0/16 (via 10.1.1.2) ! Para destino 10.20.0.5 → usa 10.0.0.0/8 (via 10.1.1.1) ! Para destino 8.8.8.8 → usa 0.0.0.0/0 (via 10.1.1.4)

🏗️ Topología de Laboratorio

Laboratorio con 3 routers (R1, R2, R3) y 3 redes LAN. El objetivo es que todas las redes se puedan comunicar usando solo rutas estáticas.
TOPOLOGÍA PC1 (192.168.1.10) | [R1] Gi0/0: 192.168.1.1/24 ←LAN1 Gi0/1: 10.0.12.1/30 ←WAN R1-R2 | [R2] Gi0/0: 10.0.12.2/30 ←WAN R1-R2 Gi0/1: 10.0.23.1/30 ←WAN R2-R3 Gi0/2: 192.168.2.1/24 ←LAN2 | [R3] Gi0/0: 10.0.23.2/30 ←WAN R2-R3 Gi0/1: 192.168.3.1/24 ←LAN3 PC3 (192.168.3.10)

⌨️ Configuración R1

R1 CONFIG R1(config)# ! Ruta hacia LAN2 (R2) R1(config)# ip route 192.168.2.0 255.255.255.0 10.0.12.2   R1(config)# ! Ruta hacia LAN3 (R3) — via R2 R1(config)# ip route 192.168.3.0 255.255.255.0 10.0.12.2   R1(config)# ! Ruta hacia WAN R2-R3 R1(config)# ip route 10.0.23.0 255.255.255.252 10.0.12.2   R1(config)# ! Verificar R1# show ip route

⌨️ Configuración R2

R2 CONFIG R2(config)# ! Ruta hacia LAN1 (R1) R2(config)# ip route 192.168.1.0 255.255.255.0 10.0.12.1   R2(config)# ! Ruta hacia LAN3 (R3) R2(config)# ip route 192.168.3.0 255.255.255.0 10.0.23.2   ! R2 ya conoce directamente: 10.0.12.0/30 y 10.0.23.0/30

⌨️ Configuración R3

R3 CONFIG R3(config)# ! Ruta hacia LAN1 (R1) — via R2 R3(config)# ip route 192.168.1.0 255.255.255.0 10.0.23.1   R3(config)# ! Ruta hacia LAN2 (R2) R3(config)# ip route 192.168.2.0 255.255.255.0 10.0.23.1   R3(config)# ! Ruta hacia WAN R1-R2 R3(config)# ip route 10.0.12.0 255.255.255.252 10.0.23.1

🌐 Configurar Ruta Default

DEFAULT ROUTE ! Método 1: ip route con next-hop Router(config)# ip route 0.0.0.0 0.0.0.0 203.0.113.1   ! Método 2: ip route con interfaz de salida Router(config)# ip route 0.0.0.0 0.0.0.0 GigabitEthernet0/0   ! Distribuir ruta default a vecinos OSPF Router(config-router)# default-information originate   ! Verificar Router# show ip route 0.0.0.0

🗑️ Eliminar una Ruta Estática

ELIMINAR RUTA ! Agregar "no" al principio del comando ip route Router(config)# no ip route 192.168.2.0 255.255.255.0 10.0.12.2   ! Eliminar la ruta default Router(config)# no ip route 0.0.0.0 0.0.0.0 203.0.113.1
⚠️
Al eliminar una ruta, el router descarta inmediatamente los paquetes hacia esa red con un mensaje ICMP Unreachable hasta que se configure una nueva ruta.

🚨 Errores Comunes en Routing Estático

❌
Error 1: Routing asimétrico — Configurar la ruta de ida pero olvidar la de vuelta.
Si R1 puede enviar a R3 pero R3 no tiene ruta hacia R1, el ping falla aunque el paquete llegue.
❌
Error 2: Next-hop incorrecto — Poner la IP de la interfaz propia en lugar del vecino.
ip route 192.168.2.0 255.255.255.0 10.0.12.1 ← (IP propia, no del vecino)
❌
Error 3: Máscara incorrecta — Poner la máscara del host en vez de la de la red.
ip route 192.168.2.0 255.255.255.255 10.0.12.2 ← (/32 en vez de /24)
❌
Error 4: Interfaz down — La ruta aparece en la tabla pero la interfaz está apagada.
Verificar con show ip interface brief que el estado sea up/up.
✅
Método sistemático: 1) Verificar interfaces up/up → 2) Verificar tabla de routing → 3) Ping con source → 4) Traceroute → 5) Debug si sigue fallando.
