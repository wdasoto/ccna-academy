🗺️ ¿Qué es el Routing?

El routing es el proceso mediante el cual un router determina la mejor ruta para reenviar un paquete hacia su red destino. Usa la tabla de routing (RIB) para tomar cada decisión.

📊 Tabla de Routing

Lista de redes conocidas con su distancia administrativa, métrica y next-hop. El router busca la coincidencia más específica (longest match).

🔢 Distancia Administrativa (AD)

Mide la confiabilidad de la fuente de la ruta. Menor AD = más confiable. Se usa para elegir entre rutas de distintas fuentes.

📏 Métrica

Mide el costo del camino. Se usa para elegir entre múltiples rutas del mismo protocolo. Depende del protocolo: saltos (RIP), costo (OSPF), compuesto (EIGRP).

🗺️ Tipos de Rutas Estáticas

TipoSintaxisUso
Next-hop (IP)ip route RED MASK NEXT-HOP-IPMás común. Se especifica la IP del router vecino.
Exit interfaceip route RED MASK INTERFAZSe especifica la interfaz de salida. Solo recomendado en P2P serial.
Fully specifiedip route RED MASK INTERFAZ NEXT-HOP-IPMás precisa. Combina ambas formas. Recomendada en Ethernet.
Default routeip route 0.0.0.0 0.0.0.0 NEXT-HOPGateway de último recurso. Coincide con cualquier destino.
Summary routeip route 10.0.0.0 255.0.0.0 NEXT-HOPUna ruta que resume múltiples redes específicas.
Host routeip route 192.168.1.100 255.255.255.255 NEXT-HOP/32 apunta a un host específico.

📌 Longest Match — La regla más importante del routing

El router siempre elige la ruta más específica (prefijo más largo) que coincida con el destino.

Destino: 10.1.1.5 → Tabla tiene: 10.0.0.0/8, 10.1.0.0/16, 10.1.1.0/24 → usa /24

⚙️ Configuración Rutas Estáticas

Rutas estáticas básicas

! Red: R1(192.168.1.0/24) ─── 10.0.0.0/30 ─── R2(172.16.0.0/24)
! En R1: agregar ruta hacia red de R2
R1(config)# ip route 172.16.0.0 255.255.0.0 10.0.0.2
! En R2: agregar ruta hacia red de R1
R2(config)# ip route 192.168.1.0 255.255.255.0 10.0.0.1
! Ruta por defecto (gateway de último recurso)
R1(config)# ip route 0.0.0.0 0.0.0.0 203.0.113.1
! Verificar
R1# show ip route
S    172.16.0.0/16    [1/0] via 10.0.0.2
C    10.0.0.0/30      directly connected, Se0/0/0
C    192.168.1.0/24   directly connected, Gi0/0
S*   0.0.0.0/0        [1/0] via 203.0.113.1

Verificación y diagnóstico

R1# show ip route static
R1# show ip route 172.16.1.5
Routing entry for 172.16.0.0/16
  Known via "static", distance 1, metric 0
  Routing Descriptor Blocks:
  * 10.0.0.2
      Route metric is 0, traffic share count is 1
R1# ping 172.16.0.1 source 192.168.1.1
R1# traceroute 172.16.0.1

📌 Rutas Flotantes (Floating Static Routes)

Una ruta flotante es una ruta estática con AD mayor que la ruta principal, usada como backup. Solo aparece en la tabla de routing cuando la ruta principal desaparece.

💡 Funcionamiento

Ruta principal: OSPF AD=110 → 172.16.0.0/24 via 10.0.0.2

Ruta flotante: ip route 172.16.0.0 255.255.0.0 10.1.1.2 200 (AD=200)

Si OSPF cae, la ruta con AD 200 activa automáticamente la ruta de backup.

! Ruta primaria (dinámica o estática con AD normal)
R1(config)# ip route 172.16.0.0 255.255.0.0 10.0.0.2
! Ruta flotante: misma red, distinto next-hop, AD mayor (150)
R1(config)# ip route 172.16.0.0 255.255.0.0 10.1.1.2 150
! La flotante NO aparece en show ip route mientras la principal esté activa
R1# show ip route 172.16.0.0
Routing entry for 172.16.0.0/16
  Known via "static", distance 1, metric 0 (ruta principal)

🧪 Laboratorio — Red con 3 routers

📋 Topología: PC-A─R1─Serial─R2─Serial─R3─PC-C

R1: LAN 192.168.1.0/24, Serial 10.0.0.0/30
R2: Serial 10.0.0.0/30 y 10.0.0.4/30
R3: LAN 192.168.3.0/24, Serial 10.0.0.4/30

R1 — Configuración

R1(config)# interface GigabitEthernet0/0
R1(config-if)# ip address 192.168.1.1 255.255.255.0
R1(config-if)# no shutdown
R1(config)# interface Serial0/0/0
R1(config-if)# ip address 10.0.0.1 255.255.255.252
R1(config-if)# no shutdown
! Rutas estáticas
R1(config)# ip route 10.0.0.4 255.255.255.252 10.0.0.2
R1(config)# ip route 192.168.3.0 255.255.255.0 10.0.0.2

R2 — Router intermedio (necesita rutas a ambos extremos)

R2(config)# ip route 192.168.1.0 255.255.255.0 10.0.0.1
R2(config)# ip route 192.168.3.0 255.255.255.0 10.0.0.6

Verificación final

R1# ping 192.168.3.1 source 192.168.1.1
!!!!!  Success rate is 100 percent
R1# traceroute 192.168.3.1
 1  10.0.0.2 (R2)
 2  10.0.0.6 (R3 serial)
 3  192.168.3.1 (PC-C)

📋 Resumen — Routing Estático

✅ Puntos críticos

  • AD Static = 1 (segunda más confiable tras Connected=0)
  • Longest match: el router siempre usa la ruta más específica
  • Ruta por defecto: ip route 0.0.0.0 0.0.0.0 → aparece como S*
  • Ruta flotante: misma red con AD mayor → backup automático
  • Las rutas estáticas no se adaptan a cambios de topología
  • show ip route → ver tabla completa | show ip route static → solo estáticas
