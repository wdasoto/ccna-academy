WAN y Servicios

Tecnologías WAN · DHCP · DNS · NTP · SNMP · Syslog · QoS

🌍 ¿Qué es una WAN?

Una WAN (Wide Area Network) es una red que cubre áreas geográficas grandes — ciudades, países o continentes. Conecta múltiples LANs a través de infraestructura de un proveedor de servicios (ISP). En CCNA, WAN abarca las tecnologías y protocolos que permiten esta conectividad de largo alcance.
🎯
CCNA Examen: Para WAN debes conocer: tipos de conexión (dedicada, conmutada), terminología (CPE, DCE, DTE, demarcación), y las tecnologías principales: MPLS, Metro Ethernet, Banda Ancha (DSL, Cable, Fibra), VPN.

🏷️ Terminología WAN

TérminoSignificadoEjemplo

📡 Tecnologías WAN

✅
MPLS vs VPN: MPLS es un servicio del ISP — privado, seguro, costoso. VPN sobre Internet es una alternativa económica que cifra el tráfico sobre la infraestructura pública del ISP. Hoy en día muchas empresas migran de MPLS a SD-WAN con VPN.

🔒 VPN — Virtual Private Network

Una VPN crea un túnel cifrado sobre una red pública (Internet), emulando una conexión privada. El tráfico se encapsula y cifra antes de enviarse, protegiéndolo de interceptación. Es la alternativa económica a los circuitos WAN dedicados.
Tipo VPNDescripciónProtocolo común
Site-to-SiteConecta dos redes permanentemente — reemplaza un circuito WANIPsec, GRE sobre IPsec
Remote AccessUsuario individual se conecta a la red corporativa desde cualquier lugarSSL/TLS, IPsec
DMVPNVPN dinámica multipunto — múltiples sitios hablan entre sí sin hub centralGRE + IPsec + NHRP

📡 DHCP — Asignación Automática de IPs

DHCP (Dynamic Host Configuration Protocol) asigna automáticamente configuración de red a los hosts: dirección IP, máscara, gateway y DNS. Opera en UDP puerto 67 (servidor) y 68 (cliente). Sin DHCP, cada host necesitaría configuración manual.
🎯
CCNA Examen: Memoriza el proceso DORA: Discover → Offer → Request → Acknowledge. Es el proceso de 4 pasos que usa broadcast para obtener una IP. También recuerda que el servidor DHCP de Cisco IOS usa el comando ip dhcp pool.

🔄 Proceso DORA

PROCESO DORA — 4 PASOS Cliente Servidor DHCP   1. DISCOVER → Broadcast 255.255.255.255 ────────────────→ "¿Hay algún servidor DHCP? Necesito una IP"   ←──────────────────────────── 2. OFFER "Te ofrezco la IP 192.168.1.10 con estos parámetros"   3. REQUEST → Broadcast: "Acepto la oferta del servidor X" ─→   ←──────────────────────────── 4. ACK "Confirmado. La IP 192.168.1.10 es tuya por X tiempo"
✅
¿Por qué broadcast en REQUEST? El cliente usa broadcast (no unicast al servidor) en el paso 3 para notificar a todos los servidores DHCP que ha elegido una oferta, permitiendo que los demás liberen sus ofertas pendientes.

⌨️ Configuración DHCP en Cisco IOS

SERVIDOR DHCP EN ROUTER CISCO ! Excluir IPs estáticas del pool (gateways, servidores, impresoras) R1(config)# ip dhcp excluded-address 192.168.1.1 192.168.1.20   ! Crear el pool DHCP R1(config)# ip dhcp pool LAN-POOL R1(dhcp-config)# network 192.168.1.0 255.255.255.0 R1(dhcp-config)# default-router 192.168.1.1 R1(dhcp-config)# dns-server 8.8.8.8 8.8.4.4 R1(dhcp-config)# lease 7 ← días de arrendamiento (default: 1 día) R1(dhcp-config)# domain-name empresa.local
DHCP RELAY (IP HELPER) — SERVIDOR EN OTRA RED ! Cuando el servidor DHCP está en una red diferente al cliente ! El router reenvía los broadcasts DHCP como unicast al servidor R1(config)# interface GigabitEthernet0/0 ← interfaz de la LAN del cliente R1(config-if)# ip helper-address 10.0.0.100 ← IP del servidor DHCP
VERIFICACIÓN DHCP R1# show ip dhcp binding ← IPs asignadas a cada MAC R1# show ip dhcp pool ← estadísticas del pool R1# show ip dhcp conflict ← IPs con conflictos detectados R1# show ip dhcp server statistics ← contadores de mensajes
🎯
CCNA Examen: ip helper-address convierte los broadcasts DHCP en unicast hacia el servidor DHCP remoto. Sin este comando, los broadcasts no cruzan los routers. Este es uno de los comandos más evaluados de DHCP en CCNA.

🔎 DNS — Sistema de Nombres de Dominio

DNS (Domain Name System) traduce nombres de dominio a direcciones IP. Cuando escribes www.cisco.com, tu dispositivo consulta un servidor DNS para obtener la IP real. Opera en UDP/TCP puerto 53. Es la "guía telefónica" de Internet.
✅
UDP vs TCP en DNS: DNS usa UDP 53 para consultas normales (respuestas pequeñas). Usa TCP 53 para transferencias de zona (copias de la base de datos DNS entre servidores) y respuestas grandes (> 512 bytes).

🏗️ Jerarquía DNS

ÁRBOL DNS JERÁRQUICO . (root) / | \ .com .org .net .edu .co ← TLD | cisco ← Dominio de segundo nivel (cisco.com) / \ www mail ← Subdominios (www.cisco.com, mail.cisco.com)
ComponenteFunciónEjemplo
Root Servers13 servidores raíz que conocen los servidores TLD — son la cima del DNSa.root-servers.net
TLD ServersServidores para .com, .org, .net, etc. — conocen los servidores autoritativosns1.verisign.com
AuthoritativeTiene la respuesta final para un dominio específicons1.cisco.com
Resolver/CacheServidor DNS del ISP o empresa — hace las consultas en nombre del cliente8.8.8.8 (Google DNS)

⌨️ DNS en Cisco IOS

CONFIGURAR DNS EN ROUTER CISCO ! Habilitar resolución de nombres (está activo por defecto) R1(config)# ip domain-lookup   ! Especificar servidor DNS a usar R1(config)# ip name-server 8.8.8.8 8.8.4.4   ! Dominio local del router R1(config)# ip domain-name empresa.local   ! Deshabilitar lookup DNS (evita delays en typos de comandos) R1(config)# no ip domain-lookup ← útil en labs para evitar esperas   ! Verificar R1# show hosts ← caché DNS local del router

⏱️ NTP — Sincronización de Tiempo

NTP (Network Time Protocol) sincroniza los relojes de todos los dispositivos de la red con una fuente de tiempo precisa. Opera en UDP puerto 123. Es crítico para la seguridad: los certificados digitales, los logs de seguridad, Kerberos y los registros de Syslog necesitan hora precisa para ser válidos.
⚠️
Sin NTP: Los logs de diferentes dispositivos tendrán horas distintas — imposible correlacionar eventos de seguridad. Los certificados SSL fallarán si el reloj del cliente difiere demasiado del servidor.

📊 Estratos NTP (Stratum)

NTP usa el concepto de stratum (estrato) para indicar la distancia desde la fuente de tiempo más precisa. Stratum 0 = fuente de referencia (reloj atómico, GPS). Cada salto NTP aumenta el stratum en 1.
JERARQUÍA DE ESTRATOS NTP Stratum 0: Reloj atómico / GPS / Radio ← fuente primaria ↓ Stratum 1: Servidor NTP primario ← time.google.com, pool.ntp.org ↓ Stratum 2: Servidor NTP corporativo ← sincroniza de stratum 1 ↓ Stratum 3: Routers y switches Cisco ← clientes NTP   Stratum 15 = máximo aceptable. Stratum 16 = no sincronizado.

⌨️ Configuración NTP en Cisco IOS

NTP CLIENTE ! Sincronizar el router con un servidor NTP externo R1(config)# ntp server time.google.com ← preferido R1(config)# ntp server pool.ntp.org prefer ← alternativo   ! Configurar zona horaria R1(config)# clock timezone COT -5 ← Colombia Time UTC-5
NTP MASTER (el router es servidor NTP) ! El router actúa como servidor NTP para la red interna R1(config)# ntp master 3 ← stratum 3 (si no tiene fuente externa)   ! Autenticación NTP (seguridad) R1(config)# ntp authenticate R1(config)# ntp authentication-key 1 md5 ClaveSeg123 R1(config)# ntp trusted-key 1   ! Verificar sincronización R1# show ntp status ← estado general y stratum actual R1# show ntp associations ← servidores configurados y su estado R1# show clock detail ← hora actual y fuente
🎯
CCNA Examen: En show ntp status busca: "Clock is synchronized" (sincronizado) vs "Clock is unsynchronized". El stratum indica la calidad — cuanto menor el número, más preciso. El símbolo * en show ntp associations indica el servidor actualmente seleccionado.

📊 SNMP — Gestión de Red

SNMP (Simple Network Management Protocol) permite monitorear y gestionar dispositivos de red de forma centralizada. Un servidor de gestión (NMS) puede leer estadísticas, configuraciones y recibir alertas de routers, switches y servidores. Opera en UDP 161 (consultas) y UDP 162 (traps).
VersiónSeguridadCaracterísticas
SNMPv1Community string (texto plano)Básico, obsoleto, sin cifrado
SNMPv2cCommunity string (texto plano)Más eficiente, bulk operations, sin cifrado
SNMPv3Usuario + cifrado AES/DES + autenticación MD5/SHASeguro, recomendado para producción

⌨️ SNMP en Cisco IOS

SNMPv2c — CONFIGURACIÓN BÁSICA ! Community string de solo lectura (para monitoreo) R1(config)# snmp-server community PUBLIC ro   ! Community string de lectura/escritura (para gestión) R1(config)# snmp-server community PRIVATE rw   ! Enviar traps al servidor NMS R1(config)# snmp-server host 192.168.1.200 version 2c PUBLIC R1(config)# snmp-server enable traps   ! Información del dispositivo R1(config)# snmp-server location "Datacenter Piso 3 Rack 5" R1(config)# snmp-server contact "[email protected]"
⚠️
Seguridad SNMP: Nunca uses community strings "public" o "private" en producción — son las predeterminadas y cualquiera las conoce. Siempre usa SNMPv3 con autenticación y cifrado, o al menos community strings únicas con ACLs.

📋 Syslog — Registro Centralizado de Eventos

Syslog envía mensajes de log de los dispositivos de red a un servidor centralizado. Cada mensaje tiene un nivel de severidad del 0 al 7. Opera en UDP 514. Es esencial para auditoría, troubleshooting y cumplimiento de seguridad.
NivelNombreDescripciónEjemplo
0EmergencySistema inutilizableFallo catastrófico del hardware
1AlertAcción inmediata requeridaTemperatura crítica del router
2CriticalCondición críticaFallo de interfaz principal
3ErrorCondición de errorInterfaz down/up
4WarningCondición de advertenciaMemoria baja, ACL log match
5NoticeCondición normal pero significativaCambio de configuración
6InformationalMensaje informativoVecino OSPF establecido
7DebuggingMensajes de debug — muy verbosoSalida de comandos debug
SYSLOG EN CISCO IOS ! Enviar logs al servidor Syslog R1(config)# logging host 192.168.1.200   ! Nivel mínimo de severidad a enviar (0=más crítico, 7=todo) R1(config)# logging trap informational ← envía niveles 0-6   ! Activar timestamps en los logs (requiere NTP configurado) R1(config)# service timestamps log datetime msec   ! Guardar logs en buffer local R1(config)# logging buffered 64000 informational   ! Verificar R1# show logging ← logs locales y configuración
🎯
Mnemónico CCNA: Every Awesome Cisco Engineer Will Need Ice Daily = Emergency, Alert, Critical, Error, Warning, Notice, Informational, Debug (niveles 0-7).

⚡ QoS — Calidad de Servicio

QoS (Quality of Service) permite priorizar ciertos tipos de tráfico sobre otros cuando la red está congestionada. Sin QoS, todos los paquetes se tratan igual — una descarga pesada puede degradar las llamadas VoIP. Con QoS, el tráfico crítico siempre tiene prioridad.
🎯
CCNA Examen: QoS tiene tres modelos principales: Best Effort (sin QoS), IntServ (reserva de recursos con RSVP) y DiffServ (marcado de clases — el más usado). Para CCNA enfócate en DiffServ: DSCP, clasificación, marcado, colas.

📊 Características QoS

CaracterísticaDescripciónAfecta a
BandwidthAncho de banda garantizado para un tipo de tráficoRendimiento sostenido
Delay (Latencia)Tiempo que tarda un paquete en llegar — crítico en VoIPVoIP, video conferencia
JitterVariación en el delay — los paquetes llegan con retardo inconsistenteVoIP, streaming
Packet LossPaquetes descartados por congestión — inaceptable en vozTodos los tipos
✅
Requerimientos VoIP (Cisco): Delay < 150ms · Jitter < 30ms · Packet Loss < 1%. Si alguno supera estos valores, la calidad de la llamada se degrada notablemente. QoS existe principalmente para garantizar estos valores.

🏷️ DiffServ — DSCP y Marcado

DiffServ clasifica el tráfico en clases y marca cada paquete con un valor DSCP (Differentiated Services Code Point) en el campo ToS del encabezado IP (6 bits). Los dispositivos de red aplican tratamiento preferencial según el valor DSCP.
Valor DSCPNombreTráfico típicoPrioridad
46 / EFExpedited ForwardingVoIP, voz en tiempo realMáxima
34 / AF41Assured Forwarding 4Video conferenciaAlta
26 / AF31Assured Forwarding 3Aplicaciones críticas, SAPMedia-Alta
18 / AF21Assured Forwarding 2Transacciones de base de datosMedia
10 / AF11Assured Forwarding 1Email, transferencias FTPNormal
0 / BEBest Effort (default)Navegación web, descargasMínima
QoS CON MQC (MODULAR QoS CLI) — CISCO IOS ! Paso 1: Class-map — identificar el tráfico R1(config)# class-map match-any VOIP-TRAFFIC R1(config-cmap)# match dscp ef ← DSCP 46 = VoIP   ! Paso 2: Policy-map — definir qué hacer con el tráfico R1(config)# policy-map QOS-POLICY R1(config-pmap)# class VOIP-TRAFFIC R1(config-pmap-c)# priority 512 ← garantizar 512 Kbps para VoIP R1(config-pmap)# class class-default R1(config-pmap-c)# fair-queue   ! Paso 3: Service-policy — aplicar a interfaz R1(config)# interface GigabitEthernet0/1 R1(config-if)# service-policy output QOS-POLICY   ! Verificar R1# show policy-map interface GigabitEthernet0/1
