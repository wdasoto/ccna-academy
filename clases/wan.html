🌍 ¿Qué es una WAN?

Una WAN (Wide Area Network) conecta redes LAN separadas geográficamente a través de infraestructura de un proveedor de servicios (ISP). Abarca ciudades, países y continentes.

Término WANDescripción
CPE (Customer Premises Equipment)Equipo en las instalaciones del cliente (router, CSU/DSU)
DCE (Data Circuit-terminating Equipment)Equipo del proveedor que termina el circuito. Pone el clock.
DTE (Data Terminal Equipment)Equipo del cliente que recibe el clock (router)
DemarcaciónPunto donde termina la responsabilidad del ISP y comienza la del cliente
Local LoopCircuito físico entre el cliente y la central del ISP (CO)
CO (Central Office)Instalaciones del ISP donde termina el local loop

⚙️ Configuración Serial WAN

HDLC (default en interfaces serial Cisco)

R1(config)# interface Serial0/0/0
R1(config-if)# ip address 10.0.0.1 255.255.255.252
R1(config-if)# encapsulation hdlc
! hdlc es el default — no hace falta este comando
! En el lado DCE (simulado en Packet Tracer), configurar clock
R1(config-if)# clock rate 64000
R1(config-if)# no shutdown

PPP con autenticación CHAP

! En R1
R1(config)# username R2 password SecretPPP!
R1(config)# interface Serial0/0/0
R1(config-if)# encapsulation ppp
R1(config-if)# ppp authentication chap
! En R2 (debe tener usuario R1 con la misma password)
R2(config)# username R1 password SecretPPP!
R2(config)# interface Serial0/0/0
R2(config-if)# encapsulation ppp
R2(config-if)# ppp authentication chap

Verificación serial

R1# show interfaces Serial0/0/0
Serial0/0/0 is up, line protocol is up
  Hardware is GT96K Serial
  Internet address is 10.0.0.1/30
  Encapsulation PPP, LCP Open
  Open: CDPCP, IPCP
  Last input 00:00:01, Last clear 00:00:00
R1# show controllers Serial0/0/0
DTE V.35, clock rate 64000   ← o DCE si es el otro extremo

🔐 VPN — Virtual Private Network

Una VPN crea un túnel cifrado sobre Internet público, ofreciendo conectividad WAN económica con seguridad.

🏢 Site-to-Site VPN

Conecta dos sedes permanentemente. Los routers de cada sede establecen el túnel. Los usuarios finales no notan la VPN.

👤 Remote Access VPN

Usuarios remotos se conectan a la red corporativa desde cualquier lugar. Requiere cliente VPN (AnyConnect, OpenVPN).

🔐 IPsec — Protocolo de seguridad VPN

  • AH (Authentication Header): autenticación e integridad. No cifra.
  • ESP (Encapsulating Security Payload): autenticación, integridad y cifrado. Más usado.
  • IKE: negocia los parámetros de seguridad (claves, algoritmos)

🚇 GRE Tunnel — Encapsulación sin cifrado

R1(config)# interface Tunnel0
R1(config-if)# ip address 172.16.0.1 255.255.255.252
R1(config-if)# tunnel source GigabitEthernet0/1
R1(config-if)# tunnel destination 203.0.113.2
R1(config-if)# tunnel mode gre ip
! Para usar con IPsec, agregar crypto map en la interfaz física

🧪 Laboratorio WAN Serial con PPP

R1─Serial─R2 con PPP CHAP

! Topología: LAN-R1(10.0.0.1/30)─Serial─(10.0.0.2/30)R2-LAN
! R1
R1(config)# username R2 password Shared@PPP!
R1(config)# interface Serial0/0/0
R1(config-if)# ip address 10.0.0.1 255.255.255.252
R1(config-if)# encapsulation ppp
R1(config-if)# ppp authentication chap
R1(config-if)# no shutdown
! R2
R2(config)# username R1 password Shared@PPP!
R2(config)# interface Serial0/0/0
R2(config-if)# ip address 10.0.0.2 255.255.255.252
R2(config-if)# encapsulation ppp
R2(config-if)# ppp authentication chap
R2(config-if)# no shutdown
! Verificar
R1# show interfaces Serial0/0/0
Serial0/0/0 is up, line protocol is up
  Encapsulation PPP, LCP Open
R1# ping 10.0.0.2
!!!!!   100%

📋 Resumen WAN

✅ Puntos críticos

  • DCE pone el clock rate en el enlace serial | DTE lo recibe
  • HDLC = default Cisco Serial, propietario, solo Cisco-Cisco
  • PPP = estándar abierto, multi-vendor, soporta CHAP/PAP y compresión
  • CHAP usa hash MD5 (más seguro) | PAP envía contraseña en texto plano
  • MPLS = más común en WAN empresarial modernsa
  • VPN Site-to-Site = IPsec entre routers | Remote Access = cliente VPN
  • GRE = túnel sin cifrado (se combina con IPsec para seguridad)
