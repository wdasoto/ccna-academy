Parsing de Outputs IOS con TextFSM

Convierte la salida de show commands (texto desestructurado) en datos estructurados Python listos para procesar, comparar y reportar. Domina TextFSM, ntc-templates y parsing avanzado de equipos Cisco.

🐍 Python 3.x 🎯 CCNA Domain 6.2 ⏱ ~90 min lectura 🔍 Regex Patterns 🧪 Proyecto Audit

El Problema del Texto Plano

Cuando ejecutas show ip route o show interfaces vía SSH, recibes texto plano diseñado para humanos, no para máquinas. Es fácil de leer, pero imposible de procesar programáticamente sin lógica compleja de split(), find() y regex enrevesados.

TextFSM soluciona esto: convierte el output semi-estructurado de Cisco IOS en listas de diccionarios Python perfectamente estructurados. Una tabla de rutas se convierte en objetos con campos protocol, network, next_hop, metric.

💡 Por qué TextFSM es estándar en networking

Desarrollado por Google para sus operaciones de red masiva. Cisco, Juniper, Arista — todos los outputs de CLI pueden parsearse con TextFSM. La librería ntc-templates contiene más de 500 templates pre-escritos y probados en producción.

Antes vs Después de TextFSM

📄

Output Crudo (String)

Gateway of last resort...
O 10.0.0.0/8 [110/2] via 192.168.1.1...
C 192.168.1.0/24 is directly...

Difícil de procesar: Requiere regex complejos y es frágil ante cambios de formato entre versiones IOS.

📊

Con TextFSM (Lista de Dicts)

[
  {proto:"O", net:"10.0.0.0/8", nh:"192.168.1.1", metric:2},
  {proto:"C", net:"192.168.1.0/24", nh:"", metric:0}
]

Listo para usar: Iterar, filtrar, exportar a JSON/Excel, comparar con inventories.

Arquitectura de Parsing

TextFSM funciona mediante templates: archivos de texto con reglas que describen cómo extraer campos del output. Cada template define:

  • Values: Variables a extraer (ej: IP_Address, Status)
  • States: Máquina de estados que recorre línea por línea
  • Reglas Regex: Patrones que identifican qué líneas procesar y qué capturar
Concepto: Template → Parsed Data
# 1. Tienes un output de show command
output_raw = """
Interface              IP-Address      OK? Method Status                Protocol
GigabitEthernet0/0    192.168.1.1     YES manual up                    up
GigabitEthernet0/1    unassigned      YES unset  administratively down down
"""

# 2. Aplicas un template de TextFSM
import textfsm
with open("cisco_ios_show_ip_interface_brief.template") as tpl:
    parser = textfsm.TextFSM(tpl)
    resultados = parser.ParseText(output_raw)

# 3. Obtienes datos estructurados
print(resultados)
# [['GigabitEthernet0/0', '192.168.1.1', 'YES', 'manual', 'up', 'up'],
#  ['GigabitEthernet0/1', 'unassigned', 'YES', 'unset', 'down', 'down']]

Anatomía de un Template TextFSM

Un template tiene dos secciones principales: la declaración de Values (variables a capturar) y la sección de States (reglas de parsing con regex).

cisco_ios_show_version.template — Estructura básica
Value VERSION (.+)
Value HOSTNAME (\S+)
Value UPTIME (.+)
Value IMAGE_FILE (\S+)
Value SERIAL (\S+)

Start
  ^Cisco IOS Software.*Version ${VERSION}
  ^${HOSTNAME} uptime is ${UPTIME}
  ^System image file is "${IMAGE_FILE}"
  ^Processor board ID ${SERIAL}
  ^#

Declaración de Values

La sintaxis es: Value [Options] Nombre Regex

Tipos de Values y modificadores
# Value simple — captura cualquier cosa hasta el final de línea
Value DESCRIPTION (.+)

# Value con restricción — solo dígitos
Value VLAN_ID (\d+)

# Value con opción Required — debe aparecer en cada registro
Value Required IP_ADDRESS (\S+)

# Value que captura lista (multivalue) — útil para múltiples IPs
Value List NEXT_HOPS (\S+)

# Value opcional — puede o no existir
Value STATUS (up|down|administratively down)

States y Reglas

TextFSM procesa el input línea por línea. Cada regla tiene la forma: ^Regex$ → Acción.

Ejemplo: Parsear tabla de interfaces
Value INTF (\S+)
Value IP (\S+)
Value STATUS (up|down|administratively down)
Value PROTOCOL (up|down)

Start
  ^${INTF}\s+${IP}\s+\S+\s+\S+\s+${STATUS}\s+${PROTOCOL} -> Record
  ^Interface -> Skip
  ^$ -> Skip
  ^.+ -> Error
💡 Keywords especiales en States
  • Record: Guarda los values actuales y empieza un nuevo registro
  • Next: Continúa al siguiente match sin guardar
  • Skip: Ignora la línea
  • Error: Línea no reconocida — útil para detectar formatos inesperados
  • Continue: Sigue procesando la misma línea con más reglas

Máquina de Estados Avanzada

Cuando el output tiene secciones diferentes (ej: encabezado, tabla, resumen), usas múltiples estados:

cisco_ios_show_ip_route.template — Múltiples estados
Value PROTOCOL (C|O|B|D|S|U|M|i|L)
Value NETWORK (\S+)
Value MASK (\d+)
Value DISTANCE (\d+)
Value METRIC (\d+)
Value NEXTHOP (\S+)
Value INTERFACE (\S+)

Start
  ^Codes: -> HeaderDescription

HeaderDescription
  ^\s+\S+ -> Continue
  ^Gateway of last resort -> Routes

Routes
  ^${PROTOCOL}\s+${NETWORK}/${MASK} -> Continue
  ^\s+\[${DISTANCE}/${METRIC}\] via ${NEXTHOP}, ${INTERFACE} -> Record
  ^\s*$$ -> Done

Done

ntc-templates: Templates Probados en Producción

Escribir templates TextFSM desde cero es tedioso y propenso a errores. El proyecto ntc-templates (Network To Code) mantiene una biblioteca de más de 500 templates probados para Cisco, Juniper, Arista, HP, Dell, y más.

Procesamiento en Python

Analizar rutas específicas
from ntc_templates.parse import parse_output

output_route = """...output de show ip route..."""

rutas = parse_output(platform="cisco_ios", command="show ip route", data=output_route)

# Filtrar solo rutas OSPF
rutas_ospf = [r for r in rutas if r['protocol'] == 'O']

# Encontrar rutas con múltiples next-hops (ECMP)
rutas_ecmp = [r for r in rutas if len(r['nexthop_ip']) > 1]

# Calcular distancia administrativa promedio
distancias = [int(r['distance']) for r in rutas if r['distance']]
prom_dist = sum(distancias) / len(distancias)

# Verificar cobertura de red 10.0.0.0/8
red_10 = [r for r in rutas if r['network'].startswith('10.')]

print(f"Total rutas: {len(rutas)} | OSPF: {len(rutas_ospf)} | ECMP: {len(rutas_ecmp)}")
💡 Truco: Detectar rutas flotantes

Compara el distance de rutas estáticas vs dinámicas. Si tienes una ruta estática con distance 250 y una OSPF con 110, la OSPF gana. Usa TextFSM para auditificar distancias administrativas inconsistentes en tu red.

Técnicas Avanzadas de Parsing

1. Captura de Múltiples Valores (List)

Cuando un registro tiene múltiples valores del mismo tipo (ej: varios next-hops ECMP, múltiples IPs en show aliases), usa el modificador List.

Value List para ECMP
Value List NEXTHOPS (\d+\.\d+\.\d+\.\d+)

Start
  ^  via \S+ \(\S+, ${NEXTHOPS}\) -> Continue
  ^  via \S+ \(\S+, ${NEXTHOPS}\) -> Record

2. Estados Anidados y Saltos Condicionales

Para outputs que cambian formato según la configuración (ej: show running-config con secciones de interface, router, line):

Navegación por secciones
Start
  ^interface -> InterfaceSection
  ^router ospf -> OspfSection
  ^line con -> LineSection

InterfaceSection
  ^! -> Start
  ^ ip address -> Continue.Record

OspfSection
  ^! -> Start
  ^ network -> Continue.Record

3. Filldown — Herencia de Valores

El modificador Filldown mantiene el último valor capturado para las siguientes líneas. Útil cuando solo aparece el nombre de la interfaz una vez pero hay múltiples subentradas.

Ejemplo: Filldown para interfaces
Value Filldown INTF (\S+)
Value IP (\d+\.\d+\.\d+\.\d+)

Start
  ^interface ${INTF}
  ^ ip address ${IP} -> Record

4. Expresiones Regulares Avanzadas

PatrónUsoEjemplo
(\S+)Cualquier string sin espaciosNombres de interfaz
(\d+\.\d+\.\d+\.\d+)IPv4Direcciones IP
(\d+)Números enterosVLANs, métricas
(.+?)Lazy match (mínimo posible)Descripciones
([^,]+)Todo hasta la comaCampos CSV-like
(?:regex)Non-capturing groupSeparadores opcionales
⚠️ Debugging de Templates

Usa textfsm.parser.TextFSM con print(parser.header) para ver las columnas detectadas. Si una línea no hace match, TextFSM la ignora silenciosamente a menos que uses la regla ^.+ -> Error al final.

Integración con Netmiko y Automatización

TextFSM brilla cuando se combina con librerías de conexión. Netmiko tiene soporte nativo para TextFSM mediante el parámetro use_textfsm=True.

Netmiko + TextFSM nativo
from netmiko import ConnectHandler
import json

device = {
    "device_type": "cisco_ios",
    "host": "192.168.1.1",
    "username": "admin",
    "password": "cisco123",
}

# use_textfsm=True activa parsing automático vía ntc-templates

    "show ip interface brief",

Flujo Completo: Conexión → Parse → Análisis → Reporte

Auditoría de rutas en múltiples routers
from netmiko import ConnectHandler
from ntc_templates.parse import parse_output
import pandas as pd

routers = ["10.0.0.1", "10.0.0.2", "10.0.0.3"]
credenciales = {"username": "admin", "password": "cisco123", "device_type": "cisco_ios"}

todas_las_rutas = []

for router in routers:
    print(f"Conectando a {router}...")
    dev = {"host": router, **credenciales}
    
    try:
        with ConnectHandler(**dev) as conn:
            output = conn.send_command("show ip route")
            
            # Parse con TextFSM
            rutas = parse_output(platform="cisco_ios", 
                               command="show ip route", 
                               data=output)
            
            # Agregar metadata del router
            for r in rutas:
                r['router_source'] = router
            
            todas_las_rutas.extend(rutas)
            
    except Exception as e:
        print(f"Error en {router}: {e}")

# Análisis con Pandas
df = pd.DataFrame(todas_las_rutas)

# Análisis: Rutas inconsistentes (misma red, diferentes next-hops)
agrupadas = df.groupby('network')['nexthop_ip'].nunique()
inconsistentes = agrupadas[agrupadas > 1]

print(f"\nRedes con rutas inconsistentes: {len(inconsistentes)}")
for red in inconsistentes.index:
    print(f"  ⚠️  {red} tiene rutas diferentes según el router")

🚀 Proyecto Final — Auditoría de Interfaces con TextFSM

Script completo de producción que conecta a múltiples switches, parsea interfaces, detecta problemas de capa física, valida descripciones, y genera un reporte HTML profesional.

💡 Qué incluye este proyecto

Conexión SSH con Netmiko, parsing con TextFSM/ntc-templates, análisis de datos (interfaces sin desc, errores CRC, estado admin-down no documentado), generación de reporte HTML con estadísticas, y manejo de errores robusto.

#!/usr/bin/env python3
"""
auditoria_interfaces.py
─────────────────────────────────────────────────
Auditoría automatizada de interfaces Cisco IOS
usando TextFSM para parseo estructurado.

Requisitos: pip install netmiko ntc-templates jinja2
─────────────────────────────────────────────────
"""

from netmiko import ConnectHandler
from ntc_templates.parse import parse_output
from jinja2 import Template
from datetime import datetime
import sys

# ══════════════════════════════════════════════════════
# CONFIGURACIÓN DE DISPOSITIVOS
# ══════════════════════════════════════════════════════
DISPOSITIVOS = [
    {"host": "192.168.1.10", "nombre": "SW-CORE-01"},
    {"host": "192.168.1.11", "nombre": "SW-ACC-01"},
    {"host": "192.168.1.12", "nombre": "SW-ACC-02"},
]

CREDENTIALS = {
    "device_type": "cisco_ios",
    "username": "admin",
    "password": "Cisco123",
    "secret": "Cisco123",
}

# ══════════════════════════════════════════════════════
# FUNCIONES DE ANÁLISIS
# ══════════════════════════════════════════════════════

def auditar_switch(dispositivo):
    """Conecta, parsea y analiza interfaces de un switch"""
    host = dispositivo['host']
    nombre = dispositivo['nombre']
    
    print(f"\n🔍 Auditando {nombre} ({host})...")
    
    config = {**CREDENTIALS, "host": host}
    hallazgos = []
    
    try:
        with ConnectHandler(**config) as conn:
            conn.enable()
            
            # Obtener y parsear interfaces
            output = conn.send_command("show interfaces")
            interfaces = parse_output(
                platform="cisco_ios",
                command="show interfaces",
                data=output
            )
            
            # Análisis de cada interfaz
            for intf in interfaces:
                problemas = []
                
                # Check 1: Sin descripción
                desc = intf.get('description', '')
                if not desc and intf['interface'].startswith('Gi'):
                    problemas.append("Sin descripción")
                
                # Check 2: Errores de input
                input_errors = int(intf.get('input_errors', 0))
                if input_errors > 0:
                    problemas.append(f"Input errors: {input_errors}")
                
                # Check 3: CRC errors
                crc = int(intf.get('crc_errors', 0))
                if crc > 0:
                    problemas.append(f"CRC errors: {crc}")
                
                # Check 4: Administratively Down sin ser loopback
                link = intf.get('link_status', '')
                if link == 'administratively down' and 'Loopback' not in intf['interface']:
                    if not desc or 'MANTENIMIENTO' not in desc.upper():
                        problemas.append("Admin-down sin tag de mantenimiento")
                
                if problemas:
                    hallazgos.append({
                        'interface': intf['interface'],
                        'problemas': '; '.join(problemas),
                        'estado': link,
                        'descripcion': desc or 'N/A'
                    })
                    
    except Exception as e:
        print(f"❌ Error en {nombre}: {e}")
        return {'nombre': nombre, 'host': host, 'error': str(e), 'hallazgos': []}
    
    print(f"✅ {nombre}: {len(hallazgos)} problemas encontrados")
    return {
        'nombre': nombre,
        'host': host,
        'total_interfaces': len(interfaces),
        'hallazgos': hallazgos
    }

def generar_reporte(resultados):
    """Genera archivo HTML con resultados"""
    total_problemas = sum(len(r['hallazgos']) for r in resultados)
    
    template = Template(HTML_TEMPLATE)
    html = template.render(
        fecha=datetime.now().strftime("%Y-%m-%d %H:%M:%S"),
        dispositivos=resultados,
        total_devices=len(resultados),
        total_problemas=total_problemas
    )
    
    filename = f"auditoria_interfaces_{datetime.now().strftime('%Y%m%d_%H%M')}.html"
    with open(filename, 'w') as f:
        f.write(html)
    
    print(f"\n📄 Reporte generado: {filename}")
