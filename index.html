🎯 TIPS

CLAVES PARA EL EXAMEN CCNA

Los conceptos más frecuentes y trampas más comunes del examen CCNA 200-301.

⚡ ROUTING

La AD (Distancia Administrativa) determina qué protocolo gana cuando hay rutas de distintas fuentes. Connected=0, Static=1, EIGRP=90, OSPF=110, RIP=120. Menor AD = más confiable.

🔀 SWITCHING

Las interfaces del router están administratively down por defecto. Las del switch están activas al conectar cable. Siempre usar no shutdown en el router.

🌳 STP

La priority STP solo acepta múltiplos de 4096 (0, 4096, 8192...61440). Default = 32768. root primary ajusta a 24576. root secondary ajusta a 28672.

🔐 SSH

SSH v2 requiere: hostname distinto de Router, ip domain-name, y crypto key generate rsa modulus 2048. Sin ip domain-name, el comando RSA falla.

🏷️ VLANs

Las VLANs se guardan en flash:vlan.dat, separado de startup-config. erase startup-config NO borra las VLANs. Para borrarlas: delete flash:vlan.dat.

🔄 NAT

Marcar siempre ip nat inside en la LAN e ip nat outside en la WAN. PAT usa overload al final del comando. Sin overload es NAT dinámico (no PAT).

🛡️ ACLs

ACL Estándar → colocar cerca del destino. ACL Extendida → cerca del origen. Toda ACL tiene un implicit deny any any al final que no aparece en la config.

⭕ OSPF

El Router ID OSPF se elige en orden: 1° manual (router-id), 2° IP más alta de Loopback activa, 3° IP más alta de cualquier interfaz activa. Siempre configurarlo manualmente.

🔑 RECUPERACIÓN

Config-register 0x2142 = ignora startup-config. 0x2102 = operación normal. Después de recuperar contraseña, siempre restaurar a 0x2102 y hacer copy run start.

⌨️ COMANDOS

copy running-config startup-config (o wr) es obligatorio para guardar cambios. Sin esto, TODO se pierde al reiniciar. Trampa clásica del examen CCNA.
